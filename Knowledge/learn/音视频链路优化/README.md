## 写作原则

本系列文档遵循以下写作原则：

### 1. 金字塔结构
- 主文章提供全景概览，明确"做什么"和"为什么"
- 子文件深入细节，解释"怎么做"和"如何优化"
- 每篇文章开头有核心结论，便于快速把握要点

### 2. 工程导向
- 强调工程优化为主、算法优化为辅的定位
- 提供具体的优化参数、配置建议和代码示例
- 包含真实案例和可量化的优化收益

### 3. 数据驱动
- 提供各环节的典型延迟数据、性能指标
- 优化建议基于ROI评估，明确优先级
- 强调度量驱动、灰度验证的工程方法论

### 4. 系统化思维
- 全链路视角，避免单点优化陷阱
- 强调各环节协同配合、瓶颈消除
- 提供分层治理、分级响应的体系化方案

## 核心概念速查表

| 术语 | 英文 | 简要解释 | 详见 |
|------|------|---------|------|
| 端到端延迟 | End-to-End Latency | 从采集到渲染的完整耗时 | 延迟优化 |
| 首帧时间 | Time to First Frame | 从请求到首帧渲染的耗时 | 首帧优化 |
| 卡顿率 | Freeze Rate | 播放不流畅的时间占比 | 卡顿治理 |
| VMAF | Video Multimethod Assessment Fusion | Netflix的感知质量指标 | 质量评估 |
| GOP | Group of Pictures | 一组关联的视频帧 | 编码优化 |
| RTT | Round Trip Time | 网络往返时延 | 传输优化 |
| FEC | Forward Error Correction | 前向纠错，冗余数据恢复丢包 | 弱网对抗 |
| ARQ | Automatic Repeat Request | 自动重传请求 | 弱网对抗 |
| NACK | Negative Acknowledgment | 否定确认，请求重传 | 弱网对抗 |
| GCC | Google Congestion Control | Google拥塞控制算法 | 传输优化 |
| BBR | Bottleneck Bandwidth and RTT | Google拥塞控制算法 | 传输优化 |
| SVC | Scalable Video Coding | 可伸缩视频编码 | 编码优化 |
| 抖动缓冲 | Jitter Buffer | 消除网络抖动的缓冲区 | 解码优化 |
| 零拷贝 | Zero Copy | 避免CPU内存拷贝的数据传输 | 延迟优化 |
| 硬编/硬解 | Hardware Encode/Decode | 使用专用硬件进行编解码 | 编解码优化 |
| MediaCodec | Android MediaCodec | Android统一编解码API | 硬件编码 |
| VideoToolbox | Apple VideoToolbox | iOS/macOS硬件编解码框架 | 硬件编码 |
