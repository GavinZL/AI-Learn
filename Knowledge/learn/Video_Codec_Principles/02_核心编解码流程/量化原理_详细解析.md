# 量化原理（Quantization）

> 量化是有损压缩的核心步骤——通过"适度的粗糙"换取"巨大的压缩"。

---

## 核心结论（TL;DR）

**量化的本质是"降低精度"——将连续或高精度的数值映射到有限个离散等级。这是视频编码中唯一的有损步骤，但通过巧妙地量化那些人眼不敏感的信息，可以在几乎不影响视觉质量的前提下实现数十倍的压缩。**

关键要点：
1. 量化步长 $\Delta$ 决定了精度损失的程度——步长越大，压缩越高，质量越差
2. 死区量化将接近零的系数直接变为零，增强稀疏性
3. QP 每增加 6，量化步长翻倍，码率大约减半
4. RDOQ（率失真优化量化）比简单四舍五入更智能

## 1. 量化的本质

### 1.1 为什么需要量化

经过变换后，我们得到了 DCT 系数。虽然能量已经集中，但系数仍然是高精度的数值（通常是 16 位甚至 32 位整数或浮点数）。

```
变换后的 DCT 系数：
┌────────────────────────────────┐
│ 892.3  -45.7   12.3   -3.8 ... │
│ -67.2    8.9   -2.1    0.7 ... │
│  15.4   -3.2    0.8   -0.3 ... │
│  -4.1    0.6   -0.2    0.1 ... │
└────────────────────────────────┘
还是需要很多比特来表示
```

**量化的作用**：用更少的比特表示这些系数，代价是损失一些精度。

### 1.2 地图精度的类比

想象你需要传达一个位置：

| 精度 | 表示方式 | 信息量 | 准确性 |
|------|---------|-------|-------|
| **高精度** | GPS 坐标 (39.9042° N, 116.4074° E) | 需要很多字符 | 精确到米 |
| **中精度** | 街道地址（北京市东城区...） | 中等 | 精确到建筑 |
| **低精度** | 城市名（北京） | 很少 | 只知道大概区域 |

量化就是在"表示精度"和"信息量"之间做取舍。

### 1.3 量化的数学表达

**量化**：将输入值 $x$ 映射到离散的量化级别 $\hat{x}$

$$\hat{x} = Q(x) = \text{round}\left(\frac{x}{\Delta}\right) \times \Delta$$

其中 $\Delta$ 是**量化步长**（Quantization Step Size）。

**反量化**：（在解码端执行）

$$x' = \hat{x} = Q^{-1}(Q(x))$$

注意：$x' \neq x$，这个差值就是**量化误差**（量化噪声）。

## 2. 标量量化

### 2.1 均匀量化

最简单的量化方式：将数值范围等分。

```
量化步长 Δ = 8 的均匀量化：

输入值 x:  -20 -12  -5   0   3   7  15  20
           │   │   │   │   │   │   │   │
           ▼   ▼   ▼   ▼   ▼   ▼   ▼   ▼
量化级别:  -2  -2  -1   0   0   1   2   2
           │   │   │   │   │   │   │   │
           ▼   ▼   ▼   ▼   ▼   ▼   ▼   ▼
反量化值: -16 -16  -8   0   0   8  16  16
```

**Python 实现**：
```python
def uniform_quantize(x, delta):
    """均匀量化"""
    return round(x / delta)

def uniform_dequantize(q, delta):
    """反量化（重建）"""
    return q * delta
```

### 2.2 死区量化（Dead Zone Quantization）

视频编码中实际使用的是**死区量化**：在零附近设置一个更宽的区间，使接近零的值直接量化为零。

```
普通均匀量化：
        -Δ    0    Δ
    ─────┼────┼────┼─────
         │-1  │ 0  │ 1
    区间: [-Δ,0) [0,Δ)

死区量化（死区宽度 = 2Δ）：
      -1.5Δ  -0.5Δ  0.5Δ  1.5Δ
    ─────┼─────┼─────┼─────┼─────
         │ -1  │  0  │  1  │
    区间:      [-Δ, Δ)
                  ↑
              更宽的"死区"
```

**为什么要用死区量化？**

1. DCT 系数中有很多接近零的小值
2. 这些小值量化后若不为零，需要额外比特编码
3. 死区让更多系数变成"真正的零"，大幅提高稀疏性

```
变换系数: [892, -46, 12, -4, 2, -1, 0, 0]
                      ↑   ↑  ↑
                     这些小值经过死区量化后变成零
                     
量化后:   [56,  -3,  1,  0, 0,  0, 0, 0]
                         ↑
                    大量的零 → 易于压缩
```

### 2.3 QP 与量化步长的关系

H.264/H.265 使用**量化参数（QP）**控制量化强度。QP 范围是 0-51。

**核心规律**：QP 每增加 6，量化步长翻倍。

$$\Delta = 2^{(QP - 4) / 6}$$

| QP | 量化步长 Δ | 相对于 QP=0 |
|----|-----------|------------|
| 0 | 0.625 | 1× |
| 6 | 1.25 | 2× |
| 12 | 2.5 | 4× |
| 18 | 5 | 8× |
| 24 | 10 | 16× |
| 30 | 20 | 32× |
| 36 | 40 | 64× |
| 42 | 80 | 128× |
| 51 | 224 | 358× |

**为什么是指数关系？**

人眼对亮度的感知是对数的（韦伯-费希纳定律），量化步长指数增长可以在主观质量上产生近似线性的变化。

## 3. 量化矩阵

### 3.1 不同频率不同量化

人眼对**低频信息**更敏感，对**高频信息**不太敏感。

利用这一特性，可以对不同位置的 DCT 系数使用不同的量化步长：

```
8×8 量化矩阵示例（JPEG 标准亮度表）：

┌────────────────────────────────────────────┐
│ 16  11  10  16  24   40   51   61  │ ← 低频：小步长（精细）
│ 12  12  14  19  26   58   60   55  │
│ 14  13  16  24  40   57   69   56  │
│ 14  17  22  29  51   87   80   62  │
│ 18  22  37  56  68  109  103   77  │
│ 24  35  55  64  81  104  113   92  │
│ 49  64  78  87 103  121  120  101  │
│ 72  92  95  98 112  100  103   99  │ ← 高频：大步长（粗糙）
└────────────────────────────────────────────┘
```

**实际量化**：

$$Q_{quantized}[u][v] = \text{round}\left(\frac{DCT[u][v]}{QMatrix[u][v] \times scale}\right)$$

### 3.2 H.264/H.265 中的量化矩阵

H.264/H.265 支持**自定义量化矩阵**，但也提供默认的"flat"（均匀）矩阵。

- **Flat 矩阵**：所有位置使用相同的量化步长
- **自定义矩阵**：专业应用可以根据内容特点调整

**H.265 默认使用 flat 矩阵**，因为现代编码器的 RDO 已经能够很好地处理不同频率的重要性。

### 3.3 缩放列表（Scaling List）

为了支持整数变换，H.264/H.265 使用**缩放列表**来实现量化矩阵的功能：

```
量化过程（H.264 4×4 块）：

1. 整数变换：Y = H × X × H^T
2. 量化：Q = (Y × MF + f) >> qbits

其中：
- MF：乘法因子（与 QP 相关的查找表）
- f：舍入偏移（帧内 f = 2^qbits/3，帧间 f = 2^qbits/6）
- qbits：移位量

这样可以用乘法和移位代替除法，适合硬件实现。
```

## 4. 矢量量化

### 4.1 概念介绍

**标量量化**：每个系数独立量化

**矢量量化（VQ）**：将多个系数作为一个向量，整体量化

```
标量量化：
(x1, x2, x3) → (Q(x1), Q(x2), Q(x3))
每个分量独立处理

矢量量化：
(x1, x2, x3) → 查找最接近的码字 c_i
整体映射到码本中的一个码字
```

### 4.2 码本（Codebook）

矢量量化需要预先训练一个**码本**，包含有限个"代表性向量"（码字）：

```
码本 C = {c1, c2, c3, ..., cN}

编码：找到与输入向量 x 最接近的码字 c_i
      i* = argmin_i ||x - c_i||

传输：只需要传输索引 i*（log2(N) 比特）

解码：用索引 i* 查表得到 c_i* 作为重建值
```

### 4.3 为什么视频编码不主要使用 VQ

1. **码本存储**：需要在编解码器两端存储相同的码本
2. **搜索复杂度**：在大码本中搜索最近邻很耗时
3. **自适应性**：固定码本难以适应不同类型的视频内容
4. **标量量化已经足够好**：配合 DCT 和熵编码，效果不比 VQ 差

**VQ 在语音编码（如 CELP）中更常见，因为语音信号的统计特性更稳定。**

## 5. 率失真优化量化（RDOQ）

### 5.1 简单量化的问题

传统量化使用四舍五入：

```python
def simple_quantize(coeff, delta):
    return round(coeff / delta)
```

但这忽略了一个问题：**量化后的值还需要熵编码**，不同的值编码代价不同！

例如：
- 量化结果为 0 → 可能只需要 1 比特（run-level 编码）
- 量化结果为 1 → 可能需要 5-10 比特

### 5.2 RDOQ 的思想

**率失真优化量化**考虑编码代价：

$$\min_{\hat{x}} \left[ D(\hat{x}) + \lambda \cdot R(\hat{x}) \right]$$

其中：
- $D$：失真（量化导致的误差）
- $R$：码率（编码这个量化值需要的比特数）
- $\lambda$：拉格朗日乘子

### 5.3 RDOQ 示例

假设某个 DCT 系数 $x = 7.4$，量化步长 $\Delta = 8$：

| 量化选择 | 量化值 | 反量化 | 失真 D | 估计码率 R | 总代价 J |
|---------|-------|-------|--------|-----------|---------|
| 四舍五入 | 1 | 8 | 0.36 | 5 bits | 0.36 + 0.5×5 = 2.86 |
| 向下取整 | 0 | 0 | 54.76 | 0.5 bits | 54.76 + 0.5×0.5 = 55.01 |

**如果 $\lambda$ 很大**（高压缩模式）：
| 量化选择 | 失真 D | 码率 R | 总代价 J (λ=10) |
|---------|--------|--------|----------------|
| 量化为 1 | 0.36 | 5 | 0.36 + 50 = 50.36 |
| 量化为 0 | 54.76 | 0.5 | 54.76 + 5 = 59.76 |

即使失真增加很多，高 $\lambda$ 下量化为 0 也可能更优！

### 5.4 软判决 vs 硬判决

| 类型 | 方法 | 特点 |
|------|------|------|
| **硬判决** | 直接四舍五入 | 简单、快速 |
| **软判决（RDOQ）** | 考虑 R-D 代价选择 | 更优的压缩效率 |

**H.265 参考软件 HM 默认启用 RDOQ**，可以提高约 3-5% 的编码效率。

## 6. 感知量化

### 6.1 人眼视觉特性

人眼对不同空间频率的敏感度不同，用**对比敏感度函数（CSF）**描述：

```
      敏感度
        ↑
        │      ╭──╮
        │    ╱    ╲
        │  ╱        ╲
        │╱            ╲
        └──────────────────→ 空间频率
          低频  中频  高频
          
人眼对中等频率最敏感，对极低和极高频率不敏感
```

### 6.2 感知量化的应用

根据 CSF，可以对人眼不敏感的频率分量施加更强的量化：

```
标准量化矩阵：    感知优化量化矩阵：
[1 1 1 1]        [1 1 2 3]
[1 1 1 1]   →    [1 1 2 4]
[1 1 1 1]        [2 2 3 5]
[1 1 1 1]        [3 4 5 6]
                  对高频更粗
```

### 6.3 AQ（自适应量化）

现代编码器（如 x264、x265）使用**自适应量化**：

- **场景复杂区域**：降低 QP（更精细），因为细节丢失明显
- **平坦区域**：提高 QP（更粗糙），因为误差容易被感知

```
┌──────────────────────────────────┐
│   平坦区域    │  复杂纹理区域    │
│   QP +2       │    QP -2         │
│   (可以粗糙)  │  (需要保留细节)  │
└──────────────────────────────────┘
```

## 7. 码率控制与 QP

### 7.1 码率控制模式

| 模式 | 全称 | QP 行为 | 适用场景 |
|------|------|---------|---------|
| **CQP** | Constant QP | 固定不变 | 质量优先、测试 |
| **CRF** | Constant Rate Factor | 动态调整保持感知质量 | 通用场景 |
| **ABR** | Average Bitrate | 动态调整达到目标平均码率 | 流媒体 |
| **CBR** | Constant Bitrate | 严格保持恒定码率 | 直播、广播 |
| **2-pass** | 两遍编码 | 第一遍分析，第二遍优化分配 | 离线高质量编码 |

### 7.2 CRF 的工作原理

CRF 模式下，编码器根据画面复杂度动态调整 QP：

```
简单场景（静态画面）：
  → 很容易编码，用较高 QP，节省码率

复杂场景（快速运动、细节丰富）：
  → 难以编码，用较低 QP，保证质量

结果：感知质量近似恒定，码率随复杂度变化
```

**CRF 值推荐**：
- x264: CRF 18-23（18 近乎无损，23 较高压缩）
- x265: CRF 22-28（因为 HEVC 效率更高）

### 7.3 QP vs 质量曲线

典型的 QP-PSNR 曲线：

```
PSNR(dB)
   ↑
50 ┼────╮
   │     ╲
45 ┼      ╲
   │       ╲
40 ┼        ╲
   │         ╲
35 ┼          ╲╲
   │            ╲╲╲
30 ┼──────────────╲─────
   └──┬──┬──┬──┬──┬──→ QP
      10 20 30 40 50

QP 从 20 增加到 30：PSNR 下降约 3-5 dB
QP 从 30 增加到 40：PSNR 下降约 5-8 dB
```

### 7.4 码率预测模型

编码器使用**码率模型**预测给定 QP 会产生多少比特：

$$R = \alpha \cdot \frac{\sigma^2}{Q^2}$$

其中：
- $R$：预测码率
- $\sigma^2$：源信号方差
- $Q$：量化步长
- $\alpha$：模型参数（需要实时更新）

通过码率模型，编码器可以选择合适的 QP 来达到目标码率。
