# 环路滤波（In-Loop Filtering）

> 环路滤波是编码器的"美颜相机"——在保持压缩效率的同时，平滑块边界、减少伪影，让重建图像更加自然。

---

## 核心结论（TL;DR）

**环路滤波在编码环路内执行，滤波后的图像作为后续帧的参考。这与单纯的后处理不同——环路滤波的效果会"传递"给后续帧，从而在整个视频序列中持续改善质量并提高预测效率。**

关键要点：
1. 块效应源于独立编码的相邻块在边界处的不连续
2. 去块滤波（Deblocking）平滑块边界，是所有标准的标配
3. SAO（H.265）通过分类+偏移补偿系统性误差
4. ALF/CDEF/LRF 是更高级的自适应滤波工具

## 1. 为什么需要环路滤波

### 1.1 块效应的成因

视频编码将图像划分为块（4×4, 8×8, 16×16 等），每个块独立进行变换、量化。

**问题**：相邻块的量化误差不一致，在块边界处产生可见的不连续——这就是**块效应（Block Artifact）**。

```
┌─────────┬─────────┐
│         │         │
│  块 A   │  块 B   │
│  亮度   │  亮度   │
│  较暗   │  较亮   │
│         │         │
├─────────┼─────────┤  ← 块边界处出现"台阶"
│         │         │
│         │         │
└─────────┴─────────┘
```

**块效应的类型**：

| 类型 | 描述 | 原因 |
|------|------|------|
| **阶梯效应** | 块边界处有明显的亮度跳变 | 量化误差不一致 |
| **网格效应** | 整个画面呈现网格状 | 所有块边界都有轻微不连续 |
| **色度块效应** | 颜色边界不自然 | 色度分量同样有块边界问题 |

### 1.2 一个直观的例子

假设两个相邻块的真实边界像素应该是：
```
块 A 边界: 120 121 122 123
块 B 边界: 124 125 126 127
```

量化后可能变成：
```
块 A 重建: 118 119 120 121  (有-2的误差)
块 B 重建: 127 128 129 130  (有+3的误差)
                ↑
           边界处跳变了 6 个亮度级别！
```

**环路滤波的目标**：平滑这个不自然的跳变。

### 1.3 振铃效应（Ringing Artifact）

除了块效应，还有一种常见的伪影是**振铃效应**：

```
原始边缘：
   低亮度区域  │  高亮度区域
   ░░░░░░░░░░░│███████████

重建后可能出现振铃：
   ░░░░░░█░░░░│░███████████
          ↑       ↑
       过冲      过冲
```

振铃是由于 DCT 变换在处理锐利边缘时高频系数被量化丢失导致的。

## 2. "环路内"的含义

### 2.1 环路滤波 vs 后处理滤波

```
编码环路：

┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   输入 → 预测 → 残差 → 变换 → 量化 → 熵编码 → 码流          │
│            ↑                   │                            │
│            │      ┌────────────┘                            │
│            │      ↓                                         │
│            │   反量化 → 反变换 → + 预测 → 重建              │
│            │                              │                 │
│            │                              ↓                 │
│            │                      ┌──────────────┐          │
│            │                      │  环路滤波    │ ← 在环路内 │
│            │                      └──────┬───────┘          │
│            │                             │                  │
│            └─────── 参考帧缓存 ←─────────┘                   │
│                                                             │
└─────────────────────────────────────────────────────────────┘

后处理滤波：

   码流 → 解码 → 重建 → 显示
                    ↓
              后处理滤波 → 显示
              （只影响显示，不影响后续帧的预测）
```

**关键区别**：

| 特性 | 环路滤波 | 后处理滤波 |
|------|---------|-----------|
| 作用位置 | 编码环路内 | 解码器输出后 |
| 影响参考帧 | 是 | 否 |
| 编解码器一致性 | 必须一致 | 可以不同 |
| 对后续帧的影响 | 改善预测效率 | 无 |

### 2.2 为什么要在环路内

**滤波后的图像作为参考帧**意味着：

1. **预测更准确**：后续帧参考的是更干净的图像，预测残差更小
2. **误差不累积**：滤波阻止了块效应随帧间预测传播和放大
3. **一致性**：编码器和解码器使用相同的参考帧

```
不使用环路滤波：
帧 0 (有块效应) → 预测帧 1 → 帧 1 (块效应累积) → 预测帧 2 → ...

使用环路滤波：
帧 0 → 滤波 → 干净的参考帧 → 预测帧 1 → 滤波 → 干净的参考帧 → ...
```

## 3. 去块滤波（Deblocking Filter）

### 3.1 基本原理

去块滤波检测块边界上的不连续，并根据一定的规则进行平滑。

**但不能无脑平滑**——真实的边缘也不能被滤掉！

```
需要区分：

真实边缘（不应该滤波）：        块效应（应该滤波）：
   ░░░░░░░│███████████         ░░░░░░░│█░░░░░░░░
   物体的边界                   同一物体内的人为边界
```

### 3.2 H.264 去块滤波

H.264 的去块滤波器是**自适应**的，根据以下因素决定滤波强度：

#### 边界强度（Boundary Strength, Bs）

| 条件 | Bs 值 | 滤波强度 |
|------|-------|---------|
| 帧内块边界 | 4 | 最强 |
| 参考帧不同/运动矢量差大 | 2 | 强 |
| 有非零残差系数 | 1 | 中等 |
| 其他情况 | 0 | 不滤波 |

#### Alpha/Beta 阈值判断

即使 Bs > 0，还需要检查边界像素的实际跳变是否超过阈值：

```
边界两侧的像素：
p3 p2 p1 p0 | q0 q1 q2 q3

判断条件：
1. |p0 - q0| < α（QP相关的阈值）
2. |p1 - p0| < β
3. |q1 - q0| < β

只有满足条件，才执行滤波（否则可能是真实边缘）
```

**滤波操作**（Bs = 1-3 时的弱滤波）：

```python
# 简化的去块滤波伪代码
def deblock_filter_4x4(p0, p1, p2, q0, q1, q2, bs, alpha, beta):
    if bs == 0:
        return  # 不滤波
    
    if abs(p0 - q0) >= alpha:
        return  # 超过阈值，可能是真实边缘
    if abs(p1 - p0) >= beta or abs(q1 - q0) >= beta:
        return
    
    # 计算滤波偏移量
    delta = clip(-tc, tc, (4*(q0 - p0) + (p1 - q1) + 4) >> 3)
    
    # 应用滤波
    p0_new = clip(0, 255, p0 + delta)
    q0_new = clip(0, 255, q0 - delta)
```

### 3.3 H.265 去块滤波的改进

H.265 简化了判断流程，并提高了并行友好性：

1. **边界强度简化**：只有 0, 1, 2 三个级别
2. **8×8 网格处理**：在 8×8 而非 4×4 边界上操作
3. **长滤波器**：对某些情况使用更长的滤波器（跨更多像素）

```
H.264：每 4 个像素一个边界判断
H.265：每 8 个像素一个边界判断（减少决策开销）
```

### 3.4 滤波顺序

去块滤波按特定顺序执行，确保不同块的边界都被处理：

```
处理顺序：
1. 先处理所有垂直边界（从左到右）
2. 再处理所有水平边界（从上到下）

┌───┬───┬───┐
│ → │ → │ → │  先垂直
├─↓─┼─↓─┼─↓─┤
│ → │ → │ → │
├─↓─┼─↓─┼─↓─┤  再水平
│   │   │   │
└───┴───┴───┘
```

## 4. SAO（样本自适应偏移）

### 4.1 概述

**SAO（Sample Adaptive Offset）**是 H.265 引入的新工具，作用于去块滤波之后。

**核心思想**：对像素进行分类，然后对每类像素加一个偏移值来补偿系统性误差。

```
去块滤波后：残留的误差往往具有规律性
SAO 的目标：识别这些规律，用简单的偏移来补偿
```

### 4.2 两种 SAO 类型

#### 边界偏移（Edge Offset, EO）

根据像素与相邻像素的关系分类：

```
方向模式（4 种）：
0: 水平 (─)    比较 p 与左右邻居
1: 垂直 (│)    比较 p 与上下邻居
2: 45° (╱)    比较 p 与对角邻居
3: 135° (╲)   比较 p 与另一对角邻居

分类（5 类）：
类别 1: 局部最小值（比两边都小）
类别 2: 边缘 -（比一边小，等于另一边）
类别 3: 边缘 +（比一边大，等于另一边）
类别 4: 局部最大值（比两边都大）
类别 0: 其他
```

对于振铃效应，边缘附近的过冲像素会被归为局部极值，然后施加负偏移来修正：

```
原始：    ░░░░░░█░░░│░███████████
                ↑       ↑
             局部极大  局部极小
             
SAO 后：   ░░░░░░░░░░│███████████
             -偏移      +偏移
```

#### 带偏移（Band Offset, BO）

将像素值范围分成 32 个"带"，选择连续的 4 个带，对其中的像素施加偏移：

```
像素值范围 [0, 255] 分成 32 带，每带 8 个值

带编号:  0   1   2   3   4   5  ... 31
像素值: [0-7][8-15][16-23]...    [248-255]

选择连续 4 带（如 12-15），对其中像素加偏移
```

**适用场景**：平坦区域中因量化产生的系统性偏移。

### 4.3 SAO 参数传输

SAO 参数按 CTU（编码树单元）传输：

```
每个 CTU 的 SAO 参数：
- sao_type（0=关闭, 1=EO, 2=BO）
- 方向/起始带
- 4 个偏移值

CTU 级别的 merge 模式：
- 可以从左侧或上方 CTU 复制 SAO 参数
- 减少参数传输开销
```

### 4.4 SAO 的效果

| 指标 | 改善幅度 | 说明 |
|------|---------|------|
| BD-Rate | -2% ~ -3% | 相同质量下码率降低 |
| PSNR | +0.1 ~ 0.3 dB | 相同码率下质量提升 |
| 主观质量 | 明显改善 | 振铃效应减少 |

## 5. ALF（自适应环路滤波）

### 5.1 概述

**ALF（Adaptive Loop Filter）**是 H.266/VVC 中使用的高级环路滤波工具。

**核心思想**：使用维纳滤波器（Wiener Filter）来最小化重建图像与原始图像之间的均方误差。

### 5.2 维纳滤波原理

**目标**：找到最优的滤波器系数 $h$，使得：

$$\min_h E[(x - \hat{x} * h)^2]$$

其中 $x$ 是原始图像，$\hat{x}$ 是重建图像。

**解**（正规方程）：

$$h = R^{-1} p$$

其中 $R$ 是重建图像的自相关矩阵，$p$ 是原始与重建图像的互相关向量。

### 5.3 ALF 的实现

#### 分类

VVC 的 ALF 对像素进行分类，每类使用不同的滤波器：

```
分类依据：
1. 局部方向（梯度方向）
2. 局部活动度（梯度强度）

分成 25 类（5 个方向 × 5 个活动度级别）
```

#### 滤波器形状

```
7×7 菱形滤波器（亮度）：

      *
    * * *
  * * * * *
* * * C * * *
  * * * * *
    * * *
      *

5×5 菱形滤波器（色度）：

    *
  * * *
* * C * *
  * * *
    *
```

#### 参数传输

- 每帧传输一组滤波器系数（最多 25 组）
- CTU 级别可以选择启用/禁用 ALF
- 支持时域层间共享滤波器参数

### 5.4 ALF 的效果

ALF 可以带来约 **4-6%** 的码率节省，但计算复杂度较高。

## 6. CDEF 和 LRF

### 6.1 CDEF（约束方向增强滤波）

**CDEF（Constrained Directional Enhancement Filter）**是 AV1 使用的环路滤波工具。

**核心思想**：
1. 检测局部的主方向
2. 沿主方向进行约束滤波（保护边缘）

```
方向检测（8 个方向）：

  3   2   1
   \  |  /
    \ | /
  4 ──+── 0
    / | \
   /  |  \
  5   6   7
```

**约束滤波**：只对与中心像素差值较小的像素进行加权平均，避免模糊边缘。

```python
# CDEF 伪代码
def cdef_filter(center, neighbors, direction, strength, damping):
    result = center << damping  # 初始化
    
    for n in neighbors_in_direction(direction):
        diff = n - center
        if abs(diff) < threshold:
            # 在阈值内，参与滤波
            result += constrain(diff, strength, damping)
    
    return (result + (1 << (damping-1))) >> damping
```

### 6.2 LRF（环路恢复滤波）

**LRF（Loop Restoration Filter）**是 AV1 的另一个环路滤波工具，包含两种模式：

#### Wiener 滤波

类似于 VVC 的 ALF，使用可分离的维纳滤波器：

```
7-tap 水平 + 7-tap 垂直 = 可分离的滤波
比直接的 7×7 滤波计算量小得多
```

#### Self-Guided 滤波

基于引导滤波（Guided Filter）的简化版本：

1. 计算局部均值和方差
2. 根据方差决定平滑程度
3. 在平坦区域强平滑，在边缘处弱平滑

```
输出 = α × 重建图像 + (1-α) × 平滑图像

α 由局部方差决定：
- 方差大（边缘）：α ≈ 1，保留原值
- 方差小（平坦）：α ≈ 0，使用平滑值
```

### 6.3 AV1 环路滤波流水线

```
重建图像
    │
    ▼
┌─────────────┐
│ Deblocking  │  去块滤波
└──────┬──────┘
       │
       ▼
┌─────────────┐
│    CDEF     │  方向增强滤波
└──────┬──────┘
       │
       ▼
┌─────────────┐
│     LRF     │  环路恢复滤波
└──────┬──────┘
       │
       ▼
   滤波后图像（作为参考帧）
```

## 7. 各标准对比

### 7.1 环路滤波工具演进

| 标准 | 去块滤波 | 其他环路滤波 |
|------|---------|-------------|
| H.264/AVC | ✓（强制） | 无 |
| H.265/HEVC | ✓（改进） | SAO |
| H.266/VVC | ✓（改进） | SAO + ALF |
| VP9 | ✓ | 无 |
| AV1 | ✓ | CDEF + LRF |

### 7.2 效率与复杂度对比

| 工具 | BD-Rate 增益 | 编码复杂度 | 解码复杂度 |
|------|-------------|-----------|-----------|
| Deblocking | 基准 | 低 | 低 |
| SAO | -2~3% | 中 | 低 |
| ALF | -4~6% | 高 | 中 |
| CDEF | -2~3% | 中 | 低 |
| LRF | -3~5% | 中 | 中 |

### 7.3 并行化考虑

| 工具 | 并行友好性 | 说明 |
|------|-----------|------|
| Deblocking | 中等 | 边界依赖，需要特殊顺序 |
| SAO | 高 | CTU 独立处理 |
| ALF | 高 | 块独立 + 滤波器共享 |
| CDEF | 高 | 超块（Superblock）独立 |
