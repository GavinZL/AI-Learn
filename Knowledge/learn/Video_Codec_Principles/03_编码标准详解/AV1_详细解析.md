# AV1 详细解析

> AV1 是由 Alliance for Open Media（AOM）开发的开放、免版税视频编码标准，结合了多家科技巨头的技术积累，旨在提供比 H.265/HEVC 更高的压缩效率，同时消除专利授权障碍。

---

## 核心结论（TL;DR）

**AV1 通过 128×128 超级块、多类型变换、三级环路滤波（Deblock + CDEF + LRF）以及复合预测等创新技术，相比 H.265 实现约 30%+ 的码率节省，且完全免版税，已成为流媒体和 WebRTC 领域的重要选择。**

关键要点：
1. 128×128 超级块配合灵活的二叉树 + 四叉树分割，比 H.265 更细致地适应图像内容
2. 三级环路滤波（Deblock → CDEF → LRF）显著提升重建图像质量
3. Film Grain Synthesis 为影视内容提供高效的胶片颗粒保留方案
4. 完全免版税，降低商业应用门槛

## 1. AV1 背景

### 1.1 Alliance for Open Media（AOM）

**AOM 成立背景**（2015年）：

| 问题 | 影响 |
|------|------|
| H.265 专利池复杂 | MPEG LA、HEVC Advance、Velos Media 三个专利池 |
| 授权费用不明确 | 商业风险大，阻碍采用 |
| 互联网需求 | 需要免费、高效的视频编码标准 |

**AOM 创始成员**：
- Google（VP9 技术贡献）
- Mozilla（Daala 技术贡献）
- Cisco（Thor 技术贡献）
- Microsoft、Amazon、Netflix、Apple（后加入）等

### 1.2 技术继承关系

```
Google VP9          Mozilla Daala       Cisco Thor
    ↘                   ↓                  ↙
     ↘                  ↓                 ↙
      ↘                 ↓                ↙
       └─────────── AV1 ────────────────┘
                   (2018)
                     
技术融合：
- VP9：超级块、Tile、参考帧管理
- Daala：PVQ、Lapped Transform 概念
- Thor：多类型变换、滤波技术
```

### 1.3 与 H.265 的竞争定位

| 维度 | H.265/HEVC | AV1 |
|------|------------|-----|
| 组织 | ITU-T + ISO | AOM 联盟 |
| 版税 | 需要授权 | 完全免费 |
| 发布时间 | 2013 | 2018 |
| 压缩效率 | 基准 | +30%（约） |
| 编码复杂度 | 基准 | 约 10-100× |
| 硬件支持 | 成熟 | 快速发展中 |

## 2. 超级块结构

### 2.1 128×128 超级块

AV1 将最大编码单元扩展到 **128×128**（H.265 为 64×64）：

```
块大小演进：
H.264:  16×16 宏块
         ↓
H.265:  64×64 CTU
         ↓
AV1:   128×128 Superblock

更大块的优势：
- 4K/8K 高分辨率内容的大面积平坦区域
- 减少模式选择和边界的开销
```

### 2.2 灵活的分区类型

AV1 支持 **10 种分区类型**，比 H.265 的纯四叉树更灵活：

```
1. NONE          2. HORZ           3. VERT           4. SPLIT
┌────────────┐   ┌────────────┐   ┌──────┬──────┐   ┌─────┬─────┐
│            │   ├────────────┤   │      │      │   ├─────┼─────┤
│            │   │            │   │      │      │   │     │     │
└────────────┘   └────────────┘   └──────┴──────┘   └─────┴─────┘

5. HORZ_A        6. HORZ_B         7. VERT_A         8. VERT_B
┌─────┬─────┐   ┌────────────┐   ┌──────┬──────┐   ┌──────┬──────┐
│     │     │   │            │   │      │      │   │      │      │
├─────┴─────┤   ├──────┬─────┤   │      ├──────┤   ├──────┤      │
│           │   │      │     │   │      │      │   │      │      │
└───────────┘   └──────┴─────┘   └──────┴──────┘   └──────┴──────┘

9. HORZ_4       10. VERT_4
┌────────────┐   ┌───┬───┬───┬───┐
├────────────┤   │   │   │   │   │
├────────────┤   │   │   │   │   │
├────────────┤   │   │   │   │   │
└────────────┘   └───┴───┴───┴───┘
```

**二叉树 + 四叉树混合分割**：

- **四叉树**（SPLIT）：等分为 4 个子块
- **二叉树**（HORZ/VERT）：等分为 2 个子块
- **非对称**（HORZ_A/B, VERT_A/B）：1:3 或 3:1 分割
- **四分**（HORZ_4/VERT_4）：等分为 4 个条状子块

### 2.3 块大小范围

AV1 支持的块大小：

| 最小块 | 最大块 | 递归深度 |
|--------|--------|----------|
| 4×4 | 128×128 | 最多 6 级 |

支持的块尺寸包括：
`128×128, 128×64, 64×128, 64×64, 64×32, 32×64, 64×16, 16×64, 32×32, 32×16, 16×32, 32×8, 8×32, 16×16, 16×8, 8×16, 16×4, 4×16, 8×8, 8×4, 4×8, 4×4`

## 3. 帧内预测

### 3.1 方向模式扩展

| 标准 | 方向模式数 | 非方向模式 |
|------|-----------|-----------|
| H.264 | 8 | DC |
| H.265 | 33 | Planar, DC |
| AV1 | 56 | Paeth, Smooth 系列, DC |

**56 种方向模式的角度分布**：
- 标称角度：8 种（与 VP9 类似）
- 角度微调：每个标称角度 ±1°、±2°、±3°

```
标称角度（度）：
45, 67, 90, 113, 135, 157, 180, 203

角度微调示例（以 90° 为例）：
87°, 88°, 89°, 90°, 91°, 92°, 93°
      │    │    │    │    │    │
   -3°  -2°  -1°  基准  +1°  +2°  +3°
```

### 3.2 Paeth 预测

Paeth 是 AV1 特有的帧内预测模式：

```
参考像素布局：
     TL   T
      ●───●
      │
   L  ●───●  当前像素 P

Paeth 预测算法：
1. base = L + T - TL
2. pL = |base - L|, pT = |base - T|, pTL = |base - TL|
3. pred = argmin(pL: L, pT: T, pTL: TL)

直觉：选择与线性预测值最接近的参考像素
```

**Paeth 的优势**：
- 适合边缘和角落区域
- 比纯方向预测更鲁棒
- 计算简单，无需乘法

### 3.3 Smooth 预测系列

AV1 提供 4 种 Smooth 模式：

| 模式 | 描述 |
|------|------|
| SMOOTH | 双向平滑插值（上 + 左） |
| SMOOTH_V | 垂直平滑（仅上方参考） |
| SMOOTH_H | 水平平滑（仅左侧参考） |
| DC | 平均值预测 |

**SMOOTH 模式的权重**：

$$pred[x][y] = \frac{w_{vert}[y] \cdot T[x] + w_{horz}[x] \cdot L[y] + (1-w_{vert}[y]-w_{horz}[x]) \cdot avg}{256}$$

其中权重随距离平滑衰减。

### 3.4 色度从亮度预测（CfL）

CfL（Chroma from Luma）利用亮度信息预测色度：

```
色度预测 = α × AC_luma + DC_chroma

其中：
- AC_luma：下采样后的亮度 AC 分量
- DC_chroma：色度 DC 预测值
- α：传输的缩放系数（-16 到 +16，步长 1/8）
```

**CfL 的优势**：
- 利用亮度色度相关性
- 对于边缘和纹理区域效果显著
- 码率开销小（仅传输 α 系数）

## 4. 帧间预测

### 4.1 参考帧管理

AV1 支持复杂的参考帧结构：

**参考帧类型**：

| 类型 | 数量 | 说明 |
|------|------|------|
| LAST | 1 | 最近的前向帧 |
| LAST2 | 1 | 第二近的前向帧 |
| LAST3 | 1 | 第三近的前向帧 |
| GOLDEN | 1 | 长期前向参考 |
| BWDREF | 1 | 最近的后向帧 |
| ALTREF2 | 1 | 第二后向参考 |
| ALTREF | 1 | 长期后向参考 |

最多可使用 **7 个参考帧**。

### 4.2 复合预测（Compound Prediction）

AV1 支持多种复合预测模式：

**基本复合预测**：
$$pred = \frac{pred_1 + pred_2 + 1}{2}$$

**加权复合预测**：
- **Distance-based**：根据 POC 距离加权
- **Wedge**：使用楔形蒙版混合
- **Difference-based**：根据预测差异自适应混合

**Wedge Compound**：

```
楔形蒙版示例：
┌─────────────────┐
│████████████░░░░│  蒙版值：
│████████████░░░░│  ████ = 使用 pred_1
│████████░░░░░░░░│  ░░░░ = 使用 pred_2
│████░░░░░░░░░░░░│
│░░░░░░░░░░░░░░░░│
└─────────────────┘
```

16 种预定义的楔形方向，适合物体边界区域。

### 4.3 全局运动补偿

AV1 支持帧级的全局运动模型：

| 模型类型 | 参数数 | 描述 |
|---------|--------|------|
| IDENTITY | 0 | 无全局运动 |
| TRANSLATION | 2 | 平移 |
| ROTZOOM | 4 | 旋转 + 缩放 |
| AFFINE | 6 | 仿射变换 |

**仿射变换**：

$$\begin{bmatrix} x' \\ y' \end{bmatrix} = \begin{bmatrix} a & b \\ c & d \end{bmatrix} \begin{bmatrix} x \\ y \end{bmatrix} + \begin{bmatrix} e \\ f \end{bmatrix}$$

适用场景：摄像机平移、旋转、缩放等。

### 4.4 翘曲运动（Warped Motion）

块级的局部运动模型，比全局运动更精细：

```
使用邻近块的 MV 拟合局部仿射变换：

    MV_1    MV_2    MV_3
      ↘      ↓      ↙
       ┌─────────┐
       │ 当前块  │ ← 拟合得到局部仿射参数
       └─────────┘
    MV_4           MV_5
```

### 4.5 OBMC（重叠块运动补偿）

减少块边界不连续性：

```
传统预测：
┌─────┬─────┐
│ A   │ B   │  边界处可能不连续
└─────┴─────┘

OBMC 预测：
┌─────┬─────┐
│ A←→ │ ←→B │  边界处混合邻居预测
└─────┴─────┘

pred_final = w1 × pred_self + w2 × pred_neighbor
```

## 5. 变换编码

### 5.1 多类型变换

AV1 支持 **4 种基本变换类型**及其组合：

| 变换类型 | 缩写 | 特性 |
|---------|------|------|
| DCT | DCT | 标准 DCT |
| ADST | ADST | 非对称 DST |
| Flip ADST | FLIPADST | 翻转的 ADST |
| Identity | IDTX | 恒等变换（跳过） |

**水平 × 垂直组合**（16 种）：

```
         水平变换
         DCT   ADST   FLIPADST   IDTX
垂直    ┌──────────────────────────────┐
变换    │ DCT_DCT ADST_DCT FLIPADST_DCT ...│
DCT     │ DCT_ADST  ...                    │
ADST    │ ...                              │
FLIP    │                                  │
IDTX    │                                  │
        └──────────────────────────────────┘
```

**变换类型选择**：
- 编码器通过 RDO 选择最佳变换类型
- 变换类型信息编码在码流中

### 5.2 变换大小

支持的变换块大小：

| 正方形 | 矩形 |
|--------|------|
| 4×4, 8×8, 16×16, 32×32, 64×64 | 4×8, 8×4, 8×16, 16×8, 16×32, 32×16 等 |

**64×64 变换的特殊处理**：
- 仅保留低频 32×32 区域的系数
- 高频系数强制置零

### 5.3 递归变换分割

类似 H.265 的 RQT，但更灵活：

```
变换块可独立于编码块进行分割：

CU: 32×32
    ┌─────────────────┐
    │                 │     可能的 TU 分割：
    │                 │  →  ┌────────┬────────┐
    │                 │     ├────────┼────────┤
    │                 │     │        │        │
    └─────────────────┘     └────────┴────────┘
                            4 个 16×16 TU
```

## 6. 环路滤波工具链

AV1 采用三级环路滤波流水线：

```
重建图像 → [去块滤波] → [CDEF] → [LRF] → 最终输出
              ↓            ↓         ↓
           边界平滑    方向滤波   自适应恢复
```

### 6.1 去块滤波

与 H.265 类似，但有优化：
- 支持更大的滤波范围
- 针对大块优化判断条件

### 6.2 CDEF（约束方向增强滤波）

**CDEF 核心思想**：
1. 检测局部纹理方向
2. 沿方向进行约束滤波

**方向检测**：

```
8 个方向（0°, 22.5°, 45°, ..., 157.5°）：

↑ ↗ → ↘ ↓ ↙ ← ↖

对于每个方向，计算像素差异的方差
选择方差最小的方向作为纹理方向
```

**约束滤波**：

$$f(d) = \begin{cases} d & |d| \leq threshold_1 \\ clamp(d, -threshold_2, threshold_2) & |d| \leq threshold_2 \\ 0 & otherwise \end{cases}$$

**滤波效果**：
- 沿纹理方向平滑
- 保持垂直于纹理的边缘
- 减少振铃效应

### 6.3 LRF（环路恢复滤波）

**两种恢复滤波器**：

#### Wiener 滤波器

```
使用分离的 7×7 Wiener 滤波：
- 水平 7 抽头 + 垂直 7 抽头
- 系数由解码器优化并编码传输

滤波公式（简化）：
y = Σ(w_i × x_i)，其中 w_i 是 Wiener 系数
```

#### Self-Guided Restoration

```
自引导滤波步骤：
1. 计算局部均值和方差
2. 生成引导图像
3. 加权混合原始和引导图像

参数：两个强度参数 (eps1, eps2)，由解码器优化
```

**LRF 区域划分**：
- 帧被划分为恢复单元（Restoration Unit）
- 每个单元独立选择：Wiener / Self-Guided / Off

## 7. Film Grain Synthesis

### 7.1 设计动机

**问题**：胶片颗粒/噪声难以压缩，消耗大量码率

**AV1 方案**：
1. 编码前：分析并去除胶片颗粒
2. 编码：压缩去噪后的图像（高效）
3. 解码后：根据参数重新合成颗粒

```
原始视频 → [去颗粒] → [AV1 编码] → [解码] → [重加颗粒] → 输出
              ↓                               ↑
              └─── 颗粒参数（少量数据）────────┘
```

### 7.2 颗粒模型参数

传输的参数包括：
- 颗粒强度曲线（分段线性）
- 自相关系数（定义颗粒的空间特性）
- 色度噪声缩放
- 随机种子

**参数示例**：

```
亮度噪声强度曲线（8 段）：
像素值：  0-32  32-64  64-96  ...  224-255
强度：     10     12     15    ...    8

色度缩放：
cb_mult = 128, cb_luma_mult = 64
cr_mult = 128, cr_luma_mult = 64
```

### 7.3 典型效果

| 场景 | 无 Film Grain | 有 Film Grain |
|------|--------------|---------------|
| 码率 | 基准 | 节省 20-40% |
| 主观质量 | 平滑（可能显得"塑料感"） | 保持原始质感 |

## 8. 熵编码

### 8.1 多符号算术编码

AV1 使用来自 Daala 的多符号算术编码器：

**与 CABAC 的区别**：

| 特性 | H.264/H.265 CABAC | AV1 daala_ec |
|------|-------------------|--------------|
| 符号处理 | 逐二进制符号 | 多符号一次编码 |
| 概率更新 | 逐符号更新 | 基于频率表 |
| 并行性 | 差 | 较好 |

### 8.2 CDF（累积分布函数）更新

AV1 使用自适应 CDF 进行概率估计：

```
概率更新示例：

初始 CDF：[0, 64, 128, 192, 256]（4 个符号均匀分布）
           ↓
观察符号 2
           ↓
更新后：  [0, 60, 136, 196, 256]（符号 2 概率增加）
```

## 10. 性能对比与编码器选择

### 10.1 BD-Rate 对比

以 H.265 HM 参考软件为基准：

| 编码器 | BD-Rate（vs HEVC） | 编码速度 |
|--------|-------------------|---------|
| AV1 libaom | -30% ~ -35% | 非常慢 |
| AV1 SVT-AV1 | -25% ~ -30% | 快 |
| AV1 rav1e | -25% ~ -30% | 中等 |

### 10.2 三大 AV1 编码器对比

| 特性 | libaom | SVT-AV1 | rav1e |
|------|--------|---------|-------|
| 开发者 | AOM/Google | Intel/Netflix | Xiph/Mozilla |
| 语言 | C | C | Rust |
| 目标 | 参考实现 | 生产效率 | 安全性 |
| 编码速度 | 最慢 | 最快 | 中等 |
| 压缩效率 | 最高 | 高 | 高 |
| 适用场景 | 离线编码 | 实时/大规模 | 研究/安全 |

### 10.3 编码参数示例

**libaom**：

```bash
# 高质量编码
aomenc --cpu-used=4 --end-usage=q --cq-level=24 \
       --threads=8 --tile-columns=2 --tile-rows=1 \
       --enable-fwd-kf=1 --kf-max-dist=240 \
       -o output.webm input.y4m

# 流媒体编码
aomenc --cpu-used=6 --end-usage=vbr --target-bitrate=2000 \
       --threads=8 --row-mt=1 \
       --lag-in-frames=25 \
       -o output.webm input.y4m
```

**SVT-AV1**：

```bash
# 高质量编码
SvtAv1EncApp -i input.y4m -b output.ivf \
             --preset 4 --crf 24 \
             --keyint 240 --enable-tf 1 \
             --film-grain 8 --film-grain-denoise 0

# 快速编码
SvtAv1EncApp -i input.y4m -b output.ivf \
             --preset 8 --crf 28 \
             --fast-decode 1
```

**rav1e**：

```bash
# 高质量编码
rav1e input.y4m -o output.ivf \
      --speed 5 --quantizer 80 \
      --keyint 240 --threads 8
```

### 10.4 关键参数说明

| 参数 | libaom | SVT-AV1 | 说明 |
|------|--------|---------|------|
| 速度 | --cpu-used (0-9) | --preset (0-13) | 越大越快 |
| 质量 | --cq-level (0-63) | --crf (0-63) | 越小越好 |
| 码率模式 | --end-usage | --rc | q/vbr/cbr |
| 并行 | --threads, --row-mt | 自动 | 多线程 |
