# VP8/VP9 详细解析

> VP8 和 VP9 是 Google 主导开发的开源视频编码标准，VP8 开启了免费视频编码的时代，VP9 则成为 YouTube 4K 视频的核心编码格式，两者共同为 AV1 的诞生奠定了技术基础。

---

## 核心结论（TL;DR）

**VP8 作为首个开源免费的 H.264 替代方案，虽性能略逊但推动了视频编码的开放生态；VP9 则通过 64×64 超级块、更多预测模式和复合预测等技术，实现了与 H.265 相当的压缩效率，成为 YouTube 和 WebRTC 的核心技术。**

关键要点：
1. VP8 与 H.264 技术架构相似，性能基本持平，但完全开源免费
2. VP9 相比 VP8 压缩效率提升约 50%，与 H.265 同代竞争
3. VP8 → VP9 → AV1 形成清晰的技术演进路线
4. WebM 容器 + VP8/VP9 成为 Web 视频的重要组合

# VP8 部分

## 1. VP8 背景与历史意义

### 1.1 发展历程

```
时间线：

2005年：On2 Technologies 开发 VP8
    ↓
2009年：Google 收购 On2（约 1.33 亿美元）
    ↓
2010年：Google 将 VP8 开源，发布 WebM 项目
    ↓
2011年：Chrome、Firefox、Opera 原生支持 VP8
    ↓
2013年：VP8 成为 WebRTC 的强制实现编解码器
```

### 1.2 WebM 项目

**WebM 容器格式**：

| 组成部分 | 技术选择 |
|---------|---------|
| 视频编码 | VP8 或 VP9 |
| 音频编码 | Vorbis 或 Opus |
| 容器格式 | Matroska（MKV）子集 |
| 授权 | BSD 许可证 |

**WebM 的定位**：为 HTML5 `<video>` 标签提供开放免费的视频格式。

### 1.3 历史意义

VP8 的开源标志着：
- **打破专利壁垒**：首个真正免费的 H.264 级别编解码器
- **推动开放标准**：为后续的 VP9、AV1 奠定基础
- **浏览器视频革命**：加速 Flash 退出，HTML5 视频崛起

## 2. VP8 技术架构

### 2.1 宏块结构

VP8 采用与 H.264 类似的 **16×16 宏块** 结构：

```
帧 → 宏块（16×16）→ 子块

亮度子分区（Y）：
┌────────────────────┐
│ 16×16（1个子块）   │
└────────────────────┘

┌─────────┬─────────┐
│   8×8   │   8×8   │
├─────────┼─────────┤
│   8×8   │   8×8   │
└─────────┴─────────┘

┌────┬────┬────┬────┐
│4×4 │4×4 │4×4 │4×4 │
├────┼────┼────┼────┤
│4×4 │4×4 │4×4 │4×4 │
├────┼────┼────┼────┤
...（共 16 个 4×4 块）
└────┴────┴────┴────┘

色度子分区（U/V）：
每个 4:2:0 宏块包含 8×8 的 U 和 V
```

### 2.2 帧内预测

**16×16 亮度帧内预测（4 种模式）**：

| 模式编号 | 名称 | 描述 |
|---------|------|------|
| 0 | DC Pred | 上方 + 左侧像素平均值 |
| 1 | V Pred | 垂直复制上方像素 |
| 2 | H Pred | 水平复制左侧像素 |
| 3 | TM Pred | True Motion 预测 |

**True Motion（TM）预测**：

```
TM 预测公式：
pred[x][y] = L[y] + T[x] - TL

其中：
- L[y]：左侧参考像素
- T[x]：上方参考像素
- TL：左上角像素

TL ─── T[0] T[1] T[2] ...
 │
L[0]    ●    ●    ●
L[1]    ●    ●    ●
L[2]    ●    ●    ●
```

**4×4 亮度帧内预测（10 种模式）**：

| 模式 | 名称 | 方向 |
|------|------|------|
| B_DC_PRED | DC | 平均值 |
| B_TM_PRED | True Motion | 梯度 |
| B_VE_PRED | Vertical | 垂直 |
| B_HE_PRED | Horizontal | 水平 |
| B_LD_PRED | Left-Down | 左下 |
| B_RD_PRED | Right-Down | 右下 |
| B_VR_PRED | Vertical-Right | 垂直偏右 |
| B_VL_PRED | Vertical-Left | 垂直偏左 |
| B_HD_PRED | Horizontal-Down | 水平偏下 |
| B_HU_PRED | Horizontal-Up | 水平偏上 |

### 2.3 帧间预测

**参考帧模型（3 帧）**：

| 参考帧 | 描述 |
|--------|------|
| Last Frame | 最近的已解码帧 |
| Golden Frame | 长期参考帧，手动更新 |
| Alt-Ref Frame | 替代参考帧，用于双向预测 |

```
参考帧使用示例：

编码顺序：I0 → P1 → P2 → P3 → P4 → ...
                ↑
显示顺序：I0 → P1 → P2 → P3 → P4 → ...

P3 可参考：
- Last = P2（前一帧）
- Golden = I0（关键帧，较远）
- Alt-Ref = P4（未来帧，如果启用双向预测）
```

**运动矢量精度**：1/4 像素（使用 6 抽头滤波器）

**运动矢量预测**：使用左边、上边、左上邻居块的 MV 进行预测

### 2.4 变换编码

**DCT + WHT 组合**：

```
4×4 DCT：用于预测残差的变换
4×4 WHT：用于 16 个 DCT DC 系数的二次变换

处理流程：
残差块 → 4×4 DCT → DC 系数提取 → 4×4 WHT
                 → AC 系数量化
```

**整数变换核**（与 H.264 类似）：

$$H_{DCT} = \begin{bmatrix} 1 & 1 & 1 & 1 \\ 2 & 1 & -1 & -2 \\ 1 & -1 & -1 & 1 \\ 1 & -2 & 2 & -1 \end{bmatrix}$$

### 2.5 熵编码

VP8 使用 **布尔算术编码器（Bool Coder）**：

```
核心思想：
- 每个符号编码为单个比特（0 或 1）
- 每个比特有对应的概率 p
- 使用算术编码压缩

概率更新：
- 帧头中传输概率更新
- 概率表在帧内保持固定
```

### 2.6 环路滤波

VP8 支持两种环路滤波模式：

**Normal Filter**：
- 根据边界强度和像素梯度决定滤波
- 最多修改边界两侧各 2 个像素

**Simple Filter**：
- 简化版本，仅用于低复杂度解码
- 仅修改边界两侧各 1 个像素

### 2.7 比特流格式

**IVF 容器**：

```
IVF 文件结构：
┌──────────────────────────────┐
│ File Header (32 bytes)       │
├──────────────────────────────┤
│ Frame Header 1               │
│ Frame Data 1                 │
├──────────────────────────────┤
│ Frame Header 2               │
│ Frame Data 2                 │
├──────────────────────────────┤
│ ...                          │
└──────────────────────────────┘
```

## 3. VP8 与 H.264 对比

### 3.1 技术对比

| 特性 | VP8 | H.264 Baseline | H.264 Main |
|------|-----|----------------|------------|
| 宏块大小 | 16×16 | 16×16 | 16×16 |
| 帧内模式（16×16） | 4 | 4 | 4 |
| 帧内模式（4×4） | 10 | 9 | 9 |
| 参考帧数 | 3 | 16 | 16 |
| B 帧 | 仅 Alt-Ref | ✗ | ✓ |
| 熵编码 | Bool Coder | CAVLC | CABAC |
| 环路滤波 | 2 种 | 1 种 | 1 种 |

### 3.2 性能对比

| 内容类型 | VP8 vs H.264 Main |
|---------|-------------------|
| 低运动 | 基本持平 |
| 高运动 | 略差 5-10% |
| 复杂纹理 | 略差 5-10% |
| 屏幕内容 | 略差 |

**总体结论**：VP8 与 H.264 Baseline 基本持平，比 H.264 Main/High 略差约 5-15%。

## 5. VP9 超级块结构

### 5.1 64×64 超级块

VP9 将基本单元从 VP8 的 16×16 宏块扩展到 **64×64 超级块**：

```
VP8:  16×16 宏块
      ↓
VP9:  64×64 超级块

超级块的优势：
- 更好地适应高分辨率内容
- 减少大面积平坦区域的开销
```

### 5.2 四叉树递归分割

VP9 支持从 64×64 到 4×4 的四叉树递归分割：

```
64×64 超级块分割示例：

┌───────────────────────────────────────────┐
│                                           │
│                  64×64                    │
│                                           │
│                                           │
└───────────────────────────────────────────┘
                    ↓ 分割
┌───────────────────┬───────────────────────┐
│                   │                       │
│      32×32        │       32×32           │
├───────────────────┼───────────────────────┤
│                   │                       │
│      32×32        │       32×32           │
└───────────────────┴───────────────────────┘
                    ↓ 继续分割
┌─────────┬─────────┬───────────────────────┐
│  16×16  │  16×16  │                       │
├─────────┼─────────┤       32×32           │
│  16×16  │  16×16  │                       │
├─────────┴─────────┼───────────────────────┤
│                   │                       │
│      32×32        │       32×32           │
└───────────────────┴───────────────────────┘
```

**支持的块大小**：64×64, 32×32, 16×16, 8×8, 4×4

### 5.3 分区模式

VP9 支持 4 种基本分区模式：

```
PARTITION_NONE     PARTITION_HORZ     PARTITION_VERT    PARTITION_SPLIT
┌────────────┐     ┌────────────┐     ┌──────┬──────┐   ┌─────┬─────┐
│            │     ├────────────┤     │      │      │   ├─────┼─────┤
│            │     │            │     │      │      │   │     │     │
└────────────┘     └────────────┘     └──────┴──────┘   └─────┴─────┘
   不分割            水平二分           垂直二分          四分割（递归）
```

## 6. VP9 预测技术

### 6.1 帧内预测改进

VP9 扩展到 **10 种帧内预测模式**：

| 模式 | 名称 | 描述 |
|------|------|------|
| DC_PRED | DC | 平均值预测 |
| V_PRED | Vertical | 垂直方向 |
| H_PRED | Horizontal | 水平方向 |
| D45_PRED | 45° | 对角方向 |
| D135_PRED | 135° | 对角方向 |
| D117_PRED | 117° | 角度方向 |
| D153_PRED | 153° | 角度方向 |
| D207_PRED | 207° | 角度方向 |
| D63_PRED | 63° | 角度方向 |
| TM_PRED | True Motion | 梯度预测 |

```
角度模式可视化：

           V(90°)
        D117 ↑ D63
           ↖ ↑ ↗
    H(180°)←   →D45
           ↙ ↓ ↘
       D153 ↓ D207
          D135
```

### 6.2 帧间预测改进

**参考帧扩展**：

| 参考帧 | VP8 | VP9 |
|--------|-----|-----|
| Last | 1 | 1 |
| Golden | 1 | 1 |
| Alt-Ref | 1 | 1 |
| **总计** | 3 | 3（但使用更灵活） |

**复合预测（Compound Prediction）**：

VP9 引入了复合预测，将两个参考帧的预测加权平均：

```
复合预测公式：
pred = (pred_ref1 + pred_ref2 + 1) >> 1

可用组合：
- Last + Golden
- Last + Alt-Ref
- Golden + Alt-Ref
```

**运动矢量精度**：1/8 像素（VP8 为 1/4）

### 6.3 运动矢量预测改进

VP9 使用更复杂的 MV 预测：

```
MV 候选来源：
1. 空间候选：上方、左侧、上右邻居块
2. 时间候选：共位参考帧的 MV
3. 零矢量：默认候选

候选列表最多 2 个，通过索引传输选择
```

## 7. VP9 变换与量化

### 7.1 多尺寸变换

VP9 支持 4 种变换大小：

| 变换大小 | 使用场景 |
|---------|---------|
| 4×4 | 高频细节区域 |
| 8×8 | 中等复杂度 |
| 16×16 | 较平坦区域 |
| 32×32 | 大面积平坦区域 |

### 7.2 DCT + ADST 组合

VP9 引入 **ADST（非对称离散正弦变换）**：

```
变换类型组合（水平 × 垂直）：

         水平
         DCT    ADST
垂直  ┌────────────────┐
DCT   │ DCT_DCT  ADST_DCT  │
ADST  │ DCT_ADST ADST_ADST │
      └────────────────────┘
```

**ADST 的优势**：
- 边界处基函数幅值小，远离边界幅值大
- 更适合帧内预测残差的统计特性

### 7.3 变换选择

变换类型根据预测模式自动选择：

| 预测模式 | 水平变换 | 垂直变换 |
|---------|---------|---------|
| DC | DCT | DCT |
| Vertical | DCT | ADST |
| Horizontal | ADST | DCT |
| 斜角 | 根据方向选择 | 根据方向选择 |

## 8. VP9 高级特性

### 8.1 分段（Segmentation）

VP9 支持将帧划分为最多 **8 个分段**：

```
分段示例：
┌─────────────────────────────────┐
│ Segment 0  │ Segment 1          │
│ (静态背景) │ (运动物体)          │
├────────────┼────────────────────┤
│ Segment 2  │ Segment 0          │
│ (ROI)      │ (静态背景)          │
└────────────┴────────────────────┘
```

**每个分段可独立设置**：
- 量化参数偏移（QP delta）
- 环路滤波强度
- 参考帧限制
- 跳过模式强制

### 8.2 可伸缩性（Scalability）

VP9 支持时间和空间可伸缩：

**时间可伸缩**：

```
3 层时间可伸缩：

Layer 2:        P           P
Layer 1:    P       P   P       P
Layer 0:  I   P   P   P   P   P   P

丢弃 Layer 2 → 帧率减半
丢弃 Layer 1+2 → 帧率 1/4
```

**空间可伸缩**：
- 基础层：低分辨率
- 增强层：高分辨率（参考基础层上采样）

### 8.3 Tile 并行

VP9 引入 Tile 进行并行编解码：

```
Tile 划分（2 列示例）：
┌─────────────────┬─────────────────┐
│                 │                 │
│     Tile 0      │     Tile 1      │
│                 │                 │
│                 │                 │
└─────────────────┴─────────────────┘

特点：
- Tile 边界不进行帧内预测
- 熵编码状态在 Tile 边界重置
- 可独立并行编解码
```

### 8.4 环路滤波改进

VP9 的去块滤波相比 VP8 更精细：

- 支持 64 级滤波强度（VP8 为 64 级）
- 可按分段设置不同滤波强度
- 更好地适应不同 QP

## 9. VP9 Profile 体系

|---------|------|---------|---------|

|------|------|

| 最大帧率 | 最高 180 fps |
| 最大码率 | 180 Mbps |

## 10. VP9 与 H.265 对比

### 10.1 技术对比

| 特性 | VP9 | H.265/HEVC |
|------|-----|------------|
| 发布年份 | 2013 | 2013 |
| 最大块大小 | 64×64 | 64×64 |
| 帧内模式数 | 10 | 35 |
| 分区类型 | 4 种 | 非对称 PU |
| 最大变换 | 32×32 | 32×32 |
| 变换类型 | DCT/ADST | DCT/DST |
| 环路滤波 | Deblock | Deblock + SAO |
| 并行工具 | Tile | Tile + WPP |
| 授权 | 免费 | 需付费 |

### 10.2 压缩效率对比

| 内容类型 | VP9 vs H.265 |
|---------|-------------|
| 低复杂度 | 基本持平 |
| 中等复杂度 | 略差 5% |
| 高复杂度 | 略差 10% |
| **平均** | **略差 5-10%** |

**结论**：VP9 在压缩效率上略逊于 H.265，但差距不大，且免版税优势明显。

### 10.3 应用场景对比

| 场景 | VP9 | H.265 |
|------|-----|-------|
| YouTube | ✓（主要格式） | 部分 |
| Netflix | ✓（部分） | ✓（主要） |
| WebRTC | ✓（常用） | 较少 |
| 广播 | 较少 | ✓（主要） |
| 蓝光 | ✗ | ✓ |

## 11. VP8 → VP9 → AV1 技术演进

### 11.1 核心技术继承

```
技术演进路线：

VP8 (2010)
│
├─ 16×16 宏块
├─ 4 种 16×16 帧内模式
├─ 3 参考帧
├─ 4×4 DCT + WHT
├─ 布尔算术编码
│
↓ 扩展块结构、更多预测模式
│
VP9 (2013)
│
├─ 64×64 超级块
├─ 10 种帧内模式
├─ 复合预测
├─ 32×32 DCT + ADST
├─ Tile 并行
│
↓ 融合 Daala、Thor 技术
│
AV1 (2018)
│
├─ 128×128 超级块
├─ 56 种方向模式 + Paeth + Smooth
├─ 更多复合预测模式
├─ 64×64 变换 + 多类型变换
├─ CDEF + LRF 滤波
├─ Film Grain Synthesis
└─ 完全免版税
```

### 11.2 关键改进汇总

| 技术点 | VP8 | VP9 | AV1 |
|--------|-----|-----|-----|
| 最大块 | 16×16 | 64×64 | 128×128 |
| 帧内模式 | 10 | 10 | 56+ |
| 分区类型 | 1 | 4 | 10 |
| 变换大小 | 4×4 | 4-32 | 4-64 |
| 变换类型 | DCT | DCT/ADST | 4 种组合 |
| 环路滤波 | Deblock | Deblock | DB+CDEF+LRF |
| 特殊功能 | - | Segmentation | Film Grain |

### 11.3 编码效率演进

以 H.264 High Profile 为基准（BD-Rate）：

| 编码器 | 相对码率 |
|--------|---------|
| VP8 | +5% ~ +15% |
| VP9 | -15% ~ -25% |
| AV1 | -35% ~ -45% |

## VP9 编码参数示例

### libvpx 编码命令

```bash
# 高质量编码
vpxenc --codec=vp9 --cpu-used=2 \
       --end-usage=q --cq-level=24 \
       --threads=8 --tile-columns=2 \
       --frame-parallel=0 --auto-alt-ref=1 --lag-in-frames=25 \
       --kf-max-dist=240 \
       -o output.webm input.y4m

# 两遍编码（最佳质量）
vpxenc --codec=vp9 --passes=2 --pass=1 \
       --cpu-used=4 --end-usage=vbr --target-bitrate=2000 \
       --fpf=firstpass.log \
       -o /dev/null input.y4m

vpxenc --codec=vp9 --passes=2 --pass=2 \
       --cpu-used=2 --end-usage=vbr --target-bitrate=2000 \
       --fpf=firstpass.log \
       --auto-alt-ref=1 --lag-in-frames=25 \
       -o output.webm input.y4m

# 实时编码
vpxenc --codec=vp9 --cpu-used=8 --rt \
       --end-usage=cbr --target-bitrate=1500 \
       --threads=4 --tile-columns=2 \
       --error-resilient=1 \
       -o output.webm input.y4m
```

### 关键参数说明

| 参数 | 说明 | 推荐值 |
|------|------|--------|
| --cpu-used | 速度/质量权衡 (0-9) | 2-4（质量）, 6-8（速度） |
| --cq-level | 恒定质量 (0-63) | 20-32 |
| --tile-columns | Tile 列数 (log2) | 1-2 |
| --auto-alt-ref | 启用 Alt-Ref | 1 |
| --lag-in-frames | 前瞻帧数 | 16-25 |
