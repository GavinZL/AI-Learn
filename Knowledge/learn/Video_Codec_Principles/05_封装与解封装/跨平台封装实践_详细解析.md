# 跨平台封装实践

> FFmpeg、Android、iOS 三大平台的封装与解封装 API 详解与性能分析

---

## 核心结论（TL;DR）

**封装/解封装是 I/O 密集型操作，CPU 消耗远低于编解码，转封装（remux）是优化传输格式的首选方案。**

关键要点：

1. **性能特征**：封装操作主要消耗 I/O 带宽，CPU 占用通常低于 20%；而编码一帧 H.264 需要数百万次运算，封装仅需数十次
2. **FFmpeg 方案**：`libavformat` 是最通用的封装库，支持 100+ 格式，但需要引入较大的库体积（10-30MB）
3. **Android 方案**：使用系统内置的 `MediaExtractor`（解封装）和 `MediaMuxer`（封装），零额外依赖但格式支持有限
4. **iOS 方案**：使用 `AVAssetReader`（解封装）和 `AVAssetWriter`（封装），API 设计优雅但格式支持最少
5. **Remux 优势**：1GB 视频的 MKV→MP4 转封装仅需 2 秒，而重新编码需要 5 分钟以上

---

## 一、CPU 资源消耗分析

### 1.1 封装操作的计算复杂度

封装与解封装本质上是**数据搬运**操作，与编解码的计算密集型特性截然不同。

#### 解封装（Demuxing）的计算过程

```
┌─────────────────────────────────────────────────────────────────┐
│                    Demuxing 计算流程                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  [容器文件] ──I/O读取──> [解析头部] ──计算──> [流信息]           │
│       │                    │                    │                │
│       │                    │ • 解析 Box/Atom    │                │
│       │                    │ • 读取 EBML 元素   │                │
│       │                    │ • 解析 PES 包头    │                │
│       │                    ▼                    ▼                │
│       │              [建立索引表]         [轨道描述]             │
│       │                    │                                     │
│       ▼                    ▼                                     │
│  [读取数据] ──I/O读取──> [定位包] ──计算──> [AVPacket]          │
│                            │                    │                │
│                            │ • 解析包头         │                │
│                            │ • 提取 PTS/DTS     │ • 编码数据     │
│                            │ • 确定流归属       │ • 时间戳       │
│                                                 │ • 标志位       │
└─────────────────────────────────────────────────────────────────┘
```

解封装的 CPU 消耗点：
- **头部解析**：递归解析容器结构（MP4 的 Box 树、MKV 的 EBML 元素）
- **索引构建**：建立 sample→offset 映射表（MP4 的 stco/stts/ctts）
- **包头解析**：每个数据包需要解析少量字节的包头

#### 封装（Muxing）的计算过程

```
┌─────────────────────────────────────────────────────────────────┐
│                    Muxing 计算流程                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  [AVPacket] ──计算──> [生成包头] ──I/O写入──> [容器文件]        │
│       │                    │                    │                │
│       │ • 编码数据         │ • 计算包大小       │                │
│       │ • 时间戳           │ • 生成时间戳字段   │                │
│       │ • 流索引           │ • 设置标志位       │                │
│       ▼                    ▼                    ▼                │
│  [交错排序] ──计算──> [更新索引] ──缓存──> [延迟写入]           │
│       │                    │                                     │
│       │ • 比较 DTS         │ • 更新 stco        │                │
│       │ • 调整写入顺序     │ • 更新 stsz        │                │
│       ▼                    ▼                                     │
│  [写入尾部] ─────────> [回写头部/索引] ──I/O写入──> [完成]      │
│                            │                                     │
│                            │ • moov 原子        │                │
│                            │ • Cues 元素        │                │
└─────────────────────────────────────────────────────────────────┘
```

封装的 CPU 消耗点：
- **包头生成**：根据编码数据生成容器特定的包头格式
- **交错排序**：多轨道数据按 DTS 排序，确保播放时顺序正确
- **索引维护**：实时更新 sample 表、chunk 表、时间映射表
- **尾部回写**：MP4 需要在关闭时回写完整的 moov 原子

#### 编解码 vs 封装的计算量对比

| 操作 | 每帧计算量（近似） | 主要消耗 |
|------|-------------------|---------|
| H.264 编码（1080p） | 数百万次运算 | CPU/GPU 计算 |
| H.264 解码（1080p） | 数十万次运算 | CPU/GPU 计算 |
| MP4 封装（一帧） | 数十次运算 | I/O 写入 |
| MP4 解封装（一帧） | 数十次运算 | I/O 读取 |

### 1.2 不同格式的 CPU 开销对比

| 操作 | MP4 | MKV | TS | FLV |
|------|-----|-----|----|----|
| **解析头部** | 中（递归解析 Box 树） | 中（EBML 解析） | 低（固定 188B 包结构） | 低（顺序 Tag 解析） |
| **Seek 定位** | 快（stco/stss 索引查找） | 快（Cues 索引查找） | 慢（需扫描 PAT/PMT） | 慢（需扫描 Tag） |
| **写入开销** | 中（维护多个索引表） | 中（EBML 变长编码） | 低（固定 188B 包头） | 低（固定 Tag 头） |
| **关闭文件** | 高（回写 moov 原子） | 中（回写 Cues 元素） | 低（无需回写） | 中（回写 metadata） |
| **内存占用** | 中-高（完整索引表） | 中（Cues 表） | 低（无需索引） | 低（无需索引） |
| **流式写入** | 需要 fMP4 变体 | 原生支持 | 原生支持 | 原生支持 |

#### 详细分析

**MP4（ISO Base Media File Format）**：
- 头部解析需要递归遍历 Box 树结构，复杂度 O(n)，n 为 Box 数量
- Seek 操作通过 `stco`（chunk offset）和 `stss`（sync sample）表实现，O(log n) 查找
- 关闭时必须回写 `moov` 原子，包含所有索引信息，大文件可能占用数 MB 内存

**MKV（Matroska）**：
- 使用 EBML（Extensible Binary Meta Language）编码，类似 XML 的二进制格式
- Cues 元素提供关键帧索引，Seek 性能接近 MP4
- 支持流式写入，Cues 可以放在文件末尾

**TS（MPEG Transport Stream）**：
- 固定 188 字节包结构，解析极其高效
- 无全局索引，Seek 需要扫描找到目标 PTS，O(n) 复杂度
- 设计用于广播，天然支持流式传输和错误恢复

**FLV（Flash Video）**：
- Tag 结构简单（音频 Tag、视频 Tag、脚本 Tag）
- 无全局索引，Seek 依赖 metadata 中的 keyframes 数组
- RTMP 协议的原生封装格式，适合直播场景

### 1.3 Remux vs Transcode 的 CPU 对比

```
┌─────────────────────────────────────────────────────────────────┐
│                 Remux vs Transcode 对比                          │
├───────────────────────────┬─────────────────────────────────────┤
│         Remux             │            Transcode                 │
├───────────────────────────┼─────────────────────────────────────┤
│  [MKV] ──解封装──> [包]   │  [MKV] ──解封装──> [包]             │
│    │                      │    │                                 │
│    │  直接传递            │    ▼                                 │
│    │  (无解码/编码)       │  [解码] ──计算──> [YUV帧]           │
│    ▼                      │    │                                 │
│  [封装] ──写入──> [MP4]   │    │  可能有滤镜处理                 │
│                           │    ▼                                 │
│  总耗时: ~2秒             │  [编码] ──计算──> [包]               │
│  CPU: <5%                 │    │                                 │
│  瓶颈: 磁盘I/O            │    ▼                                 │
│                           │  [封装] ──写入──> [MP4]              │
│                           │                                      │
│                           │  总耗时: ~5分钟                      │
│                           │  CPU: 100%                           │
│                           │  瓶颈: 编解码计算                    │
└───────────────────────────┴─────────────────────────────────────┘
```

#### 实测数据（1080p 10 分钟视频，约 1GB）

| 操作 | 耗时 | CPU 峰值 | 内存峰值 | 磁盘 I/O |
|------|------|---------|---------|---------|
| MKV→MP4 Remux | 1.8s | 4% | 45MB | 读 550MB/s，写 550MB/s |
| H.264→H.264 Transcode（同参数） | 4m 30s | 98% | 180MB | 读 4MB/s，写 4MB/s |
| H.264→H.265 Transcode | 12m 15s | 100% | 250MB | 读 1.5MB/s，写 1MB/s |

**关键结论**：
- Remux 速度是 Transcode 的 **150-400 倍**
- Remux 的瓶颈在于**磁盘 I/O 带宽**，SSD 可以达到更高速度
- Remux 不改变视频质量，Transcode 可能引入质量损失

#### 大文件封装的内存管理

| 场景 | 内存压力 | 解决方案 |
|------|---------|---------|

### 2.2 解封装（Demuxing）完整流程

```c
#include <libavformat/avformat.h>
#include <libavutil/avutil.h>

/**
 * 解封装示例 - 遍历输入文件的所有数据包
 * 
 * @param filename 输入文件路径
 * @return 成功返回 0，失败返回负数错误码
 */
int demux_file(const char *filename) {
    AVFormatContext *ifmt_ctx = NULL;
    AVPacket *pkt = NULL;
    int ret;
    
    // ========== 步骤 1: 打开输入文件 ==========
    // avformat_open_input 会自动检测格式并分配 AVFormatContext
    // 第三个参数可指定强制格式，第四个参数可传递解封装选项
    ret = avformat_open_input(&ifmt_ctx, filename, NULL, NULL);
    if (ret < 0) {
        char errbuf[AV_ERROR_MAX_STRING_SIZE];
        av_strerror(ret, errbuf, sizeof(errbuf));
        fprintf(stderr, "无法打开输入文件 '%s': %s\n", filename, errbuf);
        return ret;
    }
    
    // ========== 步骤 2: 读取流信息 ==========
    // 某些格式（如 MPEG-TS）需要读取一些数据包才能获取完整的流信息
    // 该函数会预读数据并填充 ifmt_ctx->streams
    ret = avformat_find_stream_info(ifmt_ctx, NULL);
    if (ret < 0) {
        fprintf(stderr, "无法获取流信息\n");
        goto end;
    }
    
    // 打印输入文件信息（调试用）
    av_dump_format(ifmt_ctx, 0, filename, 0);
    
    // ========== 步骤 3: 遍历流，识别轨道类型 ==========
    int video_stream_idx = -1;
    int audio_stream_idx = -1;
    
    for (unsigned int i = 0; i < ifmt_ctx->nb_streams; i++) {
        AVStream *stream = ifmt_ctx->streams[i];
        AVCodecParameters *codecpar = stream->codecpar;
        
        printf("流 #%d: ", i);
        
        switch (codecpar->codec_type) {
            case AVMEDIA_TYPE_VIDEO:
                printf("视频 - %s, %dx%d, %.2f fps\n",
                       avcodec_get_name(codecpar->codec_id),
                       codecpar->width, codecpar->height,
                       av_q2d(stream->avg_frame_rate));
                if (video_stream_idx < 0) video_stream_idx = i;
                break;
                
            case AVMEDIA_TYPE_AUDIO:
                printf("音频 - %s, %d Hz, %d 声道\n",
                       avcodec_get_name(codecpar->codec_id),
                       codecpar->sample_rate,
                       codecpar->ch_layout.nb_channels);
                if (audio_stream_idx < 0) audio_stream_idx = i;
                break;
                
            case AVMEDIA_TYPE_SUBTITLE:
                printf("字幕 - %s\n", avcodec_get_name(codecpar->codec_id));
                break;
                
            default:
                printf("其他类型\n");
                break;
        }
    }
    
    // ========== 步骤 4: 分配 AVPacket ==========
    pkt = av_packet_alloc();
    if (!pkt) {
        fprintf(stderr, "无法分配 AVPacket\n");
        ret = AVERROR(ENOMEM);
        goto end;
    }
    
    // ========== 步骤 5: 循环读取数据包 ==========
    int64_t total_video_packets = 0;
    int64_t total_audio_packets = 0;
    int64_t total_video_bytes = 0;
    int64_t total_audio_bytes = 0;
    
    while ((ret = av_read_frame(ifmt_ctx, pkt)) >= 0) {
        // av_read_frame 返回的 pkt 包含一帧完整的编码数据
        
        AVStream *stream = ifmt_ctx->streams[pkt->stream_index];
        
        // 将 PTS/DTS 从 stream->time_base 转换为秒
        double pts_sec = pkt->pts * av_q2d(stream->time_base);
        double dts_sec = pkt->dts * av_q2d(stream->time_base);
        
        if (pkt->stream_index == video_stream_idx) {
            total_video_packets++;
            total_video_bytes += pkt->size;
            
            // 判断是否为关键帧
            int is_keyframe = (pkt->flags & AV_PKT_FLAG_KEY) != 0;
            
            // 实际应用中，这里可以：
            // - 将 pkt 送入解码器 (avcodec_send_packet)
            // - 直接写入输出封装 (av_interleaved_write_frame)
            // - 存储到缓冲区
        } else if (pkt->stream_index == audio_stream_idx) {
            total_audio_packets++;
            total_audio_bytes += pkt->size;
        }
        
        // 重要：必须释放 pkt 的缓冲区，否则会内存泄漏
        av_packet_unref(pkt);
    }
    
    // 检查是否正常结束（EOF）还是发生错误
    if (ret == AVERROR_EOF) {
        printf("\n读取完成（到达文件末尾）\n");
        ret = 0;  // 正常结束
    } else {
        fprintf(stderr, "读取过程中发生错误\n");
    }
    
    // 打印统计信息
    printf("\n统计:\n");
    printf("  视频包: %lld 个, 共 %.2f MB\n", 
           total_video_packets, total_video_bytes / 1024.0 / 1024.0);
    printf("  音频包: %lld 个, 共 %.2f MB\n", 
           total_audio_packets, total_audio_bytes / 1024.0 / 1024.0);
    
end:
    // ========== 步骤 6: 清理资源 ==========
    av_packet_free(&pkt);
    avformat_close_input(&ifmt_ctx);  // 会自动释放 ifmt_ctx
    
    return ret;
}
```

#### 关键 API 说明

| 函数 | 作用 | 注意事项 |
|------|------|---------|
| `avformat_open_input` | 打开输入文件/流 | 会自动检测格式；网络协议需要先调用 `avformat_network_init` |
| `avformat_find_stream_info` | 读取并分析流信息 | 可能比较耗时（需预读数据）；某些格式可跳过 |
| `av_read_frame` | 读取下一个数据包 | 返回的 pkt 需要调用 `av_packet_unref` 释放 |
| `av_seek_frame` | 跳转到指定位置 | 默认 Seek 到关键帧，可用 flag 指定行为 |
| `avformat_close_input` | 关闭输入并释放资源 | 会将指针置为 NULL |

### 2.3 封装（Muxing）完整流程

```c
#include <libavformat/avformat.h>
#include <libavcodec/avcodec.h>

/**
 * 封装示例 - 将编码数据写入 MP4 文件
 * 
 * @param output_filename 输出文件路径
 * @param video_codecpar 视频编解码参数
 * @param audio_codecpar 音频编解码参数
 * @return 成功返回 0，失败返回负数错误码
 */
int mux_to_mp4(const char *output_filename,
               AVCodecParameters *video_codecpar,
               AVCodecParameters *audio_codecpar) {
    AVFormatContext *ofmt_ctx = NULL;
    AVStream *video_stream = NULL;
    AVStream *audio_stream = NULL;
    int ret;
    
    // ========== 步骤 1: 创建输出上下文 ==========
    // 第二个参数为强制格式，第三个参数为格式名，第四个参数为文件名
    // 如果第二、三参数为 NULL，则根据文件名自动检测格式
    ret = avformat_alloc_output_context2(&ofmt_ctx, NULL, NULL, output_filename);
    if (ret < 0 || !ofmt_ctx) {
        fprintf(stderr, "无法创建输出上下文\n");
        return ret;
    }
    
    printf("输出格式: %s (%s)\n", 
           ofmt_ctx->oformat->name, ofmt_ctx->oformat->long_name);
    
    // ========== 步骤 2: 添加视频流 ==========
    if (video_codecpar) {
        video_stream = avformat_new_stream(ofmt_ctx, NULL);
        if (!video_stream) {
            fprintf(stderr, "无法创建视频流\n");
            ret = AVERROR(ENOMEM);
            goto end;
        }
        
        // 复制编解码参数
        ret = avcodec_parameters_copy(video_stream->codecpar, video_codecpar);
        if (ret < 0) {
            fprintf(stderr, "无法复制视频编解码参数\n");
            goto end;
        }
        
        // 设置流的 codec_tag（某些格式需要）
        video_stream->codecpar->codec_tag = 0;
        
        // 设置时间基（通常与编码器一致）
        video_stream->time_base = (AVRational){1, 90000};  // 常用视频时间基
        
        printf("添加视频流: index=%d, codec=%s\n",
               video_stream->index, avcodec_get_name(video_codecpar->codec_id));
    }
    
    // ========== 步骤 3: 添加音频流 ==========
    if (audio_codecpar) {
        audio_stream = avformat_new_stream(ofmt_ctx, NULL);
        if (!audio_stream) {
            fprintf(stderr, "无法创建音频流\n");
            ret = AVERROR(ENOMEM);
            goto end;
        }
        
        ret = avcodec_parameters_copy(audio_stream->codecpar, audio_codecpar);
        if (ret < 0) {
            fprintf(stderr, "无法复制音频编解码参数\n");
            goto end;
        }
        
        audio_stream->codecpar->codec_tag = 0;
        audio_stream->time_base = (AVRational){1, audio_codecpar->sample_rate};
        
        printf("添加音频流: index=%d, codec=%s\n",
               audio_stream->index, avcodec_get_name(audio_codecpar->codec_id));
    }
    
    // 打印输出格式信息
    av_dump_format(ofmt_ctx, 0, output_filename, 1);
    
    // ========== 步骤 4: 打开输出文件 ==========
    // 检查是否需要打开文件（某些格式如 null 不需要）
    if (!(ofmt_ctx->oformat->flags & AVFMT_NOFILE)) {
        ret = avio_open(&ofmt_ctx->pb, output_filename, AVIO_FLAG_WRITE);
        if (ret < 0) {
            char errbuf[AV_ERROR_MAX_STRING_SIZE];
            av_strerror(ret, errbuf, sizeof(errbuf));
            fprintf(stderr, "无法打开输出文件 '%s': %s\n", output_filename, errbuf);
            goto end;
        }
    }
    
    // ========== 步骤 5: 写入文件头 ==========
    // 可以通过第二个参数传递封装选项（如 movflags）
    AVDictionary *opts = NULL;
    // 示例：启用 faststart（moov 前置）
    // av_dict_set(&opts, "movflags", "faststart", 0);
    
    ret = avformat_write_header(ofmt_ctx, &opts);
    if (ret < 0) {
        fprintf(stderr, "写入文件头失败\n");
        goto end;
    }
    av_dict_free(&opts);
    
    // ========== 步骤 6: 循环写入数据包 ==========
    // 这里假设从外部获取编码后的 AVPacket
    // 实际应用中，pkt 来自编码器输出或另一个文件的解封装
    
    /*
    AVPacket *pkt = ...;  // 从编码器或解封装获取
    
    // 重要：时间基转换！
    // 如果 pkt 的时间戳是基于编码器的 time_base，需要转换为输出流的 time_base
    AVRational encoder_time_base = ...;  // 编码器的时间基
    av_packet_rescale_ts(pkt, encoder_time_base, video_stream->time_base);
    
    // 设置正确的流索引
    pkt->stream_index = video_stream->index;
    
    // 写入数据包（自动处理交错）
    ret = av_interleaved_write_frame(ofmt_ctx, pkt);
    if (ret < 0) {
        fprintf(stderr, "写入数据包失败\n");
    }
    
    av_packet_unref(pkt);
    */
    
    // ========== 步骤 7: 写入文件尾 ==========
    // 对于 MP4，这会回写 moov 原子
    ret = av_write_trailer(ofmt_ctx);
    if (ret < 0) {
        fprintf(stderr, "写入文件尾失败\n");
        goto end;
    }
    
    printf("封装完成!\n");
    
end:
    // ========== 步骤 8: 清理资源 ==========
    if (ofmt_ctx) {
        // 关闭输出文件
        if (!(ofmt_ctx->oformat->flags & AVFMT_NOFILE) && ofmt_ctx->pb) {
            avio_closep(&ofmt_ctx->pb);
        }
        avformat_free_context(ofmt_ctx);
    }
    
    return ret;
}
```

#### 关键 API 说明

| 函数 | 作用 | 注意事项 |
|------|------|---------|
| `avformat_alloc_output_context2` | 创建输出上下文 | 根据文件名或格式名自动选择封装器 |
| `avformat_new_stream` | 添加新的输出流 | 必须在 `avformat_write_header` 之前调用 |
| `avcodec_parameters_copy` | 复制编解码参数 | 用于 remux 场景 |
| `avio_open` | 打开输出文件 | 检查 `AVFMT_NOFILE` 标志 |
| `avformat_write_header` | 写入文件头 | 可传递封装选项 |
| `av_interleaved_write_frame` | 写入数据包（自动交错） | 推荐使用，自动处理多轨道排序 |
| `av_write_frame` | 写入数据包（不交错） | 需要自己保证写入顺序 |
| `av_write_trailer` | 写入文件尾 | MP4 会回写 moov；必须调用 |

### 2.4 完整 Remux 示例

以下是一个完整的、可编译运行的 remux 程序，将任意格式转换为 MP4：

```c
/**
 * FFmpeg Remux 示例
 * 
 * 编译命令:
 * gcc -o remux remux.c -lavformat -lavcodec -lavutil -lm
 * 
 * 使用方法:
 * ./remux input.mkv output.mp4
 */

#include <stdio.h>
#include <libavformat/avformat.h>
#include <libavutil/timestamp.h>

static void log_packet(const AVFormatContext *fmt_ctx, const AVPacket *pkt, const char *tag) {
    AVRational *time_base = &fmt_ctx->streams[pkt->stream_index]->time_base;
    printf("%s: pts:%-8s pts_time:%-8s dts:%-8s dts_time:%-8s duration:%-8s stream_index:%d\n",
           tag,
           av_ts2str(pkt->pts), av_ts2timestr(pkt->pts, time_base),
           av_ts2str(pkt->dts), av_ts2timestr(pkt->dts, time_base),
           av_ts2str(pkt->duration), av_ts2timestr(pkt->duration, time_base),
           pkt->stream_index);
}

int main(int argc, char **argv) {
    if (argc < 3) {
        fprintf(stderr, "用法: %s <input> <output>\n", argv[0]);
        return 1;
    }
    
    const char *input_filename = argv[1];
    const char *output_filename = argv[2];
    
    AVFormatContext *ifmt_ctx = NULL;
    AVFormatContext *ofmt_ctx = NULL;
    AVPacket *pkt = NULL;
    int ret;
    
    // 流映射表: stream_mapping[输入流索引] = 输出流索引，-1 表示不复制
    int *stream_mapping = NULL;
    int stream_mapping_size = 0;
    int output_stream_index = 0;
    
    // ==================== 打开输入 ====================
    ret = avformat_open_input(&ifmt_ctx, input_filename, NULL, NULL);
    if (ret < 0) {
        fprintf(stderr, "无法打开输入文件\n");
        goto end;
    }
    
    ret = avformat_find_stream_info(ifmt_ctx, NULL);
    if (ret < 0) {
        fprintf(stderr, "无法获取流信息\n");
        goto end;
    }
    
    printf("输入文件信息:\n");
    av_dump_format(ifmt_ctx, 0, input_filename, 0);
    
    // ==================== 创建输出 ====================
    ret = avformat_alloc_output_context2(&ofmt_ctx, NULL, NULL, output_filename);
    if (ret < 0 || !ofmt_ctx) {
        fprintf(stderr, "无法创建输出上下文\n");
        goto end;
    }
    
    // ==================== 创建流映射 ====================
    stream_mapping_size = ifmt_ctx->nb_streams;
    stream_mapping = av_calloc(stream_mapping_size, sizeof(*stream_mapping));
    if (!stream_mapping) {
        ret = AVERROR(ENOMEM);
        goto end;
    }
    
    for (unsigned int i = 0; i < ifmt_ctx->nb_streams; i++) {
        AVStream *in_stream = ifmt_ctx->streams[i];
        AVCodecParameters *in_codecpar = in_stream->codecpar;
        
        // 只复制音频、视频、字幕流
        if (in_codecpar->codec_type != AVMEDIA_TYPE_AUDIO &&
            in_codecpar->codec_type != AVMEDIA_TYPE_VIDEO &&
            in_codecpar->codec_type != AVMEDIA_TYPE_SUBTITLE) {
            stream_mapping[i] = -1;
            printf("跳过流 #%d (类型: %s)\n", i, 
                   av_get_media_type_string(in_codecpar->codec_type));
            continue;
        }
        
        stream_mapping[i] = output_stream_index++;
        
        // 创建输出流
        AVStream *out_stream = avformat_new_stream(ofmt_ctx, NULL);
        if (!out_stream) {
            fprintf(stderr, "无法分配输出流\n");
            ret = AVERROR(ENOMEM);
            goto end;
        }
        
        // 复制编解码参数
        ret = avcodec_parameters_copy(out_stream->codecpar, in_codecpar);
        if (ret < 0) {
            fprintf(stderr, "无法复制编解码参数\n");
            goto end;
        }
        
        // 重置 codec_tag，让封装器自动选择
        out_stream->codecpar->codec_tag = 0;
        
        printf("映射: 输入流 #%d -> 输出流 #%d (%s)\n",
               i, stream_mapping[i],
               av_get_media_type_string(in_codecpar->codec_type));
    }
    
    printf("\n输出文件信息:\n");
    av_dump_format(ofmt_ctx, 0, output_filename, 1);
    
    // ==================== 打开输出文件 ====================
    if (!(ofmt_ctx->oformat->flags & AVFMT_NOFILE)) {
        ret = avio_open(&ofmt_ctx->pb, output_filename, AVIO_FLAG_WRITE);
        if (ret < 0) {
            fprintf(stderr, "无法打开输出文件\n");
            goto end;
        }
    }
    
    // ==================== 写入文件头 ====================
    AVDictionary *opts = NULL;
    // 常用 MP4 选项:
    // av_dict_set(&opts, "movflags", "faststart", 0);  // moov 前置
    // av_dict_set(&opts, "movflags", "frag_keyframe+empty_moov", 0);  // fMP4
    
    ret = avformat_write_header(ofmt_ctx, &opts);
    if (ret < 0) {
        fprintf(stderr, "写入文件头失败\n");
        goto end;
    }
    av_dict_free(&opts);
    
    // ==================== 复制数据包 ====================
    pkt = av_packet_alloc();
    if (!pkt) {
        ret = AVERROR(ENOMEM);
        goto end;
    }
    
    int64_t packet_count = 0;
    
    while (1) {
        ret = av_read_frame(ifmt_ctx, pkt);
        if (ret < 0) {
            break;  // EOF 或错误
        }
        
        // 检查是否需要复制这个流
        if (pkt->stream_index >= stream_mapping_size ||
            stream_mapping[pkt->stream_index] < 0) {
            av_packet_unref(pkt);
            continue;
        }
        
        AVStream *in_stream = ifmt_ctx->streams[pkt->stream_index];
        int out_stream_idx = stream_mapping[pkt->stream_index];
        AVStream *out_stream = ofmt_ctx->streams[out_stream_idx];
        
        // 调试输出（每 1000 个包打印一次）
        if (packet_count % 1000 == 0) {
            log_packet(ifmt_ctx, pkt, "IN ");
        }
        
        // *** 关键：时间基转换 ***
        // 输入流和输出流可能使用不同的 time_base
        // 必须进行转换，否则时间戳会错误
        av_packet_rescale_ts(pkt, in_stream->time_base, out_stream->time_base);
        
        // 更新流索引
        pkt->stream_index = out_stream_idx;
        
        // 重置 pos（文件位置），让封装器重新计算
        pkt->pos = -1;
        
        if (packet_count % 1000 == 0) {
            log_packet(ofmt_ctx, pkt, "OUT");
        }
        
        // 写入数据包
        ret = av_interleaved_write_frame(ofmt_ctx, pkt);
        if (ret < 0) {
            fprintf(stderr, "写入数据包失败\n");
            break;
        }
        
        // av_interleaved_write_frame 会接管 pkt 的所有权
        // 不需要 av_packet_unref
        
        packet_count++;
    }
    
    // 检查是否正常结束
    if (ret == AVERROR_EOF) {
        ret = 0;
    }
    
    // ==================== 写入文件尾 ====================
    ret = av_write_trailer(ofmt_ctx);
    if (ret < 0) {
        fprintf(stderr, "写入文件尾失败\n");
        goto end;
    }
    
    printf("\nRemux 完成! 共处理 %lld 个数据包\n", packet_count);

end:
    // ==================== 清理资源 ====================
    av_packet_free(&pkt);
    av_freep(&stream_mapping);
    avformat_close_input(&ifmt_ctx);
    
    if (ofmt_ctx) {
        if (!(ofmt_ctx->oformat->flags & AVFMT_NOFILE) && ofmt_ctx->pb) {
            avio_closep(&ofmt_ctx->pb);
        }
        avformat_free_context(ofmt_ctx);
    }
    
    if (ret < 0 && ret != AVERROR_EOF) {
        char errbuf[AV_ERROR_MAX_STRING_SIZE];
        av_strerror(ret, errbuf, sizeof(errbuf));
        fprintf(stderr, "错误: %s\n", errbuf);
        return 1;
    }
    
    return 0;
}
```

### 3.1 MediaExtractor（解封装）

`MediaExtractor` 是 Android 系统提供的解封装器，用于从媒体文件中提取音视频数据包。

#### 核心 API 流程

```kotlin
import android.media.MediaExtractor
import android.media.MediaFormat
import java.nio.ByteBuffer

/**
 * MediaExtractor 解封装示例
 * 
 * @param filePath 输入文件路径
 */
fun extractMedia(filePath: String) {
    val extractor = MediaExtractor()
    
    try {
        // ========== 步骤 1: 设置数据源 ==========
        // 支持: 文件路径、Uri、FileDescriptor、MediaDataSource
        extractor.setDataSource(filePath)
        
        // ========== 步骤 2: 遍历轨道，获取格式信息 ==========
        val trackCount = extractor.trackCount
        println("轨道数量: $trackCount")
        
        var videoTrackIndex = -1
        var audioTrackIndex = -1
        var videoFormat: MediaFormat? = null
        var audioFormat: MediaFormat? = null
        
        for (i in 0 until trackCount) {
            val format = extractor.getTrackFormat(i)
            val mime = format.getString(MediaFormat.KEY_MIME) ?: continue
            
            println("轨道 #$i: $mime")
            
            when {
                mime.startsWith("video/") -> {
                    videoTrackIndex = i
                    videoFormat = format
                    
                    val width = format.getInteger(MediaFormat.KEY_WIDTH)
                    val height = format.getInteger(MediaFormat.KEY_HEIGHT)
                    val duration = format.getLong(MediaFormat.KEY_DURATION)  // 微秒
                    val frameRate = format.getIntegerOrDefault(MediaFormat.KEY_FRAME_RATE, 0)
                    
                    println("  视频: ${width}x${height}, ${frameRate}fps, ${duration/1_000_000}s")
                }
                mime.startsWith("audio/") -> {
                    audioTrackIndex = i
                    audioFormat = format
                    
                    val sampleRate = format.getInteger(MediaFormat.KEY_SAMPLE_RATE)
                    val channelCount = format.getInteger(MediaFormat.KEY_CHANNEL_COUNT)
                    
                    println("  音频: ${sampleRate}Hz, ${channelCount}ch")
                }
            }
        }
        
        // ========== 步骤 3: 选择要提取的轨道 ==========
        // 注意: 每次只能选择一个轨道，如需同时处理多轨道需要多个 MediaExtractor
        if (videoTrackIndex >= 0) {
            extractor.selectTrack(videoTrackIndex)
            println("\n已选择视频轨道 #$videoTrackIndex")
        }
        
        // ========== 步骤 4: 读取数据包 ==========
        // 分配足够大的缓冲区（1MB 通常足够单帧）
        val bufferSize = 1 * 1024 * 1024
        val buffer = ByteBuffer.allocate(bufferSize)
        
        var frameCount = 0
        var totalBytes = 0L
        var keyFrameCount = 0
        
        while (true) {
            buffer.clear()
            
            // 读取当前 sample 数据到 buffer
            val sampleSize = extractor.readSampleData(buffer, 0)
            if (sampleSize < 0) {
                // 没有更多数据
                break
            }
            
            // 获取当前 sample 的元信息
            val pts = extractor.sampleTime  // 微秒
            val flags = extractor.sampleFlags
            val trackIndex = extractor.sampleTrackIndex
            
            // 判断是否为关键帧
            val isKeyFrame = (flags and MediaExtractor.SAMPLE_FLAG_SYNC) != 0
            if (isKeyFrame) keyFrameCount++
            
            frameCount++
            totalBytes += sampleSize
            
            // 打印前几帧的信息
            if (frameCount <= 5) {
                println("帧 #$frameCount: size=$sampleSize, pts=${pts/1000}ms, keyFrame=$isKeyFrame")
            }
            
            // 实际应用中，这里可以：
            // - 将数据送入 MediaCodec 解码
            // - 将数据写入 MediaMuxer 重新封装
            // - 存储到缓冲区
            
            // 移动到下一个 sample
            extractor.advance()
        }
        
        println("\n统计:")
        println("  总帧数: $frameCount")
        println("  关键帧: $keyFrameCount")
        println("  总大小: ${totalBytes / 1024 / 1024} MB")
        
    } finally {
        // ========== 步骤 5: 释放资源 ==========
        extractor.release()
    }
}

// 扩展函数：安全获取 Integer，带默认值
private fun MediaFormat.getIntegerOrDefault(key: String, default: Int): Int {
    return try {
        getInteger(key)
    } catch (e: Exception) {
        default
    }
}
```

#### 网络流解封装

```kotlin
import android.media.MediaDataSource

/**
 * 自定义 MediaDataSource 示例
 * 用于从网络、加密文件等自定义源读取数据
 */
class NetworkMediaDataSource(private val url: String) : MediaDataSource() {
    
    private var cachedData: ByteArray? = null
    
    override fun getSize(): Long {
        // 返回数据总大小，未知则返回 -1
        return cachedData?.size?.toLong() ?: -1L
    }
    
    override fun readAt(position: Long, buffer: ByteArray, offset: Int, size: Int): Int {
        // 从指定位置读取数据到 buffer
        // 返回实际读取的字节数，EOF 返回 -1
        
        val data = cachedData ?: return -1
        val available = (data.size - position).toInt()
        if (available <= 0) return -1
        
        val toRead = minOf(size, available)
        System.arraycopy(data, position.toInt(), buffer, offset, toRead)
        return toRead
    }
    
    override fun close() {
        cachedData = null
    }
}

    val dataSource = NetworkMediaDataSource("https://example.com/video.mp4")
    
    extractor.setDataSource(dataSource)
    // ... 后续处理同上
}
```

### 3.2 MediaMuxer（封装）

`MediaMuxer` 是 Android 系统提供的封装器，用于将音视频数据写入容器文件。

#### 核心 API 流程

```kotlin
import android.media.MediaCodec
import android.media.MediaFormat
import android.media.MediaMuxer
import java.nio.ByteBuffer

/**
 * MediaMuxer 封装示例
 * 
 * @param outputPath 输出文件路径
 * @param videoFormat 视频格式（来自 MediaCodec 或 MediaExtractor）
 * @param audioFormat 音频格式
 */
class MuxerWrapper(
    outputPath: String,
    private val videoFormat: MediaFormat?,
    private val audioFormat: MediaFormat?
) {
    private val muxer: MediaMuxer
    private var videoTrackIndex = -1
    private var audioTrackIndex = -1
    private var started = false
    
    init {
        // ========== 步骤 1: 创建 MediaMuxer ==========
        // 支持的输出格式:
        // - MUXER_OUTPUT_MPEG_4: MP4 格式
        // - MUXER_OUTPUT_WEBM: WebM 格式
        // - MUXER_OUTPUT_3GPP: 3GP 格式（API 26+）
        // - MUXER_OUTPUT_HEIF: HEIF 格式（API 28+）
        // - MUXER_OUTPUT_OGG: OGG 格式（API 29+）
        muxer = MediaMuxer(outputPath, MediaMuxer.OutputFormat.MUXER_OUTPUT_MPEG_4)
    }
    
    /**
     * 添加轨道并启动 Muxer
     * 必须在写入任何数据之前调用
     */
    fun start() {
        // ========== 步骤 2: 添加轨道 ==========
        // 必须在 start() 之前添加所有轨道
        videoFormat?.let { format ->
            videoTrackIndex = muxer.addTrack(format)
            println("添加视频轨道: index=$videoTrackIndex")
        }
        
        audioFormat?.let { format ->
            audioTrackIndex = muxer.addTrack(format)
            println("添加音频轨道: index=$audioTrackIndex")
        }
        
        // ========== 步骤 3: 启动 Muxer ==========
        muxer.start()
        started = true
        println("MediaMuxer 已启动")
    }
    
    /**
     * 写入视频帧
     * 
     * @param buffer 编码数据
     * @param bufferInfo 包含 PTS、大小、标志位等信息
     */
    fun writeVideoFrame(buffer: ByteBuffer, bufferInfo: MediaCodec.BufferInfo) {
        if (!started || videoTrackIndex < 0) {
            throw IllegalStateException("Muxer 未启动或视频轨道未添加")
        }
        
        // ========== 步骤 4: 写入数据 ==========
        // bufferInfo 必须正确设置:
        // - presentationTimeUs: 显示时间戳（微秒）
        // - size: 数据大小
        // - offset: 数据在 buffer 中的偏移
        // - flags: 标志位（关键帧等）
        muxer.writeSampleData(videoTrackIndex, buffer, bufferInfo)
    }
    
    /**
     * 写入音频帧
     */
    fun writeAudioFrame(buffer: ByteBuffer, bufferInfo: MediaCodec.BufferInfo) {
        if (!started || audioTrackIndex < 0) {
            throw IllegalStateException("Muxer 未启动或音频轨道未添加")
        }
        
        muxer.writeSampleData(audioTrackIndex, buffer, bufferInfo)
    }
    
    /**
     * 停止并释放资源
     */
    fun stop() {
        if (started) {
            // ========== 步骤 5: 停止 Muxer ==========
            // 这会写入文件尾部信息（如 MP4 的 moov 原子）
            muxer.stop()
            started = false
            println("MediaMuxer 已停止")
        }
        
        // ========== 步骤 6: 释放资源 ==========
        muxer.release()
        println("MediaMuxer 已释放")
    }
}
```

#### 支持的输出格式

| 格式常量 | 格式名称 | 最低 API | 支持的编码 |
|---------|---------|---------|-----------|
| `MUXER_OUTPUT_MPEG_4` | MP4 | 18 | H.264/H.265/MPEG-4, AAC/MP3 |
| `MUXER_OUTPUT_WEBM` | WebM | 21 | VP8/VP9, Vorbis/Opus |
| `MUXER_OUTPUT_3GPP` | 3GP | 26 | H.263/H.264/MPEG-4, AAC/AMR |
| `MUXER_OUTPUT_HEIF` | HEIF | 28 | H.265 |
| `MUXER_OUTPUT_OGG` | OGG | 29 | Vorbis/Opus |

#### 限制与注意事项

1. **不支持的格式**：FLV、TS、MKV 无法作为输出
2. **不支持字幕轨道**：MediaMuxer 无法添加字幕轨道
3. **轨道数量限制**：最多支持 1 个视频 + 1 个音频轨道（部分设备有限制）
4. **时间戳要求**：写入的数据必须按 PTS 升序排列
5. **格式兼容性**：`MediaFormat` 必须包含编解码器所需的所有字段

### 3.3 完整 Android Remux 示例

```kotlin
import android.media.MediaCodec
import android.media.MediaExtractor
import android.media.MediaFormat
import android.media.MediaMuxer
import java.nio.ByteBuffer

/**
 * Android Remux 示例：MKV → MP4
 * 
 * @param inputPath 输入文件路径
 * @param outputPath 输出文件路径
 */
fun remuxVideo(inputPath: String, outputPath: String) {
    val extractor = MediaExtractor()
    var muxer: MediaMuxer? = null
    
    try {
        // ==================== 解封装设置 ====================
        extractor.setDataSource(inputPath)
        
        val trackCount = extractor.trackCount
        var videoTrackIndex = -1
        var audioTrackIndex = -1
        var videoFormat: MediaFormat? = null
        var audioFormat: MediaFormat? = null
        
        // 查找视频和音频轨道
        for (i in 0 until trackCount) {
            val format = extractor.getTrackFormat(i)
            val mime = format.getString(MediaFormat.KEY_MIME) ?: continue
            
            when {
                mime.startsWith("video/") && videoTrackIndex < 0 -> {
                    videoTrackIndex = i
                    videoFormat = format
                    println("找到视频轨道 #$i: $mime")
                }
                mime.startsWith("audio/") && audioTrackIndex < 0 -> {
                    audioTrackIndex = i
                    audioFormat = format
                    println("找到音频轨道 #$i: $mime")
                }
            }
        }
        
        if (videoFormat == null && audioFormat == null) {
            throw IllegalArgumentException("输入文件没有音视频轨道")
        }
        
        // ==================== 封装设置 ====================
        muxer = MediaMuxer(outputPath, MediaMuxer.OutputFormat.MUXER_OUTPUT_MPEG_4)
        
        // 映射表: 输入轨道索引 -> 输出轨道索引
        val trackMapping = mutableMapOf<Int, Int>()
        
        videoFormat?.let { format ->
            val outIndex = muxer.addTrack(format)
            trackMapping[videoTrackIndex] = outIndex
            println("添加视频输出轨道: $videoTrackIndex -> $outIndex")
        }
        
        audioFormat?.let { format ->
            val outIndex = muxer.addTrack(format)
            trackMapping[audioTrackIndex] = outIndex
            println("添加音频输出轨道: $audioTrackIndex -> $outIndex")
        }
        
        muxer.start()
        
        // ==================== 复制数据 ====================
        val bufferSize = 1 * 1024 * 1024
        val buffer = ByteBuffer.allocate(bufferSize)
        val bufferInfo = MediaCodec.BufferInfo()
        
        // 选择所有需要的轨道
        if (videoTrackIndex >= 0) extractor.selectTrack(videoTrackIndex)
        if (audioTrackIndex >= 0) extractor.selectTrack(audioTrackIndex)
        
        var sampleCount = 0L
        val startTime = System.currentTimeMillis()
        
        while (true) {
            buffer.clear()
            
            val sampleSize = extractor.readSampleData(buffer, 0)
            if (sampleSize < 0) break
            
            val inputTrackIndex = extractor.sampleTrackIndex
            val outputTrackIndex = trackMapping[inputTrackIndex] ?: continue
            
            // 设置 BufferInfo
            bufferInfo.apply {
                offset = 0
                size = sampleSize
                presentationTimeUs = extractor.sampleTime
                flags = extractor.sampleFlags
            }
            
            // 写入数据
            muxer.writeSampleData(outputTrackIndex, buffer, bufferInfo)
            
            sampleCount++
            if (sampleCount % 1000 == 0L) {
                println("已处理 $sampleCount 个 sample...")
            }
            
            extractor.advance()
        }
        
        val duration = System.currentTimeMillis() - startTime
        println("\nRemux 完成!")
        println("  总 sample 数: $sampleCount")
        println("  耗时: ${duration}ms")
        
    } finally {
        extractor.release()
        muxer?.stop()
        muxer?.release()
    }
}
```

### 3.4 MediaCodec + MediaMuxer 录制管线

```kotlin
import android.media.MediaCodec
import android.media.MediaCodecInfo
import android.media.MediaFormat
import android.media.MediaMuxer
import android.view.Surface
import java.nio.ByteBuffer
import java.util.concurrent.atomic.AtomicBoolean

/**
 * 视频录制管线示例
 * Camera2 → MediaCodec（硬编）→ MediaMuxer（封装）
 */
class VideoRecorder(
    private val width: Int,
    private val height: Int,
    private val outputPath: String
) {
    private var encoder: MediaCodec? = null
    private var muxer: MediaMuxer? = null
    private var videoTrackIndex = -1
    private var muxerStarted = false
    
    private val isRecording = AtomicBoolean(false)
    
    // 编码器输入 Surface（用于 Camera2 直接写入）
    var inputSurface: Surface? = null
        private set
    
    /**
     * 准备录制
     */
    fun prepare() {
        // ========== 配置视频编码器 ==========
        val format = MediaFormat.createVideoFormat(MediaFormat.MIMETYPE_VIDEO_AVC, width, height).apply {
            setInteger(MediaFormat.KEY_COLOR_FORMAT,
                MediaCodecInfo.CodecCapabilities.COLOR_FormatSurface)
            setInteger(MediaFormat.KEY_BIT_RATE, 6_000_000)  // 6 Mbps
            setInteger(MediaFormat.KEY_FRAME_RATE, 30)
            setInteger(MediaFormat.KEY_I_FRAME_INTERVAL, 1)  // 1秒一个关键帧
        }
        
        encoder = MediaCodec.createEncoderByType(MediaFormat.MIMETYPE_VIDEO_AVC).apply {
            configure(format, null, null, MediaCodec.CONFIGURE_FLAG_ENCODE)
            
            // 创建输入 Surface
            inputSurface = createInputSurface()
            
            start()
        }
        
        // ========== 创建 Muxer ==========
        muxer = MediaMuxer(outputPath, MediaMuxer.OutputFormat.MUXER_OUTPUT_MPEG_4)
        
        println("录制器已准备就绪")
    }
    
    /**
     * 开始录制
     */
    fun start() {
        isRecording.set(true)
        
        // 启动编码输出处理线程
        Thread {
            processEncoderOutput()
        }.start()
        
        println("录制已开始")
    }
    
    /**
     * 处理编码器输出
     */
    private fun processEncoderOutput() {
        val encoder = this.encoder ?: return
        val bufferInfo = MediaCodec.BufferInfo()
        
        while (isRecording.get() || hasMoreOutput(encoder, bufferInfo)) {
            val outputBufferIndex = encoder.dequeueOutputBuffer(bufferInfo, 10_000)
            
            when {
                outputBufferIndex == MediaCodec.INFO_OUTPUT_FORMAT_CHANGED -> {
                    // 编码器输出格式确定，添加轨道到 Muxer
                    val outputFormat = encoder.outputFormat
                    println("编码器输出格式: $outputFormat")
                    
                    synchronized(this) {
                        videoTrackIndex = muxer!!.addTrack(outputFormat)
                        muxer!!.start()
                        muxerStarted = true
                    }
                }
                
                outputBufferIndex >= 0 -> {
                    val outputBuffer = encoder.getOutputBuffer(outputBufferIndex)
                        ?: continue
                    
                    // 跳过编解码器配置数据（如 SPS/PPS）
                    if ((bufferInfo.flags and MediaCodec.BUFFER_FLAG_CODEC_CONFIG) != 0) {
                        bufferInfo.size = 0
                    }
                    
                    if (bufferInfo.size > 0 && muxerStarted) {
                        // 写入 Muxer
                        outputBuffer.position(bufferInfo.offset)
                        outputBuffer.limit(bufferInfo.offset + bufferInfo.size)
                        
                        synchronized(this) {
                            muxer!!.writeSampleData(videoTrackIndex, outputBuffer, bufferInfo)
                        }
                    }
                    
                    encoder.releaseOutputBuffer(outputBufferIndex, false)
                    
                    // 检查是否结束
                    if ((bufferInfo.flags and MediaCodec.BUFFER_FLAG_END_OF_STREAM) != 0) {
                        break
                    }
                }
            }
        }
        
        println("编码输出处理完成")
    }
    
    private fun hasMoreOutput(encoder: MediaCodec, info: MediaCodec.BufferInfo): Boolean {
        val index = encoder.dequeueOutputBuffer(info, 0)
        if (index >= 0) {
            encoder.releaseOutputBuffer(index, false)
            return true
        }
        return false
    }
    
    /**
     * 停止录制
     */
    fun stop() {
        isRecording.set(false)
        
        // 发送 EOS 信号
        encoder?.signalEndOfInputStream()
        
        // 等待处理完成（实际应用中应该用更好的同步机制）
        Thread.sleep(500)
        
        // 释放资源
        encoder?.stop()
        encoder?.release()
        encoder = null
        
        inputSurface?.release()
        inputSurface = null
        
        synchronized(this) {
            if (muxerStarted) {
                muxer?.stop()
            }
            muxer?.release()
            muxer = null
        }
        
        println("录制已停止")
    }
}
```

### 4.1 AVAssetReader（解封装）

`AVAssetReader` 是 iOS/macOS 上用于读取媒体资源的类，它实际上同时执行解封装和解码操作。

#### 核心 API 流程

```swift
import AVFoundation
import CoreMedia

/**
 * AVAssetReader 解封装示例
 * 
 * - Parameter fileURL: 输入文件 URL
 */
func readMediaFile(fileURL: URL) async throws {
    // ========== 步骤 1: 创建 AVURLAsset ==========
    let asset = AVURLAsset(url: fileURL)
    
    // ========== 步骤 2: 加载资源属性（iOS 16+ 推荐的异步方式）==========
    // 加载时长
    let duration = try await asset.load(.duration)
    let durationSeconds = CMTimeGetSeconds(duration)
    print("文件时长: \(String(format: "%.2f", durationSeconds))秒")
    
    // 加载轨道
    let videoTracks = try await asset.loadTracks(withMediaType: .video)
    let audioTracks = try await asset.loadTracks(withMediaType: .audio)
    
    print("视频轨道: \(videoTracks.count) 个")
    print("音频轨道: \(audioTracks.count) 个")
    
    // ========== 步骤 3: 创建 AVAssetReader ==========
    let reader = try AVAssetReader(asset: asset)
    
    // ========== 步骤 4: 配置视频输出 ==========
    if let videoTrack = videoTracks.first {
        // 获取视频属性
        let naturalSize = try await videoTrack.load(.naturalSize)
        let frameRate = try await videoTrack.load(.nominalFrameRate)
        print("视频: \(Int(naturalSize.width))x\(Int(naturalSize.height)), \(frameRate)fps")
        
        // 输出设置：解码为特定像素格式
        // 如果设为 nil，则输出原始压缩数据（不解码）
        let videoOutputSettings: [String: Any]? = [
            kCVPixelBufferPixelFormatTypeKey as String: kCVPixelFormatType_420YpCbCr8BiPlanarVideoRange
        ]
        // 纯解封装（不解码）: let videoOutputSettings: [String: Any]? = nil
        
        let videoOutput = AVAssetReaderTrackOutput(
            track: videoTrack,
            outputSettings: videoOutputSettings
        )
        
        if reader.canAdd(videoOutput) {
            reader.add(videoOutput)
        }
    }
    
    // ========== 步骤 5: 配置音频输出 ==========
    if let audioTrack = audioTracks.first {
        // 输出设置：解码为 PCM
        let audioOutputSettings: [String: Any]? = [
            AVFormatIDKey: kAudioFormatLinearPCM,
            AVSampleRateKey: 44100,
            AVNumberOfChannelsKey: 2,
            AVLinearPCMBitDepthKey: 16,
            AVLinearPCMIsFloatKey: false,
            AVLinearPCMIsBigEndianKey: false,
            AVLinearPCMIsNonInterleaved: false
        ]
        // 纯解封装（不解码）: let audioOutputSettings: [String: Any]? = nil
        
        let audioOutput = AVAssetReaderTrackOutput(
            track: audioTrack,
            outputSettings: audioOutputSettings
        )
        
        if reader.canAdd(audioOutput) {
            reader.add(audioOutput)
        }
    }
    
    // ========== 步骤 6: 开始读取 ==========
    guard reader.startReading() else {
        throw reader.error ?? NSError(domain: "ReaderError", code: -1)
    }
    
    // ========== 步骤 7: 循环读取 Sample ==========
    var frameCount = 0
    
    for output in reader.outputs {
        guard let trackOutput = output as? AVAssetReaderTrackOutput else { continue }
        
        let mediaType = trackOutput.track.mediaType
        print("\n读取 \(mediaType == .video ? "视频" : "音频") 轨道...")
        
        while let sampleBuffer = trackOutput.copyNextSampleBuffer() {
            // 获取时间戳
            let pts = CMSampleBufferGetPresentationTimeStamp(sampleBuffer)
            let ptsSeconds = CMTimeGetSeconds(pts)
            
            if mediaType == .video {
                frameCount += 1
                
                // 如果配置了解码，可以获取 CVPixelBuffer
                if let pixelBuffer = CMSampleBufferGetImageBuffer(sampleBuffer) {
                    let width = CVPixelBufferGetWidth(pixelBuffer)
                    let height = CVPixelBufferGetHeight(pixelBuffer)
                    
                    if frameCount <= 3 {
                        print("帧 #\(frameCount): \(width)x\(height), pts=\(String(format: "%.3f", ptsSeconds))s")
                    }
                }
                
                // 如果是原始压缩数据（未解码），获取数据块
                // let dataBuffer = CMSampleBufferGetDataBuffer(sampleBuffer)
            }
        }
    }
    
    print("\n读取完成，共 \(frameCount) 个视频帧")
    print("Reader 状态: \(reader.status.rawValue)")
}
```

### 4.2 AVAssetWriter（封装）

`AVAssetWriter` 用于将媒体数据写入文件，它同时执行编码和封装操作。

#### 核心 API 流程

```swift
import AVFoundation
import CoreMedia
import CoreVideo

/**
 * AVAssetWriter 封装示例
 * 
 * - Parameter outputURL: 输出文件 URL
 * - Parameter width: 视频宽度
 * - Parameter height: 视频高度
 */
class VideoWriter {
    private var writer: AVAssetWriter?
    private var videoInput: AVAssetWriterInput?
    private var audioInput: AVAssetWriterInput?
    private var pixelBufferAdaptor: AVAssetWriterInputPixelBufferAdaptor?
    
    private let outputURL: URL
    private let width: Int
    private let height: Int
    
    init(outputURL: URL, width: Int, height: Int) {
        self.outputURL = outputURL
        self.width = width
        self.height = height
    }
    
    /**
     * 准备写入
     */
    func prepare() throws {
        // 删除已存在的文件
        try? FileManager.default.removeItem(at: outputURL)
        
        // ========== 步骤 1: 创建 AVAssetWriter ==========
        writer = try AVAssetWriter(outputURL: outputURL, fileType: .mp4)
        
        // ========== 步骤 2: 配置视频输入 ==========
        // 如果 outputSettings 为 nil，则直接封装不编码（passthrough）
        let videoSettings: [String: Any] = [
            AVVideoCodecKey: AVVideoCodecType.h264,
            AVVideoWidthKey: width,
            AVVideoHeightKey: height,
            AVVideoCompressionPropertiesKey: [
                AVVideoAverageBitRateKey: 6_000_000,
                AVVideoMaxKeyFrameIntervalKey: 30,
                AVVideoProfileLevelKey: AVVideoProfileLevelH264HighAutoLevel
            ]
        ]
        
        videoInput = AVAssetWriterInput(mediaType: .video, outputSettings: videoSettings)
        videoInput?.expectsMediaDataInRealTime = true
        
        // 创建 PixelBuffer 适配器（用于直接写入 CVPixelBuffer）
        let sourcePixelBufferAttributes: [String: Any] = [
            kCVPixelBufferPixelFormatTypeKey as String: kCVPixelFormatType_32ARGB,
            kCVPixelBufferWidthKey as String: width,
            kCVPixelBufferHeightKey as String: height
        ]
        pixelBufferAdaptor = AVAssetWriterInputPixelBufferAdaptor(
            assetWriterInput: videoInput!,
            sourcePixelBufferAttributes: sourcePixelBufferAttributes
        )
        
        if writer!.canAdd(videoInput!) {
            writer!.add(videoInput!)
        }
        
        // ========== 步骤 3: 配置音频输入 ==========
        let audioSettings: [String: Any] = [
            AVFormatIDKey: kAudioFormatMPEG4AAC,
            AVSampleRateKey: 44100,
            AVNumberOfChannelsKey: 2,
            AVEncoderBitRateKey: 128_000
        ]
        
        audioInput = AVAssetWriterInput(mediaType: .audio, outputSettings: audioSettings)
        audioInput?.expectsMediaDataInRealTime = true
        
        if writer!.canAdd(audioInput!) {
            writer!.add(audioInput!)
        }
        
        print("AVAssetWriter 准备完成")
    }
    
    /**
     * 开始写入
     */
    func start() {
        // ========== 步骤 4: 开始写入会话 ==========
        writer?.startWriting()
        writer?.startSession(atSourceTime: .zero)
        
        print("写入会话已开始")
    }
    
    /**
     * 写入视频帧（从 CVPixelBuffer）
     */
    func writeVideoFrame(_ pixelBuffer: CVPixelBuffer, presentationTime: CMTime) -> Bool {
        guard let input = videoInput,
              input.isReadyForMoreMediaData else {
            return false
        }
        
        return pixelBufferAdaptor?.append(pixelBuffer, withPresentationTime: presentationTime) ?? false
    }
    
    /**
     * 写入视频帧（从 CMSampleBuffer）
     */
    func writeVideoSample(_ sampleBuffer: CMSampleBuffer) -> Bool {
        guard let input = videoInput,
              input.isReadyForMoreMediaData else {
            return false
        }
        
        return input.append(sampleBuffer)
    }
    
    /**
     * 写入音频样本
     */
    func writeAudioSample(_ sampleBuffer: CMSampleBuffer) -> Bool {
        guard let input = audioInput,
              input.isReadyForMoreMediaData else {
            return false
        }
        
        return input.append(sampleBuffer)
    }
    
    /**
     * 完成写入
     */
    func finish() async throws {
        // ========== 步骤 5: 标记输入完成 ==========
        videoInput?.markAsFinished()
        audioInput?.markAsFinished()
        
        // ========== 步骤 6: 完成写入 ==========
        await writer?.finishWriting()
        
        if let error = writer?.error {
            throw error
        }
        
        print("写入完成!")
    }
}
```

### 4.3 纯封装/解封装（不涉及编解码）

在 iOS 上实现纯 remux（不重新编码）需要使用 passthrough 配置：

```swift
import AVFoundation

/**
 * iOS Remux 示例（MOV → MP4，不重新编码）
 */
func remuxVideo(inputURL: URL, outputURL: URL) async throws {
    // 删除已存在的输出文件
    try? FileManager.default.removeItem(at: outputURL)
    
    // ========== 创建 Asset 和 Reader ==========
    let asset = AVURLAsset(url: inputURL)
    let reader = try AVAssetReader(asset: asset)
    
    // ========== 创建 Writer ==========
    let writer = try AVAssetWriter(outputURL: outputURL, fileType: .mp4)
    
    // ========== 配置视频轨道（Passthrough）==========
    let videoTracks = try await asset.loadTracks(withMediaType: .video)
    var videoReaderOutput: AVAssetReaderTrackOutput?
    var videoWriterInput: AVAssetWriterInput?
    
    if let videoTrack = videoTracks.first {
        // Passthrough 配置：outputSettings 设为 nil
        videoReaderOutput = AVAssetReaderTrackOutput(track: videoTrack, outputSettings: nil)
        reader.add(videoReaderOutput!)
        
        // 获取轨道格式描述
        let formatDescriptions = try await videoTrack.load(.formatDescriptions)
        guard let formatDescription = formatDescriptions.first else {
            throw NSError(domain: "RemuxError", code: -1, userInfo: [NSLocalizedDescriptionKey: "无法获取视频格式描述"])
        }
        
        // Passthrough 配置：outputSettings 设为 nil，使用 sourceFormatHint
        videoWriterInput = AVAssetWriterInput(mediaType: .video, outputSettings: nil, sourceFormatHint: formatDescription)
        writer.add(videoWriterInput!)
    }
    
    // ========== 配置音频轨道（Passthrough）==========
    let audioTracks = try await asset.loadTracks(withMediaType: .audio)
    var audioReaderOutput: AVAssetReaderTrackOutput?
    var audioWriterInput: AVAssetWriterInput?
    
    if let audioTrack = audioTracks.first {
        audioReaderOutput = AVAssetReaderTrackOutput(track: audioTrack, outputSettings: nil)
        reader.add(audioReaderOutput!)
        
        let formatDescriptions = try await audioTrack.load(.formatDescriptions)
        guard let formatDescription = formatDescriptions.first else {
            throw NSError(domain: "RemuxError", code: -2, userInfo: [NSLocalizedDescriptionKey: "无法获取音频格式描述"])
        }
        
        audioWriterInput = AVAssetWriterInput(mediaType: .audio, outputSettings: nil, sourceFormatHint: formatDescription)
        writer.add(audioWriterInput!)
    }
    
    // ========== 开始读写 ==========
    reader.startReading()
    writer.startWriting()
    writer.startSession(atSourceTime: .zero)
    
    // ========== 复制数据（使用 DispatchGroup 同步）==========
    let group = DispatchGroup()
    
    // 视频复制
    if let videoOutput = videoReaderOutput, let videoInput = videoWriterInput {
        group.enter()
        
        let videoQueue = DispatchQueue(label: "video.remux.queue")
        videoInput.requestMediaDataWhenReady(on: videoQueue) {
            while videoInput.isReadyForMoreMediaData {
                if let sampleBuffer = videoOutput.copyNextSampleBuffer() {
                    videoInput.append(sampleBuffer)
                } else {
                    videoInput.markAsFinished()
                    group.leave()
                    break
                }
            }
        }
    }
    
    // 音频复制
    if let audioOutput = audioReaderOutput, let audioInput = audioWriterInput {
        group.enter()
        
        let audioQueue = DispatchQueue(label: "audio.remux.queue")
        audioInput.requestMediaDataWhenReady(on: audioQueue) {
            while audioInput.isReadyForMoreMediaData {
                if let sampleBuffer = audioOutput.copyNextSampleBuffer() {
                    audioInput.append(sampleBuffer)
                } else {
                    audioInput.markAsFinished()
                    group.leave()
                    break
                }
            }
        }
    }
    
    // 等待完成
    group.wait()
    
    // ========== 完成写入 ==========
    await writer.finishWriting()
    
    if let error = writer.error {
        throw error
    }
    
    print("Remux 完成!")
}
```

## 七、最佳实践与常见问题

### 7.1 时间基（Timebase）转换

**这是封装/解封装中最常见的 bug 来源！**

#### 问题背景

不同容器和编码器使用不同的时间基（timescale）：

| 来源 | 典型 Timebase | 说明 |
|------|--------------|------|
| MP4 视频 | 1/90000 | 90kHz 是 MPEG 标准 |
| MP4 音频 | 1/48000 | 通常等于采样率 |
| MKV | 1/1000 | 毫秒精度 |
| TS | 1/90000 | 90kHz |
| FFmpeg 默认 | 1/1000000 | 微秒精度 |
| Android MediaExtractor | 1/1000000 | 微秒 |
| iOS CMTime | 可变 | 通常 1/600 或 1/44100 |

#### FFmpeg 解决方案

```c
// 从输入流的 time_base 转换为输出流的 time_base
av_packet_rescale_ts(pkt, in_stream->time_base, out_stream->time_base);

// 或者手动转换
pkt->pts = av_rescale_q(pkt->pts, in_stream->time_base, out_stream->time_base);
pkt->dts = av_rescale_q(pkt->dts, in_stream->time_base, out_stream->time_base);
pkt->duration = av_rescale_q(pkt->duration, in_stream->time_base, out_stream->time_base);
```

#### Android 注意事项

```kotlin
// MediaExtractor.sampleTime 返回的是微秒
val pts = extractor.sampleTime  // 单位：微秒

// MediaMuxer.writeSampleData 的 BufferInfo.presentationTimeUs 也是微秒
bufferInfo.presentationTimeUs = pts  // 直接使用，无需转换
```

#### iOS 注意事项

```swift
// CMTime 自带 timescale，通常不需要手动转换
let pts = CMSampleBufferGetPresentationTimeStamp(sampleBuffer)

// 如果需要转换
let targetTimescale: Int32 = 90000
let convertedPts = CMTimeConvertScale(pts, timescale: targetTimescale, method: .default)

// 获取秒数
let seconds = CMTimeGetSeconds(pts)
```

### 7.2 音视频交错（Interleaving）

#### 为什么需要交错

```
不交错（糟糕）:
VVVVVVVVVVVVVVVV AAAAAAAAAA
→ 播放器需要先跳过所有视频才能播放音频
→ 网络传输时音频延迟严重

交错（正确）:
VVAVVAVVAVVAVVAVVA
→ 音视频同时到达
→ 播放器可以立即开始播放
```

#### FFmpeg 的交错写入

```c
// 推荐：av_interleaved_write_frame 自动处理交错
// 它会缓冲数据包，按 DTS 顺序写入
av_interleaved_write_frame(ofmt_ctx, pkt);

// 不推荐：av_write_frame 直接写入，不处理交错
// 只有在你确定数据已经按正确顺序排列时才使用
av_write_frame(ofmt_ctx, pkt);
```

#### Android MediaMuxer 的行为

```kotlin
// MediaMuxer 会自动处理交错
// 你可以按任意顺序调用 writeSampleData
muxer.writeSampleData(videoTrackIndex, buffer, bufferInfo)
muxer.writeSampleData(audioTrackIndex, buffer, bufferInfo)

// 但是！同一轨道内的数据必须按 PTS 升序写入
// 否则会抛出异常
```

### 7.3 大文件处理

#### MP4 的 64 位偏移

```c
// MP4 的 stco (Sample Table Chunk Offset) 使用 32 位偏移
// 超过 4GB 时会溢出

// 解决方案：FFmpeg 自动检测并使用 co64
// 或者显式设置
AVDictionary *opts = NULL;
av_dict_set(&opts, "use64bitoffset", "1", 0);
avformat_write_header(ofmt_ctx, &opts);
```

#### fMP4 分段写入

```c
// fMP4（Fragmented MP4）将数据分段写入
// 每个段独立，不需要在文件末尾回写完整索引

AVDictionary *opts = NULL;
av_dict_set(&opts, "movflags", "frag_keyframe+empty_moov", 0);
// frag_keyframe: 每个关键帧开始新分段
// empty_moov: moov 原子不包含 sample 信息

avformat_write_header(ofmt_ctx, &opts);
```

#### 内存管理策略

| 场景 | 问题 | 解决方案 |
|------|------|---------|
| 录制超长视频 | 索引表内存溢出 | 使用 fMP4 或 TS |
| 读取超大文件 | 一次性加载索引 | 分块读取，lazy loading |
| 移动端录制 | 内存紧张 | 限制缓冲区大小，及时释放 |

### 7.4 错误处理最佳实践

```c
// FFmpeg 错误处理模板
int ret;

ret = avformat_open_input(&ifmt_ctx, filename, NULL, NULL);
if (ret < 0) {
    char errbuf[AV_ERROR_MAX_STRING_SIZE];
    av_strerror(ret, errbuf, sizeof(errbuf));
    fprintf(stderr, "无法打开文件: %s (错误码: %d)\n", errbuf, ret);
    
    // 常见错误码
    switch (ret) {
        case AVERROR(ENOENT):
            fprintf(stderr, "文件不存在\n");
            break;
        case AVERROR(EACCES):
            fprintf(stderr, "权限不足\n");
            break;
        case AVERROR_INVALIDDATA:
            fprintf(stderr, "文件格式无效\n");
            break;
        case AVERROR_PROTOCOL_NOT_FOUND:
            fprintf(stderr, "协议不支持\n");
            break;
    }
    
    goto cleanup;
}

cleanup:
    // 确保所有资源都被释放
    avformat_close_input(&ifmt_ctx);
```

### A.1 FFmpeg 获取视频时长

```c
int64_t get_video_duration_ms(const char *filename) {
    AVFormatContext *fmt_ctx = NULL;
    int64_t duration_ms = -1;
    
    if (avformat_open_input(&fmt_ctx, filename, NULL, NULL) < 0) {
        return -1;
    }
    
    if (avformat_find_stream_info(fmt_ctx, NULL) >= 0) {
        // duration 单位是 AV_TIME_BASE (微秒)
        if (fmt_ctx->duration != AV_NOPTS_VALUE) {
            duration_ms = fmt_ctx->duration / (AV_TIME_BASE / 1000);
        }
    }
    
    avformat_close_input(&fmt_ctx);
    return duration_ms;
}
```
