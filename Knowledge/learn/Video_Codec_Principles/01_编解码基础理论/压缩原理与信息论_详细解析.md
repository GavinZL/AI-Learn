# 压缩原理与信息论

> 从信息熵到率失真理论——理解压缩的极限与权衡

---

## 核心结论（TL;DR）

**数据压缩的本质是消除冗余，信息论告诉我们压缩的理论极限在哪里，以及如何在质量与码率之间做最优权衡。**

关键要点：
1. 信息熵定义了无损压缩的理论下限——不可能压缩到比熵更小
2. 视频中存在四类冗余：空间、时间、统计、感知——编解码逐一消除
3. 有损压缩利用人眼不敏感的特性，丢弃"不重要"的信息
4. 率失真理论指导编码器在质量和码率之间做最优决策

## 1. 信息论基础：什么是"信息"？

### 1.1 信息量的直觉理解

**问题**：下面两句话，哪句包含更多"信息"？
- A："明天太阳从东边升起"
- B："明天太阳从西边升起"

**答案**：B 的信息量更大。

**原因**：信息量与"意外程度"成正比。越不可能发生的事件，一旦发生，传递的信息越多。

### 1.2 信息量的数学定义

对于概率为 $p$ 的事件，其信息量（自信息）定义为：

$$I(x) = -\log_2 p(x) \text{ bits}$$

```python
import numpy as np

def self_information(prob):
    """计算自信息（比特）"""
    return -np.log2(prob)

# 示例
print(f"50% 概率事件: {self_information(0.5):.2f} bits")   # 1 bit
print(f"25% 概率事件: {self_information(0.25):.2f} bits")  # 2 bits
print(f"1% 概率事件: {self_information(0.01):.2f} bits")   # 6.64 bits
```

**直觉**：
- 必然事件（p=1）：$I = 0$ bits，不传递任何新信息
- 抛硬币（p=0.5）：$I = 1$ bit
- 罕见事件（p→0）：$I → ∞$

### 1.3 信息熵：平均信息量

对于随机变量 $X$，其熵（Entropy）定义为所有可能结果的信息量的期望：

$$H(X) = -\sum_{x} p(x) \log_2 p(x)$$

```python
def entropy(probs):
    """计算离散分布的熵"""
    probs = np.array(probs)
    probs = probs[probs > 0]  # 忽略零概率
    return -np.sum(probs * np.log2(probs))

# 公平硬币
fair_coin = [0.5, 0.5]
print(f"公平硬币熵: {entropy(fair_coin):.2f} bits")  # 1.0 bit

# 偏置硬币
biased_coin = [0.9, 0.1]
print(f"偏置硬币熵: {entropy(biased_coin):.2f} bits")  # 0.47 bits

# 完全确定
certain = [1.0, 0.0]
print(f"确定结果熵: {entropy(certain):.2f} bits")  # 0 bits
```

**关键洞察**：
- 熵是不确定性的度量
- 分布越均匀，熵越大
- 分布越集中，熵越小（更可预测 = 更可压缩）

### 1.4 Shannon 编码定理

**无损数据压缩定理**：

> 对于独立同分布的信源，平均每个符号至少需要 $H(X)$ bits 来无损表示。

这意味着：
- 熵是压缩的理论下限
- 无论多巧妙的压缩算法，都不可能突破熵的限制

```python
# 图像像素值的熵分析
def image_entropy(image):
    """计算图像的一阶熵"""
    hist, _ = np.histogram(image.flatten(), bins=256, range=(0, 256))
    probs = hist / hist.sum()
    return entropy(probs)

# 自然图像熵约 6-7 bits/pixel（比 8 bits 有压缩空间）
# 纯噪声图像熵接近 8 bits/pixel（几乎不可压缩）
```

## 2. 视频中的四类冗余

视频压缩的核心就是识别并消除冗余。按照 MECE 原则，冗余可分为四类：

### 2.1 空间冗余（Spatial Redundancy）

**定义**：同一帧内相邻像素高度相关

```
自然图像中的空间冗余示例：
┌────────────────────┐
│  蓝  蓝  蓝  蓝  蓝 │  ← 天空区域：像素几乎相同
│  蓝  蓝  蓝  蓝  蓝 │
│  绿  绿  绿  绿  绿 │  ← 草地区域：颜色相近
│  绿  绿  绿  绿  绿 │
└────────────────────┘
```

**量化**：相邻像素的相关系数通常 > 0.9

```python
def spatial_correlation(image):
    """计算水平相邻像素的相关性"""
    left = image[:, :-1].flatten()
    right = image[:, 1:].flatten()
    return np.corrcoef(left, right)[0, 1]

# 自然图像的相关性通常 > 0.9
# 这意味着知道一个像素，可以很好地预测相邻像素
```

**消除方式**：
- 帧内预测（Intra Prediction）
- 变换编码（DCT/DST）

### 2.2 时间冗余（Temporal Redundancy）

**定义**：相邻帧之间大部分内容相同

```
相邻两帧的差异（运动区域很小）：
帧 N:                帧 N+1:              差异:
┌─────────────┐      ┌─────────────┐      ┌─────────────┐
│  背景  ●    │      │  背景   ●   │      │       ←     │
│  背景       │  →   │  背景       │  =   │             │
│  背景       │      │  背景       │      │             │
└─────────────┘      └─────────────┘      └─────────────┘
                     （球移动了一点）       （只有球移动的痕迹）
```

**量化**：典型视频相邻帧差异 < 5%

**消除方式**：
- 帧间预测（Inter Prediction）
- 运动估计与补偿（Motion Estimation & Compensation）

### 2.3 统计冗余（Statistical Redundancy）

**定义**：符号出现的概率不均匀

```python
# 自然图像的像素值分布通常不均匀
# 例如，某张图像的像素分布：
# 值 0-50:   5%
# 值 50-100: 15%
# 值 100-150: 40%  ← 集中在中间范围
# 值 150-200: 30%
# 值 200-255: 10%

# 如果所有值等概率出现，需要 log2(256) = 8 bits/pixel
# 但实际分布的熵可能只有 6 bits/pixel
```

**消除方式**：
- 熵编码（Entropy Coding）
- Huffman 编码
- 算术编码
- CABAC/CAVLC

### 2.4 感知冗余（Perceptual Redundancy）

**定义**：人眼对某些信息不敏感

```
人眼的感知特性：
1. 对亮度比色度敏感 → 4:2:0 子采样
2. 对低频比高频敏感 → 丢弃高频系数
3. 对比敏感度有限   → 量化
4. 存在掩蔽效应    → 纹理区域可更粗糙量化
```

**消除方式**：
- 量化（Quantization）
- 色度子采样
- 感知优化的码率控制

## 3. 无损压缩技术

### 3.1 游程编码（Run-Length Encoding, RLE）

**原理**：将连续重复的符号用"符号+长度"表示

```python
def rle_encode(data):
    """游程编码"""
    encoded = []
    i = 0
    while i < len(data):
        count = 1
        while i + count < len(data) and data[i + count] == data[i]:
            count += 1
        encoded.append((data[i], count))
        i += count
    return encoded

def rle_decode(encoded):
    """游程解码"""
    return ''.join(char * count for char, count in encoded)

# 示例
original = "AAAAAABBBBCCCCCCCC"
encoded = rle_encode(original)  # [('A', 6), ('B', 4), ('C', 8)]
# 原始: 18 字符 → 编码后: 6 个 (char, count) 对
```

**应用场景**：适合有大量连续重复的数据（如位图、量化后的零系数）

### 3.2 字典编码（LZ77/LZ78/LZW）

**原理**：用"回溯距离+长度"替代重复出现的字符串

```
LZ77 示例：
原始: "ABCABCABC"
编码: ABC + (距离=3, 长度=6)
解释: "ABC" + 复制前3个位置的6个字符
```

**应用**：ZIP、gzip、PNG 等

### 3.3 Huffman 编码

**原理**：高频符号用短码，低频符号用长码

```python
import heapq
from collections import Counter

def build_huffman_tree(freq):
    """构建 Huffman 树"""
    heap = [[count, [symbol, ""]] for symbol, count in freq.items()]
    heapq.heapify(heap)
    
    while len(heap) > 1:
        lo = heapq.heappop(heap)
        hi = heapq.heappop(heap)
        for pair in lo[1:]:
            pair[1] = '0' + pair[1]
        for pair in hi[1:]:
            pair[1] = '1' + pair[1]
        heapq.heappush(heap, [lo[0] + hi[0]] + lo[1:] + hi[1:])
    
    return sorted(heap[0][1:], key=lambda p: (len(p[1]), p[0]))

# 示例
text = "AAAAABBBCCDE"
freq = Counter(text)  # {'A': 5, 'B': 3, 'C': 2, 'D': 1, 'E': 1}
codes = build_huffman_tree(freq)
# 结果: A='0', B='10', C='110', D='1110', E='1111'
# A 最频繁，只用 1 bit
```

**特点**：
- 前缀码，无歧义解码
- 平均码长接近熵
- 码长必须是整数比特（限制）

### 3.4 算术编码（Arithmetic Coding）

**原理**：将整个消息映射到 [0,1) 区间的一个子区间

```python
def arithmetic_encode_simple(symbols, probs, message):
    """简化的算术编码演示"""
    # 构建累积概率表
    cum_probs = {}
    total = 0
    for s, p in zip(symbols, probs):
        cum_probs[s] = (total, total + p)
        total += p
    
    # 编码
    low, high = 0.0, 1.0
    for char in message:
        range_size = high - low
        char_low, char_high = cum_probs[char]
        high = low + range_size * char_high
        low = low + range_size * char_low
    
    return (low + high) / 2  # 返回区间中点

# 示例
symbols = ['A', 'B', 'C']
probs = [0.5, 0.3, 0.2]
code = arithmetic_encode_simple(symbols, probs, "ABC")
# 输出一个 [0,1) 之间的小数
```

**优势**：
- 理论上可以达到熵的极限
- 码长可以是小数比特
- 现代视频编解码的核心技术

## 4. 有损压缩：可接受的质量损失

### 4.1 为什么需要有损压缩？

**无损压缩的局限**：
- 自然图像的熵约 6-7 bits/pixel
- 无损压缩比最多 1.3-1.5:1
- 远远不够视频传输需求（需要 100:1+）

**有损压缩的核心思想**：
丢弃人眼"不太需要"的信息

### 4.2 JPEG 压缩流程（入门示例）

```
原始图像 → [色彩空间转换] → [下采样] → [DCT] → [量化] → [熵编码] → 压缩数据
          RGB→YCbCr      4:2:0    频域变换   丢弃高频    Huffman
```

```python
from scipy.fftpack import dct, idct
import numpy as np

def jpeg_like_compress(block_8x8, quality=50):
    """JPEG 风格的 8x8 块压缩"""
    
    # 标准 JPEG 量化表（亮度）
    Q = np.array([
        [16, 11, 10, 16, 24, 40, 51, 61],
        [12, 12, 14, 19, 26, 58, 60, 55],
        [14, 13, 16, 24, 40, 57, 69, 56],
        [14, 17, 22, 29, 51, 87, 80, 62],
        [18, 22, 37, 56, 68,109,103, 77],
        [24, 35, 55, 64, 81,104,113, 92],
        [49, 64, 78, 87,103,121,120,101],
        [72, 92, 95, 98,112,100,103, 99]
    ])
    
    # 根据质量因子缩放量化表
    if quality < 50:
        scale = 5000 / quality
    else:
        scale = 200 - 2 * quality
    Q_scaled = np.clip((Q * scale + 50) / 100, 1, 255)
    
    # DCT 变换
    dct_block = dct(dct(block_8x8.T, norm='ortho').T, norm='ortho')
    
    # 量化（核心有损步骤）
    quantized = np.round(dct_block / Q_scaled)
    
    # 反量化
    dequantized = quantized * Q_scaled
    
    # 逆 DCT
    reconstructed = idct(idct(dequantized.T, norm='ortho').T, norm='ortho')
    
    return quantized, reconstructed

# 量化后，大量高频系数变为 0，便于压缩
```

### 4.3 量化：有损压缩的核心

量化 = 用较少的离散值表示连续范围

```python
def quantize(value, step):
    """标量量化"""
    return np.round(value / step) * step

# 示例：step=10 的量化
original = [0, 3, 7, 12, 18, 23, 27, 35]
quantized = [quantize(v, 10) for v in original]
# [0, 0, 10, 10, 20, 20, 30, 40]

# 信息丢失了，但数值种类减少，更易压缩
```

**量化步长（QP）的影响**：
- QP 越大 → 量化越粗 → 压缩率越高 → 质量越差
- QP 越小 → 量化越细 → 压缩率越低 → 质量越好

## 5. 率失真理论：质量与码率的权衡

### 5.1 率失真函数

率失真函数 $R(D)$ 定义了达到失真 $D$ 所需的最小码率 $R$：

$$R(D) = \min_{p(\hat{x}|x): E[d(x,\hat{x})] \leq D} I(X; \hat{X})$$

```
        码率 R
          ↑
          │     ╱ 高质量区域
          │    ╱  （边际收益递减）
          │   ╱
          │  ╱  R(D) 曲线
          │ ╱
          │╱____________________→ 失真 D
               低质量区域
               （压缩极限）
```

**直觉理解**：
- 要求质量越高（D 越小），需要的码率越大
- 存在理论下限，无论多高码率都无法完全消除失真
- 曲线呈凸形，边际收益递减

### 5.2 失真度量

**均方误差（MSE）**：
$$MSE = \frac{1}{N}\sum_{i}(x_i - \hat{x}_i)^2$$

**峰值信噪比（PSNR）**：
$$PSNR = 10 \log_{10}\left(\frac{MAX^2}{MSE}\right) \text{ dB}$$

```python
def mse(original, compressed):
    """均方误差"""
    return np.mean((original.astype(float) - compressed.astype(float)) ** 2)

def psnr(original, compressed, max_val=255):
    """峰值信噪比"""
    m = mse(original, compressed)
    if m == 0:
        return float('inf')
    return 10 * np.log10(max_val ** 2 / m)

# PSNR 参考值：
# > 40 dB: 优秀，几乎看不出差异
# 30-40 dB: 良好，仔细看能发现差异
# 20-30 dB: 一般，明显可见差异
# < 20 dB: 差，严重失真
```

**SSIM（结构相似性）**：
更符合人眼感知的质量度量

```python
from skimage.metrics import structural_similarity as ssim

# SSIM 考虑亮度、对比度、结构三个因素
similarity = ssim(original, compressed)
# 范围 [0, 1]，越大越好
```

### 5.3 拉格朗日优化

编码器的核心决策问题：如何在给定码率下最大化质量？

**拉格朗日成本函数**：
$$J = D + \lambda R$$

其中：
- $D$：失真（如 MSE）
- $R$：码率（比特数）
- $\lambda$：拉格朗日乘子（平衡因子）

```python
def lagrangian_cost(distortion, rate, lambda_val):
    """拉格朗日成本"""
    return distortion + lambda_val * rate

# 编码器决策示例：选择最优量化参数
def find_optimal_qp(block, lambda_val, qp_range=range(1, 52)):
    """寻找最优 QP"""
    best_cost = float('inf')
    best_qp = 1
    
    for qp in qp_range:
        # 编码
        quantized, reconstructed = encode_block(block, qp)
        
        # 计算失真和码率
        distortion = mse(block, reconstructed)
        rate = estimate_bits(quantized)
        
        # 拉格朗日成本
        cost = lagrangian_cost(distortion, rate, lambda_val)
        
        if cost < best_cost:
            best_cost = cost
            best_qp = qp
    
    return best_qp
```

### 5.4 λ 与 QP 的关系

在实际编码器中，λ 通常由 QP 决定：

$$\lambda = c \cdot 2^{(QP-12)/3}$$

其中 $c$ 是经验常数。

## 6. 视频编解码的压缩策略总览

现代视频编解码遵循统一的框架：

```
原始视频
    │
    ▼
┌─────────────────────────────────────────────────────────┐
│ 1. 预测（Prediction）                                    │
│    - 帧内预测：消除空间冗余                               │
│    - 帧间预测：消除时间冗余                               │
│    输出：预测残差（原始 - 预测）                          │
└─────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────┐
│ 2. 变换（Transform）                                     │
│    - DCT/DST：能量集中到少数系数                          │
│    输出：变换系数                                        │
└─────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────┐
│ 3. 量化（Quantization）                                  │
│    - 丢弃不重要的高频系数                                 │
│    - 消除感知冗余                                        │
│    输出：量化系数（很多为 0）                             │
└─────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────┐
│ 4. 熵编码（Entropy Coding）                              │
│    - CABAC/CAVLC：利用统计特性                           │
│    - 消除统计冗余                                        │
│    输出：压缩比特流                                      │
└─────────────────────────────────────────────────────────┘
    │
    ▼
压缩视频流
```

### 6.1 各阶段的压缩贡献

| 阶段 | 消除的冗余 | 是否有损 | 压缩贡献 |
|-----|-----------|---------|---------|
| 预测 | 空间/时间 | 无损 | 60-80% |
| 变换 | 能量集中 | 无损 | 准备工作 |
| 量化 | 感知 | **有损** | 20-40% |
| 熵编码 | 统计 | 无损 | 10-20% |

### 6.2 编码模式决策

编码器需要做大量决策：

```python
class EncoderDecision:
    """编码器需要决策的内容"""
    
    # 分块决策

    inter_modes = ['Skip', 'Merge', 'Inter', 'Bi']

    transform_types = ['DCT-II', 'DST-VII', 'Skip']
    
    # 量化决策
    qp_range = range(0, 52)  # QP 0-51
    
    # 所有这些决策都通过率失真优化来选择
```

## 7. 压缩效率的评估

### 7.1 BD-Rate

BD-Rate（Bjøntegaard Delta Rate）是比较编码器效率的标准指标：

```
BD-Rate = -10% 表示：在相同质量下，新编码器节省 10% 码率
BD-Rate = +5% 表示：在相同质量下，新编码器多用 5% 码率
```

### 7.2 编码标准的演进

| 标准 | 年份 | 相对 H.264 的 BD-Rate |
|-----|------|----------------------|
| H.264/AVC | 2003 | 基准 |
| H.265/HEVC | 2013 | -50% |
| AV1 | 2018 | -30%（相对 HEVC）|

## 8. 小结

```
信息论基础
├── 信息量：I(x) = -log₂p(x)
├── 熵：H(X) = -Σp(x)log₂p(x)
└── Shannon 定理：压缩下限 = 熵

四类冗余
├── 空间冗余 → 帧内预测
├── 时间冗余 → 帧间预测
├── 统计冗余 → 熵编码
└── 感知冗余 → 量化

压缩策略
├── 无损：RLE, LZ77, Huffman, 算术编码
└── 有损：变换 + 量化

率失真优化
├── R(D) 曲线
├── J = D + λR
└── 编码器决策的核心
```
