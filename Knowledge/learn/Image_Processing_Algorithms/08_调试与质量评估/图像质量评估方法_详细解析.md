## 1. 图像质量评估概述

### 1.1 什么是图像质量评估

**图像质量评估（Image Quality Assessment, IQA）** 是量化图像视觉质量的技术，用于衡量图像处理算法的效果、编码压缩的失真程度、以及成像系统的性能表现。

```
图像质量评估的典型应用场景：

┌─────────────────────────────────────────────────────────────────┐
│                    图像/视频处理流水线                           │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   原始图像 ──→ [处理算法] ──→ 处理后图像 ──→ [质量评估] ──→ 分数  │
│      │                           │               │              │
│      │         ┌─────────────────┘               │              │
│      │         │                                 │              │
│      └─────────┼─────→ 参考图像 ─────────────────┘              │
│                │       (Ground Truth)                           │
│                │                                                │
│   应用场景：    │                                                │
│   • 编码质量控制   • 降噪效果评估   • 超分辨率验证              │
│   • ISP调优       • 算法对比       • A/B测试                    │
└─────────────────────────────────────────────────────────────────┘
```

### 1.2 评估方法分类

根据是否需要参考图像，图像质量评估分为三大类：

#### 全参考（Full-Reference, FR）

需要原始无失真图像作为参考，将待测图像与参考图像逐像素比较。

| 方法 | 描述 | 特点 |
|------|------|------|
| PSNR | 峰值信噪比 | 计算简单，广泛使用 |
| SSIM | 结构相似性 | 符合人眼感知特性 |
| VMAF | 视频多方法融合 | Netflix开发，高精度 |
| LPIPS | 深度学习感知相似度 | 基于神经网络特征 |

#### 半参考（Reduced-Reference, RR）

只需要参考图像的部分特征信息，而非完整图像。

```
半参考评估原理：

参考图像 ──→ [特征提取] ──→ 特征向量F₁ ─┐
                                        ├──→ [比较] ──→ 质量分数
待测图像 ──→ [特征提取] ──→ 特征向量F₂ ─┘

特征类型：
• 统计特征（直方图、矩等）
• 频域特征（小波系数分布）
• 结构特征（边缘、纹理）
```

#### 无参考（No-Reference, NR）

不需要参考图像，直接从待测图像本身提取特征评估质量。

| 方法 | 全称 | 适用场景 |
|------|------|----------|
| NIQE | Natural Image Quality Evaluator | 自然图像通用评估 |
| BRISQUE | Blind/Referenceless Image Spatial Quality | 空域失真检测 |
| NRQM | No-Reference Quality Metric | 超分辨率质量 |
| PIQE | Perception-based Image Quality | 感知质量评估 |

### 1.3 主观评价与客观评价

```
主观评价 vs 客观评价对比：

┌──────────────────────────────────────────────────────────────┐
│                        主观评价                               │
├──────────────────────────────────────────────────────────────┤
│  方法：让人类观察者对图像质量打分                              │
│  优点：直接反映人眼感知，最具可信度                            │
│  缺点：耗时、成本高、难以大规模应用、结果有主观性               │
│  标准：ITU-R BT.500、ITU-T P.910                              │
│  指标：MOS（Mean Opinion Score）、DMOS（Differential MOS）     │
└──────────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────────┐
│                        客观评价                               │
├──────────────────────────────────────────────────────────────┤
│  方法：使用数学模型/算法自动计算质量分数                        │
│  优点：快速、可重复、适合大规模自动化测试                       │
│  缺点：可能与人眼感知不完全一致                                │
│  目标：设计与主观评价高度相关的客观指标                         │
│  验证：通过与MOS的相关系数（PLCC/SROCC）评估指标有效性          │
└──────────────────────────────────────────────────────────────┘
```

## 2. PSNR峰值信噪比

### 2.1 数学定义

**PSNR（Peak Signal-to-Noise Ratio）** 是最广泛使用的图像质量指标，基于像素级误差计算。

#### 均方误差（MSE）

```
MSE = (1 / (M × N)) × Σᵢ Σⱼ [I(i,j) - K(i,j)]²

其中：
  I(i,j) = 参考图像在位置(i,j)的像素值
  K(i,j) = 待测图像在位置(i,j)的像素值
  M × N  = 图像尺寸（高×宽）
```

#### PSNR公式

```
PSNR = 10 × log₁₀(MAX² / MSE)
     = 20 × log₁₀(MAX / √MSE)
     = 20 × log₁₀(MAX / RMSE)

其中：
  MAX = 像素最大值（8位图像为255，10位为1023）
  RMSE = √MSE（均方根误差）

单位：dB（分贝）
```

### 2.2 PSNR取值范围与解释

```
PSNR 质量参考表：

PSNR (dB)     质量描述               典型应用场景
─────────────────────────────────────────────────────
> 50          几乎无损，人眼无法区分    无损/近无损压缩
40-50         高质量，差异难以察觉      高质量编码(CRF 18-22)
35-40         良好质量，仔细看有差异    标准质量编码(CRF 23-28)
30-35         可接受质量，差异明显      低码率编码
25-30         质量较差，失真可见        极低码率/移动网络
< 25          质量很差，严重失真        不推荐使用
─────────────────────────────────────────────────────

注意：
• 相同PSNR在不同内容上感知质量可能差异很大
• PSNR每增加6dB，MSE减少为1/4
• PSNR每增加3dB，MSE减少为1/2
```

### 2.3 分通道PSNR计算

对于彩色图像，可计算各通道独立的PSNR：

```
分通道PSNR计算：

1. 各通道独立计算：
   PSNR_R = 10 × log₁₀(255² / MSE_R)
   PSNR_G = 10 × log₁₀(255² / MSE_G)  
   PSNR_B = 10 × log₁₀(255² / MSE_B)

2. 彩色图像综合PSNR（方法一）：
   MSE_avg = (MSE_R + MSE_G + MSE_B) / 3
   PSNR_color = 10 × log₁₀(255² / MSE_avg)

3. 亮度加权PSNR（方法二）：
   Y = 0.299×R + 0.587×G + 0.114×B
   MSE_Y = (1/MN) × Σ[Y_ref - Y_test]²
   PSNR_Y = 10 × log₁₀(255² / MSE_Y)

YUV格式PSNR：
   PSNR_YUV = (6×PSNR_Y + PSNR_U + PSNR_V) / 8
   （加权考虑人眼对亮度更敏感）
```

### 2.4 PSNR的局限性

```
PSNR的主要问题：

┌─────────────────────────────────────────────────────────────┐
│  问题1：对空间位置不敏感                                      │
│  ──────────────────────────                                  │
│  整体平移1像素和局部严重失真可能有相同MSE                      │
│  但感知质量完全不同                                          │
├─────────────────────────────────────────────────────────────┤
│  问题2：对失真类型不区分                                      │
│  ──────────────────────────                                  │
│  模糊、噪声、块效应等不同失真可能PSNR相同                      │
│  但人眼对它们的敏感度不同                                     │
├─────────────────────────────────────────────────────────────┤
│  问题3：对内容不敏感                                          │
│  ──────────────────────────                                  │
│  平坦区域和纹理区域的相同误差，PSNR贡献相同                    │
│  但人眼在纹理区域对误差不太敏感                               │
├─────────────────────────────────────────────────────────────┤
│  问题4：与主观感知相关性有限                                   │
│  ──────────────────────────                                  │
│  特别是在低码率压缩场景                                       │
│  PSNR高不一定看起来好                                         │
└─────────────────────────────────────────────────────────────┘

示例：两图PSNR相同但感知质量差异大

图A：轻微模糊    →  PSNR = 32 dB  →  主观感觉还可以
图B：明显块效应  →  PSNR = 32 dB  →  主观感觉很差
```

### 2.5 PSNR计算示例

```python
import numpy as np

def calculate_psnr(img_ref, img_test, max_val=255.0):
    """
    计算PSNR
    
    参数:
        img_ref: 参考图像 (numpy array)
        img_test: 待测图像 (numpy array)
        max_val: 像素最大值
    
    返回:
        PSNR值 (dB)
    """
    # 确保数据类型为浮点数
    img_ref = img_ref.astype(np.float64)
    img_test = img_test.astype(np.float64)
    
    # 计算MSE
    mse = np.mean((img_ref - img_test) ** 2)
    
    if mse == 0:
        return float('inf')  # 完全相同
    
    # 计算PSNR
    psnr = 10 * np.log10((max_val ** 2) / mse)
    
    return psnr

def calculate_psnr_yuv(y_ref, y_test, u_ref, u_test, v_ref, v_test):
    """计算YUV加权PSNR"""
    psnr_y = calculate_psnr(y_ref, y_test)
    psnr_u = calculate_psnr(u_ref, u_test)
    psnr_v = calculate_psnr(v_ref, v_test)
    
    # 加权平均（Y权重更高）
    psnr_yuv = (6 * psnr_y + psnr_u + psnr_v) / 8
    
    return psnr_yuv, psnr_y, psnr_u, psnr_v
```

## 3. SSIM结构相似性

### 3.1 SSIM原理

**SSIM（Structural Similarity Index）** 由Wang等人于2004年提出，基于人眼视觉系统对结构信息敏感的特性设计。

```
SSIM设计理念：

人眼视觉系统特点：
├── 对亮度变化敏感 ───→ 亮度比较分量
├── 对对比度变化敏感 ──→ 对比度比较分量
└── 对结构信息敏感 ───→ 结构比较分量

SSIM = f(亮度相似性, 对比度相似性, 结构相似性)
```

### 3.2 三分量公式推导

#### 亮度比较（Luminance）

```
亮度分量衡量两图像的平均亮度差异：

μₓ = (1/N) × Σᵢ xᵢ    （图像x的均值）
μᵧ = (1/N) × Σᵢ yᵢ    （图像y的均值）

l(x,y) = (2μₓμᵧ + C₁) / (μₓ² + μᵧ² + C₁)

其中 C₁ = (K₁ × L)²，K₁ = 0.01，L = 255（动态范围）
```

#### 对比度比较（Contrast）

```
对比度分量衡量两图像的亮度波动差异：

σₓ = √[(1/N) × Σᵢ (xᵢ - μₓ)²]    （图像x的标准差）
σᵧ = √[(1/N) × Σᵢ (yᵢ - μᵧ)²]    （图像y的标准差）

c(x,y) = (2σₓσᵧ + C₂) / (σₓ² + σᵧ² + C₂)

其中 C₂ = (K₂ × L)²，K₂ = 0.03
```

#### 结构比较（Structure）

```
结构分量衡量两图像归一化后的相关性：

σₓᵧ = (1/N) × Σᵢ (xᵢ - μₓ)(yᵢ - μᵧ)    （协方差）

s(x,y) = (σₓᵧ + C₃) / (σₓσᵧ + C₃)

其中 C₃ = C₂ / 2
```

### 3.3 SSIM完整公式

```
SSIM综合公式：

SSIM(x,y) = [l(x,y)]^α × [c(x,y)]^β × [s(x,y)]^γ

标准设置 α = β = γ = 1 时：

SSIM(x,y) = (2μₓμᵧ + C₁)(2σₓᵧ + C₂)
            ─────────────────────────────────
            (μₓ² + μᵧ² + C₁)(σₓ² + σᵧ² + C₂)

取值范围：[-1, 1]
  • SSIM = 1：完全相同
  • SSIM = 0：完全不相关
  • SSIM < 0：负相关（罕见）

实际应用中通常 SSIM ∈ [0, 1]
```

### 3.4 SSIM Map与局部计算

```
SSIM局部计算（滑动窗口）：

┌─────────────────────────────────────────────────────────┐
│                                                         │
│   参考图像x            待测图像y                          │
│   ┌─────────┐          ┌─────────┐                      │
│   │    ┌───┐│          │    ┌───┐│                      │
│   │    │W₁││          │    │W₁││   → SSIM(W₁)          │
│   │    └───┘│          │    └───┘│                      │
│   │         │          │         │                      │
│   │  ┌───┐  │          │  ┌───┐  │                      │
│   │  │W₂ │  │          │  │W₂ │  │   → SSIM(W₂)         │
│   │  └───┘  │          │  └───┘  │                      │
│   └─────────┘          └─────────┘                      │
│                                                         │
│   窗口参数：                                              │
│   • 窗口大小：11×11（标准）或 8×8                          │
│   • 权重函数：高斯窗（σ = 1.5）                            │
│   • 步长：通常为1（逐像素滑动）                            │
│                                                         │
│   输出：                                                  │
│   • SSIM Map：每个位置的局部SSIM值                        │
│   • MSSIM：SSIM Map的平均值                              │
└─────────────────────────────────────────────────────────┘
```

### 3.5 MS-SSIM多尺度SSIM

```
MS-SSIM（Multi-Scale SSIM）：

在多个尺度上计算SSIM，更好模拟人眼多分辨率感知特性。

处理流程：
                    尺度1      尺度2      尺度3      尺度4      尺度5
原始图像 ─→ [降采样] ─→ 1/2 ─→ [降采样] ─→ 1/4 ─→ [降采样] ─→ 1/8 ─→ ...
    ↓           ↓           ↓           ↓           ↓
   l₁,c₁,s₁    l₂,c₂,s₂    l₃,c₃,s₃    l₄,c₄,s₄    l₅,c₅,s₅

MS-SSIM = ∏ⱼ₌₁ᴹ [cⱼ(x,y)]^βⱼ × [sⱼ(x,y)]^γⱼ × [lₘ(x,y)]^αₘ

标准权重（M=5）：
尺度    β       γ
─────────────────
1      0.0448  0.0448
2      0.2856  0.2856
3      0.3001  0.3001
4      0.2363  0.2363
5      0.1333  0.1333
```

### 3.6 SSIM Python实现

```python
import numpy as np
from scipy.ndimage import gaussian_filter

def calculate_ssim(img_ref, img_test, win_size=11, K1=0.01, K2=0.03, L=255):
    """
    计算SSIM
    
    参数:
        img_ref: 参考图像
        img_test: 待测图像
        win_size: 窗口大小
        K1, K2: 稳定常数
        L: 动态范围
    """
    C1 = (K1 * L) ** 2
    C2 = (K2 * L) ** 2
    C3 = C2 / 2
    
    img_ref = img_ref.astype(np.float64)
    img_test = img_test.astype(np.float64)
    
    # 计算均值（使用高斯滤波器）
    sigma = 1.5
    mu_x = gaussian_filter(img_ref, sigma)
    mu_y = gaussian_filter(img_test, sigma)
    
    # 计算方差和协方差
    sigma_x_sq = gaussian_filter(img_ref ** 2, sigma) - mu_x ** 2
    sigma_y_sq = gaussian_filter(img_test ** 2, sigma) - mu_y ** 2
    sigma_xy = gaussian_filter(img_ref * img_test, sigma) - mu_x * mu_y
    
    # 计算SSIM
    numerator = (2 * mu_x * mu_y + C1) * (2 * sigma_xy + C2)
    denominator = (mu_x ** 2 + mu_y ** 2 + C1) * (sigma_x_sq + sigma_y_sq + C2)
    
    ssim_map = numerator / denominator
    
    return np.mean(ssim_map), ssim_map
```

## 4. VMAF视频质量评估

### 4.1 VMAF概述

**VMAF（Video Multimethod Assessment Fusion）** 是Netflix开发的视频质量评估指标，通过机器学习融合多个基础特征。

```
VMAF设计目标：

┌─────────────────────────────────────────────────────────────┐
│                     传统指标的问题                           │
├─────────────────────────────────────────────────────────────┤
│  • PSNR对感知质量相关性低                                    │
│  • SSIM在极端场景表现不稳定                                  │
│  • 单一指标难以覆盖所有失真类型                               │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│                     VMAF解决方案                             │
├─────────────────────────────────────────────────────────────┤
│  • 提取多个感知相关的视觉特征                                 │
│  • 使用SVM/神经网络融合特征                                   │
│  • 基于大规模主观评测数据训练                                 │
│  • 与主观MOS高度相关                                         │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 VMAF核心特征

```
VMAF提取的视觉特征：

1. VIF（Visual Information Fidelity）
   ──────────────────────────────────
   • 基于自然场景统计和信息论
   • 在多个尺度上计算信息保真度
   • 对模糊和噪声敏感

2. DLM（Detail Loss Metric）  
   ──────────────────────────────────
   • 衡量细节信息的损失
   • 基于多尺度边缘检测
   • 对锐度变化敏感

3. Motion
   ──────────────────────────────────
   • 时域运动信息
   • 运动区域质量更重要
   • 考虑运动掩蔽效应

4. ADM（Anti-Aliasing Detail Metric）
   ──────────────────────────────────
   • 检测锯齿和混叠伪影
   • 用于评估抗锯齿处理效果
```

### 4.3 VMAF计算流程

```
VMAF处理流水线：

输入：参考视频 + 待测视频
                    ↓
┌─────────────────────────────────────────────────────────────┐
│                    预处理                                    │
│  • 格式转换（YUV420）                                        │
│  • 分辨率对齐                                                │
│  • 帧同步                                                   │
└─────────────────────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────────────────────┐
│                  逐帧特征提取                                │
│                                                             │
│  参考帧 ──┬──→ [VIF提取] ──→ vif_scale0, vif_scale1, ...    │
│          │                                                  │
│          ├──→ [ADM提取] ──→ adm_scale0, adm_scale1, ...    │
│          │                                                  │
│          └──→ [Motion]  ──→ motion_score                    │
│                                                             │
│  失真帧 ──┴──→ ...                                          │
└─────────────────────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────────────────────┐
│                   特征融合                                   │
│                                                             │
│  特征向量 ──→ [SVM/NN模型] ──→ VMAF分数 (0-100)              │
│                                                             │
│  模型类型：                                                  │
│  • vmaf_v0.6.1：默认模型，4K优化                             │
│  • vmaf_4k_v0.6.1：专为4K内容训练                            │
│  • vmaf_phone：移动设备观看场景                              │
└─────────────────────────────────────────────────────────────┘
                    ↓
输出：每帧VMAF分数 + 平均VMAF分数
```

### 4.4 VMAF分数解释

```
VMAF质量等级：

VMAF分数      质量描述               
────────────────────────────────────
93-100        优秀（Excellent）      
80-93         良好（Good）           
60-80         一般（Fair）           
40-60         较差（Poor）           
0-40          很差（Bad）            
────────────────────────────────────

与编码参数的关系（典型H.264编码）：
CRF 18 → VMAF ~95
CRF 23 → VMAF ~90
CRF 28 → VMAF ~80
CRF 35 → VMAF ~65
```

## 5. 感知质量指标

### 5.1 LPIPS深度学习感知相似度

**LPIPS（Learned Perceptual Image Patch Similarity）** 使用深度神经网络提取特征来评估感知相似度。

```
LPIPS原理：

┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   参考图像 ──→ [预训练CNN] ──→ 多层特征 F_ref               │
│                  (VGG/Alex)      L1, L2, ..., Ln            │
│                                      │                      │
│                                      ↓                      │
│                               [特征差异计算]                 │
│                                      ↑                      │
│                                      │                      │
│   待测图像 ──→ [预训练CNN] ──→ 多层特征 F_test              │
│                                                             │
└─────────────────────────────────────────────────────────────┘

LPIPS = Σₗ wₗ × ‖normalize(Fₗ_ref) - normalize(Fₗ_test)‖²

特点：
• 值越小表示越相似（与SSIM相反）
• 典型范围：0-1
• LPIPS < 0.1：非常相似
• LPIPS > 0.5：差异明显
```

### 5.2 无参考质量评估

#### NIQE（Natural Image Quality Evaluator）

```
NIQE原理：

1. 训练阶段（仅使用自然无失真图像）：
   自然图像集 ──→ [NSS特征提取] ──→ 多元高斯模型 MVG_natural

2. 评估阶段：
   待测图像 ──→ [NSS特征提取] ──→ 特征分布 MVG_test
                                       │
                                       ↓
                              [距离计算]
                                       ↓
                              NIQE分数（与自然图像的偏离度）

NSS特征：
• Mean Subtracted Contrast Normalized (MSCN) 系数
• 相邻MSCN系数乘积的分布参数
• 局部对比度特征

分数解释：
• 值越小越好（越接近自然图像）
• 典型范围：0-100
• NIQE < 3：高质量
• NIQE > 6：低质量
```

#### BRISQUE（Blind/Referenceless Image Spatial Quality）

```
BRISQUE特征：

1. MSCN系数计算：
   Î(i,j) = [I(i,j) - μ(i,j)] / [σ(i,j) + C]
   
   其中μ和σ为局部均值和标准差

2. 广义高斯分布拟合：
   对MSCN系数拟合GGD，提取形状参数α和尺度参数σ²

3. 相邻系数乘积的AGGD拟合：
   水平/垂直/对角方向的成对乘积
   提取η、σ_l²、σ_r²参数

4. SVM回归：
   36维特征向量 ──→ [SVM模型] ──→ BRISQUE分数

分数范围：0-100（越小越好）
```

## 6. 色彩准确性评估

### 6.1 色差公式

#### CIE76 ΔE*ab

```
最基础的色差公式：

ΔE*ab = √[(L*₁-L*₂)² + (a*₁-a*₂)² + (b*₁-b*₂)²]

在Lab色彩空间中的欧氏距离

ΔE*ab值      感知差异
───────────────────────────
< 1          不可察觉
1-2          仔细观察才能看出
2-3.5        能看出差异
3.5-5        明显差异
> 5          显著差异
───────────────────────────

局限性：Lab空间不是完全感知均匀的
```

#### CIEDE2000 (ΔE00)

```
更精确的感知色差公式：

ΔE₀₀ = √[(ΔL'/kₗSₗ)² + (ΔC'/kᶜSᶜ)² + (ΔH'/kₕSₕ)² + Rₜ(ΔC'/kᶜSᶜ)(ΔH'/kₕSₕ)]

改进点：
1. 亮度权重调整（Sₗ）
2. 色度权重调整（Sᶜ）
3. 色调权重调整（Sₕ）
4. 蓝色区域旋转项（Rₜ）
5. 中性色附近色度修正

参数设置：
• kₗ = kᶜ = kₕ = 1（标准条件）
• 纺织业：kₗ = 2

ΔE00 更符合人眼感知，是目前推荐的标准
```

### 6.2 ColorChecker评估流程

```
使用ColorChecker进行色彩准确性评估：

┌─────────────────────────────────────────────────────────────┐
│  步骤1：拍摄ColorChecker色卡                                  │
│  ────────────────────────                                   │
│  • 使用标准24色或140色ColorChecker                            │
│  • 确保均匀照明（D50或D65）                                   │
│  • 避免反光和阴影                                            │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│  步骤2：色块检测与提取                                        │
│  ────────────────────────                                   │
│  • 自动或手动检测色块位置                                     │
│  • 提取每个色块的平均RGB/Lab值                                │
│  • 排除边缘像素避免色溢                                       │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│  步骤3：色差计算                                              │
│  ────────────────────────                                   │
│  对每个色块计算：                                             │
│  • 拍摄值 vs 标准参考值                                       │
│  • ΔE*ab 或 ΔE00                                             │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│  步骤4：统计分析                                              │
│  ────────────────────────                                   │
│  • 平均ΔE（Mean ΔE）                                         │
│  • 最大ΔE（Max ΔE）                                          │
│  • ΔE95（95%分位数）                                         │
│  • 各色块的ΔE分布图                                          │
└─────────────────────────────────────────────────────────────┘

质量标准（ΔE00）：
• 优秀：平均ΔE < 1，最大ΔE < 3
• 良好：平均ΔE < 2，最大ΔE < 5  
• 可接受：平均ΔE < 3，最大ΔE < 8
```

## 7. 主观评价方法

### 7.1 MOS与DMOS

```
MOS（Mean Opinion Score）：

评分流程：
1. 招募评估者（通常20-30人）
2. 显示图像/视频
3. 评估者打分（5分制或100分制）
4. 计算平均分

5分制标准：
分数    质量描述
─────────────────
5       优秀（Excellent）
4       良好（Good）
3       一般（Fair）
2       差（Poor）
1       很差（Bad）

MOS = (1/N) × Σᵢ scoreᵢ
95%置信区间 = MOS ± 1.96 × σ/√N
```

```
DMOS（Differential MOS）：

衡量失真程度而非绝对质量：

DMOS = MOS_reference - MOS_distorted

• DMOS = 0：与参考无差异
• DMOS > 0：质量下降
• 值越大，失真越严重

优点：消除内容偏好的影响
```

### 7.2 ITU-R BT.500标准

ITU-R BT.500 主观评价标准：

## 8. 专项质量评估

### 8.1 降噪质量评估

```
降噪算法评估指标：

1. 降噪效果（噪声抑制程度）
   ─────────────────────────
   • PSNR提升：ΔPSNR = PSNR_denoised - PSNR_noisy
   • 噪声残留分析：σ_residual
   
2. 细节保留（避免过度平滑）
   ─────────────────────────
   • 边缘保持度：edge_preservation_index
   • 纹理保持度：texture_preservation_index
   • SSIM分量分析（结构分量s(x,y)）

3. 伪影检测
   ─────────────────────────
   • 块效应（blocking artifacts）
   • 振铃效应（ringing artifacts）
   • 阶梯效应（staircase effect）

综合评估流程：
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  无噪声原图  │    │   加噪图像   │    │  降噪结果   │
│ (Ground Truth)│    │   (Noisy)   │    │ (Denoised)  │
└──────┬──────┘    └──────┬──────┘    └──────┬──────┘
       │                  │                  │
       └──────────────────┼──────────────────┘
                          ↓
              ┌─────────────────────────┐
              │   质量指标计算            │
              │   • PSNR(GT, Denoised)   │
              │   • SSIM(GT, Denoised)   │
              │   • LPIPS(GT, Denoised)  │
              └─────────────────────────┘
```

### 8.2 Demosaic质量评估

```
Demosaic算法专用评估指标：

1. CPSNR（Color PSNR）
   ──────────────────
   分别计算RGB三通道PSNR，评估色彩重建准确性
   CPSNR = (PSNR_R + PSNR_G + PSNR_B) / 3

2. S-CIELAB
   ──────────────────
   空间扩展的CIELAB色差，考虑视觉空间滤波

3. 伪色检测
   ──────────────────
   • 高频区域色彩伪影（false color）
   • 边缘色溢（color fringing）
   • 拉链伪影（zipper artifact）

4. 分辨率损失
   ──────────────────
   • MTF50（50% MTF对应的空间频率）
   • 边缘锐度（edge sharpness）

测试图案：
• 星卡（Siemens Star）：检测方向依赖性
• 斜边（slanted edge）：计算MTF
• 高频纹理：检测伪色
```

### 8.3 HDR质量评估

```
HDR专用质量指标：

1. HDR-VDP-2（Visual Difference Predictor）
   ──────────────────────────────────────
   • 基于人眼视觉模型
   • 输出：可见差异概率图
   • 考虑亮度适应和对比度敏感度
   • 适用于宽动态范围场景

2. TMQI（Tone-Mapped Quality Index）
   ──────────────────────────────────────
   • 评估色调映射后的SDR图像质量
   • 包含结构保真度和统计自然度
   
   TMQI = a × S^α + (1-a) × N^β
   
   其中：
   S = 多尺度结构保真度
   N = 自然度（基于直方图）
   a, α, β = 权重参数

3. PU-PSNR / PU-SSIM
   ──────────────────────────────────────
   • 在感知均匀域计算PSNR/SSIM
   • 使用PU（Perceptual Uniformization）变换
   • 更好处理高动态范围

4. Delta E ITP
   ──────────────────────────────────────
   • ICtCp色彩空间中的色差
   • 专为HDR/WCG设计
   • 比Lab空间更感知均匀
```

### 9.3 Python评估脚本

```python
import cv2
import numpy as np
from skimage.metrics import structural_similarity as ssim
from skimage.metrics import peak_signal_noise_ratio as psnr

def evaluate_image_quality(ref_path, test_path):
    """
    综合图像质量评估
    """
    # 读取图像
    ref_img = cv2.imread(ref_path)
    test_img = cv2.imread(test_path)
    
    # 转换为RGB
    ref_rgb = cv2.cvtColor(ref_img, cv2.COLOR_BGR2RGB)
    test_rgb = cv2.cvtColor(test_img, cv2.COLOR_BGR2RGB)
    
    # 转换为灰度（用于某些指标）
    ref_gray = cv2.cvtColor(ref_img, cv2.COLOR_BGR2GRAY)
    test_gray = cv2.cvtColor(test_img, cv2.COLOR_BGR2GRAY)
    
    results = {}
    
    # PSNR
    results['psnr'] = psnr(ref_rgb, test_rgb)
    results['psnr_y'] = psnr(ref_gray, test_gray)
    
    # SSIM
    results['ssim'], _ = ssim(ref_gray, test_gray, full=True)
    
    # 分通道SSIM
    ssim_r, _ = ssim(ref_rgb[:,:,0], test_rgb[:,:,0], full=True)
    ssim_g, _ = ssim(ref_rgb[:,:,1], test_rgb[:,:,1], full=True)
    ssim_b, _ = ssim(ref_rgb[:,:,2], test_rgb[:,:,2], full=True)
    results['ssim_rgb'] = (ssim_r + ssim_g + ssim_b) / 3
    
    # MSE
    results['mse'] = np.mean((ref_rgb.astype(float) - test_rgb.astype(float)) ** 2)
    
    return results

def batch_evaluate(ref_dir, test_dir, output_csv):
    """批量评估目录中的图像"""
    import os
    import csv
    
    results = []
    ref_files = sorted(os.listdir(ref_dir))
    
    for filename in ref_files:
        ref_path = os.path.join(ref_dir, filename)
        test_path = os.path.join(test_dir, filename)
        
        if os.path.exists(test_path):
            metrics = evaluate_image_quality(ref_path, test_path)
            metrics['filename'] = filename
            results.append(metrics)
    
    # 写入CSV
    with open(output_csv, 'w', newline='') as f:
        writer = csv.DictWriter(f, fieldnames=results[0].keys())
        writer.writeheader()
        writer.writerows(results)
    
    return results

# 使用LPIPS（需要安装lpips库）
def evaluate_lpips(ref_path, test_path):
    """
    使用LPIPS评估感知相似度
    pip install lpips
    """
    import lpips
    import torch
    from PIL import Image
    from torchvision import transforms
    
    # 加载模型
    loss_fn = lpips.LPIPS(net='alex')
    
    # 预处理
    transform = transforms.Compose([
        transforms.ToTensor(),
        transforms.Normalize((0.5, 0.5, 0.5), (0.5, 0.5, 0.5))
    ])
    
    ref_img = transform(Image.open(ref_path)).unsqueeze(0)
    test_img = transform(Image.open(test_path)).unsqueeze(0)
    
    # 计算LPIPS
    with torch.no_grad():
        lpips_value = loss_fn(ref_img, test_img).item()
    
    return lpips_value
```
