## 1. 镜头阴影校正（LSC）

### 1.1 阴影形成原因

镜头阴影（Lens Shading / Vignetting）是指图像从中心向边缘亮度逐渐降低的现象。

```
阴影效果示意：

均匀光场输入 →  实际传感器输出
┌────────────────┐    ┌────────────────┐
│                │    │▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓│
│                │    │▓▓░░░░░░░░░░░░▓▓│
│    均匀白场    │ →  │▓░░░░░░░░░░░░░░▓│  边缘变暗
│                │    │▓░░░░░░░░░░░░░░▓│
│                │    │▓▓░░░░░░░░░░░░▓▓│
│                │    │▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓│
└────────────────┘    └────────────────┘
```

#### 1.1.1 余弦四次方定律（Cos⁴ Law）

对于理想薄透镜，离轴光线的衰减遵循：

$$
I(\theta) = I_0 \cdot \cos^4(\theta)
$$

其中 $\theta$ 是光线与光轴的夹角。

```
余弦四次方衰减原因分解：

cos⁴(θ) = cos(θ) × cos(θ) × cos(θ) × cos(θ)
           ↓         ↓         ↓         ↓
        光通量   入射角    像距    像面倾斜
        减少    余弦衰减  增大    投影面积
```

| 视场角 θ | cos⁴(θ) | 相对亮度 |
|---------|---------|----------|
| 0° | 1.000 | 100% |
| 15° | 0.870 | 87% |
| 30° | 0.563 | 56% |
| 45° | 0.250 | 25% |
| 60° | 0.063 | 6% |

#### 1.1.2 机械渐晕（Mechanical Vignetting）

```

         │  │███│   │   │  │
```

#### 1.1.3 CRA不匹配（Chief Ray Angle Mismatch）

```
微透镜CRA不匹配：

理想情况（CRA匹配）：          CRA不匹配：
       ↓光线                    ╲光线
   ┌───────┐                  ┌───────┐

```

传感器微透镜的设计CRA与镜头实际CRA不匹配时，边缘像素接收光子效率降低。

### 1.2 增益补偿算法

#### 1.2.1 基本原理

LSC通过对每个像素乘以补偿增益来恢复均匀亮度：

$$
I_{corrected}(x, y) = I_{raw}(x, y) \times G(x, y)
$$

其中增益 $G(x,y) \geq 1$，边缘增益大于中心。

#### 1.2.2 多项式拟合法

使用径向多项式拟合阴影分布：

$$
G(r) = 1 + a_1 r^2 + a_2 r^4 + a_3 r^6 + \ldots
$$

其中 $r = \sqrt{(x - x_c)^2 + (y - y_c)^2}$ 是到光学中心的距离。

```c
// 多项式LSC实现
void lsc_polynomial(uint16_t* raw, int W, int H,
                    float cx, float cy,  // 光学中心
                    float* coeffs, int num_coeffs) {
    float r_max = sqrtf((W/2)*(W/2) + (H/2)*(H/2));
    
    for (int y = 0; y < H; y++) {
        for (int x = 0; x < W; x++) {
            // 计算归一化半径
            float dx = x - cx;
            float dy = y - cy;
            float r = sqrtf(dx*dx + dy*dy) / r_max;  // 归一化到[0,1]
            
            // 计算多项式增益
            float gain = 1.0f;
            float r_power = r * r;  // r²
            for (int i = 0; i < num_coeffs; i++) {
                gain += coeffs[i] * r_power;
                r_power *= r * r;  // r⁴, r⁶, ...
            }
            
            // 应用增益
            int idx = y * W + x;
            float corrected = raw[idx] * gain;
            raw[idx] = (uint16_t)fminf(corrected, 65535.0f);
        }
    }
}
```

#### 1.2.3 网格插值法

将图像划分为网格，在网格节点处存储增益值，中间像素通过插值获得。

```
网格LSC原理：

标定增益网格（如 17×13）：
┌─────┬─────┬─────┬─────┬─────┐
│1.00 │1.02 │1.05 │1.02 │1.00 │
├─────┼─────┼─────┼─────┼─────┤
│1.03 │1.00 │1.00 │1.00 │1.03 │
├─────┼─────┼─────┼─────┼─────┤
│1.08 │1.02 │1.00 │1.02 │1.08 │
├─────┼─────┼─────┼─────┼─────┤
│1.03 │1.00 │1.00 │1.00 │1.03 │
├─────┼─────┼─────┼─────┼─────┤
│1.00 │1.02 │1.05 │1.02 │1.00 │
└─────┴─────┴─────┴─────┴─────┘

任意像素位置通过双线性插值获得增益
```

```c
// 网格插值LSC
typedef struct {
    int grid_w;        // 网格宽度（节点数）
    int grid_h;        // 网格高度
    float* gains_r;    // R通道增益表
    float* gains_gr;   // Gr通道增益表
    float* gains_gb;   // Gb通道增益表
    float* gains_b;    // B通道增益表
} LSC_Table;

void lsc_grid_interpolation(uint16_t* raw, int W, int H, LSC_Table* table) {
    float cell_w = (float)W / (table->grid_w - 1);
    float cell_h = (float)H / (table->grid_h - 1);
    
    for (int y = 0; y < H; y++) {
        for (int x = 0; x < W; x++) {
            // 确定所在网格单元
            float gx = x / cell_w;
            float gy = y / cell_h;
            int gx0 = (int)gx;
            int gy0 = (int)gy;
            int gx1 = gx0 + 1;
            int gy1 = gy0 + 1;
            
            // 插值权重
            float wx = gx - gx0;
            float wy = gy - gy0;
            
            // 选择对应颜色通道的增益表
            float* gains = select_gain_table(table, x, y);
            
            // 双线性插值
            float g00 = gains[gy0 * table->grid_w + gx0];
            float g01 = gains[gy0 * table->grid_w + gx1];
            float g10 = gains[gy1 * table->grid_w + gx0];
            float g11 = gains[gy1 * table->grid_w + gx1];
            
            float gain = (1-wx)*(1-wy)*g00 + wx*(1-wy)*g01 
                       + (1-wx)*wy*g10 + wx*wy*g11;
            
            // 应用增益
            int idx = y * W + x;
            raw[idx] = (uint16_t)fminf(raw[idx] * gain, 65535.0f);
        }
    }
}
```

### 1.3 各通道独立校正

由于不同波长光线的衰减特性不同，R/G/B需要独立校正：

```
各通道阴影差异：

         中心          边缘
         │              │
R通道 ───┼──────────────┼──▓▓▓  衰减较多
         │              │
G通道 ───┼──────────────┼──▓▓   中等衰减
         │              │
B通道 ───┼──────────────┼──▓    衰减较少
         │              │

原因：短波长（蓝光）CRA通常更大，边缘衰减相对较小
```

### 1.4 标定流程

```
LSC标定流程：

1. 准备均匀光源（积分球/均匀漫射板）

2. 拍摄多张RAW图像
   - 不同色温光源（可选）
   - 多帧平均降噪

3. 计算各位置相对于中心的亮度比
   G(x,y) = I_center / I(x,y)

4. 拟合/生成增益表
   - 多项式拟合：求解系数
   - 网格法：直接存储节点增益

5. 验证校正效果
   - 均匀性检查
   - 边缘增益不超阈值（如2.0×）
```

## 2. 几何畸变校正

### 2.1 畸变类型

#### 2.1.1 径向畸变（Radial Distortion）

```
畸变类型对比：

理想网格：           桶形畸变：           枕形畸变：
┌─┬─┬─┬─┬─┐         ╭─┬─┬─┬─╮          ┌──┬──┬──┬──┐
├─┼─┼─┼─┼─┤        ╱─┼─┼─┼─╲         ╱──┼──┼──┼──╲
├─┼─┼─┼─┼─┤       │─┼─┼─┼─│        │ ─┼──┼──┼─ │
├─┼─┼─┼─┼─┤       │─┼─┼─┼─│        │ ─┼──┼──┼─ │
├─┼─┼─┼─┼─┤        ╲─┼─┼─┼─╱         ╲──┼──┼──┼──╱
└─┴─┴─┴─┴─┘         ╰─┴─┴─┴─╯          └──┴──┴──┴──┘

特点：              向外凸出            向内凹陷
常见镜头：          广角镜头            长焦镜头
```

#### 2.1.2 切向畸变（Tangential Distortion）

由镜头元件与传感器不完全平行引起：

```
切向畸变成因：

理想安装：           实际安装（有倾斜）：
镜头 ─────           镜头 ─────
      │                    ╲
传感器═══           传感器═══

导致图像发生非对称形变
```

### 2.2 Brown-Conrady畸变模型

标准的镜头畸变数学模型：

#### 2.2.1 完整模型

$$
x_{distorted} = x_u(1 + k_1r^2 + k_2r^4 + k_3r^6) + [2p_1x_uy_u + p_2(r^2 + 2x_u^2)]
$$

$$
y_{distorted} = y_u(1 + k_1r^2 + k_2r^4 + k_3r^6) + [p_1(r^2 + 2y_u^2) + 2p_2x_uy_u]
$$

其中：
- $(x_u, y_u)$：无畸变的归一化坐标
- $(x_{distorted}, y_{distorted})$：畸变后坐标
- $r^2 = x_u^2 + y_u^2$
- $k_1, k_2, k_3$：径向畸变系数
- $p_1, p_2$：切向畸变系数

#### 2.2.2 参数含义

| 参数 | 含义 | 典型范围 |
|------|------|----------|
| $k_1$ | 二阶径向畸变（主要项） | -0.5 ~ +0.5 |
| $k_2$ | 四阶径向畸变 | -0.1 ~ +0.1 |
| $k_3$ | 六阶径向畸变（高阶修正） | -0.01 ~ +0.01 |
| $p_1$ | X方向切向畸变 | -0.01 ~ +0.01 |
| $p_2$ | Y方向切向畸变 | -0.01 ~ +0.01 |

```
畸变系数符号与畸变类型：

k1 < 0 → 桶形畸变
k1 > 0 → 枕形畸变
k1 ≈ 0, k2 < 0 → 复杂的"胡须"畸变
```

### 2.3 畸变校正的重采样实现

#### 2.3.1 逆向映射

畸变校正通常采用逆向映射：对于输出图像的每个像素，计算其在输入（畸变）图像中的对应位置。

```c
// 畸变校正实现
typedef struct {
    float fx, fy;    // 焦距（像素单位）
    float cx, cy;    // 主点
    float k1, k2, k3;  // 径向畸变
    float p1, p2;    // 切向畸变
} DistortionParams;

void undistort_image(uint8_t* src, uint8_t* dst, int W, int H,
                     DistortionParams* params) {
    for (int y = 0; y < H; y++) {
        for (int x = 0; x < W; x++) {
            // 归一化坐标
            float x_n = (x - params->cx) / params->fx;
            float y_n = (y - params->cy) / params->fy;
            
            // 计算径向距离
            float r2 = x_n * x_n + y_n * y_n;
            float r4 = r2 * r2;
            float r6 = r4 * r2;
            
            // 径向畸变
            float radial = 1 + params->k1 * r2 + params->k2 * r4 + params->k3 * r6;
            
            // 切向畸变
            float x_tangential = 2 * params->p1 * x_n * y_n + params->p2 * (r2 + 2 * x_n * x_n);
            float y_tangential = params->p1 * (r2 + 2 * y_n * y_n) + 2 * params->p2 * x_n * y_n;
            
            // 畸变后归一化坐标
            float x_d = x_n * radial + x_tangential;
            float y_d = y_n * radial + y_tangential;
            
            // 转回像素坐标
            float src_x = x_d * params->fx + params->cx;
            float src_y = y_d * params->fy + params->cy;
            
            // 双线性插值采样
            dst[y * W + x] = bilinear_sample(src, W, H, src_x, src_y);
        }
    }
}

// 双线性插值
uint8_t bilinear_sample(uint8_t* img, int W, int H, float x, float y) {
    if (x < 0 || x >= W - 1 || y < 0 || y >= H - 1) {
        return 0;  // 边界处理
    }
    
    int x0 = (int)x;
    int y0 = (int)y;
    float dx = x - x0;
    float dy = y - y0;
    
    float v00 = img[y0 * W + x0];
    float v01 = img[y0 * W + x0 + 1];
    float v10 = img[(y0 + 1) * W + x0];
    float v11 = img[(y0 + 1) * W + x0 + 1];
    
    float v = (1 - dx) * (1 - dy) * v00 + dx * (1 - dy) * v01
            + (1 - dx) * dy * v10 + dx * dy * v11;
    
    return (uint8_t)(v + 0.5f);
}
```

#### 2.3.2 查找表加速

预计算映射表以加速实时处理：

```c
// 预计算映射表
typedef struct {
    float* map_x;  // X坐标映射
    float* map_y;  // Y坐标映射
    int width;
    int height;
} DistortionMap;

DistortionMap* create_distortion_map(int W, int H, DistortionParams* params) {
    DistortionMap* map = malloc(sizeof(DistortionMap));
    map->width = W;
    map->height = H;
    map->map_x = malloc(W * H * sizeof(float));
    map->map_y = malloc(W * H * sizeof(float));
    
    for (int y = 0; y < H; y++) {
        for (int x = 0; x < W; x++) {
            // 计算映射（同上）
            float src_x, src_y;
            compute_distorted_position(x, y, params, &src_x, &src_y);
            
            map->map_x[y * W + x] = src_x;
            map->map_y[y * W + x] = src_y;
        }
    }
    
    return map;
}

// 使用映射表进行快速校正
void undistort_with_map(uint8_t* src, uint8_t* dst, int W, int H,
                        DistortionMap* map) {
    for (int y = 0; y < H; y++) {
        for (int x = 0; x < W; x++) {
            int idx = y * W + x;
            float src_x = map->map_x[idx];
            float src_y = map->map_y[idx];
            dst[idx] = bilinear_sample(src, W, H, src_x, src_y);
        }
    }
}
```

## 3. 横向色差校正（LCA）

### 3.1 色差的物理成因

#### 3.1.1 色散原理

不同波长的光在透镜中折射率不同，导致焦点位置不同：

$$
n(\lambda) = A + \frac{B}{\lambda^2} + \frac{C}{\lambda^4}
$$

```
色散导致的色差：

            R(红光)
白光 ──────○───────────●───────────
           │          │      G(绿光)
           │          ●─────────
           │               B(蓝光)
         透镜       ●───────
                   不同波长焦点位置不同
```

#### 3.1.2 横向色差（Lateral Chromatic Aberration, LCA）

边缘处不同颜色的放大率不同，导致颜色边缘错位：

```
横向色差效果：

理想（无LCA）：        实际（有LCA）：
    ┌──────┐              R┌──────┐
    │ 白色 │             G ┌──────┐
    │ 物体 │            B  ┌──────┐
    └──────┘               └──────┘
                       红色偏外，蓝色偏内
```

### 3.2 各通道偏移建模

#### 3.2.1 放大率差异模型

LCA可建模为各通道相对于绿色的放大率差异：

$$
r_R = r_G \cdot (1 + \alpha_R \cdot r_G^2)
$$

$$
r_B = r_G \cdot (1 + \alpha_B \cdot r_G^2)
$$

其中：
- $r_G$：绿色通道的径向距离
- $\alpha_R, \alpha_B$：红/蓝相对于绿色的偏移系数

#### 3.2.2 多项式模型

更精确的模型使用高阶多项式：

$$
\Delta r_R = a_{R1} r + a_{R2} r^2 + a_{R3} r^3
$$

$$
\Delta r_B = a_{B1} r + a_{B2} r^2 + a_{B3} r^3
$$

### 3.3 亚像素级校正算法

```c
// LCA校正
typedef struct {
    float cx, cy;      // 光学中心
    float alpha_r;     // R通道偏移系数
    float alpha_b;     // B通道偏移系数
} LCA_Params;

void correct_lca(uint8_t* r, uint8_t* g, uint8_t* b, 
                 int W, int H, LCA_Params* params) {
    // 为R和B通道创建输出缓冲
    uint8_t* r_out = malloc(W * H);
    uint8_t* b_out = malloc(W * H);
    
    float r_max = sqrtf(W*W/4 + H*H/4);
    
    for (int y = 0; y < H; y++) {
        for (int x = 0; x < W; x++) {
            // 相对于光学中心的位置
            float dx = x - params->cx;
            float dy = y - params->cy;
            float r = sqrtf(dx*dx + dy*dy);
            float r_norm = r / r_max;
            
            if (r > 0) {
                // 计算R通道的源位置
                float scale_r = 1.0f + params->alpha_r * r_norm * r_norm;
                float src_x_r = params->cx + dx * scale_r;
                float src_y_r = params->cy + dy * scale_r;
                
                // 计算B通道的源位置
                float scale_b = 1.0f + params->alpha_b * r_norm * r_norm;
                float src_x_b = params->cx + dx * scale_b;
                float src_y_b = params->cy + dy * scale_b;
                
                // 双线性插值采样
                r_out[y * W + x] = bilinear_sample(r, W, H, src_x_r, src_y_r);
                b_out[y * W + x] = bilinear_sample(b, W, H, src_x_b, src_y_b);
            } else {
                r_out[y * W + x] = r[y * W + x];
                b_out[y * W + x] = b[y * W + x];
            }
        }
    }
    
    // 复制回原数组
    memcpy(r, r_out, W * H);
    memcpy(b, b_out, W * H);
    
    free(r_out);
    free(b_out);
}
```

### 3.4 在RAW域的LCA校正

更高质量的做法是在Demosaic之前进行LCA校正：

```c
// RAW域LCA校正（校正R和B像素位置）
void correct_lca_raw(uint16_t* raw, int W, int H, 
                     LCA_Params* params) {
    uint16_t* raw_out = malloc(W * H * sizeof(uint16_t));
    memcpy(raw_out, raw, W * H * sizeof(uint16_t));
    
    float r_max = sqrtf(W*W/4 + H*H/4);
    
    for (int y = 0; y < H; y++) {
        for (int x = 0; x < W; x++) {
            int bayer_phase = (y % 2) * 2 + (x % 2);
            
            // 只处理R和B像素
            if (bayer_phase == 0) {  // R像素 (假设RGGB)
                float dx = x - params->cx;
                float dy = y - params->cy;
                float r = sqrtf(dx*dx + dy*dy);
                
                if (r > 0) {
                    float scale = 1.0f + params->alpha_r * (r/r_max) * (r/r_max);
                    float src_x = params->cx + dx * scale;
                    float src_y = params->cy + dy * scale;
                    
                    // 亚像素采样（只在同色像素间插值）
                    raw_out[y * W + x] = sample_same_color(raw, W, H, src_x, src_y, 0);
                }
            }
            else if (bayer_phase == 3) {  // B像素
                float dx = x - params->cx;
                float dy = y - params->cy;
                float r = sqrtf(dx*dx + dy*dy);
                
                if (r > 0) {
                    float scale = 1.0f + params->alpha_b * (r/r_max) * (r/r_max);
                    float src_x = params->cx + dx * scale;
                    float src_y = params->cy + dy * scale;
                    
                    raw_out[y * W + x] = sample_same_color(raw, W, H, src_x, src_y, 3);
                }
            }
            // G像素保持不变
        }
    }
    
    memcpy(raw, raw_out, W * H * sizeof(uint16_t));
    free(raw_out);
}
```

## 4. 紫边校正

### 4.1 紫边形成机理

紫边（Purple Fringing）是高对比度边缘出现的紫色或品红色伪影。

```
紫边成因：

1. 横向色差残余
   - 高对比度边缘放大了色差效应
   
2. 传感器光晕
   - 强光在传感器内部反射
   - 微透镜间串扰
   
3. 轴向色差
   - 不同波长焦平面不同
   - 导致边缘模糊带有色彩
   
紫边典型出现位置：
┌─────────────────────────┐
│         天空           │
│ ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓  │ ← 树枝轮廓
│ ▓███紫边███████████▓   │    出现紫色边缘
│ ▓███████████████████▓  │
└─────────────────────────┘
```

### 4.2 检测算法

#### 4.2.1 颜色特征检测

紫边在色彩空间中有特定特征：

```c
// 紫边检测（Lab空间）
bool detect_purple_fringe(float L, float a, float b) {
    // 紫色特征：a > 0（偏红）且 b < 0（偏蓝）
    // 高亮度边缘：L > 阈值
    
    // 饱和度
    float chroma = sqrtf(a*a + b*b);
    
    // 色相角（紫色约在-60°到-30°或300°到330°）
    float hue = atan2f(b, a) * 180.0f / M_PI;
    if (hue < 0) hue += 360.0f;
    
    bool is_purple = (hue > 270.0f && hue < 330.0f) || (hue > -90.0f && hue < -30.0f);
    bool is_saturated = chroma > 20.0f;
    
    return is_purple && is_saturated;
}
```

#### 4.2.2 边缘检测结合

只在高对比度边缘处理紫边：

```c
// 结合边缘检测的紫边掩码
void create_purple_fringe_mask(uint8_t* rgb, uint8_t* mask, int W, int H) {
    // 1. 计算边缘强度（Sobel）
    float* edge = malloc(W * H * sizeof(float));
    compute_edge_strength(rgb, edge, W, H);
    
    // 2. 转换到Lab空间
    float* lab = malloc(W * H * 3 * sizeof(float));
    rgb_to_lab(rgb, lab, W, H);
    
    // 3. 生成掩码
    for (int y = 0; y < H; y++) {
        for (int x = 0; x < W; x++) {
            int idx = y * W + x;
            
            float L = lab[idx * 3 + 0];
            float a = lab[idx * 3 + 1];
            float b = lab[idx * 3 + 2];
            
            bool is_edge = edge[idx] > 50.0f;  // 边缘阈值
            bool is_purple = detect_purple_fringe(L, a, b);
            
            mask[idx] = (is_edge && is_purple) ? 255 : 0;
        }
    }
    
    free(edge);
    free(lab);
}
```

### 4.3 消除算法

#### 4.3.1 去饱和法

降低紫边区域的色彩饱和度：

```c
void remove_purple_fringe_desaturate(uint8_t* rgb, uint8_t* mask, int W, int H) {
    for (int y = 0; y < H; y++) {
        for (int x = 0; x < W; x++) {
            int idx = y * W + x;
            
            if (mask[idx] > 0) {
                float R = rgb[idx * 3 + 0];
                float G = rgb[idx * 3 + 1];
                float B = rgb[idx * 3 + 2];
                
                // 计算亮度
                float Y = 0.299f * R + 0.587f * G + 0.114f * B;
                
                // 混合因子（基于掩码强度）
                float blend = mask[idx] / 255.0f * 0.8f;  // 最多80%去饱和
                
                // 向灰度混合
                rgb[idx * 3 + 0] = (uint8_t)(R * (1 - blend) + Y * blend);
                rgb[idx * 3 + 1] = (uint8_t)(G * (1 - blend) + Y * blend);
                rgb[idx * 3 + 2] = (uint8_t)(B * (1 - blend) + Y * blend);
            }
        }
    }
}
```

#### 4.3.2 颜色替换法

用周围非紫边像素的颜色替换：

```c
void remove_purple_fringe_replace(uint8_t* rgb, uint8_t* mask, int W, int H) {
    uint8_t* rgb_out = malloc(W * H * 3);
    memcpy(rgb_out, rgb, W * H * 3);
    
    for (int y = 2; y < H - 2; y++) {
        for (int x = 2; x < W - 2; x++) {
            int idx = y * W + x;
            
            if (mask[idx] > 0) {
                // 收集周围非紫边像素的颜色
                float sum_r = 0, sum_g = 0, sum_b = 0;
                int count = 0;
                
                for (int dy = -2; dy <= 2; dy++) {
                    for (int dx = -2; dx <= 2; dx++) {
                        int n_idx = (y + dy) * W + (x + dx);
                        if (mask[n_idx] == 0) {  // 非紫边
                            sum_r += rgb[n_idx * 3 + 0];
                            sum_g += rgb[n_idx * 3 + 1];
                            sum_b += rgb[n_idx * 3 + 2];
                            count++;
                        }
                    }
                }
                
                if (count > 0) {
                    // 保持原始亮度，替换色度
                    float orig_Y = 0.299f * rgb[idx*3+0] + 0.587f * rgb[idx*3+1] + 0.114f * rgb[idx*3+2];
                    float new_Y = 0.299f * (sum_r/count) + 0.587f * (sum_g/count) + 0.114f * (sum_b/count);
                    
                    float scale = (new_Y > 0) ? orig_Y / new_Y : 1.0f;
                    scale = fminf(scale, 1.5f);  // 限制缩放
                    
                    rgb_out[idx * 3 + 0] = (uint8_t)fminf(sum_r / count * scale, 255);
                    rgb_out[idx * 3 + 1] = (uint8_t)fminf(sum_g / count * scale, 255);
                    rgb_out[idx * 3 + 2] = (uint8_t)fminf(sum_b / count * scale, 255);
                }
            }
        }
    }
    
    memcpy(rgb, rgb_out, W * H * 3);
    free(rgb_out);
}
```

## 5. 标定方法

### 5.1 棋盘格标定流程

```
标准棋盘格标定流程：

1. 准备标定板
   ┌─┬─┬─┬─┬─┬─┬─┬─┐
   │█│░│█│░│█│░│█│░│  使用高精度打印
   ├─┼─┼─┼─┼─┼─┼─┼─┤  的棋盘格图案
   │░│█│░│█│░│█│░│█│  
   ├─┼─┼─┼─┼─┼─┼─┼─┤  常用规格：9×6, 11×8
   │█│░│█│░│█│░│█│░│
   └─┴─┴─┴─┴─┴─┴─┴─┘

2. 拍摄多角度图像
   - 覆盖整个视场
   - 不同倾斜角度
   - 10-20张通常足够

3. 角点检测
   - 亚像素精度定位
   - Harris角点 + 亚像素细化

4. 参数优化
   - Bundle Adjustment
   - 最小化重投影误差
```

#### 5.1.1 OpenCV标定代码示例

```python
import cv2
import numpy as np

# 棋盘格尺寸
BOARD_SIZE = (9, 6)
SQUARE_SIZE = 25.0  # mm

# 准备世界坐标
objp = np.zeros((BOARD_SIZE[0] * BOARD_SIZE[1], 3), np.float32)
objp[:, :2] = np.mgrid[0:BOARD_SIZE[0], 0:BOARD_SIZE[1]].T.reshape(-1, 2)
objp *= SQUARE_SIZE

# 存储点对
obj_points = []  # 3D世界坐标
img_points = []  # 2D图像坐标

# 处理每张标定图像
for img_path in calibration_images:
    img = cv2.imread(img_path)
    gray = cv2.cvtColor(img, cv2.COLOR_BGR2GRAY)
    
    # 查找角点
    ret, corners = cv2.findChessboardCorners(gray, BOARD_SIZE, None)
    
    if ret:
        # 亚像素精化
        criteria = (cv2.TERM_CRITERIA_EPS + cv2.TERM_CRITERIA_MAX_ITER, 30, 0.001)
        corners = cv2.cornerSubPix(gray, corners, (11, 11), (-1, -1), criteria)
        
        obj_points.append(objp)
        img_points.append(corners)

# 执行标定
ret, mtx, dist, rvecs, tvecs = cv2.calibrateCamera(
    obj_points, img_points, gray.shape[::-1], None, None
)

print("相机内参矩阵:\n", mtx)
print("畸变系数 (k1, k2, p1, p2, k3):\n", dist)
print("重投影误差:", ret)
```

### 5.2 工厂在线标定 vs 离线标定

| 方面 | 工厂在线标定 | 离线标定 |
|------|-------------|----------|
| **时机** | 生产线上自动完成 | 出厂后手动执行 |
| **速度** | 快（<1秒） | 慢（需多帧处理） |
| **精度** | 中等 | 高 |
| **设备** | 专用标定设备 | 普通标定板 |
| **适用** | 大批量生产 | 专业应用 |
| **参数** | 预置到模组/ISP | 写入配置文件 |

### 5.3 标定数据存储格式

#### 5.3.1 常见格式

```
1. OTP（One-Time Programmable）

   - 随模组一起出厂
   - 格式为二进制

2. XML/JSON配置文件
   <camera_calibration>

     <lsc_table>...</lsc_table>
   </camera_calibration>

   - LSC网格直接以数组存储
   - 畸变映射表预计算存储
```

#### 5.3.2 数据结构示例

```c
// 标定数据结构
typedef struct {
    // 内参
    float fx, fy;         // 焦距
    float cx, cy;         // 主点
    
    // 畸变
    float k1, k2, k3;     // 径向畸变
    float p1, p2;         // 切向畸变
    
    // LSC
    uint16_t lsc_grid_w;
    uint16_t lsc_grid_h;
    float lsc_gains_r[17*13];
    float lsc_gains_gr[17*13];
    float lsc_gains_gb[17*13];
    float lsc_gains_b[17*13];
    
    // LCA
    float lca_alpha_r;
    float lca_alpha_b;
    
    // 验证
    uint32_t checksum;
} CalibrationData;

// 从OTP读取
bool load_calibration_from_otp(CalibrationData* data, uint8_t* otp_buffer) {
    memcpy(data, otp_buffer, sizeof(CalibrationData));
    
    // 校验
    uint32_t calc_checksum = crc32(data, sizeof(CalibrationData) - 4);
    return calc_checksum == data->checksum;
}
```
