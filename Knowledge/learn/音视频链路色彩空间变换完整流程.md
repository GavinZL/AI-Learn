### 核心概念速查

| 概念 | 定义 | 关键参数 |
|-----|------|---------|
| **色彩空间 (Color Space)** | 定义颜色如何用数值表示的数学模型 | RGB原色坐标、白点 |
| **传递函数 (Transfer Function)** | 线性光信号与电信号之间的映射关系 | Gamma、PQ、HLG |
| **色域 (Color Gamut)** | 色彩空间能表示的颜色范围 | 色度图面积占比 |
| **YUV Range** | YUV数值的有效范围 | Limited [16,235] / Full [0,255] |
| **色度子采样 (Chroma Subsampling)** | 色度分量的空间分辨率降采样方式 | 4:4:4, 4:2:2, 4:2:0 |

#### 传递函数对比

| 传递函数 | 标准 | 应用场景 | 峰值亮度 |
|---------|------|---------|---------|
| BT.709 Gamma | ITU-R BT.709 | SDR视频 | 100 nit |
| sRGB | IEC 61966-2-1 | 网页/图像 | 80 nit |
| PQ (ST 2084) | SMPTE ST 2084 | HDR10/Dolby Vision | 10,000 nit |
| HLG | ITU-R BT.2100 | HDR广播 | 1,000-4,000 nit |

## 一、采集阶段

### 1.1 传感器与RAW数据

#### Bayer CFA采样原理

相机传感器使用Bayer Color Filter Array（CFA）进行颜色采样，每个像素只采集一种颜色：

```
常见Bayer排列模式：

RGGB排列（最常见）    BGGR排列           GBRG排列
┌───┬───┬───┬───┐    ┌───┬───┬───┬───┐    ┌───┬───┬───┬───┐
│ R │ G │ R │ G │    │ B │ G │ B │ G │    │ G │ B │ G │ B │
├───┼───┼───┼───┤    ├───┼───┼───┼───┤    ├───┼───┼───┼───┤
│ G │ B │ G │ B │    │ G │ R │ G │ R │    │ R │ G │ R │ G │
├───┼───┼───┼───┤    ├───┼───┼───┼───┤    ├───┼───┼───┼───┤
│ R │ G │ R │ G │    │ B │ G │ B │ G │    │ G │ B │ G │ B │
├───┼───┼───┼───┤    ├───┼───┼───┼───┤    ├───┼───┼───┼───┤
│ G │ B │ G │ B │    │ G │ R │ G │ R │    │ R │ G │ R │ G │
└───┴───┴───┴───┘    └───┴───┴───┴───┘    └───┴───┴───┴───┘
```

#### RAW数据特性

| 特性 | 说明 |
|-----|------|
| 位深 | 12-16bit（主流14bit），提供更大的动态范围 |
| 色彩空间 | 相机原生RGB空间（设备相关，非标准色彩空间） |
| 线性特性 | 传感器输出与光强成线性关系（未经Gamma校正） |
| 黑电平 | 存在黑电平偏移（Black Level），需在处理时减去 |

```c
// RAW数据读取伪代码
struct RawFrame {
    uint16_t* data;          // 原始像素数据
    int width, height;       // 分辨率
    int bit_depth;           // 位深 (12/14/16)
    int black_level;         // 黑电平值
    int white_level;         // 白电平（饱和值）
    BayerPattern pattern;    // RGGB/BGGR/GBRG/GRBG
};

// 归一化到[0,1]线性空间
float normalize_raw(uint16_t raw_value, int black_level, int white_level) {
    float normalized = (raw_value - black_level) / (float)(white_level - black_level);
    return clamp(normalized, 0.0f, 1.0f);
}
```

### 1.2 ISP处理流程中的色彩空间变换

ISP（Image Signal Processor）是将RAW数据转换为可显示图像的核心模块：

```
ISP处理流程：

RAW Data ──▶ Demosaic ──▶ AWB ──▶ CCM ──▶ Gamma ──▶ RGB→YUV ──▶ YUV Output
   │            │          │       │        │           │
   ▼            ▼          ▼       ▼        ▼           ▼
 线性        全彩RGB    白平衡   色彩校正  非线性     编码格式
 单通道      插值重建   校正     到标准空间 压缩       转换
```

#### 1.2.1 Demosaic（去马赛克/去拜耳）

将单通道Bayer数据插值为全彩RGB图像：

```c
// 简化的双线性插值Demosaic
void demosaic_bilinear(uint16_t* raw, float* rgb_out, int w, int h, BayerPattern pattern) {
    for (int y = 1; y < h - 1; y++) {
        for (int x = 1; x < w - 1; x++) {
            int idx = y * w + x;
            float r, g, b;
            
            // 根据Bayer位置判断当前像素颜色
            if (is_red_pixel(x, y, pattern)) {
                r = raw[idx];
                g = (raw[idx-1] + raw[idx+1] + raw[idx-w] + raw[idx+w]) / 4.0f;
                b = (raw[idx-w-1] + raw[idx-w+1] + raw[idx+w-1] + raw[idx+w+1]) / 4.0f;
            }
            // ... 类似处理Green和Blue像素
            
            rgb_out[idx*3 + 0] = r;
            rgb_out[idx*3 + 1] = g;
            rgb_out[idx*3 + 2] = b;
        }
    }
}
```

#### 1.2.2 AWB（自动白平衡）

校正色温偏差，使白色物体在图像中呈现为中性白：

```c
// 白平衡校正
void apply_white_balance(float* rgb, int size, float r_gain, float g_gain, float b_gain) {
    for (int i = 0; i < size; i += 3) {
        rgb[i + 0] *= r_gain;  // R通道增益
        rgb[i + 1] *= g_gain;  // G通道增益（通常为1.0）
        rgb[i + 2] *= b_gain;  // B通道增益
    }
}

// 典型白平衡增益示例
// 日光(D65): r_gain=1.0, g_gain=1.0, b_gain=1.0
// 钨丝灯(A): r_gain=0.5, g_gain=1.0, b_gain=2.0
```

#### 1.2.3 CCM色彩校正矩阵

通过3×3矩阵将相机原生RGB空间转换到标准色彩空间（如sRGB/Rec.709）：

```
CCM标定流程：

X-Rite ColorChecker ──▶ 拍摄参考色卡 ──▶ 提取色块RGB值
                                              │
                        ┌─────────────────────┘
                        ▼
              计算最小二乘拟合矩阵 ──▶ Camera RGB → CIE XYZ → sRGB/Rec.709
```

**CCM转换公式：**

```
┌─────┐   ┌─────────────────────┐   ┌──────────┐
│ R'  │   │ CCM[0][0-2]         │   │ R_camera │
│ G'  │ = │ CCM[1][0-2]         │ × │ G_camera │
│ B'  │   │ CCM[2][0-2]         │   │ B_camera │
└─────┘   └─────────────────────┘   └──────────┘
```

**双光源矩阵插值：**

相机通常标定两个光源下的CCM：
- **ColorMatrix1**: A光源 (2856K，钨丝灯)
- **ColorMatrix2**: D65光源 (6504K，日光)

```c
// 双光源CCM插值
void interpolate_ccm(float ccm_out[3][3], 
                     float ccm_A[3][3], float ccm_D65[3][3],
                     float color_temp) {
    // 计算插值权重（基于色温的倒数空间）
    float inv_temp = 1.0f / color_temp;
    float inv_A = 1.0f / 2856.0f;
    float inv_D65 = 1.0f / 6504.0f;
    
    float weight = (inv_temp - inv_A) / (inv_D65 - inv_A);
    weight = clamp(weight, 0.0f, 1.0f);
    
    // 线性插值两个矩阵
    for (int i = 0; i < 3; i++) {
        for (int j = 0; j < 3; j++) {
            ccm_out[i][j] = ccm_A[i][j] * (1 - weight) + ccm_D65[i][j] * weight;
        }
    }
}
```

#### 1.2.4 Gamma校正

将线性光信号压缩到非线性编码空间，匹配人眼感知特性：

```c
// BT.709 OETF (Gamma编码)
float bt709_oetf(float L) {
    // L: 线性光值 [0, 1]
    if (L < 0.018f) {
        return 4.5f * L;
    } else {
        return 1.099f * powf(L, 0.45f) - 0.099f;
    }
}

// sRGB OETF
float srgb_oetf(float L) {
    if (L <= 0.0031308f) {
        return 12.92f * L;
    } else {
        return 1.055f * powf(L, 1.0f / 2.4f) - 0.055f;
    }
}
```

#### 1.2.5 RGB→YUV转换

将RGB转换为YUV格式，便于压缩编码：

```c
// BT.709 RGB→YCbCr转换 (Full Range)
void rgb_to_ycbcr_bt709_full(float r, float g, float b, 
                              float* y, float* cb, float* cr) {
    *y  =  0.2126f * r + 0.7152f * g + 0.0722f * b;
    *cb = -0.1146f * r - 0.3854f * g + 0.5000f * b + 0.5f;
    *cr =  0.5000f * r - 0.4542f * g - 0.0458f * b + 0.5f;
}

// BT.709 RGB→YCbCr转换 (Limited Range, 8-bit)
void rgb_to_ycbcr_bt709_limited(uint8_t r, uint8_t g, uint8_t b,
                                 uint8_t* y, uint8_t* cb, uint8_t* cr) {
    float rf = r / 255.0f, gf = g / 255.0f, bf = b / 255.0f;
    
    float yf  =  0.2126f * rf + 0.7152f * gf + 0.0722f * bf;
    float cbf = -0.1146f * rf - 0.3854f * gf + 0.5000f * bf;
    float crf =  0.5000f * rf - 0.4542f * gf - 0.0458f * bf;
    
    // Limited Range缩放: Y=[16,235], Cb/Cr=[16,240]
    *y  = (uint8_t)(yf * 219.0f + 16.0f);
    *cb = (uint8_t)(cbf * 224.0f + 128.0f);
    *cr = (uint8_t)(crf * 224.0f + 128.0f);
}
```

### 1.3 Android Camera API的YUV输出问题

#### 核心问题

Android Camera2/CameraX API输出YUV时，**无法实时可靠地确认是Limited Range [16,235] 还是 Full Range [0,255]**。

```
问题根源：

┌─────────────────────────────────────────────────────────────┐
│                    Android Camera YUV输出                    │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Camera HAL ──▶ ISP处理 ──▶ YUV输出 ──▶ Camera2 API         │
│       │                         │              │            │
│       ▼                         ▼              ▼            │
│  厂商实现差异            Range类型不确定    API信息不完整   │
│  (Qualcomm/MTK/          (Limited/Full)    (部分设备不报告) │
│   Exynos/Kirin)                                             │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### 不同SoC厂商行为差异

| SoC厂商 | 常见行为 | 备注 |
|--------|---------|------|
| Qualcomm Snapdragon | 多数输出Limited Range | 部分高端芯片可配置 |
| MediaTek | 行为不一致 | 不同型号差异大 |
| Samsung Exynos | 多数Limited Range | 较为统一 |
| HiSilicon Kirin | 早期Full Range，后期Limited | 版本依赖 |

#### 工程解决方案

**方案1：MediaFormat查询（不可靠）**

**方案2：直方图分析法（推荐）**

```java
// 直方图分析判断YUV Range类型
public class YuvRangeDetector {
    
    public enum RangeType { LIMITED, FULL, UNKNOWN }
    
    public static RangeType detectRange(Image yuvImage, int sampleFrames) {
        int[] histogram = new int[256];
        
        // 采样Y通道数据
        Image.Plane yPlane = yuvImage.getPlanes()[0];
        ByteBuffer yBuffer = yPlane.getBuffer();
        
        while (yBuffer.hasRemaining()) {
            int y = yBuffer.get() & 0xFF;
            histogram[y]++;
        }
        
        // 分析分布特征
        int lowCount = 0, highCount = 0;
        for (int i = 0; i < 16; i++) lowCount += histogram[i];
        for (int i = 236; i < 256; i++) highCount += histogram[i];
        
        int totalPixels = yuvImage.getWidth() * yuvImage.getHeight();
        float lowRatio = lowCount / (float) totalPixels;
        float highRatio = highCount / (float) totalPixels;
        
        // 判断逻辑
        if (lowRatio < 0.001f && highRatio < 0.001f) {
            return RangeType.LIMITED;  // 几乎没有值落在[0,15]和[236,255]
        } else if (lowRatio > 0.01f || highRatio > 0.01f) {
            return RangeType.FULL;     // 有明显的边界值分布
        }
        return RangeType.UNKNOWN;
    }
}
```

## 二、前处理阶段

### 2.1 GPU渲染管线中的色彩空间

前处理阶段通常在GPU上进行，涉及YUV→RGB→处理→RGB→YUV的完整流程。

#### 输入阶段色彩空间转换

```glsl
// GLSL: YUV纹理采样与转换
uniform sampler2D yTexture;   // Y平面
uniform sampler2D uvTexture;  // UV平面 (NV12格式)
uniform mat3 yuvToRgbMatrix;  // YUV→RGB转换矩阵
uniform vec3 yuvOffset;       // YUV偏移量

vec3 sampleYuv(vec2 texCoord) {
    float y = texture(yTexture, texCoord).r;
    vec2 uv = texture(uvTexture, texCoord).rg;
    
    // 组装YUV向量
    vec3 yuv = vec3(y, uv.x, uv.y);
    
    // 减去偏移量
    yuv -= yuvOffset;
    
    // 矩阵转换得到RGB
    vec3 rgb = yuvToRgbMatrix * yuv;
    
    return clamp(rgb, 0.0, 1.0);
}
```

#### 工作空间选择

| 工作空间 | 优点 | 缺点 | 适用场景 |
|---------|-----|------|---------|
| **线性sRGB（推荐）** | 物理正确的混合运算 | 需要额外Gamma编解码 | 美颜、特效、合成 |
| **非线性sRGB** | 无需额外转换 | 混合运算不准确 | 简单滤镜、LUT |

```glsl
// Gamma解码：非线性→线性
vec3 srgbToLinear(vec3 srgb) {
    vec3 low = srgb / 12.92;
    vec3 high = pow((srgb + 0.055) / 1.055, vec3(2.4));
    return mix(low, high, step(vec3(0.04045), srgb));
}

// Gamma编码：线性→非线性
vec3 linearToSrgb(vec3 linear) {
    vec3 low = linear * 12.92;
    vec3 high = 1.055 * pow(linear, vec3(1.0/2.4)) - 0.055;
```

### 2.2 特效/美颜/滤镜处理的色彩要求

#### 2.2.1 美颜处理 - 必须在线性空间


**磨皮算法示例（线性空间）：**

```glsl
// 双边滤波磨皮（线性空间）
vec3 bilateralSkinSmooth(sampler2D tex, vec2 uv, float sigma_s, float sigma_r) {
    vec3 centerLinear = srgbToLinear(texture(tex, uv).rgb);
    vec3 sum = vec3(0.0);
    float weightSum = 0.0;
    
    for (int i = -KERNEL_SIZE; i <= KERNEL_SIZE; i++) {
        for (int j = -KERNEL_SIZE; j <= KERNEL_SIZE; j++) {
            vec2 offset = vec2(float(i), float(j)) * texelSize;
            vec3 sampleLinear = srgbToLinear(texture(tex, uv + offset).rgb);
            
            // 空间权重
            float spatialWeight = exp(-float(i*i + j*j) / (2.0 * sigma_s * sigma_s));
            
            // 范围权重（基于线性空间色差）
            vec3 diff = sampleLinear - centerLinear;
            float rangeWeight = exp(-dot(diff, diff) / (2.0 * sigma_r * sigma_r));
            
            float weight = spatialWeight * rangeWeight;
            sum += sampleLinear * weight;
            weightSum += weight;
        }
    }
    
    vec3 resultLinear = sum / weightSum;
    return linearToSrgb(resultLinear);  // 输出时转回Gamma空间
}
```

#### 2.2.2 滤镜/LUT应用

```glsl
// LUT应用 - 需明确LUT的输入空间假设
uniform sampler3D lutTexture;
uniform int lutInputSpace;  // 0=sRGB, 1=Linear

vec3 applyLut(vec3 color, int inputColorSpace) {
    vec3 lutInput = color;
    
    // 如果当前是线性空间，但LUT期望sRGB输入
    if (inputColorSpace == LINEAR_SPACE && lutInputSpace == SRGB_SPACE) {
        lutInput = linearToSrgb(color);
    }
    // 如果当前是sRGB空间，但LUT期望线性输入
    else if (inputColorSpace == SRGB_SPACE && lutInputSpace == LINEAR_SPACE) {
        lutInput = srgbToLinear(color);
    }
    
    // LUT采样（通常是33x33x33或64x64x64）
    vec3 lutCoord = lutInput * (LUT_SIZE - 1.0) / LUT_SIZE + 0.5 / LUT_SIZE;
    return texture(lutTexture, lutCoord).rgb;
}
```

### 2.3 RGB→YUV回转

处理完成后需转回YUV格式送入编码器：

```glsl
// RGB→YUV转换（需与编码器期望一致）
vec3 rgbToYuv(vec3 rgb, int standard, int range) {
    mat3 matrix;
    vec3 offset;
    
    // 选择转换矩阵
    if (standard == BT709) {
        matrix = mat3(
            0.2126,  0.7152,  0.0722,
           -0.1146, -0.3854,  0.5000,
            0.5000, -0.4542, -0.0458
        );
    } else { // BT601
        matrix = mat3(
            0.299,   0.587,   0.114,
           -0.169,  -0.331,   0.500,
            0.500,  -0.419,  -0.081
        );
    }
    
    vec3 yuv = matrix * rgb;
    
    // 应用Range缩放和偏移
    if (range == LIMITED_RANGE) {
        yuv.x = yuv.x * (219.0/255.0) + (16.0/255.0);
        yuv.yz = yuv.yz * (224.0/255.0) + vec2(128.0/255.0);
    } else {
        yuv.yz += vec2(0.5);
    }
    
    return yuv;
}
```

### 2.4 常见问题与解决方案

#### 问题1：颜色偏色

**现象：** 画面整体偏绿或偏紫

**原因：** YUV→RGB或RGB→YUV转换时BT.601和BT.709矩阵混用

```
问题示例：
编码端使用BT.601，解码端使用BT.709

BT.601 YUV ──▶ [BT.709矩阵] ──▶ RGB (偏色!)
```

**解决方案：**

```c
// 检查并统一矩阵选择
typedef enum {
    MATRIX_BT601,
    MATRIX_BT709,
    MATRIX_BT2020
} MatrixType;

MatrixType selectMatrix(int width, int height, int vui_matrix) {
    // 优先使用VUI信息
    if (vui_matrix != 0) {
        return vui_matrix == 6 ? MATRIX_BT601 : 
               vui_matrix == 1 ? MATRIX_BT709 : MATRIX_BT2020;
    }
    
    // 启发式：根据分辨率判断
    if (width <= 720 && height <= 576) {
        return MATRIX_BT601;   // SD视频
    } else if (width <= 1920 && height <= 1080) {
        return MATRIX_BT709;   // HD视频
    } else {
        return MATRIX_BT2020;  // UHD视频
    }
}
```

#### 问题2：亮度跳变

**现象：** 画面忽明忽暗，或黑色不够黑、白色过曝

**原因：** Full Range和Limited Range处理不一致

```
问题示例：
Full Range [0,255] 数据被当作 Limited Range [16,235] 处理

Y=0 ──▶ (0-16)/219 = -0.073 (裁剪为0) ──▶ 丢失暗部细节
Y=255 ──▶ (255-16)/219 = 1.091 (裁剪为1) ──▶ 丢失亮部细节
```

**解决方案：**

```glsl
// 统一的Range归一化处理
float normalizeY(float y, int rangeType) {
    if (rangeType == LIMITED_RANGE) {
        return clamp((y - 16.0/255.0) / (219.0/255.0), 0.0, 1.0);
    } else {
        return y;  // Full Range直接使用
    }
}

float denormalizeY(float y, int rangeType) {
    if (rangeType == LIMITED_RANGE) {
        return y * (219.0/255.0) + (16.0/255.0);
    } else {
        return y;
    }
}
```

#### 问题3：饱和度异常

**现象：** 颜色过于鲜艳或过于灰暗

**原因：** 在Gamma空间下进行了应该在线性空间进行的运算

```glsl
// 错误：在Gamma空间做色彩增强
vec3 wrongEnhance(vec3 gammargb, float saturation) {
    float luma = dot(gammargb, vec3(0.299, 0.587, 0.114));
    return mix(vec3(luma), gammargb, saturation);  // 结果不正确
}

// 正确：在线性空间做色彩增强
vec3 correctEnhance(vec3 gammaRgb, float saturation) {
    vec3 linear = srgbToLinear(gammaRgb);
    float luma = dot(linear, vec3(0.2126, 0.7152, 0.0722));
    vec3 result = mix(vec3(luma), linear, saturation);
    return linearToSrgb(result);
}
```

### 4.1 RTMP

RTMP（Real-Time Messaging Protocol）基于FLV封装，色彩空间信息通过VideoTagHeader传递。

#### 标准RTMP的色彩支持

```
RTMP/FLV色彩信息携带方式：
```

### 4.2 SRT

SRT（Secure Reliable Transport）基于MPEG-2 TS封装，色彩信息完整传递能力强。

```
SRT/TS色彩信息传递：

TS Packet (188 bytes)
├── Sync Byte (0x47)
├── PID (Program ID)
└── Payload
    └── PES Packet
        ├── PES Header
        └── Elementary Stream
            └── NAL Units (H.264/H.265)
                └── SPS → VUI Parameters

PAT (Program Association Table)
└── PMT PID

PMT (Program Map Table)
├── Program Info Descriptors
│   └── Video Stream Descriptor
│       └── 色彩空间描述符
└── Elementary Stream Info
    ├── Stream Type
    └── ES Info Descriptors
        └── HEVC Video Descriptor
            └── 色彩配置信息
```

**PMT中的色彩描述符：**


### 5.1 VUI参数提取流程

```
VUI解析流程：

码流 → NAL解析 → 识别SPS → 解析SPS → 提取VUI → 应用色彩设置
         │
         ▼
    ┌─────────────────────────────────────────────┐
    │ H.264: nal_unit_type == 7 (SPS)             │
    │ H.265: nal_unit_type == 33 (SPS)            │
    │ AV1: OBU_SEQUENCE_HEADER                    │
    └─────────────────────────────────────────────┘
```

#### H.264 VUI解析伪代码

```c
typedef struct {
    int colour_primaries;           // 1-255, 2=unspecified
    int transfer_characteristics;   // 1-255, 2=unspecified
    int matrix_coefficients;        // 0-255, 2=unspecified
    int video_full_range_flag;      // 0=limited, 1=full
} VUIColorInfo;

VUIColorInfo parse_h264_vui(BitstreamReader* bs, SPSData* sps) {
    VUIColorInfo color = {2, 2, 2, 0};  // 默认：unspecified
    
    if (!sps->vui_parameters_present_flag) {
        return color;
    }
    
    // 跳过aspect_ratio等字段...
    
    if (read_bit(bs)) {  // video_signal_type_present_flag
        skip_bits(bs, 3);  // video_format
        color.video_full_range_flag = read_bit(bs);
        
        if (read_bit(bs)) {  // colour_description_present_flag
            color.colour_primaries = read_bits(bs, 8);
            color.transfer_characteristics = read_bits(bs, 8);
            color.matrix_coefficients = read_bits(bs, 8);
        }
    }
    
    return color;
}
```

#### AV1 Color Config解析

```c
typedef struct {
    int color_primaries;
    int transfer_characteristics;
    int matrix_coefficients;
    int color_range;              // 0=limited (studio swing), 1=full
    int subsampling_x;
    int subsampling_y;
    int chroma_sample_position;   // 0=unknown, 1=vertical, 2=colocated
} AV1ColorConfig;

AV1ColorConfig parse_av1_color_config(BitstreamReader* bs, 
                                       SequenceHeader* seq) {
    AV1ColorConfig color;
    
    int high_bitdepth = read_bit(bs);
    if (seq->seq_profile == 2 && high_bitdepth) {
        int twelve_bit = read_bit(bs);
        seq->bit_depth = twelve_bit ? 12 : 10;
    } else {
        seq->bit_depth = high_bitdepth ? 10 : 8;
    }
    
    if (seq->seq_profile == 1) {
        color.mono_chrome = 0;
    } else {
        color.mono_chrome = read_bit(bs);
    }
    
    int color_description_present_flag = read_bit(bs);
    if (color_description_present_flag) {
        color.color_primaries = read_bits(bs, 8);
        color.transfer_characteristics = read_bits(bs, 8);
        color.matrix_coefficients = read_bits(bs, 8);
    } else {
        color.color_primaries = 2;  // unspecified
        color.transfer_characteristics = 2;
        color.matrix_coefficients = 2;
    }
    
    // ... 继续解析subsampling等
    
    return color;
}
```

### 5.2 解码器色彩空间处理

```java
        // 创建解码器
        String mime = inputFormat.getString(MediaFormat.KEY_MIME);

    public ColorInfo getOutputColorInfo() {

            // COLOR_STANDARD_BT709 = 1
            // COLOR_STANDARD_BT601_PAL = 2
            // COLOR_STANDARD_BT601_NTSC = 4
            // COLOR_STANDARD_BT2020 = 6
        }
        
        // 色彩范围

            info.range = outputFormat.getInteger(MediaFormat.KEY_COLOR_RANGE);
            // COLOR_RANGE_LIMITED = 2
            // COLOR_RANGE_FULL = 1
```

```c
typedef struct {
    int colour_primaries;
    int transfer_characteristics;
    int matrix_coefficients;
    int video_full_range_flag;

ColorParams get_color_params_with_fallback(
    const VUIColorInfo* vui, 
    int width, int height,
    int bit_depth) {
    
    ColorParams params;
    
    // 1. 优先使用VUI信息
    if (vui && vui->colour_primaries != 2) {  // 2 = unspecified
        params.colour_primaries = vui->colour_primaries;
        params.transfer_characteristics = vui->transfer_characteristics;
        params.matrix_coefficients = vui->matrix_coefficients;
        params.video_full_range_flag = vui->video_full_range_flag;
        return params;

    if (width <= 720 && height <= 576) {
        // SD视频 → BT.601
        params.colour_primaries = 5;          // BT.601-625 (PAL)
        params.transfer_characteristics = 6;   // BT.601
        params.matrix_coefficients = 5;        // BT.601-625
    } else if (width <= 1920 && height <= 1080) {
        // HD视频 → BT.709
        params.colour_primaries = 1;          // BT.709
        params.transfer_characteristics = 1;   // BT.709
        params.matrix_coefficients = 1;        // BT.709
    } else {
        // UHD视频 → BT.2020
        params.colour_primaries = 9;          // BT.2020
        params.transfer_characteristics = 
            (bit_depth > 8) ? 16 : 14;        // PQ或BT.2020-10bit
        params.matrix_coefficients = 9;        // BT.2020
    }
    
    // 3. 默认Limited Range（最保守）
    params.video_full_range_flag = 0;
    
    return params;
```

### 5.4 色彩空间转换

#### YUV→RGB转换

```glsl
// GLSL: 完整的YUV→RGB转换（支持多种标准）
uniform int uColorStandard;   // 0=BT601, 1=BT709, 2=BT2020
uniform int uColorRange;      // 0=Limited, 1=Full
uniform sampler2D yTexture;
uniform sampler2D uvTexture;

// 转换矩阵常量
const mat3 BT601_MATRIX = mat3(
    1.0,     1.0,      1.0,
    0.0,    -0.344136, 1.772,
    1.402,  -0.714136, 0.0
);

const mat3 BT709_MATRIX = mat3(
    1.0,     1.0,      1.0,
    0.0,    -0.1873,   1.8556,
    1.5748, -0.4681,   0.0
);

const mat3 BT2020_MATRIX = mat3(
    1.0,     1.0,      1.0,
    0.0,    -0.1646,   1.8814,
    1.4746, -0.5714,   0.0
);

vec3 yuvToRgb(vec2 texCoord) {
    // 采样YUV
    float y = texture(yTexture, texCoord).r;
    vec2 uv = texture(uvTexture, texCoord).rg;
    
    // Limited Range归一化
    if (uColorRange == 0) {
        y = (y - 16.0/255.0) * 255.0/219.0;
        uv = (uv - 128.0/255.0) * 255.0/224.0;
    } else {
        uv -= 0.5;
    }
    
    vec3 yuv = vec3(y, uv.x, uv.y);
    
    // 选择转换矩阵
    mat3 matrix;
    if (uColorStandard == 0) {
        matrix = BT601_MATRIX;
    } else if (uColorStandard == 1) {
        matrix = BT709_MATRIX;
    } else {
        matrix = BT2020_MATRIX;
    }
    
    vec3 rgb = matrix * yuv;
    return clamp(rgb, 0.0, 1.0);
}
```

#### HDR EOTF应用

**PQ (ST 2084) EOTF：**

```glsl
// PQ EOTF: 将PQ编码值转换为线性光值
// 输出范围: [0, 10000] nit (归一化到[0, 1]时，1.0 = 10000 nit)
const float PQ_M1 = 0.1593017578125;      // 2610/16384
const float PQ_M2 = 78.84375;             // 2523/32
const float PQ_C1 = 0.8359375;            // 3424/4096
const float PQ_C2 = 18.8515625;           // 2413/128
const float PQ_C3 = 18.6875;              // 2392/128

vec3 pqEotf(vec3 pq) {
    // PQ信号 [0,1] → 线性光 [0,1] (其中1.0 = 10000 nit)
    vec3 Np = pow(pq, vec3(1.0 / PQ_M2));
    vec3 L = pow(
        max(Np - PQ_C1, 0.0) / (PQ_C2 - PQ_C3 * Np),
        vec3(1.0 / PQ_M1)
    );
    return L;  // 线性光，峰值1.0 = 10000 nit
}

// 反函数: PQ OETF
vec3 pqOetf(vec3 L) {
    // 线性光 [0,1] → PQ信号 [0,1]
    vec3 Lm = pow(L, vec3(PQ_M1));
    vec3 N = pow(
        (PQ_C1 + PQ_C2 * Lm) / (1.0 + PQ_C3 * Lm),
        vec3(PQ_M2)
    );
    return N;
}
```

**HLG EOTF：**

```glsl
// HLG OETF参数
const float HLG_A = 0.17883277;
const float HLG_B = 0.28466892;  // 1 - 4*a
const float HLG_C = 0.55991073;  // 0.5 - a*ln(4*a)

// HLG OETF: 线性光 → HLG信号
float hlgOetf(float L) {
    if (L <= 1.0/12.0) {
        return sqrt(3.0 * L);
    } else {
        return HLG_A * log(12.0 * L - HLG_B) + HLG_C;
    }
}

// HLG EOTF: HLG信号 → 线性光
float hlgEotf(float E) {
    if (E <= 0.5) {
        return E * E / 3.0;
    } else {
        return (exp((E - HLG_C) / HLG_A) + HLG_B) / 12.0;
    }
}

// HLG OOTF (场景光→显示光，包含系统Gamma)
vec3 hlgOotf(vec3 sceneLinear, float systemGamma) {
    // BT.2100建议的系统Gamma: 1.2 (标称)
    float Ys = dot(sceneLinear, vec3(0.2627, 0.6780, 0.0593));
    return pow(Ys, systemGamma - 1.0) * sceneLinear;
}
```

```glsl
    if (actualRange == expectedRange) {
        return yuv;
    }
    
    // Limited → Full
    if (actualRange == LIMITED && expectedRange == FULL) {
        yuv.x = yuv.x * 219.0/255.0 + 16.0/255.0;
        yuv.yz = yuv.yz * 224.0/255.0;

        yuv.x = (yuv.x - 16.0/255.0) * 255.0/219.0;
        yuv.yz = yuv.yz * 255.0/224.0;
    }
    
    return yuv;
}
```

```glsl
// BT.1886 EOTF (SDR显示器参考EOTF)
// 与sRGB略有不同，用于专业视频监视器
vec3 bt1886Eotf(vec3 V, float gamma, float Lw, float Lb) {
    // V: 输入信号 [0,1]
    // gamma: 系统Gamma (通常2.4)
    // Lw: 白电平亮度 (通常100 cd/m²)
    // Lb: 黑电平亮度 (通常0.1 cd/m²)
    
    float a = pow(pow(Lw, 1.0/gamma) - pow(Lb, 1.0/gamma), gamma);
    float b = pow(Lb, 1.0/gamma) / 
              (pow(Lw, 1.0/gamma) - pow(Lb, 1.0/gamma));
    
    return a * pow(max(V + b, 0.0), vec3(gamma));
}
```

### 6.4 色调映射 (Tone Mapping)

#### 全局色调映射算法

**Reinhard算法：**

```glsl
// Reinhard全局色调映射
vec3 reinhardToneMap(vec3 hdrColor, float Lwhite) {
    // Lwhite: 映射到白色的亮度值
    float L = dot(hdrColor, vec3(0.2126, 0.7152, 0.0722));
    float Ld = L * (1.0 + L / (Lwhite * Lwhite)) / (1.0 + L);
    return hdrColor * (Ld / L);
}
```

**ACES Filmic算法：**

```glsl
// ACES Filmic色调映射 (简化版)
vec3 acesFilmicToneMap(vec3 x) {
    float a = 2.51;
    float b = 0.03;
    float c = 2.43;
    float d = 0.59;
    float e = 0.14;
    return clamp((x * (a * x + b)) / (x * (c * x + d) + e), 0.0, 1.0);
}
```

**BT.2390 EETF (推荐)：**

```glsl
// BT.2390 EETF (Electrical-Electrical Transfer Function)
// 在ICtCp色彩空间中操作，保持色调一致性

// RGB → ICtCp 转换
vec3 rgbToICtCp(vec3 rgb) {
    // RGB → LMS (BT.2020)
    mat3 rgb2lms = mat3(
        0.4122, 0.5239, 0.0639,
        0.1667, 0.7203, 0.1130,
        0.0241, 0.0754, 0.9005
    );
    vec3 lms = rgb2lms * rgb;
    
    // 应用PQ OETF到LMS
    vec3 lmsPQ = pqOetf(lms);
    
    // LMS' → ICtCp
    mat3 lms2ictcp = mat3(
        2048.0/4096.0,   2048.0/4096.0,    0.0,
        6610.0/4096.0,  -13613.0/4096.0,   7003.0/4096.0,
        17933.0/4096.0, -17390.0/4096.0,  -543.0/4096.0
    );
    return lms2ictcp * lmsPQ;
}

// BT.2390 EETF色调映射（在ICtCp空间的I通道操作）
float bt2390Eetf(float I, float Lhdr, float Lsdr) {
    // I: 输入强度 [0, 1]
    // Lhdr: 源HDR峰值亮度 (如10000)
    // Lsdr: 目标SDR峰值亮度 (如100)
    
    float KS = 1.5 * Lsdr / Lhdr - 0.5;
    float b = Lsdr / Lhdr;
    
    if (I < KS) {
        return I;
    } else {
        float t = (I - KS) / (1.0 - KS);
        float t2 = t * t;
        float t3 = t2 * t;
        // Hermite样条插值
        float P = (2.0*t3 - 3.0*t2 + 1.0) * KS +
                  (t3 - 2.0*t2 + t) * (1.0 - KS) +
                  (-2.0*t3 + 3.0*t2) * b;
        return P;
    }
}
```

```glsl
// 感知映射 (Perceptual)
// 保持整体色彩关系，压缩整个色域
vec3 perceptualGamutMap(vec3 srcColor, mat3 srcToXYZ, mat3 xyzToDst, 
                         float compressionFactor) {
    // 转换到XYZ
    vec3 xyz = srcToXYZ * srcColor;
    
    // 压缩色度（保持亮度）
    float Y = xyz.y;
    vec3 xyY = vec3(xyz.x / (xyz.x + xyz.y + xyz.z),
                    xyz.y / (xyz.x + xyz.y + xyz.z),
                    Y);
    
    // 向白点压缩色度
    vec2 whitePoint = vec2(0.3127, 0.3290);  // D65
    xyY.xy = mix(xyY.xy, whitePoint, compressionFactor);
    
    // 转回XYZ
    xyz = vec3(xyY.x * Y / xyY.y, Y, (1.0 - xyY.x - xyY.y) * Y / xyY.y);
    
    // 转换到目标色彩空间
    return xyzToDst * xyz;
}

// 色域裁剪 (Clipping)
// 简单但可能导致色调偏移
vec3 clipGamutMap(vec3 color) {
    return clamp(color, 0.0, 1.0);
}

// 饱和度映射
// 保持色调，降低饱和度
vec3 saturationGamutMap(vec3 color) {
    // 转换到HSL
    vec3 hsl = rgbToHsl(color);
    
    // 检查是否出域
    vec3 clipped = clamp(color, 0.0, 1.0);
    if (color != clipped) {
        // 降低饱和度直到在域内
        for (float s = hsl.y; s >= 0.0; s -= 0.01) {
            vec3 test = hslToRgb(vec3(hsl.x, s, hsl.z));
            if (all(greaterThanEqual(test, vec3(0.0))) && 
                all(lessThanEqual(test, vec3(1.0)))) {
                return test;
            }
        }
    }
    return color;
}
```

### 6.7 PQ EOTF完整公式

**数学定义：**

```
PQ EOTF (SMPTE ST 2084):

         ⎛ max(N^(1/m₂) - c₁, 0) ⎞^(1/m₁)
L = 10000 × ⎜ ──────────────────── ⎟
         ⎝   c₂ - c₃ × N^(1/m₂)  ⎠

其中：
- N: 输入信号值 [0, 1]
- L: 输出亮度值 [0, 10000] cd/m²

常数值：
- m₁ = 2610/16384 = 0.1593017578125
- m₂ = 2523/32 = 78.84375
- c₁ = 3424/4096 = 0.8359375 (= c₃ - c₂ + 1)
- c₂ = 2413/128 = 18.8515625
- c₃ = 2392/128 = 18.6875
```

**实现代码：**

```c
// PQ EOTF实现
double pq_eotf(double N) {
    const double m1 = 2610.0 / 16384.0;
    const double m2 = 78.84375;  // 2523/32
    const double c1 = 3424.0 / 4096.0;
    const double c2 = 2413.0 / 128.0;
    const double c3 = 2392.0 / 128.0;
    
    double Np = pow(N, 1.0 / m2);
    double L = pow(fmax(Np - c1, 0.0) / (c2 - c3 * Np), 1.0 / m1);
    
    return L * 10000.0;  // 返回cd/m²
}

// PQ OETF实现 (EOTF的逆函数)
double pq_oetf(double L) {
    const double m1 = 2610.0 / 16384.0;
    const double m2 = 78.84375;  // 2523/32
    const double c1 = 3424.0 / 4096.0;
    const double c2 = 2413.0 / 128.0;
    const double c3 = 2392.0 / 128.0;
    
    double Lm = pow(L / 10000.0, m1);
    double N = pow((c1 + c2 * Lm) / (1.0 + c3 * Lm), m2);
    
    return N;
}
```

#### 问题2：色域溢出处理

```glsl
// 柔和的色域裁剪（避免硬裁剪导致的色带）
vec3 softGamutClip(vec3 color, float threshold) {
    // threshold: 开始软裁剪的阈值 (如0.95)
    
    vec3 clipped = clamp(color, 0.0, 1.0);
    vec3 diff = color - clipped;
    
    // 如果有溢出
    if (any(notEqual(diff, vec3(0.0)))) {
        // 计算溢出程度
        float maxOverflow = max(max(abs(diff.r), abs(diff.g)), abs(diff.b));
        
        // 软压缩：使用sigmoid曲线
        float softFactor = 1.0 / (1.0 + maxOverflow * 2.0);
        return mix(color * softFactor, clipped, 
                   smoothstep(threshold, 1.0, max(max(color.r, color.g), color.b)));
    }
    
    return color;
}
```
