# 色彩管理管线设计详解

> 构建端到端一致的色彩管理工作流

---

## 1. 色彩管理管线的目标

### 1.1 核心问题

在没有色彩管理的情况下：

```
美术A的显示器 ──▶ 渲染效果 ──▶ 美术B的显示器
   (偏暖)                           (偏冷)
      │                              │
      └──────── 同一文件 ────────────┘
                   ↓
            看起来完全不同！
```

### 1.2 色彩管理的目标

| 目标 | 描述 |
|------|------|
| **一致性** | 相同内容在不同设备上显示一致 |
| **可预测性** | 创作时看到的就是最终效果 |
| **可交换性** | 文件可在不同软件间无损交换 |
| **可归档** | 长期保存，未来仍可正确显示 |

| 决策 | 选项 | 建议 |
|------|------|------|
| **工作空间** | sRGB Linear / ACEScg / Rec.2020 | ACEScg |

## 3. 各阶段详细设计

### 3.1 资产创建阶段

#### 纹理创作

**Photoshop配置**：

```
编辑 > 颜色设置
├── RGB工作空间: sRGB IEC61966-2.1
├── CMYK工作空间: 根据输出需求
├── 灰度: Gray Gamma 2.2
├── 色彩管理方案: 保留嵌入的配置文件
└── 配置文件不匹配: 询问
```

**Substance Painter配置**：

```
编辑 > 项目配置
├── 颜色空间: Linear
├── 显示: sRGB (ACES sRGB ODT)
└── 导出: 根据目标引擎
```

#### 导出规范

| 资产类型 | 格式 | 颜色空间 | 位深 |
|---------|------|---------|------|
| Albedo | PNG/TGA | sRGB | 8-bit |
| Normal | PNG | Linear | 8-bit |
| ORM | PNG | Linear | 8-bit |
| HDR Sky | EXR | Linear (Scene-Referred) | 16-bit half |
| LUT | CUBE | 根据用途 | 32^3 |

### 3.2 引擎导入阶段

#### 纹理导入设置

```cpp
struct TextureImportSettings
{
    // 颜色空间检测
    ColorSpace sourceColorSpace;
    
    // 自动检测规则
    bool autoDetectSRGB = true;  // 根据文件名/路径
    
    // 压缩设置
    TextureCompression compression;
    
    // Mip Map
    MipMapGeneration mipMaps = MipMapGeneration::SimpleAverage;
    
    // 转换到工作空间
    ColorSpace targetWorkingSpace = ColorSpace::ACEScg;
};
```

#### 导入流程

```
纹理文件
    │
    ├──▶ 检测颜色空间
    │       ├── 文件名含"_Linear" → Linear
    │       ├── 文件名含"_Normal" → Linear
    │       └── 其他 → sRGB
    │
    ├──▶ 解码到Linear
    │       ├── sRGB → sRGB EOTF
    │       └── Linear → 无需解码
    │
    ├──▶ 转换到工作空间
    │       ├── sRGB Linear → ACEScg矩阵
    │       └── ACES2065-1 → AP0_to_AP1矩阵
    │
    ├──▶ 压缩
    │       ├── BC1 (Albedo)
    │       ├── BC5 (Normal)
    │       └── BC7 (高质量)
    │
    └──▶ 生成Mip Maps
```

### 3.3 渲染阶段

#### 工作空间选择

**推荐：ACEScg (AP1)**

```hlsl
// 着色器常量
static const float3x3 sRGB_to_ACEScg = {
    0.613132, 0.339538, 0.047416,
    0.070127, 0.916353, 0.013520,
    0.020588, 0.109573, 0.869950
};

// 纹理采样并转换
float3 SampleAlbedo(Texture2D tex, float2 uv)
{
    float3 srgbColor = tex.Sample(sRGBSampler, uv).rgb;
    float3 linearColor = SRGBToLinear(srgbColor);
    return mul(sRGB_to_ACEScg, linearColor);
}
```

#### 渲染管线集成

```
┌─────────────────────────────────────────────────────────────┐
│                    Rendering Pipeline                        │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  G-Buffer Pass                                               │
│  ├── Albedo: ACEScg Linear                                   │
│  ├── Normal: World Space                                     │
│  └── Material: Roughness/Metallic (Linear)                   │
│                                                              │
│  Lighting Pass                                               │
│  ├── Direct Lighting: ACEScg                                 │
│  ├── Indirect Lighting: ACEScg                               │
│  └── Emissive: ACEScg (unbounded)                            │
│                                                              │
│  Post-Processing                                             │
│  ├── Bloom: ACEScg                                           │
│  ├── SSR/AO: Linear                                          │
│  └── Color Grading: ACEScg → AP0 → RRT → ODT                 │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 3.4 输出阶段

#### 多目标输出

```hlsl
// 输出变换选择
float3 OutputTransform(float3 acescgColor, OutputDevice device)
{
    switch(device)
    {
        case OutputDevice::sRGB_100nit:
            return ACES_sRGB_100nit(acescgColor);
            
        case OutputDevice::Rec709_100nit:
            return ACES_Rec709_100nit(acescgColor);
            
        case OutputDevice::P3D65_1000nit:
            return ACES_P3D65_1000nit(acescgColor);
            
        case OutputDevice::Rec2020_1000nit:
            return ACES_Rec2020_1000nit(acescgColor);
            
        case OutputDevice::Rec2020_2000nit:
            return ACES_Rec2020_2000nit(acescgColor);
            
        default:
            return ACES_sRGB_100nit(acescgColor);
    }
}
```

### 4.2 运行时集成

```cpp
// OCIO运行时配置
class ColorManagementSystem
{
public:
    void Initialize(const std::string& configPath);
    
    // 获取转换处理器
    Processor GetProcessor(
        const std::string& srcColorSpace,
        const std::string& dstColorSpace);
    
    // 应用颜色转换
    void ApplyTransform(
        const Processor& processor,
        Image& image);
    
    // 3D LUT生成
    Texture3D GenerateLUT(
        const Processor& processor,
        int lutSize = 32);
    
private:
    OCIO::ConstConfigRcPtr m_config;
};

// 使用示例
void ProcessTexture(Texture& texture)
{
    auto processor = g_colorSystem.GetProcessor(
        "Input - sRGB Texture",
        "ACES - ACEScg"
    );
    
    // 生成3D LUT用于运行时
    Texture3D lut = g_colorSystem.GenerateLUT(processor, 32);
    texture.SetColorTransformLUT(lut);
}
```

## 7. 性能优化

### 7.1 LUT优化

| 优化技术 | 效果 | 实现 |
|---------|------|------|
| **预计算LUT** | 避免运行时转换 | 离线生成3D LUT |
| **降低LUT分辨率** | 减少纹理缓存压力 | 32^3 → 16^3（移动端） |
| **分离LUT** | 减少采样次数 | 1D LUT + 3D LUT |
| **Async Compute** | 并行生成LUT | 每帧更新 |

### 7.2 着色器优化

```hlsl
// 优化前：多次矩阵乘法
float3 color = mul(Matrix1, mul(Matrix2, mul(Matrix3, input)));

// 优化后：预乘矩阵
static const float3x3 CombinedMatrix = mul(Matrix1, mul(Matrix2, Matrix3));
float3 color = mul(CombinedMatrix, input);
```

### 7.3 带宽优化

- 使用R11G11B10_FLOAT代替RGBA16_FLOAT
- 延迟颜色空间转换到需要时
- 使用硬件sRGB采样/写入

## 8. 最佳实践总结

### 8.1 黄金法则

1. **始终在线性空间进行光照计算**
2. **使用广色域工作空间（ACEScg）**
3. **延迟色调映射到最终输出**
4. **使用标准化的View Transform**
5. **验证跨设备一致性**
