## ACES色彩管理系统详解

### 1. ACES系统的核心作用

#### 1.1 在整个影视制作流程中的角色

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    ACES：影视制作的"通用语言"系统                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│   拍摄阶段                    后期制作                    输出交付        │
│   ┌─────────┐              ┌─────────────┐            ┌─────────────┐   │
│   │  ARRI   │───▶│   Nuke合成    │───▶│  DCI-P3影院  │   │
│   │  RED    │  ACES2065-1  │   Maya渲染    │  输出变换   │  Rec.709电视  │   │
│   │  Sony   │─────────────▶│   DaVinci调色  │──────────▶│  Rec.2020 HDR │   │
│   │  ...    │   (场景参考)   │   (场景线性工作)  │  (显示参考)  │  sRGB网络    │   │
│   └─────────┘              └─────────────┘            └─────────────┘   │
│                                                                          │
│   不同相机                   统一工作空间                  多平台交付      │
│   不同色彩空间               ACEScg/ACEScct               自动适配        │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘
```

**ACES的核心定位**：

| 特性 | 说明 |
|------|------|
| **场景参考工作流** | 保留拍摄场景的完整光学信息，而非针对特定显示设备优化 |
| **设备无关性** | 不依赖任何特定相机或显示设备的色彩特性 |
| **未来兼容性** | 为尚未发明的显示技术预留空间 |
| **开放标准** | 由Academy Software Foundation维护，免费使用 |

### 2. 主要解决的问题

#### 2.1 传统色彩管理的痛点

| 痛点 | 传统方案的问题 | ACES的解决方案 |
|------|---------------|---------------|
| **设备锁定** | 每个厂商有自己的色彩空间，素材难以跨平台使用 | 统一的ACES场景参考空间 |
| **动态范围丢失** | 早期就压缩到显示设备的动态范围 | 保留完整场景动态范围（30+档） |
| **色域不兼容** | 不同色域之间转换困难，容易失真 | AP0超广色域包含所有人眼可见颜色 |
| **版本混乱** | 同一项目多个版本，色彩不一致 | 单一源文件，多设备输出 |
| **未来不可扩展** | 新设备出现时需要重新处理所有素材 | 设备无关设计，支持未来技术 |

#### 2.2 场景参考 vs 显示参考的对比

```
┌─────────────────────────────────────────────────────────────────────────┐
│              场景参考 (Scene Referred) vs 显示参考 (Display Referred)     │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  场景参考 - ACES工作方式：                                                │
│  ┌─────────────────────────────────────────────────────────────────┐    │
│  │  场景光线强度 ──▶ 线性编码 ──▶ 完整保留 ──▶ 后期灵活处理         │    │
│  │  (10支蜡烛 = 10倍亮度)                                           │    │
│  │                                                                  │    │
│  │  优点：曝光调整是线性的，合成计算准确                              │    │
│  │  缺点：需要专业显示设备和色调映射才能正确观看                        │    │
│  └─────────────────────────────────────────────────────────────────┘    │
│                                                                          │
│  显示参考 - 传统工作方式：                                                │
│  ┌─────────────────────────────────────────────────────────────────┐    │
│  │  场景光线 ──▶ 立即压缩 ──▶ 适配显示设备 ──▶ 后期受限             │    │
│  │  (10支蜡烛 ≠ 10倍亮度，被人眼感知曲线压缩)                        │    │
│  │                                                                  │    │
│  │  优点：直接可观看，兼容性好                                        │    │
│  │  缺点：信息丢失，跨设备不一致，合成计算不准确                        │    │
│  └─────────────────────────────────────────────────────────────────┘    │
│                                                                          │
│  关键区别：                                                              │
│  • 场景参考 = 保存"原始场景的光学信息"                                  │
│  • 显示参考 = 保存"针对特定显示设备的优化结果"                          │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘
```

#### 2.3 动态范围和色域问题

**传统方案的局限**：
- Rec.709色域仅覆盖约35.9%的CIE 1931色域
- SDR动态范围约6-8档
- 一旦压缩到显示范围，无法恢复

**ACES的解决方案**：

| 参数 | ACES规格 | 说明 |
|------|---------|------|
| **色域 (AP0)** | 覆盖100%+ CIE 1931 | 包含所有人眼可见颜色 |
| **动态范围** | 支持30+档 | 远超任何现有相机和显示设备 |
| **位深** | 16bit浮点 (Half Float) | 足够精度避免量化误差 |
| **文件格式** | OpenEXR | 行业标准的HDR图像格式 |

### 3. 主要优化的点

#### 3.1 RRT（参考渲染变换）和ODT（输出设备变换）

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    ACES输出变换架构（ACES 1.x）                          │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│   ACES场景数据                                                          │
│   (线性，AP0色域)                                                        │
│        │                                                                 │
│        ▼                                                                 │
│   ┌─────────────────────────────────────────────────────────────────┐   │
│   │              Reference Rendering Transform (RRT)                 │   │
│   │                        参考渲染变换                              │   │
│   │                                                                  │   │
│   │  • 将场景线性数据转换为显示线性HDR数据                            │   │
│   │  • 应用S曲线色调映射                                              │   │
│   │  • 添加"电影感"外观（对比度、饱和度调整）                          │   │
│   │  • 输出：线性HDR显示参考数据                                       │   │
│   │                                                                  │   │
│   └─────────────────────────────────────────────────────────────────┘   │
│        │                                                                 │
│        ▼                                                                 │
│   ┌─────────────────────────────────────────────────────────────────┐   │
│   │              Output Device Transform (ODT)                       │   │
│   │                        输出设备变换                              │   │
│   │                                                                  │   │
│   │  • 根据目标设备特性进行编码                                        │   │
│   │  • 应用设备特定的传递函数（Gamma/PQ/HLG）                         │   │
│   │  • 限制到设备色域和动态范围                                        │   │
│   │  • 输出：设备特定的代码值                                          │   │
│   │                                                                  │   │
│   └─────────────────────────────────────────────────────────────────┘   │
│        │                                                                 │
│        ▼                                                                 │
│   显示设备（Rec.709/Rec.2020/DCI-P3/HDR显示器）                          │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘
```

#### 3.2 ACES 2.0的重大改进

**ACES 1.x的问题**：
- RRT产生的"ACES Look"过于对比度高
- 暗部容易去饱和
- 不同输出设备之间的匹配不够一致

**ACES 2.0的改进**：

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    ACES 2.0 新架构（DRT - 显示渲染变换）                  │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  ACES 2065-1                                                            │
│  (AP0线性)                                                               │
│      │                                                                   │
│      ▼                                                                   │
│  ┌─────────────────────────────────────────────────────────────────┐    │
│  │  ACES to JMh                                                    │    │
│  │  转换到色彩外观模型空间（基于Hellwig 2022 CAM）                    │    │
│  │  J = 明度(Lightness), M = 彩度(Colorfulness), h = 色相(Hue)      │    │
│  └─────────────────────────────────────────────────────────────────┘    │
│      │                                                                   │
│      ▼                                                                   │
│  ┌─────────────────────────────────────────────────────────────────┐    │
│  │  色调映射 (Tonescale) - 仅处理J分量                               │    │
│  │  • 保持色相不变                                                   │    │
│  │  • 更平滑的S曲线                                                  │    │
│  └─────────────────────────────────────────────────────────────────┘    │
│      │                                                                   │
│      ▼                                                                   │
│  ┌─────────────────────────────────────────────────────────────────┐    │
│  │  彩度压缩 (Chroma Compression) - 仅处理M分量                      │    │
│  │  • 根据亮度自适应调整饱和度                                        │    │
│  └─────────────────────────────────────────────────────────────────┘    │
│      │                                                                   │
│      ▼                                                                   │
│  ┌─────────────────────────────────────────────────────────────────┐    │
│  │  色域压缩 (Gamut Compression)                                     │    │
│  │  • 处理超出色域的颜色                                             │    │
│  │  • 保持视觉一致性                                                 │    │
│  └─────────────────────────────────────────────────────────────────┘    │
│      │                                                                   │
│      ▼                                                                   │
│  ┌─────────────────────────────────────────────────────────────────┐    │
│  │  JMh to Limiting RGB + 设备编码                                   │    │
│  │  转换到目标设备色彩空间并编码                                      │    │
│  └─────────────────────────────────────────────────────────────────┘    │
│                                                                          │
│  关键改进：                                                              │
│  • 在感知均匀的色彩空间中进行处理                                        │
│  • 色调、彩度、色相分离处理，避免相互影响                                 │
│  • 不同输出设备之间的一致性大幅提升                                      │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘
```

#### 3.3 S曲线特性

**ACES色调映射的S曲线设计**：

```
亮度输出
  │    ┌─────────────────────────────────────┐
  │   /                                       \\
  │  /    高光压缩区                            \\    阴影压缩区
  │ /     (保护高光细节)                         \\   (提升暗部可见性)
  │/                                              \\
  ├──────────────────────────────────────────────────\\───────────────
  │                                                    \\            │
  │    线性区                                           \\           │
  │    (保持中调对比度)                                  \\          │
  │                                                       \\         │
  │                                                        \\        │
  └────────────────────────────────────────────────────────\\───────┘
  0                                                        输入亮度
                                                           (场景线性)

特性说明：
• 阴影区：提升暗部，避免死黑
• 线性区：保持中灰层次（18%灰映射到约50%显示亮度）
• 高光区：压缩高光，保留细节
• 整体：模拟胶片响应曲线，具有"电影感"
```

#### 3.4 ACES的色彩空间家族

| 色彩空间 | 用途 | 技术规格 |
|---------|------|---------|
| **ACES2065-1** | 存档、交换 | AP0色域，16bit半浮点，线性 |
| **ACEScg** | CG渲染、合成工作 | AP1色域（更实用），线性 |
| **ACEScct** | 调色工作 | AP1色域，对数编码，类似LogC |
| **ACESproxy** | 实时预览 | AP1色域，10bit整数，低带宽 |

### 4. 权威标准依据

#### 4.1 SMPTE ST 2065-1:2021

**核心技术规范**：

> **AP0原色坐标**（来自ACES官方文档）：
> | 原色 | CIE x | CIE y |
> |------|-------|-------|
> | Red | 0.73470 | 0.26530 |
> | Green | 0.00000 | 1.00000 |
> | Blue | 0.00010 | -0.07700 |
>
> **白点**：
> | | CIE x | CIE y |
> |--|-------|-------|
> | White | 0.32168 | 0.33767 |
>
> **传递函数**：线性（Linear）

**关键特性**：
- 使用16bit半精度浮点（Half Float）编码
- 支持超过30档的动态范围
- 色域覆盖整个CIE 1931可见光谱

#### 4.2 SMPTE RDD 18

**ACES工作流规范**：
- 定义IDT（输入设备变换）的开发标准
- 规定RRT和ODT的技术要求
- 建立ACES色彩管理的实施指南

#### 4.3 Academy Software Foundation

**开源项目维护**：
- ACES色彩管理系统的开源实现
- 提供参考转换矩阵和查找表（LUT）
- 维护AMF（ACES Metadata File）标准

### 7. 总结

ACES色彩管理系统的核心价值在于：

| 维度 | 传统方案 | ACES方案 |
|------|---------|---------|
| **工作方式** | 显示参考，针对特定设备 | 场景参考，设备无关 |
| **色彩空间** | 多种不兼容的色彩空间 | 统一的ACES空间家族 |
| **动态范围** | 早期压缩，信息丢失 | 完整保留，后期灵活 |
| **跨平台** | 需要多次转换，容易失真 | 统一中转，一致性高 |
| **未来兼容** | 新技术需要重新处理 | 支持未来任何显示技术 |
| **标准化** | 厂商私有方案 | SMPTE国际标准 |

**ACES就像色彩世界的"国际通用语"和"万能集装箱"**，它让不同设备、不同软件、不同平台之间的色彩交换变得简单可靠，同时为影视制作提供了面向未来的色彩管理基础设施。
