# 色调映射算法详解

> 从Reinhard到ACES：色调映射算法的演进与实践

---

## 1. 色调映射概述

### 1.1 为什么需要色调映射

HDR场景亮度范围：0.001 - 10000+ nit  
SDR显示范围：0.1 - 100 nit

直接截断（Clip）会导致：
- 高光完全丢失（过曝）
- 暗部细节消失（欠曝）
- 整体对比度降低

### 1.2 色调映射的目标

1. **动态范围压缩**：将HDR映射到显示范围
2. **细节保留**：避免硬裁切，保持渐变
3. **对比度增强**：提升中灰区域对比度
4. **视觉美感**：保持或增强图像的艺术效果

### 1.3 色调映射的分类

| 类型 | 特点 | 代表算法 |
|------|------|---------|
| **全局算子** | 每个像素独立处理，相同输入相同输出 | Reinhard, ACES |
| **局部算子** | 考虑邻域信息，可能产生光晕 | Durand, bilateral filter |
| **基于梯度** | 压缩梯度场，保持细节 | Fattal |
| **混合方法** | 结合多种技术 | 现代游戏引擎 |

## 2. 经典全局算子

### 2.1 Reinhard算子

**原始论文**：Reinhard et al., "Photographic Tone Reproduction for Digital Images", SIGGRAPH 2002

**基本思想**：模拟摄影中的区域曝光法（Zone System）

**公式**：

$$
L_d(x, y) = \frac{L(x, y)}{1 + L(x, y)}
$$

其中 $L$ 是场景亮度，$L_d$ 是显示亮度。

**带白点的变体**：

$$
L_d(x, y) = \frac{L(x, y)(1 + \frac{L(x, y)}{L_{white}^2})}{1 + L(x, y)}
$$

**特性**：
- 简单高效
- 柔和的高光压缩
- 可能产生"灰蒙蒙"的效果

### 2.2 Reinhard扩展（Extended Reinhard）

引入局部适应：

```hlsl
float ReinhardExtended(float x, float maxWhite)
{
    float numerator = x * (1.0 + x / (maxWhite * maxWhite));
    return numerator / (1.0 + x);
}
```

### 2.3 Uncharted 2 Tonemapper

John Hable为《神秘海域2》开发的Tonemapper：

```hlsl
float3 Uncharted2Tonemap(float3 x)
{
    float A = 0.15; // shoulder strength
    float B = 0.50; // linear strength
    float C = 0.10; // linear angle
    float D = 0.20; // toe strength
    float E = 0.02; // toe numerator
    float F = 0.30; // toe denominator
    
    return ((x * (A * x + C * B) + D * E) / (x * (A * x + B) + D * F)) - E / F;
}
```

**特性**：
- 参数可调
- 更好的暗部控制
- 广泛使用于游戏

### 2.4 Filmic Tonemappers

模拟胶片响应曲线的Tonemapper：

**Hejl-Richardson**：

```hlsl
float3 HejlRichardson(float3 hdr)
{
    float3 x = max(0, hdr - 0.004);
    return (x * (6.2 * x + 0.5)) / (x * (6.2 * x + 1.7) + 0.06);
}
```

**特性**：
- 接近胶片S曲线
- 自然的高光压缩
- 暗部提升

## 3. ACES Filmic Tonemapper

### 3.1 RRT Tone Scale

ACES的色调映射分为两个阶段：

**RRT（Reference Rendering Transform）**：
- 将Scene-Referred转换到Output-Referred
- 使用分段B-spline曲线

**ODT（Output Device Transform）**：
- 适配特定显示设备
- 进一步的色调映射

### 3.2 S曲线特性

理想的Filmic Tonemapping曲线：

```
输出亮度
  1.0 │                    ╭──────
      │                 ╭──╯
 0.86 │              ╭──╯        ← 白点（ACES SDR）
      │           ╭──╯
 0.18 │──────────╯               ← 中灰
      │      ╭──╯
      │   ╭──╯
  0.0 │╭──╯
      └────────────────────────▶ 输入亮度（log scale）
      0.001   0.01   0.1    1   10  100  1000
```

**关键特征**：
- **Toe（暗部）**：提升暗部，避免死黑
- **Linear（线性段）**：保持中灰对比度
- **Shoulder（肩部）**：柔和压缩高光

### 3.3 ACES S曲线参数

ACES 1.0使用的分段spline参数：

```
节点（Log2亮度）:  -6.5  -4.0  -2.0   0.0   2.0
输出值:            0.02  0.10  0.35  0.70  0.90
```

**ACES 2.0改进**：
- 统一为Single Stage Tone Scale（SSTS）
- 更简洁的参数集
- 更好的HDR支持

### 3.4 参数调整指南

| 参数 | 效果 | 调整建议 |
|------|------|---------|
| **Slope** | 整体对比度 | 增加使对比度更高 |
| **Toe** | 暗部提升 | 增加提亮暗部 |
| **Shoulder** | 高光压缩 | 增加使高光更柔和 |
| **Black Clip** | 黑电平 | 通常保持为0 |
| **White Clip** | 白电平 | 控制最大输出 |

## 4. 局部色调映射

### 4.1 双边滤波（Bilateral Filter）

**Durand & Dorsey方法**：

1. 计算对数亮度
2. 双边滤波分解为基层（Base）和细节层（Detail）
3. 压缩基层
4. 重建：压缩后的基层 + 细节层

```
输入图像 → 对数亮度 → 双边滤波 → 基层
                              ↓
                         细节层 = 输入 - 基层
                              ↓
                    压缩基层 + 细节层 → 输出
```

**优点**：
- 保留局部细节
- 避免全局方法的"扁平化"

**缺点**：
- 计算开销大
- 可能产生光晕（Halo）

### 4.2 局部适应Tonemapping

基于人眼局部适应特性：

```hlsl
float LocalAdaptation(float centerLum, float surroundLum)
{
    // 根据周围亮度调整局部对比度
    float adaptation = lerp(centerLum, surroundLum, adaptationFactor);
    return centerLum / (adaptation + epsilon);
}
```

## 5. 游戏引擎中的实现

### 5.1 UE4 Filmic Tonemapper

```hlsl
// PostProcessTonemap.usf

float3 FilmToneMap(float3 Color)
{
    // 参数（来自Post Process Volume）
    float Slope = GetTonemapperSlope();
    float Toe = GetTonemapperToe();
    float Shoulder = GetTonemapperShoulder();
    float BlackClip = 0.0;
    float WhiteClip = 0.04;
    
    // 应用S曲线
    float3 numerator = Color * (Slope * Color + Toe * 0.5) + BlackClip;
    float3 denominator = Color * (Slope * Color + 0.5) + Shoulder;
    
    return numerator / denominator - BlackClip / Shoulder;
}
```

### 5.2 Unity HDRP Tonemapper

Unity提供多种Tonemapping模式：

| 模式 | 特点 |
|------|------|
| **None** | 线性映射，无色调映射 |
| **Neutral** | 轻微压缩，保持最大动态范围 |
| **ACES** | 标准ACES Filmic |
| **Custom** | 可调整参数的曲线 |

### 5.3 自定义Tonemapper

基于多项式的Tonemapper：

```hlsl
float3 CustomTonemap(float3 x)
{
    // 三次多项式拟合S曲线
    float3 a = x * x;
    float3 b = a * x;
    
    // 系数可调
    float c0 = 0.1;  // 常数项
    float c1 = 0.8;  // 线性项
    float c2 = 0.3;  // 二次项
    float c3 = -0.2; // 三次项（控制肩部）
    
    return saturate(c0 + c1 * x + c2 * a + c3 * b);
}
```

## 6. HDR色调映射

### 6.1 HDR与SDR的区别

| 特性 | SDR Tonemapping | HDR Tonemapping |
|------|----------------|----------------|
| 目标范围 | 0-100 nit | 0-1000+ nit |
| 压缩程度 | 大（10000→100） | 小（10000→1000） |
| 细节保留 | 有限 | 更多 |
| 白点 | 固定 | 可调整 |

### 6.2 BT.2390 EETF

ITU-R BT.2390定义的EETF（Electro-Optical Transfer Function）：

用于将内容亮度映射到显示设备亮度：

```
输入亮度 ──▶ 归一化 ──▶ Hermite曲线映射 ──▶ 反归一化 ──▶ 输出亮度
```

**关键参数**：
- $L_{max}$：内容最大亮度
- $L_{min}$：内容最小亮度
- $T_{max}$：显示最大亮度
- $T_{min}$：显示最小亮度

### 6.3 动态色调映射

根据场景内容动态调整：

```hlsl
float DynamicTonemapping(float luminance, float avgLuminance, float maxLuminance)
{
    // 根据场景平均亮度调整曝光
    float keyValue = 0.18; // 中灰
    float exposure = keyValue / avgLuminance;
    
    // 根据场景最大亮度调整压缩
    float compression = 1.0 / (1.0 + log2(maxLuminance / 1000.0));
    
    float exposed = luminance * exposure;
    return FilmicTonemap(exposed, compression);
}
```

## 7. 色调映射的评估

### 7.1 客观指标

| 指标 | 计算方法 | 目标 |
|------|---------|------|
| **对比度保留** | 局部对比度比值 | 接近1 |
| **细节保留** | 梯度相似度 | 高 |
| **色偏** | 颜色差异 | 低 |
| **时间一致性** | 帧间差异 | 低 |

### 7.2 主观评估

- 自然感：是否看起来像真实场景
- 细节可见：高光/暗部是否有细节
- 对比度：是否有足够的层次感
- 色彩：是否保持原始色彩关系

## 8. 最佳实践

### 8.1 参数选择建议

| 场景类型 | 推荐Tonemapper | 参数调整 |
|---------|---------------|---------|
| 写实风格 | ACES Filmic | 默认参数 |

### 8.2 调试技巧

1. **使用直方图**：检查亮度分布
2. **分离通道**：检查各通道压缩情况
3. **对比参考**：与影视HDR内容对比
4. **多设备测试**：在不同显示器上验证
