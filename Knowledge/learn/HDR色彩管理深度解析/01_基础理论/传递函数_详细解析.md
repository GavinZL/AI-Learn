# 传递函数：OETF与EOTF详解

> 深入理解Gamma校正、PQ、HLG等传递函数的原理与应用

---

## 1. 为什么需要传递函数

### 1.1 人眼视觉的非线性特性

人眼对亮度的感知遵循**韦伯-费希纳定律（Weber-Fechner Law）**：

> 感知到的亮度变化与物理亮度的相对变化成正比，而非绝对变化。

**Barten Ramp模型**描述了人眼在不同亮度下可察觉的最小亮度差（JND, Just Noticeable Difference）：

```
可察觉亮度差 (%)
    ▲
 10%│    ╭────╮
    │   ╱      ╲
  5%│  ╱        ╲
    │ ╱          ╲
  1%│╱            ╲
    └──────────────────▶ 亮度 (log scale)
      0.01   1   100   10000 nit
```

**关键洞察**：
- 在暗部（<1 nit），人眼可察觉约1%的亮度差
- 在亮部（>1000 nit），人眼可察觉约10%的亮度差
- 这意味着暗部需要更精细的量化

### 1.2 编码效率优化

假设使用8-bit整数存储（256个等级）：

**线性编码的问题**：
- 暗部：0-1 nit 只有约2-3个等级 → 明显色阶
- 亮部：100-1000 nit 有约230个等级 → 过度量化

**非线性编码的优势**：
- 将更多编码值分配给暗部
- 亮部使用较粗的量化（人眼察觉不到）

## 2. Gamma校正

### 2.1 Gamma的双重角色

Gamma既描述一种物理现象，也是一种编码策略。理解这一点对正确使用传递函数至关重要。

#### 角色一：CRT物理特性（历史起源）

早期CRT显示器存在固有的非线性响应：

```
CRT输出亮度 = (输入电压)^γ
```

其中 $γ ≈ 2.2-2.5$。

为了得到正确的显示结果，需要预先对信号进行补偿：

```
存储信号 = (场景亮度)^(1/γ)
```

这就是**EOTF**的物理来源——显示器"自带"Gamma 2.2的转换特性。

#### 角色二：感知编码策略（现代核心）

现代LCD已经没有CRT的非线性特性，但我们**主动保留**Gamma编码，因为它服务于**人眼视觉特性**：

| 人眼特性 | 说明 |
|---------|------|
| 对暗部更敏感 | 能分辨0.01和0.02的亮度差异 |
| 对亮部较迟钝 | 难以区分0.8和0.9的亮度差异 |

**Gamma编码（OETF）**在存储/传输阶段将更多比特位分配给暗部，实现感知均匀编码：

```
线性场景亮度 L → 编码值 V = L^(1/2.2)  [压缩暗部，扩展亮部范围]
```

#### 完整链路中的两次Gamma

```
场景线性光 ──[OETF: Gamma 1/2.2]──▶ 存储/传输（非线性信号）
                                              ↓
显示亮度 ◄──[EOTF: Gamma 2.2]────── 显示器解码（还原线性光）
```

| 阶段 | 函数类型 | Gamma作用 | 目的 |
|------|---------|-----------|------|
| **拍摄/编码** | OETF | 感知均匀编码 | 优化比特分配，匹配人眼敏感度 |
| **显示/解码** | EOTF | 还原线性光 | 补偿CRT历史 + 正确显示 |

**关键洞察**：Gamma公式（如sRGB OETF）描述的是在**内容创作/存储阶段**应用的编码函数，其核心目的是利用人眼特性在有限位深下最大化视觉质量，而非仅仅"矫正显示器"。

### 2.2 现代Gamma编码

尽管LCD可以线性响应，Gamma编码仍被保留，因为：

1. **兼容性**：海量历史内容使用Gamma编码
2. **感知优化**：匹配人眼对暗部的敏感度
3. **噪声抑制**：暗部信号被放大，压缩传输噪声影响

### 2.3 sRGB传递函数

sRGB定义了分段传递函数：

**OETF（编码，Linear → sRGB）**：

$$
V = \begin{cases}
12.92 \cdot L & L \leq 0.0031308 \\
1.055 \cdot L^{1/2.4} - 0.055 & L > 0.0031308
\end{cases}
$$

**EOTF（解码，sRGB → Linear）**：

$$
L = \begin{cases}
\frac{V}{12.92} & V \leq 0.04045 \\
\left(\frac{V + 0.055}{1.055}\right)^{2.4} & V > 0.04045
\end{cases}
$$

**为什么分段？**

- 线性段（接近0）：避免在暗部出现无限大斜率，减少数值精度问题
- 幂函数段：主要的Gamma压缩

**等效Gamma值**：

整体等效Gamma约为2.2，但在不同亮度区域略有变化：
- 暗部：接近线性（Gamma ≈ 1）
- 中灰：Gamma ≈ 2.2
- 高光：Gamma ≈ 2.4

### 2.4 Rec. 709传递函数

Rec. 709与sRGB的三原色相同，但传递函数略有不同：

**OETF**：

$$
V = \begin{cases}
4.5 \cdot L & L < 0.018 \\
1.099 \cdot L^{0.45} - 0.099 & L \geq 0.018
\end{cases}
$$

**关键区别**：

| 特性 | sRGB | Rec. 709 |
|------|------|---------|
| 线性段阈值 | 0.00313 | 0.018 |
| 线性段斜率 | 12.92 | 4.5 |
| 幂指数 | 1/2.4 ≈ 0.4167 | 0.45 |
| 等效Gamma | ~2.2 | ~2.4 |

**EOTF（BT.1886）**：

Rec. 709本身未定义EOTF，BT.1886补充：

$$
L = \left(\frac{V + a}{1 + a}\right)^{2.4}
$$

其中 $a$ 由显示器的黑电平和白电平决定。

## 3. PQ（Perceptual Quantizer）

### 3.1 设计目标

PQ（ST 2084）由杜比实验室开发，设计目标：

1. **感知均匀**：每个编码等级对应相同的感知差异
2. **绝对亮度**：编码值直接对应物理亮度（尼特）
3. **宽动态范围**：支持0.0001 - 10000 尼特
4. **前向兼容**：为未来更高亮度显示器预留空间

### 3.2 Barten Ramp与PQ设计

PQ基于Barten Ramp模型设计，确保：

$$
\frac{\Delta L}{L} \approx \text{JND阈值}
$$

在整个亮度范围内，相邻编码值的亮度差都接近或低于人眼可察觉阈值。

**不同编码方式的对比**：

```
亮度差 (%)
    ▲
    │    ╭──── 10-bit Gamma
 10%│   ╱
    │  ╱   ╭─── 12-bit PQ
  5%│ ╱   ╱
    │╱   ╱
  1%├────╱────── JND阈值
    │
    └──────────────────▶ 亮度 (log)
      0.1  1  10  100  1000 10000 nit
```

### 3.3 PQ EOTF公式

**EOTF（PQ → Linear，单位：尼特）**：

$$
L = \left(\frac{\max(N^{1/m_2} - c_1, 0)}{c_2 - c_3 \cdot N^{1/m_2}}\right)^{1/m_1}
$$

其中：
- $N$：归一化信号值（0-1）
- $m_1 = 2610/4096 \times 1/4 = 0.1593017578125$
- $m_2 = 2523/4096 \times 128 = 78.84375$
- $c_1 = 3424/4096 = 0.8359375$
- $c_2 = 2413/4096 \times 32 = 18.8515625$
- $c_3 = 2392/4096 \times 32 = 18.6875$

**OETF（Linear → PQ）**：

$$
N = \left(\frac{c_1 + c_2 \cdot L^{m_1}}{1 + c_3 \cdot L^{m_1}}\right)^{m_2}
$$

### 3.4 PQ的关键特性

#### 3.4.1 绝对亮度编码

PQ值直接对应物理亮度：

| PQ值 | 亮度（尼特） | 典型场景 |
|------|------------|---------|
| 0.0 | 0 | 纯黑 |
| 0.1 | ~0.1 | 极暗阴影 |
| 0.5 | ~100 | SDR白点 |
| 0.75 | ~1000 | HDR显示器峰值 |
| 1.0 | 10000 | HDR标准上限 |

#### 3.4.2 与Gamma的对比

| 特性 | Gamma | PQ |
|------|-------|-----|
| 参考点 | 相对（0-1） | 绝对（尼特） |
| 动态范围 | 约2个数量级 | 5个数量级 |
| 感知均匀性 | 较差 | 优秀 |
| 位深需求 | 10-bit勉强 | 10-bit足够，12-bit理想 |
| 兼容性 | 广泛 | 需要HDR支持 |

### 3.5 PQ的计算优化

PQ计算涉及多次幂运算，在着色器中需要优化：

**HLSL实现**：
```hlsl
float3 ST2084ToLinear(float3 pq)
{
    const float m1 = 0.1593017578125;
    const float m2 = 78.84375;
    const float c1 = 0.8359375;
    const float c2 = 18.8515625;
    const float c3 = 18.6875;
    const float C = 10000.0;
    
    float3 Np = pow(pq, 1.0 / m2);
    float3 L = Np - c1;
    L = max(0.0, L);
    L = L / (c2 - c3 * Np);
    L = pow(L, 1.0 / m1);
    return L * C;
}
```

**优化建议**：
- 预计算 $1/m_1$ 和 $1/m_2$
- 使用快速幂近似（如果对精度要求不高）
- 在LUT生成阶段完成PQ转换，运行时查表

## 4. HLG（Hybrid Log-Gamma）

### 4.1 设计目标

HLG由BBC和NHK联合开发，设计目标：

1. **向后兼容**：同一信号可在SDR和HDR显示器上显示
2. **无元数据**：不依赖动态元数据
3. **信号简单**：编码端简单，解码端适配

### 4.2 HLG原理

HLG使用**双曲线设计**：

**低亮度区域（0-1，相对值）**：
- 使用与Gamma类似的曲线
- SDR显示器可直接显示合理效果

**高亮度区域（>1）**：
- 使用对数曲线
- HDR显示器可恢复高亮细节

### 4.3 HLG OETF公式

**OETF（Scene → HLG）**：

$$
E' = \begin{cases}
\frac{\sqrt{3 \cdot E}}{2} & 0 \leq E \leq \frac{1}{12} \\
a \cdot \ln(E - b) + c & \frac{1}{12} < E \leq 1
\end{cases}
$$

其中：
- $a = 0.17883277$
- $b = 0.28466892$
- $c = 0.55991073$

**EOTF（HLG → Display）**：

HLG的EOTF依赖于显示器的峰值亮度，需要系统伽马调整：

$$
E = \begin{cases}
\frac{4 \cdot E'^2}{3} & E' \leq \frac{1}{2} \\
\frac{\exp((E' - c)/a) + b}{12} & E' > \frac{1}{2}
\end{cases}
$$

### 4.4 HLG系统伽马

HLG使用**系统伽马（System Gamma）**来适应不同显示环境：

$$
\gamma_{system} = 1.2 + 0.42 \cdot \log_{10}(L_{peak}/1000)
$$

其中 $L_{peak}$ 是显示器峰值亮度（尼特）。

**典型值**：

| 环境 | 显示器峰值 | 系统Gamma |
|------|-----------|----------|
| 暗室（影院） | 48 nit | 1.2 |
| 家庭客厅 | 300 nit | 1.4 |
| 明亮房间 | 1000 nit | 1.6 |

### 4.5 PQ vs HLG 对比

| 特性 | PQ | HLG |
|------|-----|-----|
| 设计目标 | 最佳图像质量 | 广播兼容性 |
| 绝对/相对 | 绝对亮度 | 相对亮度 |
| 元数据 | 需要 | 不需要 |
| SDR兼容性 | 差（需要色调映射） | 好（可直接显示） |
| 主要应用 | 影视制作、游戏、流媒体 | 广播电视、直播 |
| 典型位深 | 10-bit/12-bit | 10-bit |
| 峰值亮度 | 固定10000 nit | 取决于显示器 |

## 5. 传递函数的选择

### 5.1 应用场景决策树

```
                    开始
                      │
              ┌───────┴───────┐
              ▼               ▼
        需要SDR兼容？     纯HDR内容？
              │               │
         是 ◄─┤               ├─► 使用PQ
              │               │
              ▼               │
        广播/直播场景？        │
              │               │
         是 ◄─┤               │
              │               │
              ▼               │
        使用HLG ◄─────────────┘
              │
              ▼
        使用Gamma + 色调映射
```

### 5.2 游戏引擎中的选择

| 场景 | 推荐方案 | 理由 |
|------|---------|------|
| PC游戏HDR | PQ + 自定义Tonemapping | 精确控制，最佳质量 |
| 主机游戏 | PQ（PS5/Xbox支持） | 标准化，一致性 |
| 移动端HDR | HLG或PQ子集 | 兼容性考虑 |
| 云游戏 | HLG | 网络波动下的鲁棒性 |

## 6. 传递函数的常见误区

### 6.1 "Gamma 2.2"的误解

**误区**：sRGB就是Gamma 2.2

**事实**：
- sRGB是分段函数，等效Gamma在2.0-2.4之间变化
- 纯Gamma 2.2曲线与sRGB有可见差异
- 在暗部，sRGB的线性段与纯Gamma 2.2差异明显

### 6.2 "Linear工作流"的误解

**误区**：Linear工作流意味着没有颜色空间

**事实**：
- Linear工作流仍需要指定RGB颜色空间（如sRGB Linear、ACEScg）
- "Linear"仅指传递函数，不是完整的颜色空间定义
- 不同Linear空间的色域可能不同

### 6.3 PQ的"绝对亮度"误解

**误区**：PQ编码的内容在任何显示器上亮度都相同

**事实**：
- PQ编码的是**内容创作者意图的亮度**
- 实际显示亮度受显示器能力限制
- 需要色调映射（Tonemapping）适配不同显示器

## 7. 总结

| 传递函数 | 核心公式 | 适用场景 |
|---------|---------|---------|
| **sRGB** | 分段幂函数 | SDR显示、网页、通用图像 |
| **Rec. 709** | 类似sRGB，参数不同 | 广播电视（SDR） |
| **PQ** | 基于Barten Ramp的复杂曲线 | HDR影视、游戏、专业制作 |
| **HLG** | 混合Gamma-Log曲线 | HDR广播、需要SDR兼容 |
