## 1. HDR相机成像：从有限传感器到无限动态范围

**核心观点**：HDR相机技术的本质是突破单次曝光的动态范围限制，通过时域或空域复用捕获完整的场景亮度信息。

### 1.1 为什么需要HDR采集

真实世界的亮度范围远超传统传感器的单次捕获能力，这是HDR采集技术存在的根本原因。

#### 1.1.1 真实场景动态范围 vs 传感器动态范围

| 场景类型 | 动态范围(stops) | 亮度范围(cd/m²) | 典型示例 |
|---------|----------------|-----------------|---------|
| **室内均匀光照** | 6-8 | 10 - 500 | 办公室、教室 |
| **室外阴天** | 10-12 | 100 - 10,000 | 多云天气户外 |
| **室外晴天** | 14-16 | 100 - 100,000 | 阳光直射场景 |
| **逆光人像** | 18-20 | 0.1 - 100,000 | 窗前人像、日落人像 |
| **夜景城市** | 20-24 | 0.001 - 10,000 | 霓虹灯+暗巷 |
| **星空银河** | 24+ | 0.0001 - 1,000 | 天文摄影 |

**传感器动态范围对比**：

| 传感器类型 | 单帧动态范围(dB) | 等效stops | 典型应用 |
|-----------|-----------------|----------|---------|
| **消费级手机CMOS** | 60-72 | 10-12 | 手机摄影 |
| **专业APS-C** | 72-78 | 12-13 | 无反相机 |
| **全画幅CMOS** | 78-84 | 13-14 | 专业单反/无反 |
| **中画幅CCD/CMOS** | 84-90 | 14-15 | 商业摄影 |
| **电影摄影机** | 84-90 | 14-15 | ARRI、RED |

> 📌 **关键洞察**：即使最顶级的单帧传感器（~15 stops）也无法完整捕获逆光或夜景场景（20+ stops）的全部亮度信息。HDR采集不是"锦上添花"，而是**还原真实场景的必要手段**。

#### 1.1.2 传统单帧曝光的局限性

单帧曝光面临的核心矛盾：

```
                    曝光选择困境
                         │
        ┌────────────────┼────────────────┐
        ▼                ▼                ▼
    短曝光           中等曝光          长曝光
    (保高光)         (折中)           (保暗部)
        │                │                │
        ▼                ▼                ▼
   暗部死黑         两头损失        高光过曝
   噪声大           细节丢失         信息裁切
```

**量化分析**：

假设传感器动态范围为12 stops，场景动态范围为18 stops：

$$
DR_{loss} = DR_{scene} - DR_{sensor} = 18 - 12 = 6 \text{ stops}
$$

无论如何选择曝光点，都有6 stops的信息被裁切或淹没在噪声中。

### 1.2 HDR采集技术分类总览

HDR采集技术按照信息复用维度分为两大类：

```
                          HDR采集技术分类
                                │
           ┌────────────────────┼────────────────────┐
           ▼                                         ▼
      时域方法                                   空域方法
   (多帧时间复用)                             (单帧空间复用)
           │                                         │
    ┌──────┼──────┐                    ┌─────────────┼─────────────┐
    ▼      ▼      ▼                    ▼             ▼             ▼
 Bracketing  Staggered  Burst      Split-Pixel   DOL-HDR    Quad Bayer
 (顺序曝光)  (交替行)   (连续拍摄)  (大小像素)   (数字重叠)   (像素合并)
    │          │         │             │             │             │
    ▼          ▼         ▼             ▼             ▼             ▼

 运动伪影   运动/DR    计算量大     伪影        略低DR       分辨率选择
```

| 方法类别 | 代表技术 | DR扩展量 | 运动伪影 | 实时性 | 典型应用 |
|---------|---------|---------|---------|-------|---------|
| **时域多帧** | Exposure Bracketing | +6~10 stops | 严重 | 差 | 风景摄影 |

| **空域分像素** | Split-Pixel | +3~4 stops | 无 | 优秀 | 安防监控 |
| **空域重叠** | DOL-HDR | +4~6 stops | 轻微 | 优秀 | 车载相机 |
| **混合方案** | 多帧+传感器HDR | +6~8 stops | 轻微 | 好 | 旗舰手机 |

### 1.3 HDR采集的核心挑战

HDR采集技术需要解决四大核心挑战：

#### 1.3.1 运动伪影（Ghost Artifacts）

当场景或相机在多帧采集期间发生运动时，融合后的图像会出现"鬼影"：

```
帧1(长曝光)    帧2(短曝光)     直接融合结果
┌─────────┐   ┌─────────┐    ┌─────────┐
│    ○    │   │      ○  │    │ ○  ○    │  ← 鬼影
│  ┌──┐   │   │  ┌──┐   │    │  ┌──┐   │
│  │人│   │   │  │人│   │    │  │人│   │
│  └──┘   │   │  └──┘   │    │  └──┘   │
└─────────┘   └─────────┘    └─────────┘
   位置A         位置B        物体重叠
```

#### 1.3.2 噪声累积与放大

短曝光帧用于恢复高光细节，但其暗部SNR极低：

$$
SNR_{short} = \frac{Signal_{short}}{Noise_{short}} = \frac{t_{short}}{t_{long}} \times SNR_{long}
$$

当曝光比为1:16时，短曝光暗部的SNR仅为长曝光的1/16。

#### 1.3.3 计算开销

| 处理阶段 | 计算复杂度 | 4K@30fps预算 | 关键瓶颈 |
|---------|-----------|-------------|---------|
| 多帧对齐 | O(N×W×H×S) | ~5ms | 搜索范围S |
| 运动检测 | O(N×W×H) | ~2ms | 帧差计算 |
| HDR合并 | O(N×W×H) | ~1ms | 权重计算 |
| 去鬼影 | O(N×W×H×K) | ~3ms | 邻域大小K |
| **总计** | — | ~11ms | 帧间隔33ms |

#### 1.3.4 实时性约束

视频HDR与照片HDR的关键差异：

| 约束项 | HDR照片 | HDR视频 |
|-------|--------|--------|
| 处理时间 | 数秒~数分钟 | <33ms(30fps) |
| 帧间一致性 | 不需要 | 必须保证 |
| 存储/传输 | 单帧 | 连续流 |
| AE策略 | 单次决策 | 持续追踪 |

## 2. 时域HDR：多帧合成技术

**核心观点**：通过不同曝光时间的多帧图像融合，在时间维度扩展动态范围。

### 2.1 多帧HDR基本原理

#### 2.1.1 长短曝光的信息互补

多帧HDR的核心思想是利用不同曝光帧的**信息互补性**：

```
场景亮度分布          长曝光响应            短曝光响应           融合结果
                    (捕获暗部)           (捕获高光)
     ▲                 ▲                   ▲                  ▲
 亮度│ ████           │ ████饱和          │ ████有效         │ ████
     │ ████           │ ████              │ ████              │ ████
     │ ████           │ ████有效          │ ████              │ ████
     │ ████           │ ████              │ ████噪声          │ ████
     │ ████           │ ████              │ ████              │ ████
     └──────────     └──────────         └──────────        └──────────
       暗→亮           暗部细节            高光细节            完整DR
```

**核心公式**：场景辐照度的估计

$$
L_{scene}(x) = \frac{I(x)}{t \cdot g}
$$

其中：
- $L_{scene}(x)$：像素x的场景辐照度
- $I(x)$：传感器输出的数字值（DN）
- $t$：曝光时间
- $g$：传感器增益（ISO）

#### 2.1.2 曝光比设计策略

**2帧HDR策略**（最常用）：

| 曝光比 | DR扩展 | 典型场景 | 优缺点 |
|-------|-------|---------|-------|
| 1:4 | +2 stops | 轻度HDR | 融合自然，运动伪影小 |
| 1:16 | +4 stops | 标准HDR | 平衡效果，手机常用 |
| 1:64 | +6 stops | 极限HDR | 高光恢复强，融合困难 |

**3帧/4帧HDR策略**（专业场景）：

```
3帧策略示例：
曝光序列: [1/60s, 1/250s, 1/1000s]
曝光比:   [1 : 4 : 16]
DR扩展:   +4 stops (相邻帧) → 总计约+6 stops

4帧策略示例：
曝光序列: [1/30s, 1/125s, 1/500s, 1/2000s]
曝光比:   [1 : 4 : 16 : 64]
DR扩展:   总计约+8 stops
```

> 📌 **关键洞察**：帧数增加带来的DR收益是**递减**的，因为噪声和运动伪影的累积会抵消部分收益。实践中3帧已是较好平衡点。

### 2.2 长短曝光合并算法

#### 2.2.1 线性加权融合

HDR合并的标准公式：

$$
L_{HDR}(x) = \frac{\sum_{i=1}^{N} w_i(x) \cdot \frac{I_i(x)}{t_i}}{\sum_{i=1}^{N} w_i(x)}
$$

其中：
- $N$：曝光帧数
- $w_i(x)$：第i帧在像素x的权重
- $I_i(x)$：第i帧像素x的强度值
- $t_i$：第i帧的曝光时间

#### 2.2.2 权重函数设计

**帽形函数（Hat Function）**：

$$
w(I) = \begin{cases}
\frac{I - I_{min}}{I_{mid} - I_{min}} & I_{min} \leq I < I_{mid} \\
\frac{I_{max} - I}{I_{max} - I_{mid}} & I_{mid} \leq I \leq I_{max} \\
0 & \text{otherwise}
\end{cases}
$$

**三角函数的物理意义**：

```
权重
  1 │        ╱╲
    │       ╱  ╲
    │      ╱    ╲
    │     ╱      ╲
  0 │────╱────────╲────
    0   0.2  0.5  0.8  1.0  像素值(归一化)
         │         │
       噪声区     饱和区
      权重低     权重低
```

- **低像素值区域**（<0.2）：信噪比低，降低权重
- **高像素值区域**（>0.8）：接近饱和，非线性响应，降低权重
- **中间区域**（0.2-0.8）：信噪比高且线性，最高权重

#### 2.2.3 Python代码示例：多帧HDR合成

```python
"""
多帧HDR合成算法实现
实现线性加权融合的HDR合并流程
"""
import numpy as np
from typing import List, Tuple

def hat_weight(intensity: np.ndarray, 
               i_min: float = 0.05, 
               i_max: float = 0.95) -> np.ndarray:
    """
    计算帽形权重函数
    
    Args:
        intensity: 归一化像素强度 [0, 1]
        i_min: 最小有效强度阈值
        i_max: 最大有效强度阈值
    
    Returns:
        权重数组，形状与输入相同
    """
    i_mid = (i_min + i_max) / 2
    weight = np.zeros_like(intensity)
    
    # 上升段: i_min -> i_mid
    mask_rise = (intensity >= i_min) & (intensity < i_mid)
    weight[mask_rise] = (intensity[mask_rise] - i_min) / (i_mid - i_min)
    
    # 下降段: i_mid -> i_max
    mask_fall = (intensity >= i_mid) & (intensity <= i_max)
    weight[mask_fall] = (i_max - intensity[mask_fall]) / (i_max - i_mid)
    
    return weight

def merge_hdr_linear(frames: List[np.ndarray], 
                     exposure_times: List[float],
                     black_level: float = 0.0,
                     white_level: float = 1.0) -> np.ndarray:
    """
    线性加权HDR合并
    
    Args:
        frames: 多帧图像列表，每帧形状为(H, W, 3)，值域[0, 1]
        exposure_times: 对应的曝光时间列表（秒）
        black_level: 传感器黑电平（归一化）
        white_level: 传感器白电平（归一化）
    
    Returns:
        HDR图像，场景参考线性值
    """
    assert len(frames) == len(exposure_times), "帧数与曝光时间数量不匹配"
    
    # 初始化累加器
    weighted_sum = np.zeros_like(frames[0], dtype=np.float64)
    weight_sum = np.zeros_like(frames[0], dtype=np.float64)
    
    for frame, t in zip(frames, exposure_times):
        # 1. 黑电平校正
        corrected = (frame - black_level) / (white_level - black_level)
        corrected = np.clip(corrected, 0, 1)
        
        # 2. 计算权重
        # 对每个通道独立计算权重，取最小值避免色彩通道饱和
        weights_rgb = hat_weight(corrected)
        weight = np.min(weights_rgb, axis=2, keepdims=True)
        weight = np.broadcast_to(weight, corrected.shape)
        
        # 3. 归一化到场景辐照度
        # L = I / t (假设增益已归一化)
        radiance = corrected / t
        
        # 4. 加权累加
        weighted_sum += weight * radiance
        weight_sum += weight
    
    # 5. 计算最终HDR值
    # 避免除零
    weight_sum = np.maximum(weight_sum, 1e-10)
    hdr = weighted_sum / weight_sum
    
    return hdr

def demo_hdr_merge():
    """
    演示HDR合并流程
    """
    # 模拟3帧不同曝光的输入
    H, W = 100, 100
    
    # 曝光时间: 1/250s, 1/60s, 1/15s
    exposure_times = [0.004, 0.0167, 0.0667]
    
    # 模拟场景：中心有高亮光源
    scene_radiance = np.ones((H, W, 3)) * 0.18  # 18%灰
    
    # 添加高亮区域（模拟太阳或灯光）
    center_y, center_x = H // 2, W // 2
    y, x = np.ogrid[:H, :W]
    dist = np.sqrt((x - center_x)**2 + (y - center_y)**2)
    bright_mask = dist < 20
    scene_radiance[bright_mask] = 100.0  # 100倍于中灰的高光
    
    # 模拟传感器响应（线性，带饱和）
    frames = []
    for t in exposure_times:
        response = scene_radiance * t
        response = np.clip(response, 0, 1)  # 模拟饱和
        # 添加噪声
        noise = np.random.normal(0, 0.01, response.shape)
        response = np.clip(response + noise, 0, 1)
        frames.append(response)
    
    # HDR合并
    hdr_result = merge_hdr_linear(frames, exposure_times)
    
    print(f"输入帧数: {len(frames)}")
    print(f"曝光范围: {min(exposure_times):.4f}s - {max(exposure_times):.4f}s")
    print(f"HDR结果动态范围: {hdr_result.max() / max(hdr_result.min(), 1e-6):.1f}x")
    
    return hdr_result

if __name__ == "__main__":
    demo_hdr_merge()
```

### 2.3 运动补偿与配准

多帧HDR的关键前处理是**图像配准（Registration）**，确保不同曝光帧的场景对齐。

#### 2.3.1 全局运动估计（Homography）

适用于相机平移、旋转导致的全局运动：

$$
\begin{bmatrix} x' \\ y' \\ 1 \end{bmatrix} = H \cdot \begin{bmatrix} x \\ y \\ 1 \end{bmatrix}
= \begin{bmatrix} h_{11} & h_{12} & h_{13} \\ h_{21} & h_{22} & h_{23} \\ h_{31} & h_{32} & 1 \end{bmatrix} \cdot \begin{bmatrix} x \\ y \\ 1 \end{bmatrix}
$$

**实现流程**：

```
帧1 ──▶ 特征提取(SIFT/ORB) ──┐
                             ├──▶ 特征匹配 ──▶ RANSAC ──▶ H矩阵 ──▶ 变换帧2
帧2 ──▶ 特征提取(SIFT/ORB) ──┘
```

#### 2.3.2 光流（Optical Flow）方法

适用于局部运动（如场景中的移动物体）：

**Lucas-Kanade假设**：邻域内像素运动一致

$$
I(x, y, t) = I(x + u, y + v, t + 1)
$$

展开并近似：

$$
I_x u + I_y v + I_t = 0
$$

其中 $I_x, I_y, I_t$ 分别是图像的空间和时间梯度。

#### 2.3.3 块匹配（Block Matching）

在搜索窗口内寻找最佳匹配块：

$$
(u^*, v^*) = \arg\min_{u,v} \sum_{(i,j) \in B} |I_1(x+i, y+j) - I_2(x+i+u, y+j+v)|
$$

| 方法 | 计算复杂度 | 精度 | 适用场景 |
|-----|-----------|-----|---------|
| Homography | O(N特征点) | 像素级 | 纯相机运动 |
| 光流 | O(W×H×迭代) | 亚像素级 | 任意运动 |
| 块匹配 | O(W×H×S²) | 块级 | 实时系统 |

### 2.4 去鬼影（Ghost Removal）

#### 2.4.1 鬼影成因分析

鬼影的产生源于**运动区域的错误融合**：

```
时刻t1: 物体在位置A        时刻t2: 物体在位置B
┌─────────────────┐       ┌─────────────────┐
│                 │       │                 │
│    ┌───┐        │       │        ┌───┐    │
│    │ A │        │       │        │ B │    │
│    └───┘        │       │        └───┘    │
│                 │       │                 │
└─────────────────┘       └─────────────────┘

直接融合结果:
┌─────────────────┐
│                 │
│    ┌───┐┌───┐   │  ← 两个位置都有残影
│    │ A'││ B'│   │     形成"鬼影"
│    └───┘└───┘   │
│                 │
└─────────────────┘
```

#### 2.4.2 基于参考帧的检测方法

选择一帧作为参考（通常是中间曝光帧），检测其他帧的运动区域：

$$
M_i(x) = \begin{cases}
1 & |I_i(x) - I_{ref}(x) \cdot \frac{t_i}{t_{ref}}| > \tau \\
0 & \text{otherwise}
\end{cases}
$$

其中：
- $M_i(x)$：运动掩码
- $\tau$：检测阈值（考虑噪声）

#### 2.4.3 运动检测 + 融合权重调整

```
┌────────────────────────────────────────────────────────────────────┐
│                      去鬼影融合流程                                  │
├────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  多帧输入 ──▶ 配准 ──▶ 运动检测 ──┬──▶ 权重调整 ──▶ HDR融合        │
│                          │        │                                 │
│                          ▼        │                                 │
│                      运动掩码M    │                                 │
│                          │        │                                 │
│                          ▼        │                                 │
│                   w'_i = w_i × (1-M_i)  ← 运动区域降权               │
│                                                                     │
└────────────────────────────────────────────────────────────────────┘
```

**权重调整公式**：

$$
w'_i(x) = w_i(x) \cdot (1 - M_i(x)) + w_i(x) \cdot M_i(x) \cdot \mathbb{1}_{i=ref}
$$

含义：运动区域只使用参考帧的值。

#### 2.4.4 基于深度学习的去鬼影

**HDRNet架构**（Google, 2017）：

```
输入: 多帧LDR ──▶ U-Net编码器 ──▶ 特征金字塔 ──▶ Attention ──▶ HDR输出
                     │                              │
                     └──────── 跳跃连接 ────────────┘
```

| 方法 | 优点 | 缺点 |
|-----|------|------|
| 传统检测 | 可解释、可调 | 阈值敏感 |
| 深度学习 | 端到端、效果好 | 训练数据依赖、计算量大 |

### 2.5 Bracketing vs Staggered模式

#### 2.5.1 顺序曝光（Bracketing）

传统的包围曝光模式：

```
时间轴 ──────────────────────────────────────────────▶

    │←── 长曝光 ──→│←── 中曝光 ──→│←── 短曝光 ──→│
    ████████████████  ████████████    ████████
    
    总采集时间: t_long + t_mid + t_short + 读出开销
```

**特点**：
- 每帧独立完整曝光
- 帧间时延大（典型50-200ms）
- 运动伪影风险高

#### 2.5.2 交替行曝光（Staggered/Interleaved）

在单次读出周期内交替曝光：

```
传感器行号
    0 │ ████████████████ ← 长曝光行
    1 │ ████████         ← 短曝光行
    2 │ ████████████████ ← 长曝光行
    3 │ ████████         ← 短曝光行
    ...
    
    时间 ──────────────────────────────▶
         │←────── 单帧周期 ──────→│
```

**特点**：
- 同一帧内包含多种曝光
- 帧间时延接近零
- 垂直分辨率降低50%

#### 2.5.3 对比分析

| 特性 | Bracketing | Staggered |
|-----|-----------|-----------|
| **帧间时延** | 50-200ms | <1ms |
| **运动伪影** | 严重 | 轻微 |
| **空间分辨率** | 完整 | 降低(行方向) |
| **DR扩展** | 更高(可用更多帧) | 受限(通常2帧) |
| **实时性** | 差 | 优秀 |
| **适用场景** | 静态风景、三脚架 | 手持拍摄、视频 |

> 📌 **关键洞察**：现代手机HDR普遍采用Staggered模式的变体（如Google HDR+的Burst融合），在保持高帧率的同时通过多帧降噪弥补单帧DR不足。

## 3. 空域HDR：单帧传感器技术

**核心观点**：通过传感器像素级设计，在单次曝光内同时捕获多种曝光信息，从根本上消除时域运动伪影。

### 3.1 Split-Pixel技术

#### 3.1.1 大小光电二极管设计

Split-Pixel技术在每个像素单元内集成两个不同面积的光电二极管：

```
传统像素               Split-Pixel像素
┌─────────────┐       ┌─────────────┐
│             │       │    大PD     │
│    单PD     │       │ (高感光度) │
│             │       ├─────────────┤
│             │       │ 小PD(低感) │
└─────────────┘       └─────────────┘
   1个信号              2个信号
```

**工作原理**：

| 光电二极管 | 面积比 | 感光度 | 捕获范围 |
|-----------|-------|--------|---------|
| 大PD | ~70% | 高 | 暗部细节 |
| 小PD | ~30% | 低 | 高光细节 |

**等效DR扩展**：

$$
DR_{split} = DR_{sensor} + \log_2(\frac{A_{large}}{A_{small}})
$$

若面积比为2.3:1，则DR扩展约1.2 stops。

### 3.2 DOL-HDR（Digital Overlap）

#### 3.2.1 行交替曝光原理

DOL-HDR在同一帧内让不同行使用不同的曝光起始时间，实现"数字重叠"：

```
行号  曝光时序（时间→）
      │←──────────── 帧周期 ──────────────→│
  0   │████████████████████████████████████│  长曝光(LE)
  1   │            ████████████████████████│  短曝光(SE)
  2   │████████████████████████████████████│  长曝光(LE)
  3   │            ████████████████████████│  短曝光(SE)
  ...
  N   │████████████████████████████████████│  
      │←─ LE开始 ─→│←─ SE开始 ─→│←─ 读出 ─→│
```

#### 3.2.2 时序图详解

```
                    DOL-HDR时序（2曝光模式）
                    
时间 ──────────────────────────────────────────────────────────▶

长曝光:  │←────────────── 曝光时间 LE ──────────────────→│
         ████████████████████████████████████████████████
         
短曝光:                    │←──── 曝光时间 SE ────→│
                           ██████████████████████████
         
读出:                                              │═══│
                                                   全局读出

重叠区:              │←─────── 重叠(Overlap) ──────→│
                     两种曝光同时进行的区域
```

**关键参数**：

| 参数 | 定义 | 典型值 |
|-----|------|-------|
| LE (Long Exposure) | 长曝光时间 | 30ms |
| SE (Short Exposure) | 短曝光时间 | 2ms |
| 曝光比 | LE / SE | 15:1 |
| 重叠时间 | 两曝光同时进行的时间 | 2ms |

#### 3.2.3 运动伪影特征与处理

DOL-HDR的运动伪影呈现**水平条纹**特征：

```
静止场景                运动物体场景
┌─────────────┐        ┌─────────────┐
│ LE ████████ │        │ LE ████████ │ ← 物体位置A
│ SE ████████ │        │ SE ███████→ │ ← 物体位置B（移动）
│ LE ████████ │        │ LE ████████ │
│ SE ████████ │        │ SE ███████→ │
└─────────────┘        └─────────────┘
                       运动物体在LE/SE行出现位置偏移
```

**处理策略**：
1. **行间插值**：利用相邻行信息平滑过渡
2. **运动检测**：识别运动区域，降低短曝光权重
3. **帧间参考**：结合前后帧信息补偿

## 4. HDR RAW处理与合成

**核心观点**：HDR RAW处理的目标是将多通道传感器数据融合为高精度的场景参考线性图像（Scene-referred Linear）。

### 4.1 线性化与对齐

#### 4.1.1 各通道黑电平校正

不同曝光通道的黑电平可能不同：

$$
I_{corrected,i}(x) = I_{raw,i}(x) - BL_i
$$

其中 $BL_i$ 是第i个曝光通道的黑电平。

**黑电平来源**：
- 暗电流（Dark Current）
- 固定模式噪声（FPN）
- 电路偏置

```python
def correct_black_level(raw_channels: dict, 
                        black_levels: dict) -> dict:
    """
    多通道黑电平校正
    
    Args:
        raw_channels: {'LE': array, 'SE': array, ...}
        black_levels: {'LE': bl_le, 'SE': bl_se, ...}
    
    Returns:
        校正后的通道字典
    """
    corrected = {}
    for name, data in raw_channels.items():
        bl = black_levels.get(name, 0)
        corrected[name] = np.maximum(data - bl, 0)
    return corrected
```

#### 4.1.2 曝光归一化

将所有通道归一化到统一的场景辐照度尺度：

$$
L_i(x) = \frac{I_{corrected,i}(x)}{t_i \cdot g_i}
$$

其中：
- $t_i$：第i通道的曝光时间
- $g_i$：第i通道的增益（ISO）

### 4.2 HDR合并到线性域

#### 4.2.1 场景参考线性的定义

**Scene-referred Linear**：图像值与场景实际辐照度成正比的线性表示。

```
场景辐照度          传感器响应          场景参考线性
(物理量)           (数字值)            (重建值)
                                       
   L_scene    ────▶    I_raw    ────▶    L_estimated
                                       
   [cd/m²]           [DN/count]         [相对或绝对亮度]
```

**关键特性**：
- 保留完整的动态范围信息
- 支持物理准确的后期处理（如曝光调整）
- 是色彩管理流程的标准输入

#### 4.2.2 从传感器DN到绝对亮度的转换

完整的转换链路：

$$
L_{scene}(x) [cd/m^2] = \frac{I_{raw}(x) - BL}{g \cdot t \cdot K \cdot QE(\lambda)}
$$

其中：
- $K$：系统增益常数（由标定确定）
- $QE(\lambda)$：量子效率（波长相关）

> 📌 **关键洞察**：在实际ISP中，通常使用**相对场景亮度**而非绝对值，因为绝对标定需要已知的参考光源。关键是保持**线性关系**。

### 4.3 HDR降噪策略

#### 4.3.1 利用多帧信息的联合降噪

多曝光帧提供了丰富的降噪信息：

```
┌────────────────────────────────────────────────────────────────┐
│                    多帧联合降噪策略                              │
├────────────────────────────────────────────────────────────────┤
│                                                                 │
│   场景区域          最佳信息来源        降噪策略                 │
│   ─────────────────────────────────────────────────────────    │
│   极暗区域          长曝光             时域滤波(多帧平均)        │
│   中间调            长曝光+短曝光      加权融合                  │
│   高光区域          短曝光             空域滤波                  │
│   运动区域          短曝光(单帧)       自适应空域滤波            │
│                                                                 │
└────────────────────────────────────────────────────────────────┘
```

#### 4.3.2 暗部噪声 vs 亮部细节的权衡

暗部降噪强度与细节保留的平衡：

```python
def adaptive_denoise_strength(luminance: np.ndarray, 
                               base_strength: float = 0.5) -> np.ndarray:
    """
    自适应降噪强度计算
    
    暗部: 高降噪强度(噪声明显)
    亮部: 低降噪强度(保留细节)
    
    Args:
        luminance: 亮度图 [0, 1]
        base_strength: 基础降噪强度
    
    Returns:
        逐像素降噪强度图
    """
    # 基于亮度的自适应强度
    # 暗部(L<0.1): strength接近1
    # 亮部(L>0.5): strength接近base_strength
    strength = base_strength + (1 - base_strength) * np.exp(-luminance * 5)
    return np.clip(strength, base_strength, 1.0)
```

### 5.2 PQ(ST.2084)编码路径

PQ编码将场景参考线性值映射到感知均匀的非线性域：

$$
N = \left(\frac{c_1 + c_2 \cdot L^{m_1}}{1 + c_3 \cdot L^{m_1}}\right)^{m_2}
$$

**HDR相机到PQ的完整流程**：

```
HDR线性场景值(相对)
        │
        │ 1. 绝对化: L_abs = L_rel × 参考白亮度
        ▼
绝对亮度值(cd/m²)
        │
        │ 2. 归一化: L_norm = L_abs / 10000
        ▼
归一化亮度(0-1)
        │
        │ 3. PQ OETF
        ▼
PQ编码值(10-bit: 0-1023)
```

**关键参数对应**：

| 场景参考值 | 绝对亮度(假设白=203cd/m²) | PQ码值(10-bit) |
|-----------|-------------------------|----------------|
| 0.18 (中灰) | 36.5 cd/m² | ~415 |
| 1.0 (SDR白) | 203 cd/m² | ~581 |
| 5.0 (高光) | 1015 cd/m² | ~769 |
| 49.0 (峰值) | 10000 cd/m² | 1023 |

### 5.3 HLG编码路径

HLG使用相对亮度编码，更适合广播场景：

```
HDR线性场景值
        │
        │ 1. 相对化: 以场景白点为参考
        ▼
相对亮度值(0-12倍参考白)
        │
        │ 2. HLG OETF (分段函数)
        ▼
HLG编码值(10-bit)
        │
        │ 3. 系统Gamma调整(显示端)
        ▼
适配不同显示器的输出
```

**HLG的广播兼容性**：

| 显示设备 | 处理方式 | 效果 |
|---------|---------|------|
| SDR电视 | 直接显示下半曲线 | 可接受的SDR图像 |
| HDR电视(1000nit) | 完整HLG EOTF | 标准HDR效果 |
| HDR电视(4000nit) | 调整系统Gamma | 更高亮度峰值 |

### 6.2 HDR视频的AE策略

#### 6.2.1 双通道AE

HDR视频需要同时管理长曝光和短曝光：

```
场景亮度检测
      │
      ├──────────────────────────────────┐
      ▼                                  ▼
 长曝光AE目标                        短曝光AE目标
 (暗部细节)                          (高光细节)
      │                                  │
      ▼                                  ▼
┌─────────────┐                   ┌─────────────┐
│ 目标: 中灰  │                   │ 目标: 高光  │
│ 在传感器    │                   │ 不饱和      │
│ 线性范围内  │                   │             │
└─────────────┘                   └─────────────┘
      │                                  │
      └──────────────┬───────────────────┘
                     ▼
              曝光比计算
         Ratio = t_LE / t_SE
                     │
                     ▼
            约束: 保持曝光比稳定
            (避免帧间跳变)
```

#### 6.2.2 帧间一致性

HDR视频AE的关键约束是**帧间平滑过渡**：

$$
EV_{target}(n) = EV_{current}(n-1) + \alpha \cdot (EV_{metered} - EV_{current}(n-1))
$$

其中：
- $\alpha$：平滑因子（0.1-0.3）
- $n$：帧序号

```python
def smooth_ae_transition(ev_current: float, 
                         ev_metered: float,
                         alpha: float = 0.15,
                         max_step: float = 0.3) -> float:
    """
    平滑AE过渡，避免帧间闪烁
    
    Args:
        ev_current: 当前帧曝光值(EV)
        ev_metered: 测光建议曝光值(EV)
        alpha: 平滑因子
        max_step: 单帧最大EV变化
    
    Returns:
        下一帧目标曝光值
    """
    delta = ev_metered - ev_current
    
    # 限制单帧变化幅度
    delta = np.clip(delta, -max_step, max_step)
    
    # 指数平滑
    ev_next = ev_current + alpha * delta
    
    return ev_next
```

### 6.3 HDR视频的压缩友好性

#### 6.3.1 码率分配与质量一致性

HDR视频编码面临的挑战：

| 挑战 | 原因 | 解决策略 |
|-----|------|---------|
| 高位深 | 10/12-bit vs 8-bit | HEVC/AV1原生支持 |
| 宽色域 | Rec.2020 vs Rec.709 | 子采样优化(4:2:0→4:2:2) |
| 高动态范围 | 更多细节层次 | 提高码率或使用自适应QP |
| 亮度分布不均 | 高光集中编码 | 基于亮度的码率分配 |

#### 6.3.2 PQ编码的压缩特性

PQ的感知均匀编码对压缩有利：

```
亮度域分布          PQ编码后分布
(真实场景)          (编码域)

    │   ▓▓▓            │ ░░░░░░░░░░
高光│   ▓▓▓▓           │ ░░░░░░░░░░░
    │   ▓▓▓▓▓          │ ░░░░░░░░░░░░
中调│   ▓▓▓▓▓▓▓        │ ░░░░░░░░░░░░
    │   ▓▓▓▓▓▓▓▓       │ ░░░░░░░░░░░░
暗部│   ▓▓▓▓▓▓▓▓▓▓     │ ░░░░░░░░░░░░
    └───────────       └─────────────
    集中在暗部          分布更均匀
    (压缩效率低)        (压缩效率高)
```

> 📌 **关键洞察**：PQ编码不仅服务于显示，也是**压缩友好**的设计——将感知重要的信息均匀分布，使编码器更高效分配比特。
