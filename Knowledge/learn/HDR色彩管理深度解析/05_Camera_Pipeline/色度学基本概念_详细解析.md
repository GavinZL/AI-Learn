## 1. 色度学与光度学基础导论

**核心观点**：光度学以人眼视觉响应为加权函数，将物理辐射量转换为视觉感知量，是相机系统设计和图像质量评估的理论基础。

### 1.1 辐射度学 vs 光度学

#### 1.1.1 基本概念对比

辐射度学（Radiometry）研究电磁辐射的纯物理测量，不考虑人眼的响应特性；光度学（Photometry）则专门研究可见光范围内、以人眼响应为加权的光学测量。

| 比较维度 | 辐射度学（Radiometry） | 光度学（Photometry） |
|---------|----------------------|---------------------|
| 研究对象 | 全电磁辐射谱 | 可见光（380-780nm） |
| 加权函数 | 无（等权） | 人眼视觉响应 V(λ) |
| 功率单位 | 瓦特（W） | 流明（lm） |
| 强度单位 | W/sr | 坎德拉（cd） |
| 照射度单位 | W/m² | 勒克斯（lx） |
| 辐射度单位 | W/(m²·sr) | cd/m²（nit） |
| 应用场景 | 红外/紫外测量、遥感 | 照明设计、相机系统 |

#### 1.1.2 转换关系

从辐射量到光度量的转换需要乘以光谱光视效率函数 $V(\lambda)$ 并积分：

$$\text{光度量} = K_m \int_{380}^{780} V(\lambda) \cdot \text{辐射量}(\lambda) \, d\lambda$$

其中：
- $K_m = 683$ lm/W：最大光谱光视效率（对应555nm）
- $V(\lambda)$：CIE标准明视觉光谱光视效率函数
- 积分范围：可见光波长380nm至780nm

> 📌 **关键洞察**：相机传感器的响应是辐射度量的（基于光子能量），而图像最终呈现给人眼时需要考虑光度学特性。这是ISP中色彩管理的物理基础。

### 1.3 CIE 标准观察者

#### 1.3.1 1931 CIE 2° 标准观察者

1931年，国际照明委员会（CIE）基于Wright和Guild的颜色匹配实验，定义了2°视场的标准观察者。此标准观察者用三个颜色匹配函数 $\bar{x}(\lambda)$、$\bar{y}(\lambda)$、$\bar{z}(\lambda)$ 描述人眼的色彩响应。

其中 $\bar{y}(\lambda)$ 等于明视觉光谱光视效率函数 $V(\lambda)$，这是光度学与色度学的关键连接点。

#### 1.3.2 1964 CIE 10° 标准观察者

针对大视场条件，CIE于1964年补充定义了10°标准观察者。主要差异：

| 特性 | 2° 观察者 (1931) | 10° 观察者 (1964) |
|-----|-----------------|------------------|
| 视场角 | 2° | 10° |
| 实验条件 | 中央凹视觉 | 包含周边视觉 |
| 短波响应 | 较低 | 较高 |
| 应用场景 | 精确色彩匹配 | 大面积色彩评估 |

## 2. 光通量（Luminous Flux）

**核心观点**：光通量是人眼感知的光功率总量，是所有光度学量的基础，在相机系统中用于评估光源特性和照明条件。

### 2.1 物理定义

#### 2.1.1 基本定义

光通量（Luminous Flux，符号 $\Phi_v$）定义为人眼感知的光功率，是辐射通量（Radiant Flux）经人眼光谱响应加权后的结果。

$$\Phi_v = K_m \int_{0}^{\infty} V(\lambda) \cdot \Phi_e(\lambda) \, d\lambda$$

其中：
- $\Phi_v$：光通量，单位：流明（lm）
- $K_m$：最大光谱光视效率，683 lm/W
- $V(\lambda)$：光谱光视效率函数
- $\Phi_e(\lambda)$：光谱辐射通量，单位：W/nm

#### 2.1.2 单位：流明（Lumen, lm）

流明的SI定义：在555nm波长处，1瓦特的辐射功率等于683流明的光通量。

$$1 \text{ lm} = \frac{1}{683} \text{ W (at 555nm)}$$

> 📌 **关键洞察**：683这个数字源于历史上对坎德拉的定义，保证新旧单位系统的兼容性。

### 2.2 与辐射通量的转换

#### 2.2.1 单色光的转换

对于波长为 $\lambda$ 的单色光：

$$\Phi_v = 683 \times V(\lambda) \times \Phi_e$$

例如：1W的555nm绿光产生683 lm，但1W的650nm红光仅产生683×0.107≈73 lm。

#### 2.2.2 宽谱光的转换

对于连续光谱的光源（如白光）：

$$\Phi_v = 683 \int_{380}^{780} V(\lambda) \cdot \frac{d\Phi_e}{d\lambda} \, d\lambda$$

#### 2.2.3 Python计算示例

```python
import numpy as np
from scipy import integrate

def calculate_luminous_flux(wavelengths: np.ndarray, 
                            spectral_power: np.ndarray,
                            v_lambda: np.ndarray) -> float:
    """
    计算光通量（流明）
    
    参数:
        wavelengths: 波长数组，单位nm
        spectral_power: 光谱功率分布，单位W/nm
        v_lambda: 对应波长的V(λ)值
    
    返回:
        光通量，单位lm
    """
    Km = 683  # lm/W，最大光谱光视效率
    
    # 积分计算光通量
    integrand = v_lambda * spectral_power
    luminous_flux = Km * integrate.trapezoid(integrand, wavelengths)
    
    return luminous_flux

# 示例：计算单色555nm光（1W）的光通量
wavelength_mono = np.array([554, 555, 556])
power_mono = np.array([0, 1, 0])  # 1W集中在555nm
v_lambda_mono = np.array([0.995, 1.0, 0.995])

flux = calculate_luminous_flux(wavelength_mono, power_mono, v_lambda_mono)
print(f"555nm单色光（1W）的光通量: {flux:.1f} lm")  # 输出: ~683 lm
```

## 3. 发光强度（Luminous Intensity）

**核心观点**：发光强度描述光源在特定方向上的发光能力，是SI七个基本单位之一坎德拉的定义基础，在相机中用于分析点光源成像特性。

### 3.1 物理定义

#### 3.1.1 基本定义

发光强度（Luminous Intensity，符号 $I_v$）定义为光源在某一方向上单位立体角内发出的光通量：

$$I_v = \frac{d\Phi_v}{d\Omega}$$

其中：
- $I_v$：发光强度，单位：坎德拉（cd）
- $\Phi_v$：光通量，单位：流明（lm）
- $\Omega$：立体角，单位：球面度（sr）

#### 3.1.2 单位：坎德拉（Candela, cd）

坎德拉是SI七个基本单位之一。2019年新定义：

> 坎德拉是光源在给定方向上的发光强度，该光源发出频率为 $540 \times 10^{12}$ Hz（约555nm）的单色辐射，其在该方向上的辐射强度为 $\frac{1}{683}$ W/sr。

这意味着：

$$1 \text{ cd} = 1 \text{ lm/sr} = \frac{1}{683} \text{ W/sr (at 555nm)}$$

### 3.2 立体角的定义与计算

#### 3.2.1 立体角的几何定义

立体角（Solid Angle，符号 $\Omega$）是三维空间中角度的推广，定义为球面上截取的面积与球半径平方的比值：

$$\Omega = \frac{A}{r^2}$$

其中：
- $A$：球面上截取的面积
- $r$：球的半径

单位：球面度（steradian, sr）

#### 3.2.2 常见立体角值

| 几何形状 | 立体角 (sr) | 说明 |
|---------|------------|------|
| 完整球面 | $4\pi \approx 12.57$ | 全空间 |
| 半球面 | $2\pi \approx 6.28$ | 上/下半空间 |
| 圆锥（半角θ） | $2\pi(1-\cos\theta)$ | 轴对称锥体 |
| 小角度锥（θ很小） | $\pi\theta^2$ | 近似值 |
| 1°×1°方形 | $3.05 \times 10^{-4}$ | 天文观测常用 |

#### 3.2.3 圆锥立体角计算

```python
import numpy as np

def cone_solid_angle(half_angle_deg: float) -> float:
    """
    计算圆锥的立体角
    
    参数:
        half_angle_deg: 圆锥半角，单位度
    
    返回:
        立体角，单位sr
    """
    theta = np.radians(half_angle_deg)
    omega = 2 * np.pi * (1 - np.cos(theta))
    return omega

# 示例：不同半角的圆锥立体角
angles = [1, 5, 10, 30, 60, 90]
for angle in angles:
    omega = cone_solid_angle(angle)
    print(f"半角{angle:3d}°的圆锥立体角: {omega:.4f} sr")

# 输出:
# 半角  1°的圆锥立体角: 0.0010 sr
# 半角  5°的圆锥立体角: 0.0239 sr
# 半角 10°的圆锥立体角: 0.0955 sr
# 半角 30°的圆锥立体角: 0.8418 sr
# 半角 60°的圆锥立体角: 3.1416 sr
# 半角 90°的圆锥立体角: 6.2832 sr（半球）
```

### 3.3 点光源的发光强度分布

#### 3.3.1 各向同性点光源

理想的各向同性点光源在所有方向上发光强度相等：

$$I_v = \frac{\Phi_v}{4\pi}$$

例如：1000 lm的全向点光源，各方向发光强度 = 1000/(4π) ≈ 79.6 cd。

### 3.4 朗伯体的发光特性

#### 3.4.1 朗伯余弦定律

朗伯体（Lambertian Surface）是一种理想的漫反射表面，其在任意观察方向上的亮度相等，但发光强度遵循余弦定律：

$$I_v(\theta) = I_0 \cos\theta$$

其中：
- $I_0$：法线方向（θ=0°）的发光强度
- $\theta$：观察方向与表面法线的夹角

#### 3.4.2 朗伯体的光通量计算

朗伯体向半球空间发出的总光通量：

$$\Phi_v = \int_{\text{hemisphere}} I_0 \cos\theta \, d\Omega = \pi I_0$$

这导出了亮度 $L$ 与发光强度 $I_0$ 的关系：

$$L = \frac{I_0}{A}$$

### 3.5 发光强度与相机的关联

#### 3.5.1 点光源成像

当相机拍摄点光源（如星星、远处灯光）时，成像的亮度取决于：

1. **光源的发光强度** $I_v$
2. **距离的平方反比** $1/r^2$
3. **镜头光圈** $f/\#$
4. **大气透过率** $\tau_{atm}$

#### 3.5.2 星光效应与衍射

强点光源在相机中产生的衍射星芒与发光强度有关：

$$\text{星芒长度} \propto \sqrt{I_v / r^2}$$

## 4. 照度（Illuminance）

**核心观点**：照度是单位面积上接收的光通量，直接决定传感器的入射光能量，是AE算法的核心输入参数和相机性能测试的关键指标。

### 4.1 物理定义

#### 4.1.1 基本定义

照度（Illuminance，符号 $E$ 或 $E_v$）定义为单位面积上入射的光通量：

$$E = \frac{d\Phi_v}{dA}$$

其中：
- $E$：照度，单位：勒克斯（lx）
- $\Phi_v$：入射光通量，单位：流明（lm）
- $A$：接收面积，单位：m²

#### 4.1.2 单位：勒克斯（Lux, lx）

$$1 \text{ lx} = 1 \text{ lm/m}^2$$

勒克斯是照明工程和摄影中最常用的光度学单位。

### 4.2 点光源的照度公式

#### 4.2.1 平方反比定律

点光源在距离 $r$ 处、与光轴夹角为 $\theta$ 的平面上产生的照度：

$$E = \frac{I_v \cos\theta}{r^2}$$

其中：
- $I_v$：光源在该方向的发光强度（cd）
- $r$：光源到被照面的距离（m）
- $\theta$：入射角（光线与被照面法线的夹角）

#### 4.2.2 直接照射（θ=0°）

当光线垂直入射时（θ=0°，cos θ=1）：

$$E = \frac{I_v}{r^2}$$

#### 4.2.3 照度计算示例

```python
import numpy as np

def point_source_illuminance(intensity: float, 
                             distance: float, 
                             incident_angle_deg: float = 0) -> float:
    """
    计算点光源产生的照度
    
    参数:
        intensity: 发光强度，单位cd
        distance: 距离，单位m
        incident_angle_deg: 入射角，单位度
    
    返回:
        照度，单位lx
    """
    theta = np.radians(incident_angle_deg)
    E = intensity * np.cos(theta) / (distance ** 2)
    return E

# 示例：100 cd的点光源
I = 100  # cd
distances = [0.5, 1.0, 2.0, 3.0, 5.0]
print("距离(m) | 照度(lx)")
print("-" * 20)
for d in distances:
    E = point_source_illuminance(I, d)
    print(f"  {d:.1f}   |  {E:.1f}")

# 输出:
# 距离(m) | 照度(lx)
# --------------------
#   0.5   |  400.0
#   1.0   |  100.0
#   2.0   |  25.0
#   3.0   |  11.1
#   5.0   |  4.0
```

### 4.3 面光源的照度积分

#### 4.3.1 积分公式

对于扩展面光源，接收点的照度需要对整个发光面积分：

$$E = \int_A \frac{L \cos\theta_s \cos\theta_r}{r^2} \, dA_s$$

其中：
- $L$：面光源的亮度（cd/m²）
- $\theta_s$：发射角（面光源法线与光线的夹角）
- $\theta_r$：入射角（接收面法线与光线的夹角）
- $r$：面光源微元到接收点的距离
- $dA_s$：面光源微元面积

#### 4.3.2 朗伯面光源的简化

对于无限大的朗伯面光源，接收面上的照度：

$$E = \pi L$$

这意味着亮度为 $L$ 的朗伯漫射面（如均匀阴天天空）产生的地面照度约为 $\pi L$。

### 4.5 传感器如何响应照度变化

#### 4.5.1 曝光量的定义

传感器的曝光量（Exposure，符号 $H$）是照度与曝光时间的乘积：

$$H = E \times t$$

其中：
- $H$：曝光量，单位：lx·s
- $E$：传感器面照度，单位：lx
- $t$：曝光时间（快门速度），单位：s

#### 4.5.2 传感器输出与照度的关系

传感器输出的DN值（Digital Number）与照度的关系：

$$DN = f(E \times t \times G)$$

对于线性响应传感器：

$$DN = k \times E \times t \times G + DN_{offset}$$

其中：
- $k$：传感器的光电转换系数
- $G$：模拟增益（ISO相关）
- $DN_{offset}$：黑电平偏移

#### 4.5.3 从DN值反推照度

```python
def estimate_illuminance(dn_value: float, 
                         exposure_time: float,
                         iso: int,
                         aperture: float,
                         calibration_factor: float = 12.5) -> float:
    """
    从传感器DN值估算场景照度
    
    参数:
        dn_value: 传感器输出值（归一化到0-1）
        exposure_time: 曝光时间，单位s
        iso: ISO感光度
        aperture: 光圈f值
        calibration_factor: 标定系数（典型值12.5）
    
    返回:
        估算的场景照度，单位lx
    """
    # 基于曝光方程反推
    # EV = log2(aperture^2 / exposure_time)
    # E = calibration_factor * 2^EV / iso
    
    ev = np.log2(aperture**2 / exposure_time)
    illuminance = calibration_factor * (2**ev) / (iso / 100)
    
    # 使用DN值修正（假设18%灰对应DN=0.18）
    correction = dn_value / 0.18
    illuminance = illuminance * correction
    
    return illuminance

# 示例：典型室内场景
dn = 0.18  # 18%灰
t = 1/60   # 快门1/60s
iso = 400
f_number = 2.8

E_estimated = estimate_illuminance(dn, t, iso, f_number)
print(f"估算场景照度: {E_estimated:.0f} lx")
```

## 5. 亮度（Luminance）

**核心观点**：亮度是人眼实际感知到的发光表面明亮程度，是HDR成像和显示技术的核心指标，PQ曲线以绝对亮度（cd/m²）为参考建立了场景到显示的统一映射。

### 5.1 物理定义

#### 5.1.1 基本定义

亮度（Luminance，符号 $L$ 或 $L_v$）定义为发光表面在某一方向上单位投影面积的发光强度：

$$L = \frac{dI_v}{dA \cos\theta} = \frac{d^2\Phi_v}{dA \cos\theta \cdot d\Omega}$$

其中：
- $L$：亮度，单位：cd/m²（也称 nit）
- $I_v$：发光强度，单位：cd
- $A$：发光面积，单位：m²
- $\theta$：观察方向与表面法线的夹角
- $\Omega$：观察方向的立体角

#### 5.1.2 单位：坎德拉每平方米（cd/m², nit）

$$1 \text{ cd/m}^2 = 1 \text{ nit}$$

"nit" 源自拉丁语 "nitere"（发光），在显示行业广泛使用。

其他常见单位换算：
| 单位名称 | 符号 | 与 cd/m² 的换算 |
|---------|------|----------------|
| 尼特 | nit | 1 nit = 1 cd/m² |
| 英尺朗伯 | fL | 1 fL = 3.426 cd/m² |
| 朗伯 | L | 1 L = 3183 cd/m² |
| 阿泊斯 | asb | 1 asb = 0.318 cd/m² |

### 5.2 亮度与照度的本质区别

| 单位 | lx (lm/m²) | cd/m² (nit) |
| 相机对应 | 传感器入射光 | 显示输出、场景表面 |

- $L$：表面亮度（cd/m²）
- $E$：入射照度（lx）
- $\rho$：表面反射率（0-1）
- $\pi$：源于朗伯余弦定律的积分

#### 5.4.2 PQ曲线与绝对亮度

PQ（Perceptual Quantizer，SMPTE ST 2084）曲线的革命性在于以**绝对亮度**作为参考：

$$L = \left( \frac{\max(V^{1/m_2} - c_1, 0)}{c_2 - c_3 V^{1/m_2}} \right)^{1/m_1} \times 10000$$

其中：
- $L$：绝对亮度，单位：cd/m²
- $V$：电信号值（0-1归一化）
- $m_1, m_2, c_1, c_2, c_3$：PQ曲线常数

**关键特性**：
- 输出0对应0.0001 cd/m²
- 输出1对应10000 cd/m²
- 基于人眼对比敏感度优化量化
- 使用12bit可覆盖全部亮度范围

```python
import numpy as np

def pq_eotf(signal: np.ndarray) -> np.ndarray:
    """
    PQ EOTF：从电信号到绝对亮度（cd/m²）
    
    参数:
        signal: 归一化电信号值（0-1）
    
    返回:
        绝对亮度，单位cd/m²
    """
    # PQ曲线常数
    m1 = 0.1593017578125
    m2 = 78.84375
    c1 = 0.8359375
    c2 = 18.8515625
    c3 = 18.6875
    
    # 峰值亮度
    L_max = 10000  # cd/m²
    
    # EOTF计算
    Vm2 = np.power(signal, 1/m2)
    numerator = np.maximum(Vm2 - c1, 0)
    denominator = c2 - c3 * Vm2
    L = np.power(numerator / denominator, 1/m1) * L_max
    
    return L

# 示例：不同信号值对应的亮度
signals = np.array([0.0, 0.25, 0.5, 0.75, 1.0])
luminances = pq_eotf(signals)
print("PQ信号值 | 亮度(cd/m²)")
print("-" * 25)
for s, l in zip(signals, luminances):
    print(f"  {s:.2f}   |  {l:.2f}")
    
# 输出:
# PQ信号值 | 亮度(cd/m²)
# -------------------------
#   0.00   |  0.00
#   0.25   |  0.50
#   0.50   |  36.86
#   0.75   |  543.77
#   1.00   |  10000.00
```

### 5.5 亮度与相机系统的关系

#### 5.5.1 传感器与场景亮度

传感器捕获的是辐射照度（W/m²），但最终需要与场景亮度建立对应关系：

$$L_{scene} = \frac{\pi \cdot E_{sensor}}{T \cdot \tau_{lens}}$$

其中：
- $L_{scene}$：场景亮度（cd/m²）
- $E_{sensor}$：传感器面照度（lx）
- $T$：镜头T值
- $\tau_{lens}$：镜头透过率

#### 5.5.2 从像素值估算场景亮度

```python
def pixel_to_scene_luminance(pixel_value: float,
                             max_luminance: float = 10000,
                             gamma: float = 2.2,
                             is_pq: bool = False) -> float:
    """
    从像素值估算场景亮度

        pixel_value: 归一化像素值（0-1）
```

## 6. 流明与勒克斯的工程应用

**核心观点**：流明效率评估光源性能，勒克斯量化场景条件——两者在相机测试、AE调校和HDR标定中有直接的工程应用价值。

### 6.1 流明效率（光效）

#### 6.1.1 定义

流明效率（Luminous Efficacy）定义为光源发出的光通量与消耗的电功率之比：

$$\eta = \frac{\Phi_v}{P}$$

其中：
- $\eta$：流明效率，单位：lm/W
- $\Phi_v$：光通量，单位：lm
- $P$：电功率，单位：W

理论最大值为683 lm/W（单色555nm光），实际光源因光谱分布和能量转换损耗而低于此值。

#### 6.1.2 不同光源的流明效率对比

| 光源类型 | 流明效率 (lm/W) | 色温 (K) | 显色指数 (CRI) | 寿命 (h) |
|---------|----------------|---------|---------------|---------|
| 白炽灯 | 10-18 | 2700 | 100 | 1,000 |
| 卤素灯 | 15-25 | 3000 | 100 | 2,000 |
| 紧凑型荧光灯 | 50-70 | 2700-6500 | 80-90 | 8,000 |
| T5荧光灯 | 80-105 | 3000-6500 | 80-90 | 20,000 |
| 金属卤化物灯 | 80-120 | 3000-6000 | 70-95 | 10,000 |
| 高压钠灯 | 100-140 | 2000 | 20-30 | 24,000 |
| LED（普通） | 80-120 | 2700-6500 | 80-90 | 25,000 |
| LED（高效） | 150-200 | 2700-6500 | 80-95 | 50,000 |
| 太阳光（参考） | ~95 | 5500 | 100 | - |

#### 6.2.2 照度测量方法

```python
import numpy as np

def measure_illuminance_via_camera(dn_mean: float,
                                   exposure_time: float,
                                   iso: int,
                                   aperture: float,
                                   sensor_k: float = 0.12) -> float:
    """
    通过相机输出估算场景照度（用于现场测量）
    
    参数:
        dn_mean: 图像平均DN值（归一化0-1）
        exposure_time: 曝光时间（s）
        iso: ISO感光度
        aperture: 光圈f值
        sensor_k: 传感器标定系数
    
    返回:
        估算照度（lx）
    """
    # 基于曝光方程
    # H = E * t (曝光量 = 照度 × 时间)
    # DN = k * H * G (DN值 = 系数 × 曝光量 × 增益)
    
    gain = iso / 100  # 相对增益
    
    # 镜头因子：考虑光圈的光通量影响
    lens_factor = aperture ** 2
    
    # 反推照度
    E = (dn_mean * lens_factor) / (sensor_k * exposure_time * gain)
    
    return E

# 示例：从相机数据估算照度
dn = 0.18  # 18%灰目标
t = 1/125  # 快门
iso = 100
f = 5.6

E_estimated = measure_illuminance_via_camera(dn, t, iso, f)
print(f"估算场景照度: {E_estimated:.0f} lx")
```

### 6.3 EV值与照度的对应关系

#### 6.3.1 EV值的定义

曝光值（Exposure Value, EV）是描述曝光量的对数标度：

$$EV = \log_2\left(\frac{N^2}{t}\right)$$

其中：
- $N$：光圈f值
- $t$：曝光时间（s）

在ISO 100条件下，EV值与照度的关系：

$$EV_{100} = \log_2\left(\frac{E \times S}{C}\right)$$

其中：
- $E$：照度（lx）
- $S$：ISO感光度
- $C$：常数（通常取250）

```python
import numpy as np

def ev_to_illuminance(ev: float, iso: int = 100) -> float:
    """
    EV值转照度
    
    参数:
        ev: 曝光值
        iso: ISO感光度
    
    返回:
        照度（lx）
    """
    C = 250  # 标定常数

def illuminance_to_ev(E: float, iso: int = 100) -> float:
    """
    照度转EV值

        iso: ISO感光度
    
    返回:
        曝光值
    """
    C = 250  # 标定常数
    ev = np.log2(E * iso / C)
    return ev
```

### 7.2 传感器光电转换与照度的关系

#### 7.2.1 辐射照度响应特性

传感器实际响应的是辐射照度（W/m²），而非光度照度（lx）。两者的关系：

$$E_e = \frac{E_v}{K_m \cdot V(\lambda_{eff})}$$

其中：
- $E_e$：辐射照度（W/m²）
- $E_v$：光度照度（lx）
- $K_m$：683 lm/W
- $V(\lambda_{eff})$：等效波长的光谱光视效率

#### 7.2.2 OECF（光电转换函数）

OECF（Opto-Electronic Conversion Function）描述了从曝光量到数字输出的转换：

$$DN = f(H) = f(E \times t)$$

典型的线性传感器OECF：

```python
import numpy as np
import matplotlib.pyplot as plt

def linear_oecf(exposure: np.ndarray, 
                gain: float = 1.0,
                black_level: float = 64,
                saturation: float = 4095) -> np.ndarray:
    """
    线性传感器OECF模型
    
    参数:
        exposure: 相对曝光量（归一化）
        gain: 模拟增益
        black_level: 黑电平DN值
        saturation: 饱和DN值
    
    返回:
        DN输出值
    """
    # 线性响应
    dn = exposure * gain * (saturation - black_level) + black_level
    # 限制在有效范围
    dn = np.clip(dn, black_level, saturation)
    return dn

# 绘制OECF曲线描述
exposures = np.linspace(0, 1.2, 100)
dn_values = linear_oecf(exposures)

# 输出关键点
print("曝光量(相对) | DN值  | 描述")
print("-" * 40)

print(f"    0.18     |  {linear_oecf(np.array([0.18]))[0]:.0f}  | 18%灰")
print(f"    0.50     |  {linear_oecf(np.array([0.5]))[0]:.0f} | 中间调")
```

#### 7.2.3 从DN到物理照度的标定

相机标定建立DN值与物理照度的对应关系：

```python
def calibrate_dn_to_illuminance(dn_values: np.ndarray,
                                illuminance_ref: np.ndarray,
                                exposure_time: float,
                                iso: int) -> dict:
    """
    相机照度标定
    
    参数:
        dn_values: 不同照度下的DN响应
        illuminance_ref: 参考照度值（lx）
        exposure_time: 曝光时间（s）
        iso: ISO设置
    
    返回:
        标定参数字典
    """
    # 归一化DN值
    dn_normalized = dn_values / 4095  # 12bit
    
    # 线性回归求标定系数
    # DN = k * E * t * G + offset
    G = iso / 100
    X = illuminance_ref * exposure_time * G
    
    # 最小二乘法
    A = np.vstack([X, np.ones(len(X))]).T
    k, offset = np.linalg.lstsq(A, dn_normalized, rcond=None)[0]
    
    return {
        'sensitivity_k': k,
        'offset': offset,
        'exposure_time': exposure_time,
        'iso': iso
    }

# 示例标定数据
lux_ref = np.array([10, 50, 100, 200, 500, 1000, 2000])
dn_measured = np.array([150, 450, 800, 1400, 2600, 3800, 4095])

cal_params = calibrate_dn_to_illuminance(dn_measured, lux_ref, 1/60, 100)
print(f"标定系数 k: {cal_params['sensitivity_k']:.6f}")
print(f"偏移量: {cal_params['offset']:.4f}")
```

#### 7.3.2 18%灰的物理含义

AE算法的目标通常是使场景的平均亮度对应"中间灰"。18%灰（更准确是18.4%）的反射率选择基于：

1. **几何平均值**：在对数亮度空间中，18%灰位于黑（0%）和白（100%）的几何中心
2. **实测统计**：自然场景的平均反射率约为12-18%
3. **历史标准**：柯达在1941年确立的曝光标准

$$\sqrt{0 \times 100} \approx 0 \text{ (不适用)}$$
$$\sqrt{1\% \times 100\%} = 10\% \text{ (几何均值)}$$
$$\text{实测校正} \rightarrow 18\%$$

#### 7.3.3 AE曝光计算

```python
def calculate_ae_exposure(target_ev: float,
                          scene_illuminance: float,
                          iso: int = 100) -> dict:
    """
    计算AE曝光参数
    
    参数:
        target_ev: 目标曝光值
        scene_illuminance: 场景照度（lx）
        iso: ISO设置
    
    返回:
        曝光参数字典
    """
    # 从照度计算场景EV
    C = 250  # 标定常数
    scene_ev = np.log2(scene_illuminance * iso / C)
    
    # 计算需要的曝光补偿
    ev_compensation = target_ev - scene_ev
    
    # 典型参数组合（优先调整快门）
    aperture_options = [1.4, 2.0, 2.8, 4.0, 5.6, 8.0, 11, 16]
    
    best_params = None
    min_error = float('inf')
    
    for aperture in aperture_options:
        # EV = log2(N²/t)
        required_ev = scene_ev + ev_compensation
        shutter_time = (aperture ** 2) / (2 ** required_ev)
        
        # 检查快门范围（1/8000s - 30s）
        if 1/8000 <= shutter_time <= 30:
            error = abs(np.log2(shutter_time) - np.log2(1/60))  # 偏好1/60s
            if error < min_error:
                min_error = error
                best_params = {
                    'aperture': aperture,
                    'shutter': shutter_time,
                    'iso': iso,
                    'ev': required_ev
                }
    
    return best_params

# 示例：室内场景
indoor_lux = 300
params = calculate_ae_exposure(target_ev=8, scene_illuminance=indoor_lux, iso=400)
if params:
    print(f"推荐曝光参数:")
    print(f"  光圈: f/{params['aperture']}")
    print(f"  快门: 1/{1/params['shutter']:.0f}s")
    print(f"  ISO: {params['iso']}")
    print(f"  EV: {params['ev']:.1f}")
```

### 7.4 HDR成像中光度学概念的应用

#### 7.4.1 场景动态范围的光度学定义

场景动态范围（Scene Dynamic Range）定义为场景中最亮与最暗部分的亮度比：

$$DR = \frac{L_{max}}{L_{min}} = \frac{E_{max}}{E_{min}}$$

转换为档数（stops）：

$$DR_{stops} = \log_2\left(\frac{L_{max}}{L_{min}}\right)$$

#### 7.4.2 多曝光融合与辐照度线性化

HDR多曝光融合的关键步骤是将不同曝光的图像转换到统一的辐照度空间：

```python
def linearize_exposure(dn_image: np.ndarray,
                       exposure_time: float,
                       response_curve: np.ndarray = None) -> np.ndarray:
    """
    曝光线性化：将DN值转换为相对辐照度
    
    参数:
        dn_image: DN值图像（0-4095）
        exposure_time: 曝光时间（s）
        response_curve: 相机响应曲线（可选）
    
    返回:
        相对辐照度图像
    """
    # 默认线性响应
    if response_curve is None:
        # 减去黑电平，归一化
        linear = (dn_image - 64) / (4095 - 64)
    else:
        # 应用逆响应曲线
        linear = response_curve[dn_image.astype(int)]
    
    # 除以曝光时间得到相对辐照度
    irradiance = linear / exposure_time
    
    return irradiance

def merge_hdr_exposures(images: list,
                        exposure_times: list,
                        weights: list = None) -> np.ndarray:
    """
    HDR多曝光融合
    
    参数:
        images: DN值图像列表
        exposure_times: 对应曝光时间列表
        weights: 融合权重（可选）
    
    返回:
        HDR辐照度图像
    """
    # 线性化所有曝光
    irradiances = []
    for img, t in zip(images, exposure_times):
        irr = linearize_exposure(img, t)
        irradiances.append(irr)
    
    # 权重融合（简化版：基于曝光度的权重）
    if weights is None:
        weights = []
        for img in images:
            # 中间曝光区域权重高
            w = np.exp(-((img/4095 - 0.5)**2) / 0.08)
            weights.append(w)
    
    # 加权平均
    numerator = sum(w * irr for w, irr in zip(weights, irradiances))
    denominator = sum(weights) + 1e-10
    hdr_irradiance = numerator / denominator
    
    return hdr_irradiance
```

### 7.5 与AWB/AF的关联

#### 7.5.1 AWB与照度/色温的关系

自动白平衡（AWB）需要识别场景光源的色温，而色温与光源的光谱分布密切相关：

| 光源类型 | 色温 (K) | 典型照度 (lx) | AWB策略 |
|---------|---------|--------------|---------|
| 烛光 | 1850 | 10-50 | 暖色补偿 |
| 白炽灯 | 2700 | 50-300 | 暖色补偿 |
| 卤素灯 | 3000 | 100-500 | 中性偏暖 |
| 荧光灯 | 4000 | 300-1000 | 绿色补偿 |
| 日光（中午） | 5500 | 10000-100000 | 中性 |
| 阴天 | 6500 | 1000-10000 | 蓝色补偿 |
| 阴影 | 8000 | 1000-5000 | 强蓝色补偿 |

#### 7.5.2 AF与照度的关系

自动对焦在低照度下面临的挑战：

- **对比度AF**：照度低→信噪比低→对比度检测困难
- **相位检测AF**：光线不足→相位差信号弱
- **激光/红外AF**：主动发射，不受环境照度影响

## 附录A：Python工具函数汇总

```python
"""
色度学计算工具函数集
用于Camera Pipeline中的光度学计算
"""

import numpy as np
from typing import Tuple, Dict, Optional

# ============ 基础转换函数 ============

def ev_to_illuminance(ev: float, iso: int = 100) -> float:
    """EV值转照度 (lx)"""
    C = 250
    return C * (2 ** ev) / iso

def illuminance_to_ev(E: float, iso: int = 100) -> float:
    """照度(lx)转EV值"""
    C = 250
    return np.log2(E * iso / C)

def luminance_to_illuminance(L: float, reflectance: float = 0.18) -> float:
    """朗伯体亮度(cd/m²)转照度(lx)"""
    return L * np.pi / reflectance

def illuminance_to_luminance(E: float, reflectance: float = 0.18) -> float:
    """照度(lx)转朗伯体亮度(cd/m²)"""
    return E * reflectance / np.pi

# ============ PQ曲线函数 ============

def pq_oetf(L: np.ndarray) -> np.ndarray:
    """PQ OETF: 亮度(cd/m²) → 电信号(0-1)"""
    m1, m2 = 0.1593017578125, 78.84375
    c1, c2, c3 = 0.8359375, 18.8515625, 18.6875
    Y = L / 10000
    Ym1 = np.power(Y, m1)
    V = np.power((c1 + c2 * Ym1) / (1 + c3 * Ym1), m2)
    return V

def pq_eotf(V: np.ndarray) -> np.ndarray:
    """PQ EOTF: 电信号(0-1) → 亮度(cd/m²)"""
    m1, m2 = 0.1593017578125, 78.84375
    c1, c2, c3 = 0.8359375, 18.8515625, 18.6875
    Vm2 = np.power(V, 1/m2)
    L = np.power(np.maximum(Vm2 - c1, 0) / (c2 - c3 * Vm2), 1/m1) * 10000
    return L

# ============ 曝光计算函数 ============

def calculate_exposure(aperture: float, shutter: float, iso: int) -> float:
    """计算曝光值EV"""
    return np.log2(aperture**2 / shutter) - np.log2(iso / 100)

def exposure_triangle(target_ev: float, 
                      fixed_param: str,
                      fixed_value: float,
                      iso: int = 100) -> Dict:
    """
    曝光三角计算
    fixed_param: 'aperture' 或 'shutter'
    """
    if fixed_param == 'aperture':
        aperture = fixed_value
        shutter = aperture**2 / (2**(target_ev + np.log2(iso/100)))
        return {'aperture': aperture, 'shutter': shutter, 'iso': iso}
    else:
        shutter = fixed_value
        aperture = np.sqrt(shutter * 2**(target_ev + np.log2(iso/100)))
        return {'aperture': aperture, 'shutter': shutter, 'iso': iso}

# ============ 传感器相关函数 ============

def dn_to_exposure(dn: float, 
                   black_level: float = 64,
                   saturation: float = 4095,
                   gain: float = 1.0) -> float:
    """DN值转相对曝光量"""
    return (dn - black_level) / ((saturation - black_level) * gain)

def exposure_to_dn(exposure: float,
                   black_level: float = 64,
                   saturation: float = 4095,
                   gain: float = 1.0) -> float:
    """相对曝光量转DN值"""
    dn = exposure * gain * (saturation - black_level) + black_level
    return np.clip(dn, black_level, saturation)

# ============ 动态范围计算 ============

def dynamic_range_db(max_signal: float, noise_floor: float) -> float:
    """计算动态范围(dB)"""
    return 20 * np.log10(max_signal / noise_floor)

def dynamic_range_stops(max_signal: float, noise_floor: float) -> float:
    """计算动态范围(档)"""
    return np.log2(max_signal / noise_floor)
```
