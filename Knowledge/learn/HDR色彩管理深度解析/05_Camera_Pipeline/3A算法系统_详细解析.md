## 1. 3A算法系统总论

**核心观点**：3A是Camera Pipeline的"大脑"，通过闭环反馈控制实现自动化成像，使相机能够在复杂多变的环境中持续输出高质量图像。

### 1.2 3A的共性特征

尽管AE、AWB、AF解决不同问题，但它们共享四大核心特征：

| 特征 | 描述 | AE体现 | AWB体现 | AF体现 |
|------|------|--------|--------|--------|
| **闭环控制** | 输出反馈到输入形成闭环 | 亮度→曝光调整 | 色温→增益调整 | 锐度→对焦调整 |
| **统计驱动** | 基于图像统计做决策 | 亮度直方图 | R/G/B均值 | 锐度评价函数 |
| **实时约束** | 必须在帧间隔内完成 | 1-3帧收敛 | 1-5帧收敛 | 10-30帧收敛 |
| **场景自适应** | 针对不同场景调整策略 | 逆光检测 | 混合光源 | 运动目标 |

#### 控制论视角

从控制论角度，3A可以建模为：

$$
u(k) = u(k-1) + K_p \cdot e(k) + K_i \cdot \sum_{i=0}^{k} e(i) + K_d \cdot [e(k) - e(k-1)]
$$

其中：
- $u(k)$：第k帧的控制量（曝光/增益/焦点位置）
- $e(k)$：第k帧的误差（目标值 - 当前值）
- $K_p, K_i, K_d$：比例、积分、微分系数

## 2. 自动曝光(AE)

**核心观点**：AE的目标是在任意光照条件下找到最优曝光参数组合，使图像亮度和动态范围满足后续处理和人眼感知需求。

### 2.1 测光原理与模式

测光是AE的第一步——将图像分区并计算各区域亮度。

#### 2.1.1 矩阵测光(Multi-zone Metering)

将画面划分为多个区域，对每个区域赋予不同权重：

#### 2.1.2 中央重点测光

以高斯分布为中央区域赋予更高权重：

$$
W(x, y) = \exp\left(-\frac{(x-x_c)^2 + (y-y_c)^2}{2\sigma^2}\right)
$$

其中：
- $(x_c, y_c)$：图像中心坐标
- $\sigma$：权重衰减半径，通常为图像宽度的20-30%

#### 2.1.3 点测光(Spot Metering)

仅对画面1-5%的小区域测光，适用于高对比度场景：

#### 曝光值(EV)计算

曝光值EV是对数尺度的亮度量化：

$$
EV = \log_2\frac{N^2}{t}
$$

其中：
- $N$：光圈F值
- $t$：快门时间（秒）

**EV与场景亮度对应关系**：

| EV值 | 典型场景 | 照度(lux) |
|------|----------|-----------|
| -6 | 星空 | 0.0001 |
| 0 | 烛光室内 | 10 |
| 5 | 室内灯光 | 320 |
| 10 | 阴天户外 | 10,000 |
| 15 | 晴天直射 | 100,000 |

#### 等效曝光与曝光补偿

**等效曝光原则**：改变一个参数1档，需用另一参数补偿1档以保持亮度不变。

| 调整 | 效果 | 副作用 |
|------|------|--------|
| 快门×2 | 亮度+1EV | 运动模糊风险 |
| 光圈开大1档 | 亮度+1EV | 景深变浅 |
| ISO×2 | 亮度+1EV | 噪声增加 |

### 2.3 AE收敛算法

#### 2.3.1 简单比例控制(P控制)

最基本的AE算法——根据当前亮度与目标亮度的差值线性调整：

$$
\Delta_{exp} = K_p \times (L_{target} - L_{current})
$$

```python
def ae_p_control(current_luminance: float, 
                  target_luminance: float = 118,
                  kp: float = 0.5) -> float:
    """
    比例控制AE算法
    
    Args:
        current_luminance: 当前帧平均亮度 (0-255)
        target_luminance: 目标亮度，通常为中灰值18%灰对应的118
        kp: 比例增益，控制收敛速度
        
    Returns:
        曝光调整量 (EV档)
    """
    # 计算亮度误差
    error = target_luminance - current_luminance
    
    # 转换为EV调整量 (每差18单位约1EV)
    ev_adjust = kp * (error / 18.0)
    
    # 限制单次调整幅度，防止震荡
    ev_adjust = max(-2.0, min(2.0, ev_adjust))
    
    return ev_adjust
```

#### 2.3.2 PID控制器在AE中的应用

完整的PID控制提供更稳定的收敛：

```python
class AE_PID_Controller:
    """
    PID控制器实现的AE算法
    
    特点:
    - P(比例): 快速响应亮度变化
    - I(积分): 消除稳态误差
    - D(微分): 抑制超调和震荡
    """
    
    def __init__(self, kp: float = 0.4, ki: float = 0.1, kd: float = 0.2):
        self.kp = kp  # 比例系数
        self.ki = ki  # 积分系数
        self.kd = kd  # 微分系数
        
        self.integral = 0.0      # 积分累积
        self.prev_error = 0.0    # 上一帧误差
        self.target = 118.0      # 目标亮度 (18%灰)
        
    def compute(self, current_luminance: float) -> float:
        """
        计算曝光调整量
        
        Args:
            current_luminance: 当前帧平均亮度
            
        Returns:
            EV调整量
        """
        # 计算误差
        error = self.target - current_luminance
        
        # 比例项
        p_term = self.kp * error
        
        # 积分项 (带抗饱和)
        self.integral += error
        self.integral = max(-100, min(100, self.integral))  # 限幅
        i_term = self.ki * self.integral
        
        # 微分项
        d_term = self.kd * (error - self.prev_error)
        self.prev_error = error
        
        # 综合输出
        ev_adjust = (p_term + i_term + d_term) / 18.0
        
        return max(-2.0, min(2.0, ev_adjust))
    
    def reset(self):
        """场景切换时重置控制器状态"""
        self.integral = 0.0
        self.prev_error = 0.0
```

#### 2.3.3 亮度直方图分析法

直方图提供比平均亮度更丰富的信息：

```python
import numpy as np

def ae_histogram_analysis(histogram: np.ndarray, 
                           target_percentile: float = 0.5) -> dict:
    """
    基于直方图的AE分析
    
    Args:
        histogram: 256-bin亮度直方图
        target_percentile: 目标亮度百分位
        
    Returns:
        AE分析结果字典
    """
    total_pixels = histogram.sum()
    
    # 计算累积直方图
    cumsum = np.cumsum(histogram) / total_pixels
    
    # 中位数亮度
    median_idx = np.searchsorted(cumsum, target_percentile)
    
    # 高光溢出比例 (>250)
    highlight_ratio = histogram[250:].sum() / total_pixels
    
    # 暗部欠曝比例 (<10)
    shadow_ratio = histogram[:10].sum() / total_pixels
    
    # 动态范围利用率
    non_zero_range = np.where(histogram > 0)[0]
    if len(non_zero_range) > 0:
        dynamic_range = (non_zero_range[-1] - non_zero_range[0]) / 255.0
    else:
        dynamic_range = 0.0
    
    # 曝光建议
    ev_adjust = 0.0
    
    if highlight_ratio > 0.02:  # 超过2%高光溢出
        ev_adjust -= 0.5  # 减曝光
    elif shadow_ratio > 0.1:    # 超过10%欠曝
        ev_adjust += 0.5  # 加曝光
    else:
        # 根据中位数调整
        ev_adjust = (128 - median_idx) / 50.0
    
    return {
        'median_luminance': median_idx,
        'highlight_ratio': highlight_ratio,
        'shadow_ratio': shadow_ratio,
        'dynamic_range': dynamic_range,
        'ev_adjust': max(-2.0, min(2.0, ev_adjust))
    }
```

> 📌 **关键洞察**：直方图分析可以识别高动态范围场景——当高光溢出和暗部欠曝同时存在时，表明场景动态范围超出传感器能力，需要启用HDR模式。

```python
def anti_flicker_shutter(target_time_ms: float, 
                          power_frequency: int = 50) -> float:
    """
    防闪烁快门速度量化
    
    Args:
        target_time_ms: 目标曝光时间(毫秒)
        power_frequency: 电源频率 (50Hz/60Hz)
        
    Returns:
        量化后的曝光时间(毫秒)
    """
    # 亮度波动周期 = 1000 / (2 * 电源频率) 毫秒
    flicker_period = 1000.0 / (2 * power_frequency)
    
    # 50Hz: 10ms, 60Hz: 8.33ms
    
    if target_time_ms < flicker_period:
        # 曝光时间短于一个周期，无法完全消除闪烁
        # 尽量选择接近的整数倍
        return flicker_period
    else:
        # 量化到最近的整数倍周期
        n_periods = round(target_time_ms / flicker_period)
        return n_periods * flicker_period

# 示例: 50Hz环境下的合法快门速度
# 1/100s (10ms), 1/50s (20ms), 1/33s (30ms), 1/25s (40ms)...
```

```python
def backlight_ae_compensation(face_region_luminance: float,
                               background_luminance: float,
                               target_face_luminance: float = 140) -> float:
    """
    逆光场景曝光补偿

        曝光补偿量(EV)

    # 计算背光程度
    backlight_ratio = background_luminance / max(face_region_luminance, 1)
    
    # 判断是否为逆光场景
    is_backlight = backlight_ratio > 2.0 and face_region_luminance < 100
    
    if is_backlight:
        # 基于面部亮度计算补偿
        face_ev_adjust = np.log2(target_face_luminance / max(face_region_luminance, 1))

        max_compensation = 2.0  # 最多+2EV
        return min(face_ev_adjust, max_compensation)
    
    return 0.0
```

## 3. 自动白平衡(AWB)

**核心观点**：AWB的本质是估计场景光源色温，通过调整R/B增益消除色偏，还原物体在标准光源下的真实颜色。

### 3.1 色温与光源模型

#### 黑体辐射与色温定义

色温(Color Temperature)源自黑体辐射理论——理想黑体在不同温度下发出不同颜色的光：

$$
M(\lambda, T) = \frac{2\pi hc^2}{\lambda^5} \cdot \frac{1}{e^{hc/\lambda k_B T} - 1}
$$

其中：
- $M(\lambda, T)$：光谱辐射出射度
- $\lambda$：波长
- $T$：绝对温度(开尔文)
- $h$：普朗克常数，$c$：光速，$k_B$：玻尔兹曼常数

```

├─────────────────────────────────────────────────────────────────────────────────────┤
│                                                                                      │
│   CIE xy色度图中的黑体轨迹:                                                          │
│                                                                                      │
│     y ↑                                                                              │
```

### 3.2 AWB算法分类与原理

#### 3.2.1 灰世界假设(Gray World)

**假设**：场景中所有颜色的平均值应该是中性灰。

$$
\bar{R} = \bar{G} = \bar{B} = Gray
$$

```python
import numpy as np

def gray_world_awb(image: np.ndarray) -> tuple:
    """
    灰世界假设AWB算法
    
    Args:
        image: RGB图像 (H, W, 3)
        
    Returns:
        (r_gain, g_gain, b_gain): 白平衡增益
    """
    # 计算各通道均值
    r_mean = np.mean(image[:, :, 0])
    g_mean = np.mean(image[:, :, 1])
    b_mean = np.mean(image[:, :, 2])
    
    # 计算总体均值作为灰度目标
    gray_mean = (r_mean + g_mean + b_mean) / 3.0
    
    # 计算增益 (以G通道为基准更稳定)
    r_gain = g_mean / max(r_mean, 1e-6)
    g_gain = 1.0
    b_gain = g_mean / max(b_mean, 1e-6)
    
    return r_gain, g_gain, b_gain

# 优点: 简单快速，适合色彩丰富的场景
# 缺点: 大面积单色场景会失效 (如蓝天、绿草地)
```

#### 3.2.2 白点检测(White Patch / Max-RGB)

**假设**：图像中最亮的区域是白色（高光反射）。

```python
def white_patch_awb(image: np.ndarray, percentile: float = 99) -> tuple:
    """
    白点检测AWB算法
    
    Args:
        image: RGB图像
        percentile: 亮度百分位阈值
        
    Returns:
        (r_gain, g_gain, b_gain): 白平衡增益
    """
    # 计算亮度
    luminance = 0.299 * image[:,:,0] + 0.587 * image[:,:,1] + 0.114 * image[:,:,2]
    
    # 找到高亮区域
    threshold = np.percentile(luminance, percentile)
    bright_mask = luminance >= threshold
    
    # 在高亮区域计算各通道均值
    r_white = np.mean(image[:,:,0][bright_mask])
    g_white = np.mean(image[:,:,1][bright_mask])
    b_white = np.mean(image[:,:,2][bright_mask])
    
    # 归一化增益
    max_val = max(r_white, g_white, b_white)
    r_gain = max_val / max(r_white, 1e-6)
    g_gain = max_val / max(g_white, 1e-6)
    b_gain = max_val / max(b_white, 1e-6)
    
    return r_gain, g_gain, b_gain

# 优点: 适合存在高光点的场景
# 缺点: 高亮区域非白色时失效，如彩色LED灯
```

#### 3.2.3 色温估计法

基于R/B比值推断色温：

```python
def rb_ratio_to_cct(r_mean: float, b_mean: float) -> float:
    """
    基于R/B比值估计色温
    
    使用经验公式，适用于大多数自然光源
    
    Args:
        r_mean: R通道均值
        b_mean: B通道均值
        
    Returns:
        估计的色温(K)
    """
    # R/B比值
    rb_ratio = r_mean / max(b_mean, 1e-6)
    
    # 经验公式 (需根据传感器特性标定)
    # 高色温(蓝光多): rb_ratio小
    # 低色温(红光多): rb_ratio大
    
    if rb_ratio > 1.5:
        # 低色温 (2000K - 4000K)
        cct = 2000 + (1.5 / rb_ratio) * 2000
    elif rb_ratio < 0.8:
        # 高色温 (7000K - 12000K)
        cct = 7000 + (0.8 - rb_ratio) * 10000
    else:
        # 中间色温 (4000K - 7000K)
        cct = 4000 + (1.5 - rb_ratio) * 4286
    
    return max(2000, min(12000, cct))
```

#### 3.2.4 基于统计的AWB

在色度空间中进行统计分析，提取光源信息：

```python
def statistical_awb(image: np.ndarray) -> tuple:
    """
    基于色度统计的AWB算法
    
    在R/G-B/G色度空间中寻找主光源
    """
    # 避免除零
    g_channel = np.maximum(image[:,:,1], 1)
    
    # 计算色度坐标
    rg = image[:,:,0] / g_channel  # R/G
    bg = image[:,:,2] / g_channel  # B/G
    
    # 有效像素过滤 (排除过暗和过亮)
    luminance = image.mean(axis=2)
    valid_mask = (luminance > 20) & (luminance < 240)
    
    rg_valid = rg[valid_mask]
    bg_valid = bg[valid_mask]
    
    # 在Planckian轨迹附近搜索
    # 使用2D直方图找到主要聚类
    hist, xedges, yedges = np.histogram2d(
        rg_valid.flatten(), 
        bg_valid.flatten(),
        bins=64, 
        range=[[0.5, 2.0], [0.5, 2.0]]
    )
    
    # 找到峰值位置
    peak_idx = np.unravel_index(hist.argmax(), hist.shape)
    rg_peak = (xedges[peak_idx[0]] + xedges[peak_idx[0]+1]) / 2
    bg_peak = (yedges[peak_idx[1]] + yedges[peak_idx[1]+1]) / 2
    
    # 计算增益
    r_gain = 1.0 / rg_peak
    g_gain = 1.0
    b_gain = 1.0 / bg_peak
    
    # 归一化
    min_gain = min(r_gain, g_gain, b_gain)
    r_gain /= min_gain
    g_gain /= min_gain
    b_gain /= min_gain
    
    return r_gain, g_gain, b_gain
```

### 3.3 AWB与CCM协同

白平衡和色彩校正矩阵需要联合优化。

#### 多光源CCM插值

不同色温下需要不同的CCM：

$$
CCM(T) = \alpha \cdot CCM_{D65} + (1-\alpha) \cdot CCM_A
$$

其中 $\alpha = \frac{T - T_A}{T_{D65} - T_A}$，T为估计色温。

```python
def interpolate_ccm(cct: float, 
                     ccm_a: np.ndarray, 
                     ccm_d65: np.ndarray) -> np.ndarray:
    """
    基于色温插值CCM矩阵
    
    Args:
        cct: 估计的相关色温
        ccm_a: A光源(2856K)标定的CCM
        ccm_d65: D65光源(6504K)标定的CCM
        
    Returns:
        插值后的CCM矩阵 (3×3)
    """
    # 色温归一化到[0,1]
    alpha = (cct - 2856) / (6504 - 2856)
    alpha = np.clip(alpha, 0, 1)
    
    # 线性插值
    ccm = (1 - alpha) * ccm_a + alpha * ccm_d65
    
    return ccm
```

### 3.4 色彩恒常性理论

色彩恒常性(Color Constancy)是AWB的理论基础——人眼能够在不同光照下识别物体的"真实"颜色。

#### Von Kries色适应模型

经典的对角线适应模型：

$$
\begin{bmatrix} L' \\ M' \\ S' \end{bmatrix} = 
\begin{bmatrix} k_L & 0 & 0 \\ 0 & k_M & 0 \\ 0 & 0 & k_S \end{bmatrix}
\begin{bmatrix} L \\ M \\ S \end{bmatrix}
$$

其中L、M、S为锥细胞响应，$k_L, k_M, k_S$为适应系数。

#### Bradford变换

更精确的色适应变换，考虑通道间耦合：

$$
\begin{bmatrix} R' \\ G' \\ B' \end{bmatrix} = M_{adapt} \begin{bmatrix} R \\ G \\ B \end{bmatrix}
$$

其中Bradford变换矩阵为：

$$
M_{Bradford} = \begin{bmatrix} 
0.8951 & 0.2664 & -0.1614 \\
-0.7502 & 1.7135 & 0.0367 \\
0.0389 & -0.0685 & 1.0296
\end{bmatrix}
$$

### 3.5 AWB典型失败案例

| 失败场景 | 原因 | 表现 | 解决方案 |
|----------|------|------|----------|
| **单色场景** | 灰世界假设失效 | 整体偏向补色 | 检测单色场景，降低校正强度 |
| **混合光源** | 多个色温共存 | 色彩分裂 | 分区域AWB、HDR模式 |
| **高饱和度** | 强色彩主导统计 | 偏色 | 排除高饱和像素 |
| **低光场景** | 噪声干扰统计 | 色彩漂移 | 增加时域滤波 |
| **人造特殊光源** | 非连续光谱 | 严重偏色 | 光源数据库匹配 |

## 4. 自动对焦(AF)

**核心观点**：AF的核心挑战是在最短时间内找到最佳焦点位置，同时保持对运动目标的持续追踪。现代AF系统融合多种技术实现速度与精度的平衡。

### 4.1 对焦原理与景深

#### 薄透镜公式

光学对焦遵循薄透镜方程：

$$
\frac{1}{f} = \frac{1}{d_o} + \frac{1}{d_i}
$$

其中：
- $f$：焦距
- $d_o$：物距（被摄物体到镜头的距离）
- $d_i$：像距（镜头到传感器的距离）

```
┌─────────────────────────────────────────────────────────────────────────────────────┐
│                          薄透镜成像原理                                               │
├─────────────────────────────────────────────────────────────────────────────────────┤

│              物体                 镜头                  传感器                        │
│                │                   │                      │                          │

│      ●─────────┼───────────────────┼──────────────────────┼─────● 清晰像点           │
│     物点       │                   │                      │                          │
│                │                   │                      │                          │
│      ○─────────┼───────────────────┼──────────────────────┼─────○ 模糊圆(失焦)       │
│   较近物体     │                   │                      │      (CoC)               │

```

#### 弥散圆(CoC)与景深

当物体不在焦平面时，形成弥散圆而非点：

$$
CoC = \frac{|d_i - d_i'|}{d_i'} \times D
$$

其中 $D$ 为光圈直径。

**景深计算**：

$$
DOF = \frac{2 \times N \times c \times d_o^2}{f^2}
$$

其中：
- $DOF$：景深
- $N$：光圈F值
- $c$：可接受弥散圆直径
- $d_o$：对焦距离
- $f$：焦距

### 4.2 对比度检测AF(CDAF)

CDAF通过分析图像锐度来寻找最佳焦点。

#### 锐度评价函数

```python
import numpy as np
from scipy import ndimage

def brenner_focus_measure(image: np.ndarray) -> float:
    """
    Brenner锐度评价函数
    
    计算相邻像素差的平方和
    
    Args:
        image: 灰度图像
        
    Returns:
        锐度值 (越大越清晰)
    """
    # 水平差分
    diff_h = image[:, 2:] - image[:, :-2]
    return np.sum(diff_h ** 2)


def laplacian_focus_measure(image: np.ndarray) -> float:
    """
    Laplacian锐度评价函数
    
    基于二阶导数，对边缘敏感
    """
    laplacian_kernel = np.array([[0, 1, 0],
                                  [1, -4, 1],
                                  [0, 1, 0]])
    laplacian = ndimage.convolve(image.astype(float), laplacian_kernel)
    return np.var(laplacian)


def tenengrad_focus_measure(image: np.ndarray) -> float:
    """
    Tenengrad锐度评价函数
    
    基于Sobel梯度，常用于AF
    """
    # Sobel算子
    sobel_x = ndimage.sobel(image.astype(float), axis=1)
    sobel_y = ndimage.sobel(image.astype(float), axis=0)
    
    # 梯度幅度
    gradient_magnitude = np.sqrt(sobel_x**2 + sobel_y**2)
    
    # 阈值过滤噪声
    threshold = np.mean(gradient_magnitude)
    return np.sum(gradient_magnitude[gradient_magnitude > threshold] ** 2)
```

#### 搜索算法

```python
class CDAF_HillClimbing:
    """
    对比度AF爬山搜索算法
    
    包含粗搜和精搜两个阶段
    """
    
    def __init__(self, lens_range: tuple = (0, 1023), 
                 coarse_step: int = 64, 
                 fine_step: int = 8):
        self.min_pos, self.max_pos = lens_range
        self.coarse_step = coarse_step
        self.fine_step = fine_step
        
    def search(self, focus_measure_func) -> int:
        """
        执行AF搜索
        
        Args:
            focus_measure_func: 输入镜头位置，返回锐度值的函数
            
        Returns:
            最佳镜头位置
        """
        # 阶段1: 粗搜索
        positions = list(range(self.min_pos, self.max_pos + 1, self.coarse_step))
        measures = [focus_measure_func(pos) for pos in positions]
        
        # 找到粗搜索峰值
        peak_idx = np.argmax(measures)
        coarse_peak = positions[peak_idx]
        
        # 阶段2: 精搜索 (在峰值附近)
        fine_start = max(self.min_pos, coarse_peak - self.coarse_step)
        fine_end = min(self.max_pos, coarse_peak + self.coarse_step)
        
        fine_positions = list(range(fine_start, fine_end + 1, self.fine_step))
        fine_measures = [focus_measure_func(pos) for pos in fine_positions]
        
        # 返回精搜索峰值
        best_idx = np.argmax(fine_measures)
        return fine_positions[best_idx]
```

```python
def pdaf_defocus_calculation(left_signal: np.ndarray, 
                              right_signal: np.ndarray,
                              calibration_factor: float = 0.1) -> tuple:
    """
    PDAF离焦量计算
    
    Args:
        left_signal: 左半像素信号
        right_signal: 右半像素信号
        calibration_factor: 标定系数 (像素偏移到镜头移动的转换)
        
    Returns:
        (phase_diff, lens_move): 相位差和建议镜头移动量
    """
    # 相关性计算求相位差
    correlation = np.correlate(left_signal, right_signal, mode='full')
    peak_idx = np.argmax(correlation)
    center_idx = len(left_signal) - 1
    
    # 相位差 (像素单位)
    phase_diff = peak_idx - center_idx
    
    # 亚像素精度优化 (抛物线拟合)
    if 0 < peak_idx < len(correlation) - 1:
        y0, y1, y2 = correlation[peak_idx-1:peak_idx+2]
        subpixel_offset = 0.5 * (y0 - y2) / (y0 - 2*y1 + y2)
        phase_diff += subpixel_offset
    
    # 转换为镜头移动量
    lens_move = phase_diff * calibration_factor
    
    return phase_diff, lens_move
```

### 4.5 连续AF(CAF)与追焦

连续AF需要预测目标运动，提前调整焦点。

```python
class ContinuousAF:
    """
    连续自动对焦算法
    
    使用卡尔曼滤波预测目标距离
    """
    
    def __init__(self):
        # 状态: [距离, 速度]
        self.state = np.array([1.0, 0.0])  # 初始距离1m，静止
        
        # 状态转移矩阵 (匀速模型)
        self.F = np.array([[1, 0.033],   # dt = 33ms (30fps)
                           [0, 1]])
        
        # 观测矩阵
        self.H = np.array([[1, 0]])
        
        # 过程噪声协方差
        self.Q = np.array([[0.01, 0],
                           [0, 0.1]])
        
        # 观测噪声协方差
        self.R = np.array([[0.05]])
        
        # 误差协方差
        self.P = np.eye(2)
        
    def predict(self) -> float:
        """
        预测下一帧的目标距离
        
        Returns:
            预测距离
        """
        # 状态预测
        self.state = self.F @ self.state
        self.P = self.F @ self.P @ self.F.T + self.Q
        
        return self.state[0]
    
    def update(self, measured_distance: float) -> float:
        """
        根据测量值更新预测
        
        Args:
            measured_distance: 当前帧测量的目标距离
            
        Returns:
            滤波后的距离估计
        """
        # 卡尔曼增益
        S = self.H @ self.P @ self.H.T + self.R
        K = self.P @ self.H.T @ np.linalg.inv(S)
        
        # 状态更新
        z = np.array([measured_distance])
        y = z - self.H @ self.state
        self.state = self.state + K @ y
        
        # 协方差更新
        self.P = (np.eye(2) - K @ self.H) @ self.P
        
        return self.state[0]
```

```
│   │  默认: AE > AF > AWB (曝光正确是一切的基础)                           │        │
│   └───────────────────────────────────────────────────────────────────────┘        │
│                                                                                      │
└─────────────────────────────────────────────────────────────────────────────────────┘
```

### 5.2 帧级流水线时序

3A算法在帧级Pipeline中的时序关系：

```
┌─────────────────────────────────────────────────────────────────────────────────────┐
│                          3A帧级时序 (30fps示例)                                       │
├─────────────────────────────────────────────────────────────────────────────────────┤
│                                                                                      │
│   时间 ──────────────────────────────────────────────────────────────────────▶       │
│                                                                                      │
│   帧N:     │◄─────── 33.3ms ────────►│                                             │
│            ┌───────────────────────────┐                                            │
│   传感器   │       曝光积分           │                                            │
│            └───────────────────────────┘                                            │
│                           │                                                         │
│                           ▼                                                         │
│            ┌───────────────────────────┐                                            │
│   统计HW   │  BLC → 统计窗口计算      │                                            │
│            └───────────────────────────┘                                            │
│                                  │                                                  │
│                                  ▼                                                  │
│                         ┌───────────────┐                                          │
│   3A算法(CPU/DSP)       │ AE/AWB/AF计算 │                                          │
│                         └───────┬───────┘                                          │
│                                 │                                                   │
│                                 ▼ 参数下发                                          │
│   帧N+1:   │◄─────── 33.3ms ────────►│                                             │
│            ┌───────────────────────────┐                                            │
│   传感器   │  应用新曝光参数，积分    │ ← 帧N的3A结果在帧N+1生效                   │
│            └───────────────────────────┘                                            │
│                                                                                      │
│   时延分析:                                                                          │
│   - 统计延迟: ~0.5帧 (曝光结束后开始统计)                                           │
│   - 算法延迟: ~0.3帧 (3A计算时间)                                                   │
│   - 生效延迟: ~1帧 (下一帧才应用新参数)                                             │
│   - 总延迟: ~2帧 (66ms @ 30fps)                                                     │
│                                                                                      │
│   对收敛速度的影响:                                                                   │
│   - 2帧延迟意味着控制器的响应带宽受限                                                │
│   - 需要保守的PID参数防止振荡                                                        │
│   - 高帧率(60/120fps)可以降低绝对延迟                                               │
│                                                                                      │
└─────────────────────────────────────────────────────────────────────────────────────┘
```

#### 统计窗口配置

```python
class AWB_Statistics_Config:
    """
    AWB统计窗口配置
    
    定义统计引擎采样区域和参数
    """
    
    def __init__(self, image_width: int, image_height: int):
        self.image_width = image_width
        self.image_height = image_height
        
        # 统计窗口网格 (通常32×32或64×48)
        self.grid_h = 32
        self.grid_v = 32
        
        # 每个窗口的尺寸
        self.window_w = image_width // self.grid_h
        self.window_h = image_height // self.grid_v
        
        # 采样模式
        self.subsample_ratio = 4  # 每4×4像素采样一次
        
        # 有效范围过滤
        self.min_value = 16    # 排除过暗像素
        self.max_value = 240   # 排除过亮像素
        
    def get_window_config(self) -> dict:
        """返回统计窗口配置"""
        return {
            'grid_size': (self.grid_h, self.grid_v),
            'window_size': (self.window_w, self.window_h),
            'subsample': self.subsample_ratio,
            'valid_range': (self.min_value, self.max_value),
            'total_windows': self.grid_h * self.grid_v
        }
```

### 5.3 3A状态机协同

场景切换时需要协调三个模块的重新收敛：

```
┌─────────────────────────────────────────────────────────────────────────────────────┐
│                          场景切换时的3A协同                                           │
├─────────────────────────────────────────────────────────────────────────────────────┤
│                                                                                      │
│   事件: 检测到场景突变 (亮度变化>2EV 或 色温变化>1000K 或 主体距离变化>20%)          │
│                                                                                      │
│   时间 ──────────────────────────────────────────────────────────────────────▶       │
│         │                                                                           │
│   t0    │ 场景切换检测                                                              │
│         │    │                                                                      │
│         │    ▼                                                                      │
│   AE    │ ╔═══════════════════════╗                                                │
│         │ ║ SEARCHING (1-3帧)     ║ → CONVERGING → CONVERGED                       │
│         │ ╚═══════════════════════╝                                                │
│         │    │                                                                      │
│         │    │ AE稳定后启动AWB                                                      │
│         │    ▼                                                                      │
│   AWB   │    ╔═══════════════════════╗                                             │
│         │    ║ SEARCHING (1-3帧)     ║ → CONVERGING → CONVERGED                    │
│         │    ╚═══════════════════════╝                                             │
│         │         │                                                                 │
│         │         │ 曝光和白平衡稳定后AF有意义                                       │
│         │         ▼                                                                 │
│   AF    │         ╔═══════════════════════════════════╗                            │
│         │         ║ SEARCHING (PDAF 1帧 / CDAF 5-10帧) ║ → CONVERGED               │
│         │         ╚═══════════════════════════════════╝                            │
│         │                                                                           │
│   总耗时:  静态场景: ~200ms (~6帧@30fps)                                           │
│            运动场景: ~100ms (~3帧@30fps, AF优先)                                   │
│                                                                                      │
│   协同规则:                                                                          │
│   1. AE必须先收敛到±0.5EV范围，否则AWB/AF无法准确工作                               │
│   2. AWB变化会轻微影响AE统计，需要小幅迭代补偿                                       │
│   3. AF搜索期间禁止AE大幅调整（避免锐度波动干扰）                                    │
│   4. 稳态时三者独立维持，小扰动各自处理                                              │
│                                                                                      │
└─────────────────────────────────────────────────────────────────────────────────────┘
```

> 📌 **关键洞察**：3A协同的核心原则是"曝光是基础"——在曝光严重错误时，AWB的色温估计和AF的锐度评估都不可靠。因此，大多数实现会在AE大幅调整期间暂停或降权AWB/AF的输出。
