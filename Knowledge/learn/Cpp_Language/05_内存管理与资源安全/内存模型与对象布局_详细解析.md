# 内存模型与对象布局详细解析

> **核心结论**：C++ 对象的内存布局由编译器根据类型特征（POD、Standard Layout、Trivial）决定。理解虚函数表、继承布局和对齐规则是优化内存使用和诊断复杂问题的关键。placement new 允许在预分配内存上构造对象，是内存池等高级技术的基石。

---

## 1. Why — 为什么需要理解对象布局

### 1.1 内存布局的实际影响

```cpp
// 同样的数据，不同的布局，不同的大小
struct BadLayout {
    char a;      // 1 byte
    int b;       // 4 bytes (3 bytes padding)
    char c;      // 1 byte (3 bytes padding)
    double d;    // 8 bytes

struct GoodLayout {
    double d;    // 8 bytes
    int b;       // 4 bytes
    char a;      // 1 byte
    char c;      // 1 byte (2 bytes padding)
};  // 可能为 16 bytes

// 理解对象布局可以：
// 1. 优化内存使用
// 2. 理解虚函数调用机制
// 3. 正确使用 placement new
// 4. 进行底层内存操作
```

## 2. C++ 对象内存布局基础

### 2.1 类型分类详解

```cpp
// C++ 类型分类层次

// 1. Trivial 类型
// - 平凡的默认构造函数
// - 平凡的拷贝/移动构造函数
// - 平凡的拷贝/移动赋值运算符
// - 平凡的析构函数
// - 可以使用 memcpy 复制
struct TrivialType {
    int x;
    int y;
};

static_assert(std::is_trivial_v<TrivialType>);

// 2. Standard Layout 类型
// - 所有非静态成员有相同的访问控制
// - 没有虚函数或虚基类
// - 没有引用成员
// - 基类没有非静态成员（或没有非静态数据成员）
// - 与 C 结构体布局兼容
struct StandardLayout {
    int x;
    int y;
};

static_assert(std::is_standard_layout_v<StandardLayout>);

// 3. POD (Plain Old Data) 类型
// - 既是 Trivial 又是 Standard Layout
// - C++20 已弃用，但概念仍有用
struct PODType {
    int x;
    int y;
};

static_assert(std::is_pod_v<PODType>);

// 4. 非 POD 类型
struct NonTrivial {
    std::string s;  // 有非平凡构造函数
    int x;
};

static_assert(!std::is_trivial_v<NonTrivial>);
```

### 2.2 内存对齐规则

```cpp
#include <cstddef>
#include <iostream>

// 对齐规则
void alignmentRules() {
    // 1. 基本类型的对齐要求
    std::cout << "alignof(char) = " << alignof(char) << std::endl;      // 1
    std::cout << "alignof(short) = " << alignof(short) << std::endl;    // 2
    std::cout << "alignof(int) = " << alignof(int) << std::endl;        // 4
    std::cout << "alignof(double) = " << alignof(double) << std::endl;  // 8
    
    // 2. 结构体的对齐要求 = 最大成员的对齐要求
    struct S {
        char c;
        double d;
    };
    std::cout << "alignof(S) = " << alignof(S) << std::endl;  // 8
    
    // 3. 结构体大小 = 各成员大小之和 + padding，向上对齐到对齐边界
    std::cout << "sizeof(S) = " << sizeof(S) << std::endl;  // 16 (1 + 7 padding + 8)
}

// 手动控制对齐
void controlAlignment() {
    // alignas 指定对齐要求
    struct alignas(64) CacheLine {
        int data[16];  // 64 字节，对齐到缓存行
    };
    
    static_assert(alignof(CacheLine) == 64);
    static_assert(sizeof(CacheLine) == 64);
    
    // 检查对齐
    CacheLine cl;
    if (reinterpret_cast<uintptr_t>(&cl) % 64 == 0) {
        std::cout << "CacheLine is 64-byte aligned" << std::endl;
    }
}
```

## 3. 虚函数与虚函数表

### 3.1 虚函数表机制

```cpp
// 虚函数表（vtable）机制详解

class Base {
public:
    virtual void func1() {}
    virtual void func2() {}
    void nonVirtual() {}
    
    int x;
};

class Derived : public Base {
public:
    void func1() override {}  // 覆盖
    virtual void func3() {}   // 新增
    
    int y;
};

// 内存布局（典型实现）：
// 
// Base 对象：
// ┌─────────────────┐
// │ vptr (8 bytes)  │ ────────┐
// ├─────────────────┤         │
// │ x (4 bytes)     │         │
// │ padding (4)     │         │
// └─────────────────┘         │
//                             ▼
//                      Base vtable:
//                      ┌──────────────┐
//                      │ &Base::func1 │
//                      ├──────────────┤
//                      │ &Base::func2 │
//                      └──────────────┘
//
// Derived 对象：
// ┌─────────────────┐
// │ vptr (8 bytes)  │ ────────┐
// ├─────────────────┤         │
// │ x (4 bytes)     │         │
// │ padding (4)     │         │
// ├─────────────────┤         │
// │ y (4 bytes)     │         │
// │ padding (4)     │         │
// └─────────────────┘         │
//                             ▼
//                      Derived vtable:
//                      ┌─────────────────┐
//                      │ &Derived::func1 │  (覆盖)
//                      ├─────────────────┤
//                      │ &Base::func2    │  (继承)
//                      ├─────────────────┤
//                      │ &Derived::func3 │  (新增)
//                      └─────────────────┘

void vtableExample() {
    Base b;
    Derived d;
    
    // 获取 vptr（未定义行为，仅用于演示）
    using VTable = void(**)();
    VTable baseVptr = *reinterpret_cast<VTable*>(&b);
    VTable derivedVptr = *reinterpret_cast<VTable*>(&d);
    
    // vptr 指向不同的 vtable
    std::cout << "Base vtable: " << baseVptr << std::endl;
    std::cout << "Derived vtable: " << derivedVptr << std::endl;
}
```

### 3.2 多重继承的虚函数表

```cpp
// 多重继承的内存布局
class Base1 {
public:
    virtual void func1() {}
    int a;
};

class Base2 {
public:
    virtual void func2() {}
    int b;
};

class Derived : public Base1, public Base2 {
public:
    void func1() override {}
    void func2() override {}
    virtual void func3() {}
    int c;
};

// 内存布局：
// 
// Derived 对象：
// ┌─────────────────┐
// │ vptr1 (8)       │ ────────┐
// ├─────────────────┤         │
// │ a (4)           │         │
// │ padding (4)     │         │
// ├─────────────────┤         │
// │ vptr2 (8)       │ ────┐   │
// ├─────────────────┤     │   │
// │ b (4)           │     │   │
// │ padding (4)     │     │   │
// ├─────────────────┤     │   │
// │ c (4)           │     │   │
// │ padding (4)     │     │   │
// └─────────────────┘     │   │
//                         │   │
//                         ▼   ▼
//                    vtable1:  vtable2:
//                    ┌──────┐  ┌──────┐
//                    │func1 │  │func2 │
//                    ├──────┤  └──────┘
//                    │func3 │
//                    └──────┘

void multipleInheritanceExample() {
    Derived d;
    
    std::cout << "sizeof(Derived) = " << sizeof(Derived) << std::endl;
    // 典型值：32 (两个 vptr + 三个 int + padding)
    
    // 指针调整
    Base1* b1 = &d;
    Base2* b2 = &d;  // 可能需要调整指针
    
    std::cout << "b1 = " << b1 << std::endl;
    std::cout << "b2 = " << b2 << std::endl;
    // b2 可能等于 b1 + 16
}
```

### 3.3 虚继承的内存布局

```cpp
// 虚继承解决菱形继承问题
class GrandBase {
public:
    virtual void func() {}
    int x;
};

class Base1 : public virtual GrandBase {
public:
    int a;
};

class Base2 : public virtual GrandBase {
public:
    int b;
};

class Derived : public Base1, public Base2 {
public:
    int c;
};

// 内存布局（典型实现）：
//
// Derived 对象：
// ┌─────────────────┐
// │ vptr (Base1)    │ ────────┐
// ├─────────────────┤         │
// │ a (4)           │         │
// │ padding (4)     │         │
// ├─────────────────┤         │
// │ vptr (Base2)    │ ────┐   │
// ├─────────────────┤     │   │
// │ b (4)           │     │   │
// │ padding (4)     │     │   │
// ├─────────────────┤     │   │
// │ c (4)           │     │   │
// │ padding (4)     │     │   │
// ├─────────────────┤     │   │
// │ GrandBase 部分  │     │   │
// │ ├─ vptr         │◄────┼───┼──┐
// │ ├─ x (4)        │     │   │  │
// │ └─ padding (4)  │     │   │  │
// └─────────────────┘     │   │  │
//                         │   │  │
//                         ▼   ▼  ▼
//                      vtable 包含偏移量信息

void virtualInheritanceExample() {
    Derived d;
    
    std::cout << "sizeof(Derived) = " << sizeof(Derived) << std::endl;
    // 典型值：48 (三个 vptr + 四个 int + padding)
    
    // 只有一份 GrandBase
    Base1* b1 = &d;
    Base2* b2 = &d;
    GrandBase* gb1 = b1;
    GrandBase* gb2 = b2;
    
    // gb1 和 gb2 指向同一个 GrandBase 对象
    std::cout << "gb1 == gb2: " << (gb1 == gb2) << std::endl;  // true
}
```

## 5. Placement New

### 5.1 基本用法

```cpp
#include <new>
#include <iostream>

// placement new：在预分配内存上构造对象
void placementNewBasic() {
    // 1. 分配原始内存
    alignas(int) char buffer[sizeof(int)];
    
    // 2. 在 buffer 上构造 int
    int* p = new (buffer) int(42);
    
    std::cout << "Value: " << *p << std::endl;  // 42
    std::cout << "Address: " << p << std::endl;
    std::cout << "Buffer: " << static_cast<void*>(buffer) << std::endl;
    // 地址相同
    
    // 3. 显式调用析构函数
    p->~int();
    
    // 4. buffer 在栈上，自动释放
}

// 数组的 placement new
void placementNewArray() {
    alignas(std::string) char buffer[10 * sizeof(std::string)];
    
    // 构造多个对象
    std::string* strings = reinterpret_cast<std::string*>(buffer);
    for (int i = 0; i < 10; ++i) {
        new (&strings[i]) std::string("Hello " + std::to_string(i));
    }
    
    // 使用
    for (int i = 0; i < 10; ++i) {
        std::cout << strings[i] << std::endl;
    }
    
    // 显式析构（逆序）
    for (int i = 9; i >= 0; --i) {
        strings[i].~basic_string();
    }
}
```

### 5.2 内存池实现

```cpp
#include <new>
#include <vector>
#include <memory>
#include <cstddef>

// 简单的内存池
template<typename T, size_t BlockSize = 1024>
class MemoryPool {
    struct Block {
        alignas(T) char data[BlockSize * sizeof(T)];
        Block* next;
    };
    
    Block* blocks_ = nullptr;
    T* freeList_ = nullptr;
    size_t usedInCurrentBlock_ = 0;
    
public:
    ~MemoryPool() {
        while (blocks_) {
            Block* next = blocks_->next;
            delete blocks_;
            blocks_ = next;
        }
    }
    
    template<typename... Args>
    T* allocate(Args&&... args) {
        // 优先从 free list 分配
        if (freeList_) {
            T* obj = freeList_;
            freeList_ = *reinterpret_cast<T**>(freeList_);
            new (obj) T(std::forward<Args>(args)...);
            return obj;
        }
        
        // 检查当前 block 是否已满
        if (!blocks_ || usedInCurrentBlock_ >= BlockSize) {
            Block* newBlock = new Block;
            newBlock->next = blocks_;
            blocks_ = newBlock;
            usedInCurrentBlock_ = 0;
        }
        
        // 在当前 block 分配
        T* obj = reinterpret_cast<T*>(blocks_->data) + usedInCurrentBlock_;
        ++usedInCurrentBlock_;
        new (obj) T(std::forward<Args>(args)...);
        return obj;
    }
    
    void deallocate(T* obj) {
        if (obj) {
            obj->~T();
            // 添加到 free list
            *reinterpret_cast<T**>(obj) = freeList_;
            freeList_ = obj;
        }
    }
};

// 使用示例
class Widget {
    int value_;
public:
    explicit Widget(int v) : value_(v) {
        std::cout << "Widget(" << v << ") constructed" << std::endl;
    }
    ~Widget() {
        std::cout << "Widget(" << value_ << ") destroyed" << std::endl;
    }
};

void memoryPoolExample() {
    MemoryPool<Widget> pool;
    
    Widget* w1 = pool.allocate(1);
    Widget* w2 = pool.allocate(2);
    Widget* w3 = pool.allocate(3);
    
    pool.deallocate(w2);
    
    Widget* w4 = pool.allocate(4);  // 可能复用 w2 的内存
}
```

### 5.3 对象池与对齐

```cpp
#include <new>
#include <cstddef>
#include <cstdint>

// 考虑对齐的对象池
template<typename T>
class AlignedObjectPool {
    static constexpr size_t alignment = alignof(T);
    static constexpr size_t size = sizeof(T);
    
    struct alignas(alignment) Slot {
        char data[size];
        bool used = false;
    };
    
    std::vector<Slot> slots_;
    
public:
    explicit AlignedObjectPool(size_t capacity) : slots_(capacity) {}
    
    template<typename... Args>
    T* acquire(Args&&... args) {
        for (auto& slot : slots_) {
            if (!slot.used) {
                slot.used = true;
                return new (slot.data) T(std::forward<Args>(args)...);
            }
        }
        return nullptr;  // 池已满
    }
    
    void release(T* obj) {
        if (obj) {
            obj->~T();
            // 找到对应的 slot 并标记为未使用
            for (auto& slot : slots_) {
                if (reinterpret_cast<T*>(slot.data) == obj) {
                    slot.used = false;
                    break;
                }
            }
        }
    }
};
```

## 6. 小对象优化（SSO/SBO）

### 6.1 std::string 的 SSO

```cpp
#include <string>
#include <iostream>

// std::string 的小对象优化（SSO）
void ssoExample() {
    // 短字符串存储在对象内部（无堆分配）
    std::string shortStr = "Hello";  // 可能无堆分配
    
    // 长字符串存储在堆上
    std::string longStr(1000, 'x');  // 堆分配
    
    std::cout << "sizeof(std::string) = " << sizeof(std::string) << std::endl;
    // 典型值：32 字节（3 个指针或 SSO buffer）
    
    // SSO 阈值通常是 15-22 字符（取决于实现）
}

// 简化的 SSO string 实现（概念）
class SSOString {
    static constexpr size_t SSO_THRESHOLD = 15;
    
    union {
        struct {
            char* data_;
            size_t size_;
            size_t capacity_;
        } heap_;  // 堆分配模式
        
        struct {
            char buffer_[SSO_THRESHOLD + 1];
            unsigned char size_;  // SSO 模式下的大小
        } sso_;  // SSO 模式
    };
    
    bool isSSO() const { return sso_.size_ & 0x80; }
    
public:
    SSOString(const char* str) {
        size_t len = strlen(str);
        if (len <= SSO_THRESHOLD) {
            // 使用 SSO
            memcpy(sso_.buffer_, str, len + 1);
            sso_.size_ = static_cast<unsigned char>(len) | 0x80;
        } else {
            // 使用堆分配
            heap_.capacity_ = len * 2;
            heap_.data_ = new char[heap_.capacity_];
            memcpy(heap_.data_, str, len + 1);
            heap_.size_ = len;
        }
    }
    
    ~SSOString() {
        if (!isSSO() && heap_.data_) {
            delete[] heap_.data_;
        }
    }
};
```

## 7. 面试要点

### 7.1 高频面试题

#### Q1: sizeof(一个有虚函数的类) 是多少？

**答案**：
- 包含虚函数的类至少有一个 vptr（虚函数表指针）
- 典型大小：vptr（8 bytes，64位系统）+ 成员变量 + padding

```cpp
class Base {
    virtual void func();
    int x;
};
// sizeof(Base) = 16 (vptr:8 + x:4 + padding:4)

class Derived : public Base {
    int y;
};
// sizeof(Derived) = 16 (Base:16 + y:4 + padding:4 可能合并)
```

#### Q2: 虚继承的内存开销是什么？

**答案**：
- 每个虚基类增加一个 vptr
- 需要额外的间接寻址访问虚基类成员
- 对象布局更复杂，访问速度略慢

```cpp
class Base { int x; };
class VDerived : public virtual Base { int y; };
// sizeof(VDerived) 可能为 24 (两个 vptr + x + y + padding)
```

#### Q3: placement new 的应用场景？

**答案**：
- 内存池实现
- 容器（如 vector）的元素构造
- 需要在特定地址构造对象
- 避免频繁的堆分配/释放

```cpp
// 内存池
template<typename T>
class Pool {
    char buffer[1024];
public:
    T* allocate() {
        return new (buffer) T();
    }
};
```

#### Q4: 什么是空基类优化（EBO）？

**答案**：
- 空类作为基类时不占用额外空间
- 空类作为成员时至少占用 1 byte
- 标准库利用 EBO 实现 enable_shared_from_this 等

```cpp
class Empty {};
class Derived1 : Empty { int x; };  // sizeof = 4 (EBO)
class Derived2 { Empty e; int x; }; // sizeof = 8 (无 EBO)
```

#### Q5: 如何计算结构体的 padding？

**答案**：
- 每个成员的偏移量必须是其对齐值的倍数
- 结构体大小必须是最大对齐值的倍数
- 成员按大小降序排列可减少 padding

```cpp
struct Example {
    char a;   // offset 0
    // 3 bytes padding
    int b;    // offset 4
    char c;   // offset 8
    // 3 bytes padding
};  // sizeof = 12
```

## 8. 最佳实践

### 8.1 内存布局优化

```cpp
// 1. 按大小降序排列成员
struct Optimized {
    double d;    // 8
    long long l; // 8
    int i;       // 4
    short s;     // 2
    char c;      // 1
    char pad;    // 1
};  // 32 bytes，无浪费

// 2. 使用 #pragma pack 谨慎（可能影响性能）
#pragma pack(push, 1)
struct Packed {
    char c;
    int i;
};  // 5 bytes，但可能有性能损失
#pragma pack(pop)

// 3. 使用 alignas 控制对齐
struct alignas(64) CacheLineAligned {
    int data[16];
};  // 对齐到缓存行，避免 false sharing
```

### 8.2 对象构造与析构

```cpp
// 1. 使用 placement new 时注意对齐
alignas(MyClass) char buffer[sizeof(MyClass)];
MyClass* obj = new (buffer) MyClass();
// ...
obj->~MyClass();

// 2. 批量构造/析构
void constructRange(MyClass* begin, MyClass* end) {
    for (MyClass* p = begin; p != end; ++p) {
        new (p) MyClass();
    }
}

void destroyRange(MyClass* begin, MyClass* end) {
    for (MyClass* p = end; p != begin; ) {
        (--p)->~MyClass();
    }
}

// 3. 使用 std::uninitialized_fill / std::destroy
#include <memory>
std::uninitialized_fill(begin, end, value);
std::destroy(begin, end);
```

## 9. 常见陷阱

### 9.1 未定义行为：对象生命周期

```cpp
// 陷阱：访问已析构的对象
char buffer[sizeof(int)];
int* p = new (buffer) int(42);
p->~int();

int x = *p;  // UB！对象已析构

// 陷阱：重复析构
std::string* s = new std::string("hello");
s->~basic_string();
s->~basic_string();  // UB！重复析构
```

### 9.2 对齐问题

```cpp
// 陷阱：未对齐的 placement new
char buffer[sizeof(int)];  // 可能未对齐到 4 字节边界
// int* p = new (buffer) int(42);  // 可能未定义行为

// 正确做法
alignas(int) char buffer[sizeof(int)];
int* p = new (buffer) int(42);  // 安全
```

### 9.3 继承布局陷阱

```cpp
// 陷阱：假设基类在派生类开头
class Base { int x; };
class Derived : public Base { int y; };

Base* b = new Derived();
// 不要假设：
// Base* b2 = reinterpret_cast<Base*>(
//     reinterpret_cast<char*>(b) + sizeof(int));  // 危险！

// 正确做法：使用 static_cast
Derived* d = static_cast<Derived*>(b);
```

## 10. 总结

### 核心要点回顾

1. **类型分类**：POD、Trivial、Standard Layout 决定对象的内存特性
2. **对齐规则**：成员偏移量必须是其对齐值的倍数
3. **虚函数表**：每个有虚函数的类有一个 vptr，指向 vtable
4. **继承布局**：单继承、多继承、虚继承有不同的内存布局
5. **placement new**：在预分配内存上构造对象，内存池的基础
6. **SSO/SBO**：小对象优化减少堆分配

### 与内存优化的关系

对象布局知识是 [../cpp_memory_optimization/](../cpp_memory_optimization/) 中硬件层优化的基础：
- 缓存行对齐（alignas(64)）避免 false sharing
- 紧凑布局提高缓存命中率
- placement new 实现高效内存池

理解对象布局是编写高性能、低层 C++ 代码的必备技能。
