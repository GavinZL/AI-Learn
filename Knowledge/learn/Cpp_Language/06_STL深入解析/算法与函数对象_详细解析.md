## 3. 排序算法深度解析

### 3.1 std::sort 的实现：IntroSort

**IntroSort（内省排序）**：结合快速排序、堆排序和插入排序的混合算法。

```
┌─────────────────────────────────────────────────────────────────┐
│                    IntroSort 算法流程                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│                      开始排序                                     │
│                          │                                       │
│                          ↓                                       │
│                ┌─────────────────┐                               │
│                │ 元素数量 < 16 ? │                               │
│                └────────┬────────┘                               │
│                   是    │    否                                   │
│                   ↓     │                                        │
│           ┌──────────┐  │                                        │
│           │插入排序   │  │                                        │
│           │(小数组优化)│ │                                        │
│           └──────────┘  │                                        │
│                         ↓                                        │
│                ┌─────────────────┐                               │
│                │递归深度 > 2log n?│                               │
│                └────────┬────────┘                               │
│                   是    │    否                                   │
│                   ↓     │                                        │
│           ┌──────────┐  │                                        │
│           │堆排序    │  │  (防止快排最坏情况)                      │
│           │(保证O(nlogn)│ │                                        │
│           └──────────┘  │                                        │
│                         ↓                                        │
│                ┌─────────────────┐                               │
│                │  快速排序       │                               │
│                │  (分区+递归)    │                               │
│                └─────────────────┘                               │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

**为什么 IntroSort 优于纯快排**：

| 问题 | 纯快排 | IntroSort |
|-----|--------|-----------|
| 最坏情况 | O(n²)（已排序输入） | O(n log n) |
| 递归深度 | 最坏 O(n) | 最坏 O(log n) |
| 小数组效率 | 递归开销大 | 切换插入排序 |

### 3.2 排序算法选择

```cpp
// C++17 - 排序算法选择指南
#include <algorithm>
#include <vector>
#include <iostream>
#include <random>

void sorting_demo() {
    std::vector<int> v = {5, 2, 8, 1, 9, 3, 7, 4, 6};
    
    // 1. std::sort - 通用排序，不保证稳定
    std::sort(v.begin(), v.end());  // O(n log n)
    
    // 2. std::stable_sort - 稳定排序，保持相等元素顺序
    std::vector<std::pair<int, std::string>> data = {
        {3, "apple"}, {1, "banana"}, {3, "cherry"}, {2, "date"}
    };
    std::stable_sort(data.begin(), data.end(),
        [](const auto& a, const auto& b) { return a.first < b.first; });
    // 结果：(1,banana), (2,date), (3,apple), (3,cherry)
    // 注意：(3,apple) 仍在 (3,cherry) 之前
    
    // 3. std::partial_sort - 只需要前 k 个最小/最大
    std::vector<int> v2 = {9, 8, 7, 6, 5, 4, 3, 2, 1};
    std::partial_sort(v2.begin(), v2.begin() + 3, v2.end());
    // 前 3 个是最小的 3 个且有序：{1, 2, 3, ...}
    // 其余元素顺序未定义
    
    // 4. std::nth_element - 只需要第 n 小的元素
    std::vector<int> v3 = {5, 1, 4, 2, 3};
    std::nth_element(v3.begin(), v3.begin() + 2, v3.end());
    // v3[2] == 3（第 3 小的元素）
    // v3[0..1] <= 3, v3[3..4] >= 3，但各自内部无序
    
    // 5. std::partition - 按条件分组
    std::vector<int> v4 = {1, 2, 3, 4, 5, 6};
    auto it = std::partition(v4.begin(), v4.end(),
                             [](int x) { return x % 2 == 0; });
    // 偶数在前，奇数在后
}
```

**排序算法选择决策树**：

```
                         需要排序？
                            │
              ┌─────────────┴─────────────┐
              │                           │
         需要完全排序              只需要部分结果
              │                           │
     ┌────────┴────────┐         ┌───────┴───────┐
     │                 │         │               │
 需要稳定          不需要稳定    第k小元素      前k个有序
     │                 │         │               │
stable_sort          sort    nth_element   partial_sort
  O(n log n)       O(n log n)   O(n)         O(n log k)
  额外 O(n)         额外 O(log n)
```

## 4. 自定义比较器与严格弱序

### 4.1 严格弱序（Strict Weak Ordering）

**定义**：比较器 `comp(a, b)` 必须满足：
1. **反自反性**：`comp(a, a)` 必须为 false
2. **反对称性**：`comp(a, b)` 为 true 则 `comp(b, a)` 为 false
3. **传递性**：`comp(a, b)` 且 `comp(b, c)` 则 `comp(a, c)`
4. **不可比性的传递性**：若 a 与 b 不可比（`!comp(a,b) && !comp(b,a)`），
   且 b 与 c 不可比，则 a 与 c 不可比

### 4.2 违反严格弱序的后果

```cpp
// C++17 - 错误的比较器示例
#include <algorithm>
#include <vector>
#include <iostream>

// 错误示例1：<= 而不是 <
bool wrong_compare1(int a, int b) {
    return a <= b;  // 错误！违反反自反性
}

// 错误示例2：非传递性
struct Point { double x, y; };
bool wrong_compare2(const Point& a, const Point& b) {
    return a.x < b.x || a.y < b.y;  // 错误！非传递
}

// 正确示例：严格的 <
bool correct_compare(int a, int b) {
    return a < b;  // 正确
}

void strict_weak_order_demo() {
    std::vector<int> v = {3, 1, 4, 1, 5, 9, 2, 6};
    
    // 错误！可能导致：
    // - 无限循环
    // - 未定义行为
    // - 排序结果不正确
    // std::sort(v.begin(), v.end(), wrong_compare1);  // 危险！
    
    // 正确写法
    std::sort(v.begin(), v.end(), correct_compare);
    // 或直接使用默认 <
    std::sort(v.begin(), v.end());
}
```

### 4.3 常见比较器写法

```cpp
// C++17 - 比较器的正确写法
#include <algorithm>
#include <string>
#include <tuple>

struct Person {
    std::string name;
    int age;
    double score;
};

void comparator_examples() {
    std::vector<Person> people = {/*...*/};
    
    // 方式1：lambda
    std::sort(people.begin(), people.end(),
        [](const Person& a, const Person& b) {
            return a.age < b.age;
        });
    
    // 方式2：多字段排序（正确处理相等情况）
    std::sort(people.begin(), people.end(),
        [](const Person& a, const Person& b) {
            if (a.age != b.age) return a.age < b.age;
            return a.name < b.name;  // age 相等时按 name 排序
        });
    
    // 方式3：使用 std::tie（推荐，简洁且不易出错）
    std::sort(people.begin(), people.end(),
        [](const Person& a, const Person& b) {
            return std::tie(a.age, a.name) < std::tie(b.age, b.name);
        });
    
    // 方式4：C++20 的 <=>
    // auto cmp = [](const Person& a, const Person& b) {
    //     return std::tie(a.age, a.name) <=> std::tie(b.age, b.name);
    // };
    
    // 方式5：降序排序
    std::sort(people.begin(), people.end(), std::greater<>());
    // 或
    std::sort(people.begin(), people.end(),
        [](const Person& a, const Person& b) {
            return a.age > b.age;  // 注意 > 符号
        });
}
```

## 5. 函数对象与 Lambda

### 5.1 函数对象（Functor）

```cpp
// C++17 - 函数对象详解
#include <algorithm>
#include <vector>
#include <functional>

// 自定义函数对象
struct Multiplier {
    int factor;
    
    Multiplier(int f) : factor(f) {}
    
    int operator()(int x) const {
        return x * factor;
    }
};

// 有状态的函数对象
struct Accumulator {
    int sum = 0;
    
    void operator()(int x) {
        sum += x;
    }
};

void functor_demo() {
    std::vector<int> v = {1, 2, 3, 4, 5};
    
    // 使用自定义函数对象
    std::transform(v.begin(), v.end(), v.begin(), Multiplier(2));
    // v: {2, 4, 6, 8, 10}
    
    // 使用 std::function 存储任意可调用对象
    std::function<int(int)> f = Multiplier(3);
    std::function<int(int)> g = [](int x) { return x + 1; };
    
    // 有状态函数对象
    Accumulator acc;
    acc = std::for_each(v.begin(), v.end(), acc);
    // acc.sum == 30
}
```

### 5.2 Lambda 深度解析

```cpp
// C++17 - Lambda 完整语法
#include <algorithm>
#include <vector>
#include <iostream>

void lambda_demo() {
    int x = 10;
    int y = 20;
    std::vector<int> v = {1, 2, 3, 4, 5};
    
    // 基本语法：[捕获列表](参数列表) -> 返回类型 { 函数体 }
    auto lambda1 = [](int a, int b) -> int {
        return a + b;
    };
    
    // 捕获方式
    auto by_value = [x]() { return x; };        // 值捕获
    auto by_ref = [&x]() { x++; };              // 引用捕获
    auto all_by_value = [=]() { return x + y; }; // 全部值捕获
    auto all_by_ref = [&]() { x++; y++; };       // 全部引用捕获
    auto mixed = [=, &x]() { x = y; };           // 混合捕获
    
    // C++14: 初始化捕获（移动语义）
    auto ptr = std::make_unique<int>(42);
    auto move_capture = [p = std::move(ptr)]() { return *p; };
    
    // C++17: 泛型 Lambda（auto 参数）
    auto generic = [](const auto& a, const auto& b) {
        return a < b;
    };
    generic(1, 2);      // int
    generic(1.0, 2.0);  // double
    
    // C++17: constexpr Lambda
    constexpr auto square = [](int n) constexpr { return n * n; };
    static_assert(square(3) == 9);
    
    // C++20: 模板 Lambda
    // auto templated = []<typename T>(const T& a, const T& b) {
    //     return a < b;
    // };
}

// Lambda vs std::bind
void lambda_vs_bind() {
    auto add = [](int a, int b) { return a + b; };
    
    // Lambda 方式：更清晰，性能更好
    auto add5_lambda = [add](int x) { return add(x, 5); };
    
    // std::bind 方式：通常不推荐
    auto add5_bind = std::bind(add, std::placeholders::_1, 5);
    
    // 性能：Lambda 通常内联，bind 通常有运行时开销
}
```

## 6. C++17 并行算法

### 6.1 执行策略

```cpp
// C++17 - 执行策略详解
#include <algorithm>
#include <execution>
#include <vector>
#include <iostream>

void execution_policy_demo() {
    std::vector<int> v(1000000);
    std::iota(v.begin(), v.end(), 1);
    
    // 四种执行策略：
    
    // 1. std::execution::seq - 顺序执行（默认）
    std::sort(std::execution::seq, v.begin(), v.end());
    
    // 2. std::execution::par - 并行执行
    // 线程安全的要求更高，可能并行调用
    std::for_each(std::execution::par, v.begin(), v.end(),
        [](int& x) { x *= 2; });
    
    // 3. std::execution::par_unseq - 并行向量化
    // 可能使用 SIMD 指令，要求函数调用不阻塞
    std::transform(std::execution::par_unseq, v.begin(), v.end(), v.begin(),
        [](int x) { return x * x; });
    
    // 4. C++20: std::execution::unseq - 向量化（单线程 SIMD）
    // std::for_each(std::execution::unseq, v.begin(), v.end(), ...);
}
```

### 6.2 并行算法注意事项

```cpp
// C++17 - 并行算法的正确使用
#include <algorithm>
#include <execution>
#include <vector>
#include <mutex>
#include <atomic>

// 错误：并行算法中的数据竞争
void parallel_wrong() {
    std::vector<int> v(10000);
    int sum = 0;
    
    // 错误！多线程同时写入 sum，数据竞争！
    std::for_each(std::execution::par, v.begin(), v.end(),
        [&sum](int x) { sum += x; });  // 数据竞争！
}

// 正确方式1：使用锁（性能差）
void parallel_with_mutex() {
    std::vector<int> v(10000);
    int sum = 0;
    std::mutex mtx;
    
    std::for_each(std::execution::par, v.begin(), v.end(),
        [&sum, &mtx](int x) {
            std::lock_guard lock(mtx);
            sum += x;
        });
}

// 正确方式2：使用原子变量
void parallel_with_atomic() {
    std::vector<int> v(10000);
    std::atomic<int> sum{0};
    
    std::for_each(std::execution::par, v.begin(), v.end(),
        [&sum](int x) { sum.fetch_add(x, std::memory_order_relaxed); });
}

// 正确方式3：使用 reduce/transform_reduce（推荐）
void parallel_with_reduce() {
    std::vector<int> v(10000);
    std::iota(v.begin(), v.end(), 1);
    
    // 并行归约，自动处理线程安全
    int sum = std::reduce(std::execution::par, v.begin(), v.end(), 0);
    
    // 并行转换+归约
    int sum_of_squares = std::transform_reduce(
        std::execution::par,
        v.begin(), v.end(),
        0,
        std::plus<>(),
        [](int x) { return x * x; }
    );
}
```

## 7. 分配器与 PMR

### 7.1 分配器基础

```cpp
// C++17 - 分配器接口
#include <memory>
#include <vector>
#include <iostream>

// 自定义分配器（简化版）
template<typename T>
class TrackingAllocator {
public:
    using value_type = T;
    
    TrackingAllocator() = default;
    
    template<typename U>
    TrackingAllocator(const TrackingAllocator<U>&) {}
    
    T* allocate(size_t n) {
        std::cout << "Allocating " << n << " objects of size " 
                  << sizeof(T) << "\n";
        return static_cast<T*>(::operator new(n * sizeof(T)));
    }
    
    void deallocate(T* p, size_t n) {
        std::cout << "Deallocating " << n << " objects\n";
        ::operator delete(p);
    }
    
    template<typename U>
    bool operator==(const TrackingAllocator<U>&) const { return true; }
};

void custom_allocator_demo() {
    std::vector<int, TrackingAllocator<int>> v;
    
    v.push_back(1);  // 输出：Allocating 1 objects...
    v.push_back(2);  // 可能触发扩容
    v.push_back(3);
}
```

### 7.2 PMR（多态分配器）

```cpp
// C++17 - 多态分配器
#include <memory_resource>
#include <vector>
#include <string>
#include <array>
#include <iostream>

void pmr_demo() {
    // 1. 栈内存资源（固定大小缓冲区）
    std::array<std::byte, 1024> buffer;
    std::pmr::monotonic_buffer_resource pool{buffer.data(), buffer.size()};
    
    // 2. 使用 PMR vector，从池中分配
    std::pmr::vector<int> v{&pool};
    for (int i = 0; i < 100; ++i) {
        v.push_back(i);  // 从池中分配，非常快
    }
    // 注意：monotonic_buffer_resource 只增长不释放
    // 析构时一次性释放所有内存
    
    // 3. PMR string
    std::pmr::string s{&pool};
    s = "Hello, PMR World!";
    
    // 4. 嵌套容器
    std::pmr::vector<std::pmr::string> strings{&pool};
    strings.emplace_back("Hello");
    strings.emplace_back("World");
    
    // 5. unsynchronized_pool_resource（单线程高性能）
    std::pmr::unsynchronized_pool_resource fast_pool;
    std::pmr::vector<double> fast_vec{&fast_pool};
    
    // 6. synchronized_pool_resource（线程安全）
    std::pmr::synchronized_pool_resource thread_safe_pool;
    std::pmr::vector<double> safe_vec{&thread_safe_pool};
}
```

**PMR 的优势**：

| 特性 | 传统分配器 | PMR |
|-----|-----------|-----|
| 类型擦除 | 每个分配器类型生成不同容器代码 | 运行时多态 |
| 内存池 | 需要模板特化 | 可配置上游资源 |
| 容器共享 | 复杂的类型推导 | 简单的指针传递 |
| 性能 | 依赖全局分配器 | 可使用栈内存池 |

## 8. 面试要点

### Q1: std::sort 使用什么算法？为什么？

**答案**：

`std::sort` 使用 **IntroSort**（内省排序）：
1. 默认使用**快速排序**
2. 当递归深度超过 `2 * log(n)` 时，切换到**堆排序**
3. 当子数组长度小于某个阈值（通常16）时，切换到**插入排序**

**为什么这样设计**：
- 快排：平均 O(n log n)，实际最快
- 堆排序：保证最坏情况 O(n log n)
- 插入排序：小数组时常数因子最小

**面试加分**：可以手写 IntroSort 的核心逻辑框架。

### Q2: 为什么比较器必须满足严格弱序？

**答案**：

违反严格弱序会导致**未定义行为**，可能表现为：
1. **无限循环**：排序算法无法确定元素最终位置
2. **访问越界**：迭代器计算错误
3. **数据损坏**：元素被错误交换

**常见错误**：
```cpp
// 错误：使用 <=
std::sort(v.begin(), v.end(), [](int a, int b) { return a <= b; });

// 正确：使用 <
std::sort(v.begin(), v.end(), [](int a, int b) { return a < b; });
```

**原理**：排序算法假设 `comp(a, a)` 为 false，用于确定元素最终位置。如果违反，算法可能永远无法"安放"元素。

### Q3: parallel algorithms 什么时候应该使用？

**答案**：

**适合使用并行算法**：
1. 数据量大（通常 > 10000 元素）
2. 每个元素的操作是独立的（无数据竞争）
3. 元素操作有一定计算量（不只是简单赋值）
4. 机器有多核 CPU

**不适合使用**：
1. 数据量小（线程创建开销 > 计算）
2. 操作需要同步（如修改共享状态）
3. 操作可能阻塞（如 I/O）
4. 使用 `par_unseq` 时函数不能有副作用

**最佳实践**：
```cpp
// 根据数据量选择策略
if (data.size() > 100000) {
    std::sort(std::execution::par, data.begin(), data.end());
} else {
    std::sort(data.begin(), data.end());
}
```

### Q4: std::function 和 lambda 有什么区别？

**答案**：

| 维度 | Lambda | std::function |
|-----|--------|---------------|
| 类型 | 唯一的未命名类型 | 类型擦除的包装器 |
| 性能 | 可内联，零开销 | 虚调用，可能堆分配 |
| 存储 | 只能模板推导 | 可以存储、复制 |
| 灵活性 | 编译期固定 | 运行时可替换 |

```cpp
// Lambda：高性能，适合作为算法参数
std::sort(v.begin(), v.end(), [](auto a, auto b) { return a < b; });

// std::function：灵活性，适合存储回调
std::function<void(int)> callback = [](int x) { /*...*/ };
callback = another_lambda;  // 可以重新赋值
```

### Q5: 如何实现一个自定义分配器？

**答案**：

```cpp
template<typename T>
class MyAllocator {
public:
    using value_type = T;
    
    MyAllocator() = default;
    template<typename U> MyAllocator(const MyAllocator<U>&) {}
    
    T* allocate(size_t n) {
        return static_cast<T*>(::operator new(n * sizeof(T)));
    }
    
    void deallocate(T* p, size_t) {
        ::operator delete(p);
    }
    
    template<typename U>
    bool operator==(const MyAllocator<U>&) const { return true; }
};
```

**关键点**：
1. 必须定义 `value_type`
2. `allocate` 返回对齐的内存
3. `deallocate` 参数匹配 `allocate` 返回的指针
4. 提供相等比较（决定容器是否可以共享内存）

## 10. 常见陷阱

### 陷阱1：比较器未满足严格弱序

```cpp
// 错误
std::sort(v.begin(), v.end(), [](int a, int b) { return a <= b; });
// 可能导致无限循环或崩溃
```

### 陷阱2：并行算法中的数据竞争

```cpp
// 错误
int sum = 0;
std::for_each(std::execution::par, v.begin(), v.end(),
    [&sum](int x) { sum += x; });  // 数据竞争！
```

### 陷阱3：remove 不删除元素

```cpp
std::vector<int> v = {1, 2, 3, 2, 4};
std::remove(v.begin(), v.end(), 2);
// v.size() 仍是 5！
// 正确：erase-remove 惯用法
v.erase(std::remove(v.begin(), v.end(), 2), v.end());
```

### 陷阱4：std::function 的性能陷阱

```cpp
// 如果不需要类型擦除，不要使用 std::function
template<typename F>
void process(F f) { f(); }  // 好，可内联

void process(std::function<void()> f) { f(); }  // 差，虚调用
```
