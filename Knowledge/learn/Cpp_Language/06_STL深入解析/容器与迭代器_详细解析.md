# 容器与迭代器详细解析

> **核心结论**：选择正确的容器是 C++ 性能优化的第一步。`vector` 是大多数场景的默认选择；需要频繁中间插入时考虑 `deque` 或 `list`；需要有序关联查找时 `map` 是首选，无序查找优先 `unordered_map`。理解迭代器失效规则是避免并发 Bug 和内存错误的关键。

## 2. 序列容器

### 2.1 std::vector

**实现原理**：动态数组，连续内存存储。

```
┌─────────────────────────────────────────────────────────────────┐
│                     vector 内存布局                               │
├─────────────────────────────────────────────────────────────────┤
│  begin()                    end()              end() of capacity │
│     ↓                         ↓                       ↓          │
│  ┌───┬───┬───┬───┬───┬───┬───┬───┬───┬───┬───┬───┐              │
│  │ 1 │ 2 │ 3 │ 4 │ 5 │ 6 │ ? │ ? │ ? │ ? │ ? │ ? │              │
│  └───┴───┴───┴───┴───┴───┴───┴───┴───┴───┴───┴───┘              │
│  ├─────────────────size─────────────┤                            │
│  ├──────────────────capacity────────────────────────┤            │
└─────────────────────────────────────────────────────────────────┘
```

**关键特性**：

| 特性 | 说明 |
|-----|------|
| 内存布局 | 连续内存，缓存友好 |
| 访问复杂度 | O(1) 随机访问 |
| 尾部插入/删除 | 均摊 O(1) |
| 中间插入/删除 | O(n) |
| 扩容策略 | MSVC: 1.5x, GCC/Clang: 2x |

**扩容策略详解**：

```cpp
// C++17 - 观察扩容行为
#include <vector>
#include <iostream>

void demonstrate_growth() {
    std::vector<int> v;
    size_t last_capacity = 0;
    
    for (int i = 0; i < 100; ++i) {
        v.push_back(i);
        if (v.capacity() != last_capacity) {
            std::cout << "size=" << v.size() 
                      << ", capacity=" << v.capacity()
                      << ", ratio=" << (double)v.capacity() / last_capacity
                      << "\n";
            last_capacity = v.capacity();
        }
    }
}

// GCC 输出示例：
// size=1, capacity=1, ratio=inf
// size=2, capacity=2, ratio=2
// size=3, capacity=4, ratio=2
// size=5, capacity=8, ratio=2
// size=9, capacity=16, ratio=2
```

**1.5x vs 2x 扩容对比**：

| 策略 | 实现 | 优点 | 缺点 |
|-----|------|-----|------|
| **2x** | GCC, Clang | 无法复用之前内存，保证均摊 O(1) | 内存浪费可能较高 |
| **1.5x** | MSVC | 更节省内存，更少浪费 | 理论上可能复用已释放内存 |

### 2.2 std::deque

**实现原理**：分段连续内存（分段数组 / 中控器结构）。

```
┌─────────────────────────────────────────────────────────────────┐
│                     deque 内存布局                               │
├─────────────────────────────────────────────────────────────────┤
│                      map (中控器)                                │
│                    ┌─────────────┐                              │
│                    │ * │ * │ * │ *│  ← 指向各个固定大小缓冲区      │
│                    └───┴─┬─┴─┬─┴──┘                              │
│                          │   │                                   │
│         ┌────────────────┘   └────────────────┐                 │
│         ↓                                     ↓                 │
│    ┌─────────┐                          ┌─────────┐             │
│    │ ? │ ? │ 1 │ 2 │                    │ 5 │ 6 │ 7 │ ? │       │
│    └─────────┘                          └─────────┘             │
│              ↑   ↑                            ↑                 │
│           begin()                         end()                 │
└─────────────────────────────────────────────────────────────────┘
```

**关键特性**：

| 特性 | 说明 |
|-----|------|
| 内存布局 | 分段连续，map + fixed-size chunks |
| 访问复杂度 | O(1) 随机访问（略慢于 vector） |
| 两端插入/删除 | O(1) |
| 中间插入/删除 | O(n) |
| 迭代器失效 | 仅被删元素失效 |

```cpp
// C++17 - deque vs vector 两端插入对比
#include <deque>
#include <vector>
#include <chrono>
#include <iostream>

void compare_front_insertion() {
    const int N = 100000;
    
    // vector 头部插入
    std::vector<int> vec;
    auto start = std::chrono::high_resolution_clock::now();
    for (int i = 0; i < N; ++i) {
        vec.insert(vec.begin(), i);  // O(n) 每次移动所有元素
    }
    auto vec_time = std::chrono::high_resolution_clock::now() - start;
    
    // deque 头部插入
    std::deque<int> deq;
    start = std::chrono::high_resolution_clock::now();
    for (int i = 0; i < N; ++i) {
        deq.push_front(i);  // O(1)
    }
    auto deq_time = std::chrono::high_resolution_clock::now() - start;
    
    std::cout << "vector: " << std::chrono::duration_cast<std::chrono::milliseconds>(vec_time).count() << "ms\n";
    std::cout << "deque: " << std::chrono::duration_cast<std::chrono::milliseconds>(deq_time).count() << "ms\n";
    // 典型输出：vector: 2800ms, deque: 2ms
}
```

### 2.3 std::list 和 std::forward_list

**实现原理**：双向链表 / 单向链表。

```
┌─────────────────────────────────────────────────────────────────┐
│                  list (双向链表) 内存布局                         │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│    ┌────────┐    ┌────────┐    ┌────────┐    ┌────────┐         │
│    │prev|val│←──→│prev|val│←──→│prev|val│←──→│prev|val│         │
│    └────────┘    └────────┘    └────────┘    └────────┘         │
│    ↑                                              ↑              │
│  begin()                                       end()             │
│                                                                  │
├─────────────────────────────────────────────────────────────────┤
│                forward_list (单向链表) 内存布局                    │
│                                                                  │
│    ┌────────┐    ┌────────┐    ┌────────┐    ┌────────┐         │
│    │val│next│───→│val│next│───→│val│next│───→│val│next│───→ null│
│    └────────┘    └────────┘    └────────┘    └────────┘         │
│    ↑                                              ↑              │
│  before_begin()                               end()              │
└─────────────────────────────────────────────────────────────────┘
```

**关键特性对比**：

| 特性 | list | forward_list |
|-----|------|--------------|
| 内存开销 | 每节点 2 指针 + 数据 | 每节点 1 指针 + 数据 |
| 双向遍历 | ✓ | ✗ |
| 反向迭代器 | ✓ | ✗ |
| size() | O(1) | 无此函数（O(n) 设计） |
| 插入位置 | insert(pos, val) | insert_after(pos, val) |

```cpp
// C++17 - forward_list 的正确使用
#include <forward_list>
#include <iostream>

void forward_list_operations() {
    std::forward_list<int> fl = {1, 2, 3, 4, 5};
    
    // 在值为 2 的元素后插入 10
    auto it = fl.begin();
    while (it != fl.end() && *it != 2) ++it;
    if (it != fl.end()) {
        fl.insert_after(it, 10);  // 注意：insert_after，不是 insert
    }
    
    // 删除值为 3 的元素
    auto prev = fl.before_begin();
    auto curr = fl.begin();
    while (curr != fl.end() && *curr != 3) {
        ++prev;
        ++curr;
    }
    if (curr != fl.end()) {
        fl.erase_after(prev);  // 需要"前一个"迭代器
    }
}
```

### 2.4 std::array

**实现原理**：固定大小数组的包装器，零开销抽象。

```cpp
// C++17 - array 的零开销特性
#include <array>
#include <cstring>

void array_demo() {
    std::array<int, 5> arr = {1, 2, 3, 4, 5};
    
    // 与 C 数组完全等价
    int c_arr[5] = {1, 2, 3, 4, 5};
    static_assert(sizeof(arr) == sizeof(c_arr), "Same size!");
    
    // 提供 STL 接口
    for (auto& x : arr) x *= 2;  // 范围 for
    arr.at(0) = 100;             // 带边界检查
    arr.fill(0);                 // 填充
    arr.swap(arr);               // 交换
    
    // 编译期大小
    constexpr auto size = arr.size();  // 编译期常量
}
```

## 3. 关联容器

### 3.1 红黑树实现原理

**核心结构**：所有标准关联容器（set/map/multiset/multimap）基于红黑树实现。

```
┌─────────────────────────────────────────────────────────────────┐
│                      红黑树基本性质                               │
├─────────────────────────────────────────────────────────────────┤
│  1. 每个节点是红色或黑色                                         │
│  2. 根节点是黑色                                                 │
│  3. 叶子节点（NIL）是黑色                                        │
│  4. 红色节点的子节点必须是黑色                                    │
│  5. 从任一节点到其每个叶子的所有路径包含相同数目的黑色节点         │
├─────────────────────────────────────────────────────────────────┤
│  这些性质保证了最长路径不超过最短路径的 2 倍，从而保证 O(log n)    │
└─────────────────────────────────────────────────────────────────┘

                    示例树结构（map<int, string>）：
                            B:8
                          ╱     ╲
                       R:3       R:10
                      ╱   ╲        ╱
                   B:1   B:6   B:9
```

**时间复杂度保证**：

| 操作 | 平均 | 最坏 | 说明 |
|-----|------|------|------|
| 查找 | O(log n) | O(log n) | 平衡树保证 |
| 插入 | O(log n) | O(log n) | 可能触发 rebalance |
| 删除 | O(log n) | O(log n) | 可能触发 rebalance |
| 遍历 | O(n) | O(n) | 中序遍历有序 |

### 3.2 std::map 和 std::set

```cpp
// C++17 - map 的完整用法示例
#include <map>
#include <string>
#include <iostream>

void map_operations() {
    std::map<std::string, int> scores;
    
    // 插入方式对比
    scores["Alice"] = 90;                    // 方式1: 下标（默认构造 + 赋值）
    scores.insert({"Bob", 85});              // 方式2: insert（返回 pair<iterator, bool>）
    scores.emplace("Charlie", 92);           // 方式3: emplace（原地构造）
    scores.try_emplace("David", 88);         // C++17: 键存在时不覆盖
    
    // C++17: insert_or_assign
    scores.insert_or_assign("Alice", 95);    // 键存在则赋值，不存在则插入
    
    // 查找方式对比
    // 方式1: count（只判断存在）
    if (scores.count("Alice")) { /* ... */ }
    
    // 方式2: find（推荐，避免两次查找）
    if (auto it = scores.find("Alice"); it != scores.end()) {
        std::cout << "Alice's score: " << it->second << "\n";
    }
    
    // C++17: contains
    if (scores.contains("Alice")) { /* ... */ }
    
    // 有序遍历（按 key 排序）
    for (const auto& [name, score] : scores) {  // C++17 结构化绑定
        std::cout << name << ": " << score << "\n";
    }
}
```

### 3.3 map vs unordered_map 选择指南

```
┌─────────────────────────────────────────────────────────────────┐
│                    选择决策树                                     │
├─────────────────────────────────────────────────────────────────┤
│                          需要有序遍历？                           │
│                          ╱        ╲                             │
│                        是          否                            │
│                         ↓           ↓                           │
│                      map       需要 top-K 查询？                 │
│                                   ╱        ╲                    │
│                                 是          否                   │
│                                  ↓           ↓                  │
│                          堆/priority_queue  unordered_map        │
└─────────────────────────────────────────────────────────────────┘
```

| 维度 | map | unordered_map |
|-----|-----|---------------|
| 查找复杂度 | O(log n) | 平均 O(1)，最坏 O(n) |
| 有序性 | ✓ 天然有序 | ✗ 无序 |
| 内存开销 | 较小（3指针/节点） | 较大（哈希表 + 链表） |
| 最坏情况 | 稳定 O(log n) | 可能退化到 O(n) |
| Key 要求 | 支持 `<` 比较 | 支持哈希 + `==` |
| 适用场景 | 有序遍历、范围查询 | 高频查找、缓存 |

## 4. 无序容器

### 4.1 哈希表实现原理

**实现结构**：开链法（separate chaining）解决冲突。

```
┌─────────────────────────────────────────────────────────────────┐
│                unordered_map 内部结构                            │
├─────────────────────────────────────────────────────────────────┤
│  bucket array                                                    │
│  ┌─────┬─────┬─────┬─────┬─────┬─────┬─────┬─────┐              │
│  │  *  │  *  │null │  *  │null │  *  │  *  │null │              │
│  └──┬──┴──┬──┴─────┴──┬──┴─────┴──┬──┴──┬──┴─────┘              │
│     │     │           │           │     │                       │
│     ↓     ↓           ↓           ↓     ↓                       │
│   ┌───┐ ┌───┐       ┌───┐       ┌───┐ ┌───┐                     │
│   │K:V│ │K:V│       │K:V│       │K:V│ │K:V│                     │
│   └─┬─┘ └─┬─┘       └─┬─┘       └─┬─┘ └─┬─┘                     │
│     │     │           │           │     │                       │
│     ↓     ↓           ↓           ↓     ↓                       │
│   ┌───┐ ┌───┐       ┌───┐       null  ┌───┐                     │
│   │K:V│ │K:V│       │K:V│             │K:V│                     │
│   └───┘ └───┘       └───┘             └───┘                     │
└─────────────────────────────────────────────────────────────────┘

负载因子 = 元素数量 / bucket 数量
当负载因子 > max_load_factor() 时触发 rehash
```

### 4.2 负载因子与 rehash 策略

```cpp
// C++17 - 理解 rehash 行为
#include <unordered_map>
#include <iostream>

void rehash_demo() {
    std::unordered_map<int, std::string> um;
    
    // 默认 max_load_factor = 1.0
    std::cout << "Default max_load_factor: " << um.max_load_factor() << "\n";
    
    // 预留空间，避免多次 rehash
    um.reserve(1000);  // 一次分配足够容量
    
    // 或者设置 bucket 数量
    um.rehash(100);
    
    // 自定义负载因子阈值
    um.max_load_factor(0.75f);  // 降低负载因子，减少冲突
    
    for (int i = 0; i < 100; ++i) {
        um[i] = "value" + std::to_string(i);
        std::cout << "size=" << um.size() 
                  << ", buckets=" << um.bucket_count()
                  << ", load_factor=" << um.load_factor() << "\n";
    }
}
```

### 4.3 自定义哈希函数

```cpp
// C++17 - 自定义类型的哈希支持
#include <unordered_map>
#include <string>

struct Person {
    std::string name;
    int age;
    
    bool operator==(const Person& other) const {
        return name == other.name && age == other.age;
    }
};

// 方式1: 特化 std::hash
namespace std {
    template<>
    struct hash<Person> {
        size_t operator()(const Person& p) const {
            // 组合哈希的标准技巧
            size_t h1 = hash<string>{}(p.name);
            size_t h2 = hash<int>{}(p.age);
            return h1 ^ (h2 << 1);  // 或使用 boost::hash_combine
        }
    };
}

// 方式2: 传入自定义哈希
struct PersonHash {
    size_t operator()(const Person& p) const {
        return hash<string>{}(p.name) ^ (hash<int>{}(p.age) << 1);
    }
};

void custom_hash_demo() {
    // 使用方式1
    std::unordered_map<Person, std::string> map1;
    
    // 使用方式2
    std::unordered_map<Person, std::string, PersonHash> map2;
}
```

## 5. 容器适配器

### 5.1 stack、queue、priority_queue

**实现原理**：基于底层容器的包装器，提供受限接口。

```
┌─────────────────────────────────────────────────────────────────┐
│                       容器适配器关系                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   stack (栈)                queue (队列)         priority_queue  │
│   ┌───────────┐            ┌───────────┐       ┌───────────────┐ │
│   │ push()    │            │ push()    │       │ push()        │ │
│   │ pop()     │            │ pop()     │       │ pop()         │ │
│   │ top()     │            │ front()   │       │ top()         │ │
│   │           │            │ back()    │       │               │ │
│   └─────┬─────┘            └─────┬─────┘       └───────┬───────┘ │
│         │                        │                     │         │
│         ↓                        ↓                     ↓         │
│    ┌─────────┐             ┌─────────┐          ┌───────────┐    │
│    │ deque   │             │ deque   │          │ vector    │    │
│    │ (默认)   │             │ (默认)   │          │ (默认)    │    │
│    └─────────┘             └─────────┘          └───────────┘    │
└─────────────────────────────────────────────────────────────────┘
```

```cpp
// C++17 - priority_queue 完整示例
#include <queue>
#include <vector>
#include <iostream>

void priority_queue_demo() {
    // 默认大顶堆
    std::priority_queue<int> max_heap;
    max_heap.push(3);
    max_heap.push(1);
    max_heap.push(4);
    max_heap.push(1);
    max_heap.push(5);
    
    std::cout << "Max heap pop order: ";
    while (!max_heap.empty()) {
        std::cout << max_heap.top() << " ";  // 5 4 3 1 1
        max_heap.pop();
    }
    std::cout << "\n";
    
    // 小顶堆
    std::priority_queue<int, std::vector<int>, std::greater<int>> min_heap;
    min_heap.push(3);
    min_heap.push(1);
    min_heap.push(4);
    
    std::cout << "Min heap pop order: ";
    while (!min_heap.empty()) {
        std::cout << min_heap.top() << " ";  // 1 3 4
        min_heap.pop();
    }
    std::cout << "\n";
    
    // 自定义比较器
    struct Task {
        int priority;
        std::string name;
    };
    
    auto cmp = [](const Task& a, const Task& b) {
        return a.priority < b.priority;  // 大顶堆
    };
    std::priority_queue<Task, std::vector<Task>, decltype(cmp)> tasks(cmp);
}
```

## 6. 迭代器分类与特性

### 6.1 迭代器层次结构

```
┌─────────────────────────────────────────────────────────────────┐
│                     迭代器层次结构                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   Contiguous (C++17) ────────────────────────────────────────┐  │
│       RandomAccess ──────────────────────────────────────┐   │  │
│           Bidirectional ─────────────────────────────┐   │   │  │
│               Forward ───────────────────────────┐   │   │   │  │
│                   Input ──────────────────────┐  │   │   │   │  │
│                   Output ─────────────────────┼──┼───┼───┼───┼──┤
│                                               │  │   │   │   │  │
│  可读 (+)                    ✓  ✓   ✓   ✓   ✓  │
```

### 7.2 安全删除模式

```cpp
// C++17 - 安全的迭代器删除模式
#include <vector>
#include <list>
#include <map>

// 错误示范：vector 删除后继续使用迭代器
void unsafe_vector_erase() {
    std::vector<int> v = {1, 2, 3, 4, 5};
    for (auto it = v.begin(); it != v.end(); ++it) {
        if (*it % 2 == 0) {
            v.erase(it);  // 错误！it 失效，++it 未定义行为
        }
    }
}

// 正确方法1：使用返回值
void safe_vector_erase_v1() {
    std::vector<int> v = {1, 2, 3, 4, 5};
    for (auto it = v.begin(); it != v.end(); /* 不自增 */) {
        if (*it % 2 == 0) {
            it = v.erase(it);  // erase 返回下一个有效迭代器
        } else {
            ++it;
        }
    }
}

// 正确方法2：使用 remove-erase 惯用法（推荐）
void safe_vector_erase_v2() {
    std::vector<int> v = {1, 2, 3, 4, 5};
    v.erase(std::remove_if(v.begin(), v.end(), 
                           [](int x) { return x % 2 == 0; }),
             v.end());
}

// list/map 可以安全删除后继续使用
void safe_list_erase() {
    std::list<int> l = {1, 2, 3, 4, 5};
    for (auto it = l.begin(); it != l.end(); ) {
        if (*it % 2 == 0) {
            l.erase(it++);  // 安全：后置++ 先保存旧值
        } else {
            ++it;
        }
    }
}

// C++17：map 的提取删除
void map_extract_erase() {
    std::map<int, std::string> m = {{1, "one"}, {2, "two"}, {3, "three"}};
    
    // 提取节点，不影响其他迭代器
    auto node = m.extract(2);
    node.key() = 4;  // 可以修改 key
    m.insert(std::move(node));  // 重新插入
}
```

```cpp
void node_handle_demo() {

    std::map<int, std::string> m2 = {{4, "four"}, {5, "five"}};

    node.key() = 20;
    m2.insert(std::move(node));

}
```

## 10. 面试要点

### Q1: vector 扩容策略 1.5x vs 2x 的区别是什么？

**答案**：

1. **GCC/Clang 使用 2x**：保证容量增长几何级数，确保均摊 O(1) 的 push_back。更大的扩容步长减少了扩容次数。

2. **MSVC 使用 1.5x**：更节省内存，且理论上允许复用已释放的内存（因为 (1.5)^n ≈ 2^(n/2)，较早释放的内存可能被后续使用）。

3. **面试加分点**：2x 策略可以证明均摊 O(1)：`1 + 2 + 4 + ... + n = O(n)`，分摊到 n 次操作为 O(1)。而 1.5x 的证明类似，常数因子略差。

### Q2: 什么时候应该用 deque 而不是 vector？

**答案**：

使用 deque 的场景：
1. **需要频繁头部插入/删除**：deque 两端都是 O(1)
2. **数据量很大且增长不确定**：deque 分段内存，不会像 vector 一样一次性分配大块
3. **需要保证迭代器稳定性**：deque 两端插入不失效现有迭代器

不使用 deque 的场景：
1. **需要与 C API 交互**：vector 连续内存可直接传递 `data()`
2. **需要高性能随机访问**：vector 缓存更友好
3. **内存受限**：deque 内存碎片更多

### Q3: map 和 unordered_map 如何选择？

**答案**：

| 选择 map | 选择 unordered_map |
|----------|-------------------|
| 需要有序遍历 | 只需要快速查找 |
| 需要范围查询（lower_bound 等） | 不需要顺序 |
| key 没有好的哈希函数 | key 支持高效哈希 |
| 要求稳定性能（不能接受最坏 O(n)） | 接受平均 O(1) |
| 内存敏感 | 查找性能优先 |

**关键面试点**：unordered_map 最坏情况是 O(n)，发生在：
- 所有 key 哈希冲突
- 恶意构造的输入（DoS 攻击场景）

### Q4: 迭代器失效规则是什么？如何避免？

**答案**：

**核心规则**：
1. **vector**：插入可能全部失效（扩容），删除使被删位置之后失效
2. **deque**：两端操作安全，中间操作全部失效
3. **list/forward_list/关联容器**：仅被删元素失效

**安全模式**：
```cpp
// vector 安全删除
for (auto it = v.begin(); it != v.end(); ) {
    if (condition) it = v.erase(it);
    else ++it;
}

// 或使用 remove-erase 惯用法
v.erase(std::remove_if(v.begin(), v.end(), pred), v.end());
```

### Q5: 如何实现一个线程安全的容器？

**答案**：

**方案1：外部加锁（推荐）**
```cpp
template<typename T>
class ThreadSafeVector {
    std::vector<T> data;
    mutable std::mutex mtx;
public:
    void push_back(const T& value) {
        std::lock_guard lock(mtx);
        data.push_back(value);
    }
    // 注意：不要返回迭代器或引用（锁释放后失效）
};
```

**方案2：读写锁（读多写少）**
```cpp
template<typename K, typename V>
class ThreadSafeMap {
    std::map<K, V> data;
    mutable std::shared_mutex mtx;
public:
    std::optional<V> get(const K& key) const {
        std::shared_lock lock(mtx);  // 读锁
        auto it = data.find(key);
        if (it != data.end()) return it->second;
        return std::nullopt;
    }
    void set(const K& key, const V& value) {
        std::unique_lock lock(mtx);  // 写锁
        data[key] = value;
    }
};
```

**面试陷阱**：不要返回内部引用或迭代器，这会绕过锁保护。

## 11. 最佳实践

1. **默认选择 vector**：大多数场景最优，缓存友好

2. **reserve() 提前分配**：已知大小时避免多次扩容
   ```cpp
   std::vector<int> v;
   v.reserve(10000);  // 避免多次 reallocation
   ```

3. **emplace 代替 insert**：避免临时对象
   ```cpp
   map.emplace(key, value);  // 原地构造
   // vs
   map.insert({key, value});  // 构造 pair 临时对象
   ```

4. **使用 shrink_to_fit 释放内存**
   ```cpp
   std::vector<int> v(10000);
   v.clear();
   v.shrink_to_fit();  // 请求释放内存
   ```

5. **注意 string_view 生命周期**：不要返回指向临时对象的 string_view

## 12. 常见陷阱

### 陷阱1：vector 扩容后迭代器失效

```cpp
// 错误
std::vector<int> v = {1, 2, 3};
auto it = v.begin();
v.push_back(4);  // 可能扩容
*it;  // 未定义行为！

// 正确
auto it = v.begin();
v.push_back(4);
it = v.begin();  // 重新获取
```

### 陷阱2：reserve vs resize 混淆

```cpp
std::vector<int> v;
v.reserve(10);   // 仅分配容量，size() = 0
v.resize(10);    // 分配并构造 10 个元素，size() = 10

v.reserve(10);
v[0] = 1;  // 错误！越界，size() 还是 0
```

### 陷阱3：unordered_map 的哈希 DoS 攻击

```cpp
// 攻击场景：恶意构造相同哈希的 key
// 所有 key 落到同一个 bucket，查找退化为 O(n)

// 防御：使用安全哈希（如 SipHash）
// 或者限制输入、使用 map 替代
```

### 陷阱4：remove 不会删除元素

```cpp
std::vector<int> v = {1, 2, 3, 2, 4};
std::remove(v.begin(), v.end(), 2);  // v 变成 {1, 3, 4, 2, 4}，size 不变！
// 正确写法：
v.erase(std::remove(v.begin(), v.end(), 2), v.end());
```
