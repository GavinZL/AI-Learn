## 核心结论

**C++ 运行时多态的实现基于 vtable（虚函数表）+ vptr（虚函数指针），代价是间接调用无法内联。**

1. **vtable 机制**：每个多态类有一张虚函数表，每个对象有一个指向 vtable 的隐藏指针
2. **虚函数开销**：间接调用（2-5 CPU 周期）vs 直接调用（1-2 周期 + 内联优化）
3. **RTTI 开销**：typeid/dynamic_cast 依赖 vtable 中的类型信息
4. **多重继承**：对象布局复杂化，this 指针可能需要调整

## 第一部分：虚函数机制

### 1.1 虚函数基础

```cpp
class Base {
public:
    virtual void foo() { std::cout << "Base::foo\n"; }
    virtual void bar() { std::cout << "Base::bar\n"; }
    void nonVirtual() { std::cout << "Base::nonVirtual\n"; }
};

class Derived : public Base {
public:
    void foo() override { std::cout << "Derived::foo\n"; }
    // bar() 未重写，使用 Base::bar
    void nonVirtual() { std::cout << "Derived::nonVirtual\n"; }  // 隐藏，非重写
};

void demo() {
    Base* ptr = new Derived;
    ptr->foo();          // Derived::foo（虚函数调用）
    ptr->bar();          // Base::bar（未重写）
    ptr->nonVirtual();   // Base::nonVirtual（非虚函数，静态绑定）
}
```

### 1.2 虚函数调用机制

```
┌──────────────────────────────────────────────────────────────────────┐
│                    虚函数调用流程                                     │
├──────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  代码：                                                              │
│  Base* ptr = new Derived;                                           │
│  ptr->foo();                                                        │
│                                                                      │
│  汇编层面：                                                          │
│  1. 从 ptr 指向的对象中读取 vptr                                     │
│  2. 从 vtable 中获取 foo 的函数指针（偏移量固定）                     │
│  3. 间接调用该函数指针                                               │
│                                                                      │
│  伪代码：                                                            │
│  ┌────────────────────────────────────────────────────────────┐     │
│  │ // ptr->foo() 的展开                                       │     │
│  │ void** vtable = *(void***)ptr;  // 读取 vptr               │     │
│  │ void (*foo_ptr)(Base*) = (void(*)(Base*))vtable[0];        │     │
│  │ foo_ptr(ptr);  // 调用，传入 this 指针                      │     │
│  └────────────────────────────────────────────────────────────┘     │
│                                                                      │
│  关键：调用哪个函数取决于运行时对象的实际类型                         │
│                                                                      │
└──────────────────────────────────────────────────────────────────────┘
```

### 1.3 虚析构函数

```cpp
class Base {
public:
    virtual ~Base() { std::cout << "Base dtor\n"; }  // 虚析构函数
};

class Derived : public Base {
    int* data_;
public:
    Derived() : data_(new int[100]) {}
    ~Derived() { 
        delete[] data_; 
        std::cout << "Derived dtor\n"; 
    }
};

void test() {
    Base* ptr = new Derived;
    delete ptr;  // 正确调用 Derived::~Derived() → Base::~Base()
}

// 如果 Base::~Base() 不是虚函数：
// delete ptr 只调用 Base::~Base()，Derived::~Derived() 不被调用
// 导致 data_ 内存泄漏！
```

**规则**：当类可能被继承且通过基类指针删除时，析构函数必须是虚函数。

### 2.2 vtable 的组成

```
┌──────────────────────────────────────────────────────────────────────┐
│                    vtable 完整结构（典型实现）                        │
├──────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  vtable 通常包含：                                                   │
│  1. 虚函数指针（按声明顺序排列）                                     │

│  3. offset-to-top（用于多重继承的 this 调整）                        │

│  │ ┌─────────────────────┐                                     │     │

│  │ ├─────────────────────┤                                     │     │
│  │ │ type_info pointer   │  RTTI 信息                          │     │
│  │ ├─────────────────────┤                                     │     │
│  │ │ vfunc[0] (destructor)│                                    │     │
│  │ ├─────────────────────┤                                     │     │
│  │ │ vfunc[1] (foo)      │                                     │     │
│  │ ├─────────────────────┤                                     │     │
│  │ │ vfunc[2] (bar)      │                                     │     │
│  │ └─────────────────────┘                                     │     │
│  └────────────────────────────────────────────────────────────┘     │
│                                                                      │
│  注意：vtable 存储在只读数据段（.rodata）                           │
│                                                                      │
└──────────────────────────────────────────────────────────────────────┘
```

### 2.3 对象大小计算

```cpp
// 无虚函数
struct Plain {
    int a;
    double b;
};
// sizeof(Plain) = 16（对齐后）

// 有虚函数
struct WithVirtual {
    int a;
    virtual void foo();
};
// sizeof(WithVirtual) = 16（64位系统）
// = 8（vptr）+ 4（int a）+ 4（padding）

// 空类
struct Empty {};
// sizeof(Empty) = 1（C++ 规定空类大小至少为 1）

// 空基类优化（EBO）
struct EmptyBase {};
struct Derived : EmptyBase {
    int x;
};
// sizeof(Derived) = 4（空基类不占用空间）

// 无 EBO 的情况
struct Member {
    EmptyBase e;  // 成员，非基类
    int x;
};
// sizeof(Member) = 8（空成员占 1 字节 + 3 字节 padding）
```

## 第三部分：override 与 final

### 3.1 override 关键字（C++11）

```cpp
class Base {
public:
    virtual void foo(int x);
    virtual void bar() const;
};

class Good : public Base {
public:
    void foo(int x) override;     // OK：正确重写
    void bar() const override;    // OK：正确重写
};

class Bad : public Base {
public:
    void foo(double x) override;  // 编译错误：签名不匹配
    void bar() override;          // 编译错误：缺少 const
    void baz() override;          // 编译错误：Base 没有 baz
};
```

**最佳实践**：总是使用 `override`，让编译器帮你检查拼写和签名错误。

### 3.2 final 关键字（C++11）

```cpp
// 禁止类被继承
class FinalClass final {
    // ...
};

class CannotDerive : public FinalClass {};  // 编译错误

// 禁止虚函数被重写
class Base {
public:
    virtual void foo() final;
};

class Derived : public Base {
public:
    void foo() override;  // 编译错误：foo 是 final
};
```

**最佳实践**：谨慎使用 `final`，可能阻碍扩展性。但可以用于优化（帮助编译器去虚拟化）。

### 3.3 override + final 的优化意义

```cpp
class Base {
public:
    virtual void foo();
};

class Derived final : public Base {
public:
    void foo() override final;  // 同时使用
};

void callFoo(Derived& d) {
    d.foo();  // 编译器可以确定调用 Derived::foo
              // 因为 Derived 是 final，不会有更多派生类
}
```

## 第四部分：RTTI 运行时类型信息

### 4.1 typeid 运算符

```cpp
#include <typeinfo>

class Base { virtual void foo() {} };
class Derived : public Base {};

void test() {
    Base* ptr = new Derived;
    
    // 静态类型
    std::cout << typeid(ptr).name() << std::endl;    // "P4Base"（指针类型）
    
    // 动态类型（多态类型）
    std::cout << typeid(*ptr).name() << std::endl;   // "7Derived"
    
    // 比较类型
    if (typeid(*ptr) == typeid(Derived)) {
        std::cout << "ptr points to Derived\n";
    }
    
    // type_info 对象
    const std::type_info& info = typeid(*ptr);
    std::cout << info.name() << std::endl;  // 实现定义的名称
}
```

### 4.2 dynamic_cast

```cpp
class Base { virtual void foo() {} };
class Derived : public Base { 
public: 
    void derivedMethod() { std::cout << "Derived method\n"; } 
};

void test() {
    Base* b1 = new Derived;
    Base* b2 = new Base;
    
    // 指针转换：失败返回 nullptr
    Derived* d1 = dynamic_cast<Derived*>(b1);  // 成功
    Derived* d2 = dynamic_cast<Derived*>(b2);  // 返回 nullptr
    
    if (d1) {
        d1->derivedMethod();  // 安全调用
    }
    
    // 引用转换：失败抛出 std::bad_cast
    try {
        Base& ref = *b2;
        Derived& dref = dynamic_cast<Derived&>(ref);
    } catch (const std::bad_cast& e) {
        std::cout << "Cast failed: " << e.what() << std::endl;
    }
}
```

### 4.3 RTTI 的代价

```cpp
// RTTI 的开销：
// 1. 代码大小：type_info 对象存储在只读数据段
// 2. 运行时开销：dynamic_cast 需要遍历继承链

// 如果不需要 RTTI，可以禁用（编译选项：-fno-rtti）
// 但会禁止 dynamic_cast 和 typeid

// 替代方案：自定义类型标记
class Shape {
public:
    enum class Kind { Circle, Rectangle, Triangle };
    virtual Kind kind() const = 0;
};

class Circle : public Shape {
public:
    Kind kind() const override { return Kind::Circle; }
    double radius() const { return radius_; }
private:
    double radius_;
};

void process(Shape* s) {
    if (s->kind() == Shape::Kind::Circle) {
        Circle* c = static_cast<Circle*>(s);  // 比 dynamic_cast 快
        // ...
    }
}
```

## 第五部分：多重继承与虚继承

### 5.1 多重继承的内存布局

```
┌──────────────────────────────────────────────────────────────────────┐
│                    多重继承内存布局                                   │
├──────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  class A {                          class B {                        │
│  public:                            public:                          │
│      int a_;                            int b_;                      │
│      virtual void foo();                virtual void bar();          │
│  };                                 };                               │
│                                                                      │
│  class C : public A, public B {                                     │
│  public:                                                            │
│      int c_;                                                         │
│      void foo() override;                                           │
│      void bar() override;                                           │
│  };                                                                 │
│                                                                      │
│  C 对象布局：                                                        │
│  ┌──────────────┐                                                   │
│  │   A::vptr    │ ──→ C 的 A-vtable（包含 C::foo）                  │
│  ├──────────────┤                                                   │
│  │     a_       │                                                   │
│  ├──────────────┤                                                   │
│  │   B::vptr    │ ──→ C 的 B-vtable（包含 C::bar）                  │
│  ├──────────────┤                                                   │
│  │     b_       │                                                   │
│  ├──────────────┤                                                   │
│  │     c_       │                                                   │
│  └──────────────┘                                                   │
│                                                                      │
│  关键：                                                              │
│  - C 有两个 vptr（每个基类一个）                                    │
│  - C* 到 B* 的转换需要 this 指针调整                                │
│  - sizeof(C) 可能 > sizeof(A) + sizeof(B)（额外的 vptr）            │
│                                                                      │
└──────────────────────────────────────────────────────────────────────┘
```

### 5.2 this 指针调整

```cpp
class A { public: virtual void foo() {} int a; };
class B { public: virtual void bar() {} int b; };
class C : public A, public B { public: int c; };

void test() {
    C* c = new C;
    
    // 静态转换
    A* a = c;  // a == c（无偏移）
    B* b = c;  // b == c + sizeof(A)（有偏移）
    
    // 打印地址
    std::cout << c << std::endl;  // 例如 0x1000
    std::cout << a << std::endl;  // 0x1000
    std::cout << b << std::endl;  // 0x1008（假设 sizeof(A) == 16）
    
    // 通过 B* 调用虚函数时，this 会自动调整回 C*
    b->bar();  // this 在函数内部是 C*，不是 B*
}
```

### 5.4 虚继承解决方案

```cpp
class Animal { public: int age; };
class Winged : virtual public Animal { };
class Legged : virtual public Animal { };
class Bat : public Winged, public Legged { };

// Bat 对象布局（虚继承）
// ┌──────────────────┐
// │ Winged part      │
// │   ├─ vbptr       │ ──┐
// ├──────────────────┤   │
// │ Legged part      │   │
// │   ├─ vbptr       │ ──┤
// ├──────────────────┤   │
// │ Animal (shared)  │ ←─┘
// │   ├─ age         │
// └──────────────────┘

Bat bat;
bat.age = 10;  // OK：只有一份 age
```

**虚继承的代价**：
- 对象大小增加（vbptr 指向虚基类表）
- 访问虚基类成员需要间接寻址
- 构造函数复杂度增加（最远派生类负责构造虚基类）

## 第六部分：性能分析

### 6.1 虚函数调用开销

```cpp
// 性能基准测试（示意）
#include <chrono>
#include <iostream>

class VirtualBase {
public:
    virtual int compute(int x) { return x * 2; }
};

class VirtualDerived : public VirtualBase {
public:
    int compute(int x) override { return x * 3; }
};

class Concrete {
public:
    int compute(int x) { return x * 2; }
};

void benchmark() {
    const int N = 100'000'000;
    
    // 虚函数调用
    VirtualBase* v = new VirtualDerived;
    auto start = std::chrono::high_resolution_clock::now();
    volatile int result = 0;
    for (int i = 0; i < N; ++i) {
        result = v->compute(i);
    }
    auto end = std::chrono::high_resolution_clock::now();
    std::cout << "Virtual: " 
              << std::chrono::duration_cast<std::chrono::milliseconds>(end - start).count() 
              << "ms\n";
    
    // 直接调用
    Concrete c;
    start = std::chrono::high_resolution_clock::now();
    for (int i = 0; i < N; ++i) {
        result = c.compute(i);
    }
    end = std::chrono::high_resolution_clock::now();
    std::cout << "Direct: " 
              << std::chrono::duration_cast<std::chrono::milliseconds>(end - start).count() 
              << "ms\n";
}

// 典型结果（N = 1亿）：
// Virtual: ~300ms（无法内联）
// Direct:  ~50ms（内联后可能被优化为空循环）
```

### 6.2 性能对比表

| 操作 | 相对开销 | 说明 |
|------|----------|------|
| 直接函数调用 | 1x（基准） | 可内联，编译器可高度优化 |
| 虚函数调用 | 1.5-2x | 间接调用，无法内联 |
| dynamic_cast（指针） | 2-10x | 需遍历继承链 |
| dynamic_cast（引用） | 2-10x + 异常 | 失败时抛出异常 |
| typeid | 1.5-2x | 直接访问 vtable 中的 type_info |

### 6.3 去虚拟化优化

```cpp
class Base {
public:
    virtual void foo() { /* ... */ }
};

void callDirect(Base& b) {
    b.foo();  // 可能去虚拟化
}

void test() {
    Base b;
    callDirect(b);  // 编译器可以看到 b 的实际类型
                    // 可能直接调用 Base::foo，无需虚函数分发
}

// final 帮助去虚拟化
class FinalDerived final : public Base {
public:
    void foo() override { /* ... */ }
};

void callFinal(FinalDerived& d) {
    d.foo();  // 一定调用 FinalDerived::foo
              // 因为 FinalDerived 是 final，不会有更多派生类
}
```

## 面试要点

### Top 3 高频考点

#### 1. vtable 实现原理

**Q：虚函数是如何实现的？**

```cpp
// 回答要点：
// 1. 每个多态类有一张 vtable（虚函数表）
// 2. vtable 存储虚函数指针，按声明顺序排列
// 3. 每个对象有一个 vptr，指向对应类的 vtable
// 4. 虚函数调用通过 vptr -> vtable -> 函数指针 间接完成

// 内存位置
// - vtable 在只读数据段（.rodata）
// - vptr 在对象内存布局的开头
```

**追问：构造函数可以是虚函数吗？**

不可以。原因：
1. 构造时对象类型尚未完整
2. vptr 在构造函数执行后才被设置
3. 构造基类时，vptr 指向基类的 vtable

#### 2. 虚析构函数的必要性

**Q：什么时候需要虚析构函数？**

```cpp
// 规则：当类可能被继承，且可能通过基类指针删除时

class Base {
public:
    virtual ~Base() {}  // 必须 virtual
};

class Derived : public Base {
    int* data_;
public:
    Derived() : data_(new int[100]) {}
    ~Derived() { delete[] data_; }  // 需要被调用
};

Base* ptr = new Derived;
delete ptr;  // 需要 virtual ~Base() 才能正确调用 ~Derived
```

#### 3. 菱形继承与虚继承

**Q：什么是菱形继承问题？如何解决？**

```cpp
// 问题：多重继承同一基类导致成员重复
class A { public: int x; };
class B : public A {};
class C : public A {};
class D : public B, public C {};

D d;
d.x = 1;  // 编译错误：二义性

// 解决：虚继承
class B : virtual public A {};
class C : virtual public A {};
class D : public B, public C {};

D d;
d.x = 1;  // OK：只有一份 x

// 代价：对象大小增加，访问稍慢（间接寻址）
```

## 最佳实践

### 1. 虚函数设计

```cpp
// ✓ 推荐：析构函数设为 virtual（如果可能被继承）
class Base {
public:
    virtual ~Base() = default;
};

// ✓ 推荐：使用 override 确保正确重写
class Derived : public Base {
public:
    void foo() override;  // 编译器检查签名
};

// ✓ 推荐：使用 NVI 模式
class Widget {
public:
    void process() {          // 非虚公共接口
        validate();           // 前置检查
        doProcess();          // 调用私有虚函数
        log();                // 后置处理
    }
private:
    virtual void doProcess() = 0;  // 私有虚函数
};
```

### 2. 避免过度使用 RTTI

```cpp
// ✗ 避免：大量的 dynamic_cast
void process(Shape* s) {
    if (auto* c = dynamic_cast<Circle*>(s)) {
        // ...
    } else if (auto* r = dynamic_cast<Rectangle*>(s)) {
        // ...
    }
    // 扩展困难，违反开闭原则
}

// ✓ 推荐：使用虚函数
class Shape {
public:
    virtual void process() = 0;
};

class Circle : public Shape {
public:
    void process() override { /* ... */ }
};
```

### 3. 空基类优化

```cpp
// 利用 EBO 减少对象大小
template<typename Allocator>
class Container : private Allocator {  // 私有继承
    // 如果 Allocator 是空类，不增加 sizeof(Container)
};

// 对比成员方案
template<typename Allocator>
class Container2 {
    Allocator alloc_;  // 即使空类，也占至少 1 字节
};
```

## 常见陷阱

### 陷阱 1：构造/析构中调用虚函数

```cpp
class Base {
public:
    Base() { foo(); }  // 调用 Base::foo，而非 Derived::foo！
    virtual void foo() { std::cout << "Base\n"; }
};

class Derived : public Base {
public:
    void foo() override { std::cout << "Derived\n"; }
};

Derived d;  // 输出 "Base"，而非 "Derived"

// 原因：构造 Base 部分时，Derived 部分尚未构造
// vptr 指向 Base 的 vtable

// 最佳实践：避免在构造/析构中调用虚函数
```

### 陷阱 2：默认参数与虚函数

```cpp
class Base {
public:
    virtual void foo(int x = 10) { 
        std::cout << "Base: " << x << "\n"; 
    }
};

class Derived : public Base {
public:
    void foo(int x = 20) override { 
        std::cout << "Derived: " << x << "\n"; 
    }
};

Base* b = new Derived;
b->foo();  // 输出 "Derived: 10"
// 虚函数动态绑定，但默认参数静态绑定！
// 使用 Base 的默认值 10，而非 Derived 的 20

// 最佳实践：避免在虚函数中使用默认参数
```

### 陷阱 3：析构函数中的异常

```cpp
class BadExample {
public:
    ~BadExample() {
        throw std::runtime_error("Error!");  // 危险！
    }
};

void dangerous() {
    BadExample obj;
    throw std::runtime_error("Another error");
    // obj 析构时抛出第二个异常
    // 程序直接 terminate！
}

// 最佳实践：析构函数应该 noexcept
class GoodExample {
public:
    ~GoodExample() noexcept {
        try {
            // 可能抛出异常的代码
        } catch (...) {
            // 吞掉异常或记录日志
        }
    }
};
```

### 陷阱 4：切片问题

```cpp
class Base { public: virtual void foo() { std::cout << "Base\n"; } };
class Derived : public Base { public: void foo() override { std::cout << "Derived\n"; } };

void bad(Base b) {  // 按值传递，发生切片
    b.foo();  // 总是输出 "Base"
}

void good(const Base& b) {  // 引用传递，保留多态
    b.foo();  // 正确调用 Derived::foo
}

Derived d;
bad(d);   // 切片：Derived 部分被"切掉"
good(d);  // 正确
```

### 陷阱 5：多重继承的二义性

```cpp
class A { public: void foo() {} };
class B { public: void foo() {} };
class C : public A, public B {};

C c;
c.foo();  // 编译错误：A::foo 还是 B::foo？

// 修正：显式指定
c.A::foo();  // OK
c.B::foo();  // OK

// 或使用 using 声明
class C : public A, public B {
public:
    using A::foo;  // 选择 A::foo
};
```
