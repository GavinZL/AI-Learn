## 核心结论

**C++ 设计惯用法的核心目标是：资源安全 + 异常安全 + 零开销抽象。**

1. **RAII**：资源获取即初始化，C++ 资源管理的基石
2. **Rule of Five**：如果需要自定义析构/拷贝/移动，则需要定义全部五个
3. **PImpl**：编译防火墙，隐藏实现细节，改善 ABI 稳定性
4. **CRTP**：静态多态，编译期实现多态，零运行时开销

## 第一部分：SOLID 原则在 C++ 中的体现

### 1.1 单一职责原则（SRP）

**定义**：一个类应该只有一个引起它变化的原因。

```cpp
// ✗ 违反 SRP：一个类做太多事
class UserManager {
public:
    void createUser(const std::string& name);
    void deleteUser(int id);
    void sendEmail(int userId, const std::string& message);
    void logOperation(const std::string& operation);
    void validateUser(const User& user);
};

// ✓ 遵循 SRP：职责分离
class UserRepository {
public:
    void create(const User& user);
    void remove(int id);
    User find(int id);
};

class EmailService {
public:
    void send(int userId, const std::string& message);
};

class UserValidator {
public:
    bool validate(const User& user);
};

class UserService {
    UserRepository& repo_;
    EmailService& email_;
    UserValidator& validator_;
public:
    void createUser(const std::string& name) {
        User user{name};
        if (validator_.validate(user)) {
            repo_.create(user);
            email_.send(user.id(), "Welcome!");
        }
    }
};
```

### 1.2 开闭原则（OCP）

**定义**：软件实体应该对扩展开放，对修改关闭。

```cpp
// ✗ 违反 OCP：添加新形状需要修改现有代码
class AreaCalculator {
public:
    double calculate(Shape* shape) {
        if (shape->type == ShapeType::Circle) {
            return 3.14 * shape->radius * shape->radius;
        } else if (shape->type == ShapeType::Rectangle) {
            return shape->width * shape->height;
        }
        // 添加新形状需要修改这里！
    }
};

// ✓ 遵循 OCP：通过虚函数扩展
class Shape {
public:
    virtual ~Shape() = default;
    virtual double area() const = 0;
};

class Circle : public Shape {
    double radius_;
public:
    Circle(double r) : radius_(r) {}
    double area() const override { return 3.14159 * radius_ * radius_; }
};

class Rectangle : public Shape {
    double width_, height_;
public:
    Rectangle(double w, double h) : width_(w), height_(h) {}
    double area() const override { return width_ * height_; }
};

// 添加新形状只需添加新类，无需修改现有代码
class Triangle : public Shape {
    double base_, height_;
public:
    Triangle(double b, double h) : base_(b), height_(h) {}
    double area() const override { return 0.5 * base_ * height_; }
};

double totalArea(const std::vector<std::unique_ptr<Shape>>& shapes) {
    double total = 0;
    for (const auto& s : shapes) {
        total += s->area();  // 多态调用
    }
    return total;
}
```

### 1.3 里氏替换原则（LSP）

**定义**：子类对象必须能够替换父类对象，且程序行为正确。

```cpp
// ✗ 违反 LSP：子类行为与父类契约不一致
class Rectangle {
protected:
    double width_, height_;
public:
    virtual void setWidth(double w) { width_ = w; }
    virtual void setHeight(double h) { height_ = h; }
    double area() const { return width_ * height_; }
};

class Square : public Rectangle {
public:
    void setWidth(double w) override {
        width_ = height_ = w;  // 正方形必须相等
    }
    void setHeight(double h) override {
        width_ = height_ = h;  // 正方形必须相等
    }
};

void test(Rectangle& r) {
    r.setWidth(5);
    r.setHeight(4);
    assert(r.area() == 20);  // Square 违反了这个契约！
}

// ✓ 遵循 LSP：不将 Square 作为 Rectangle 的子类
// 或使用不可变设计
class Shape {
public:
    virtual double area() const = 0;
    virtual ~Shape() = default;
};

class Rectangle : public Shape {
    const double width_, height_;
public:
    Rectangle(double w, double h) : width_(w), height_(h) {}
    double area() const override { return width_ * height_; }
};

class Square : public Shape {
    const double side_;
public:
    Square(double s) : side_(s) {}
    double area() const override { return side_ * side_; }
};
```

### 1.4 接口隔离原则（ISP）

**定义**：客户端不应该依赖它不使用的方法。

```cpp
// ✗ 违反 ISP：胖接口
class Worker {
public:
    virtual void work() = 0;
    virtual void eat() = 0;
    virtual void sleep() = 0;
};

class Robot : public Worker {
public:
    void work() override { /* ... */ }
    void eat() override { /* 不需要，但必须实现 */ }
    void sleep() override { /* 不需要，但必须实现 */ }
};

// ✓ 遵循 ISP：接口分离
class Workable {
public:
    virtual void work() = 0;
    virtual ~Workable() = default;
};

class Eatable {
public:
    virtual void eat() = 0;
    virtual ~Eatable() = default;
};

class Sleepable {
public:
    virtual void sleep() = 0;
    virtual ~Sleepable() = default;
};

class Human : public Workable, public Eatable, public Sleepable {
public:
    void work() override { /* ... */ }
    void eat() override { /* ... */ }
    void sleep() override { /* ... */ }
};

class Robot : public Workable {
public:
    void work() override { /* ... */ }
    // 只实现需要的接口
};
```

### 1.5 依赖倒置原则（DIP）

**定义**：高层模块不应依赖低层模块，两者都应依赖抽象。

```cpp
// ✗ 违反 DIP：高层直接依赖低层
class LightBulb {
public:
    void turnOn() { /* ... */ }
    void turnOff() { /* ... */ }
};

class Switch {
    LightBulb& bulb_;
    bool on_ = false;
public:
    Switch(LightBulb& bulb) : bulb_(bulb) {}
    void toggle() {
        if (on_) bulb_.turnOff();
        else bulb_.turnOn();
        on_ = !on_;
    }
};

// ✓ 遵循 DIP：依赖抽象接口
class Switchable {
public:
    virtual void turnOn() = 0;
    virtual void turnOff() = 0;
    virtual ~Switchable() = default;
};

class LightBulb : public Switchable {
public:
    void turnOn() override { /* ... */ }
    void turnOff() override { /* ... */ }
};

class Fan : public Switchable {
public:
    void turnOn() override { /* ... */ }
    void turnOff() override { /* ... */ }
};

class Switch {
    Switchable& device_;  // 依赖抽象
    bool on_ = false;
public:
    Switch(Switchable& device) : device_(device) {}
    void toggle() {
        if (on_) device_.turnOff();
        else device_.turnOn();
        on_ = !on_;
    }
};
```

## 第二部分：Rule of Zero/Three/Five

### 2.1 Rule of Zero

**规则**：如果可能，让编译器生成所有特殊成员函数。

```cpp
// Rule of Zero：使用标准库组件自动管理资源
class User {
    std::string name_;
    std::vector<int> scores_;
public:
    User(const std::string& name) : name_(name) {}
    // 编译器自动生成：
    // - 析构函数：自动调用 name_ 和 scores_ 的析构
    // - 拷贝构造/赋值：深拷贝
    // - 移动构造/赋值：高效移动
};

User u1("Alice");
User u2 = u1;      // 拷贝构造
User u3 = std::move(u1);  // 移动构造
```

### 2.2 Rule of Three

**规则**：如果需要析构函数、拷贝构造或拷贝赋值中的任何一个，就需要全部三个。

```cpp
// Rule of Three：手动管理资源
class Buffer {
    int* data_;
    size_t size_;
public:
    Buffer(size_t size) : data_(new int[size]), size_(size) {}
    
    ~Buffer() { delete[] data_; }  // 1. 析构函数
    
    Buffer(const Buffer& other)    // 2. 拷贝构造
        : data_(new int[other.size_]), size_(other.size_) {
        std::copy(other.data_, other.data_ + size_, data_);
    }
    
    Buffer& operator=(const Buffer& other) {  // 3. 拷贝赋值
        if (this != &other) {
            delete[] data_;
            data_ = new int[other.size_];
            size_ = other.size_;
            std::copy(other.data_, other.data_ + size_, data_);
        }
        return *this;
    }
};
```

### 2.3 Rule of Five

**规则**：在现代 C++ 中，还需要考虑移动语义。

```cpp
// Rule of Five：完整版本
class Buffer {
    int* data_;
    size_t size_;
public:
    // 构造函数
    explicit Buffer(size_t size) : data_(new int[size]), size_(size) {}
    
    // 1. 析构函数
    ~Buffer() { delete[] data_; }
    
    // 2. 拷贝构造
    Buffer(const Buffer& other) 
        : data_(new int[other.size_]), size_(other.size_) {
        std::copy(other.data_, other.data_ + size_, data_);
    }
    
    // 3. 拷贝赋值
    Buffer& operator=(const Buffer& other) {
        if (this != &other) {
            Buffer temp(other);  // Copy-and-Swap 惯用法
            swap(*this, temp);
        }
        return *this;
    }
    
    // 4. 移动构造
    Buffer(Buffer&& other) noexcept 
        : data_(other.data_), size_(other.size_) {
        other.data_ = nullptr;
        other.size_ = 0;
    }
    
    // 5. 移动赋值
    Buffer& operator=(Buffer&& other) noexcept {
        if (this != &other) {
            delete[] data_;
            data_ = other.data_;
            size_ = other.size_;
            other.data_ = nullptr;
            other.size_ = 0;
        }
        return *this;
    }
    
    friend void swap(Buffer& a, Buffer& b) noexcept {
        std::swap(a.data_, b.data_);
        std::swap(a.size_, b.size_);
    }
};
```

## 第三部分：PImpl 惯用法

### 3.1 什么是 PImpl

**PImpl（Pointer to Implementation）**：将类的实现细节隐藏在另一个类中，通过指针访问。

```cpp
// ===== Widget.h =====
class Widget {
public:
    Widget();
    ~Widget();
    
    void doSomething();
    int getValue() const;
    
private:
    class Impl;  // 前向声明
    std::unique_ptr<Impl> pImpl_;  // 指向实现
};

// ===== Widget.cpp =====
#include "Widget.h"
#include <vector>
#include <string>

class Widget::Impl {
public:
    void doSomething() { /* 复杂实现 */ }
    int getValue() const { return value_; }
    
private:
    std::vector<int> data_;
    std::string name_;
    int value_ = 42;
    // 修改这些成员不需要重编 Widget.h
};

Widget::Widget() : pImpl_(std::make_unique<Impl>()) {}
Widget::~Widget() = default;  // 需要在 cpp 中定义

void Widget::doSomething() { pImpl_->doSomething(); }
int Widget::getValue() const { return pImpl_->getValue(); }
```

### 3.2 PImpl 的优势


### 3.3 PImpl 与特殊成员函数

```cpp
class Widget {
public:
    Widget();
    ~Widget();
    
    // 拷贝操作需要自定义（unique_ptr 不可拷贝）
    Widget(const Widget& other);
    Widget& operator=(const Widget& other);
    
    // 移动操作可以默认
    Widget(Widget&& other) noexcept = default;
    Widget& operator=(Widget&& other) noexcept = default;
    
private:
    class Impl;
    std::unique_ptr<Impl> pImpl_;
};

// 拷贝构造实现
Widget::Widget(const Widget& other) 
    : pImpl_(std::make_unique<Impl>(*other.pImpl_)) {}

Widget& Widget::operator=(const Widget& other) {
    if (this != &other) {
        *pImpl_ = *other.pImpl_;  // 深拷贝 Impl
    }
    return *this;
}
```

### 3.4 PImpl 的变体

```cpp
// 变体1：使用 shared_ptr（可拷贝）
class SharedWidget {
    struct Impl;
    std::shared_ptr<Impl> pImpl_;  // 自动支持拷贝
public:
    SharedWidget();
    // 拷贝语义：共享同一实现（注意语义）
};

// 变体2：使用裸指针（需要手动管理）
class RawWidget {
    struct Impl;
    Impl* pImpl_;  // 需要手动 delete
public:
    RawWidget();
    ~RawWidget();
    // 需要实现 Rule of Five
};

// 变体3：Fast PImpl（固定大小缓冲区）
template<size_t Size, size_t Align>
class FixedPImpl {
    alignas(Align) unsigned char buffer_[Size];
    Impl* get() { return reinterpret_cast<Impl*>(buffer_); }
public:
    FixedPImpl() { new (buffer_) Impl(); }
    ~FixedPImpl() { get()->~Impl(); }
    // 无堆分配，但大小需预先知道
};
```

## 第四部分：CRTP 奇异递归模板模式

### 4.1 CRTP 基本概念

**CRTP（Curiously Recurring Template Pattern）**：派生类作为基类的模板参数。

```cpp
// CRTP 基本结构
template<typename Derived>
class Base {
public:
    void interface() {
        // 调用派生类的实现
        static_cast<Derived*>(this)->implementation();
    }
    
    void baseMethod() {
        std::cout << "Base method\n";
    }
};

class Derived : public Base<Derived> {
public:
    void implementation() {
        std::cout << "Derived implementation\n";
    }
};

// 使用
Derived d;
d.interface();  // 调用 Derived::implementation
```

### 4.2 CRTP 实现静态多态

```cpp
// 静态多态：编译期确定调用，无虚函数开销
template<typename Derived>
class Shape {
public:
    double area() const {
        return static_cast<const Derived*>(this)->areaImpl();
    }
    
    void draw() const {
        static_cast<const Derived*>(this)->drawImpl();
    }
};

class Circle : public Shape<Circle> {
    double radius_;
public:
    Circle(double r) : radius_(r) {}
    
    double areaImpl() const { return 3.14159 * radius_ * radius_; }
    void drawImpl() const { std::cout << "Drawing Circle\n"; }
};

class Rectangle : public Shape<Rectangle> {
    double width_, height_;
public:
    Rectangle(double w, double h) : width_(w), height_(h) {}
    
    double areaImpl() const { return width_ * height_; }
    void drawImpl() const { std::cout << "Drawing Rectangle\n"; }
};

// 模板函数，编译期多态
template<typename T>
void printArea(const Shape<T>& shape) {
    std::cout << "Area: " << shape.area() << "\n";
}
```

### 4.3 CRTP 的常见应用

#### Mixin 风格

```cpp
// 为类添加功能
template<typename Derived>
class Comparable {
public:
    bool operator!=(const Derived& other) const {
        return !(*static_cast<const Derived*>(this) == other);
    }
    
    bool operator>(const Derived& other) const {
        return other < *static_cast<const Derived*>(this);
    }
    
    bool operator<=(const Derived& other) const {
        return !(other < *static_cast<const Derived*>(this));
    }
    
    bool operator>=(const Derived& other) const {
        return !(*static_cast<const Derived*>(this) < other);
    }
};

class Point : public Comparable<Point> {
    int x_, y_;
public:
    Point(int x, int y) : x_(x), y_(y) {}
    
    bool operator==(const Point& other) const {
        return x_ == other.x_ && y_ == other.y_;
    }
    
    bool operator<(const Point& other) const {
        return x_ < other.x_ || (x_ == other.x_ && y_ < other.y_);
    }
};

// 现在 Point 自动拥有 !=, >, <=, >= 运算符
```

#### 统计对象数量

```cpp
template<typename Derived>
class Counter {
    static inline int count_ = 0;
public:
    Counter() { ++count_; }
    Counter(const Counter&) { ++count_; }
    ~Counter() { --count_; }
    
    static int getCount() { return count_; }
};

class Widget : public Counter<Widget> {
    // ...
};

class Gadget : public Counter<Gadget> {
    // ...
};

Widget w1, w2;
std::cout << Widget::getCount();  // 2

```

```cpp
class FluentBuilder {
```

## 第五部分：其他重要惯用法

### 5.1 NVI（Non-Virtual Interface）模式

```cpp
class Widget {
public:
    // 非虚公共接口
    void process(int value) {
        validate(value);      // 前置检查
        doProcess(value);     // 调用实际实现
        logResult(value);     // 后置处理
    }
    
    int getResult() const { return result_; }
    
    virtual ~Widget() = default;
    
private:
    int result_ = 0;
    
    // 私有虚函数，由派生类实现
    virtual void doProcess(int value) = 0;
    
    void validate(int value) {
        if (value < 0) throw std::invalid_argument("negative value");
    }
    
    void logResult(int value) {
        std::cout << "Processed: " << value << " -> " << result_ << "\n";
    }
};

class ConcreteWidget : public Widget {
private:
    void doProcess(int value) override {
        // 只需实现核心逻辑
        // 前置检查和后置处理由基类保证
    }
};
```

### 5.2 Copy-and-Swap 惯用法

```cpp
class Resource {
    int* data_;
    size_t size_;
    
public:
    Resource(size_t size) : data_(new int[size]), size_(size) {}
    ~Resource() { delete[] data_; }
    
    // 拷贝构造
    Resource(const Resource& other) 
        : data_(new int[other.size_]), size_(other.size_) {
        std::copy(other.data_, other.data_ + size_, data_);
    }
    
    // 移动构造
    Resource(Resource&& other) noexcept 
        : data_(nullptr), size_(0) {
        swap(*this, other);
    }
    
    // 统一的赋值运算符（拷贝和移动）
    Resource& operator=(Resource other) noexcept {
        swap(*this, other);  // 交换资源
        return *this;        // other 析构时释放旧资源
    }
    
    friend void swap(Resource& a, Resource& b) noexcept {
        std::swap(a.data_, b.data_);
        std::swap(a.size_, b.size_);
    }
};

// 优点：
// 1. 代码复用：赋值运算符使用拷贝/移动构造
// 2. 异常安全：资源交换后才释放旧资源
// 3. 自赋值安全：通过拷贝临时对象实现
```

### 5.3 Attorney-Client 惯用法

```cpp
// 问题：友元关系是全部授予，无法部分授权
class Secret {
    int privateData_;
    
    friend class Friend;  // Friend 可访问所有私有成员
    
    void secretMethod1() { /* ... */ }
    void secretMethod2() { /* ... */ }
};

// 解决：使用 Attorney 类作为中介
class Secret {
private:
    int privateData_;
    void secretMethod1() { /* ... */ }
    void secretMethod2() { /* ... */ }
    
    // Attorney 类作为友元
    friend class SecretAttorney;
};

class SecretAttorney {
public:
    // 只暴露特定的方法
    static int getPrivateData(Secret& s) { return s.privateData_; }
    static void callSecretMethod1(Secret& s) { s.secretMethod1(); }
    // 不暴露 secretMethod2
};

class PartialFriend {
public:
    void accessSecret(Secret& s) {
        // 通过 Attorney 访问
        int data = SecretAttorney::getPrivateData(s);
        SecretAttorney::callSecretMethod1(s);
        // SecretAttorney::callSecretMethod2(s);  // 编译错误：无法访问
    }
};
```

### 5.4 强类型 typedef

```cpp
// 问题：typedef/using 不创建新类型
using Meter = double;
using Second = double;

void travel(Meter distance, Second time) {}

void test() {
    travel(100.0, 10.0);  // OK
    travel(10.0, 100.0);  // 也 OK！单位搞反了
}

// 解决：强类型包装
template<typename T, typename Tag>
class StrongType {
    T value_;
public:
    explicit StrongType(T value) : value_(std::move(value)) {}
    T& get() { return value_; }
    const T& get() const { return value_; }
};

struct MeterTag {};
struct SecondTag {};

using Meter = StrongType<double, MeterTag>;
using Second = StrongType<double, SecondTag>;

void travel(Meter distance, Second time) {}

void test() {
    travel(Meter(100.0), Second(10.0));  // OK
    // travel(Second(10.0), Meter(100.0));  // 编译错误！类型不匹配
}
```

### Top 3 高频考点

#### 1. Rule of Five

**Q：什么时候需要自定义析构函数？需要同时定义哪些函数？**

```cpp
// 需要自定义析构函数的情况：
// 1. 管理裸指针
// 2. 管理文件句柄、socket 等
// 3. 有自定义资源释放逻辑

// 如果定义了任何一个，应该定义全部五个：
// - 析构函数
// - 拷贝构造
// - 拷贝赋值
// - 移动构造
// - 移动赋值

// 最佳实践：优先使用 Rule of Zero
class Good {
    std::unique_ptr<int[]> data_;  // 自动管理
    // 无需自定义任何特殊成员函数
};
```

#### 2. PImpl 惯用法

**Q：什么是 PImpl？有什么优缺点？**

```cpp
// 优点：
// 1. 编译防火墙：修改实现无需重编客户端
// 2. ABI 稳定性：添加成员不改变头文件中的对象大小
// 3. 减少头文件依赖

// 1. 一次堆分配开销
```

**Q：CRTP 和虚函数有什么区别？**

|------|------|--------|
| 绑定时间 | 编译期 | 运行期 |
| 性能 | 无间接调用开销 | 间接调用开销 |
| 内联 | 可内联 | 不可内联 |
| 灵活性 | 需要知道类型 | 完全动态 |
| 代码膨胀 | 可能 | 无 |

## 常见陷阱

### 陷阱 1：移动构造函数未标记 noexcept

```cpp
class Container {
    std::vector<int> data_;
public:
    // ✗ 可能的问题：未标记 noexcept
    Container(Container&& other)  // 没有 noexcept
        : data_(std::move(other.data_)) {}
};

// 影响：std::vector::push_back 等操作会优先使用拷贝而非移动
// 因为移动构造不是 noexcept 的

// ✓ 修正
Container(Container&& other) noexcept
    : data_(std::move(other.data_)) {}
```

### 陷阱 2：PImpl 和析构函数

```cpp
// Widget.h
class Widget {
    class Impl;
    std::unique_ptr<Impl> pImpl_;
public:
    ~Widget() = default;  // 错误！在头文件中定义
    // 编译错误：Impl 是不完整类型
};

// 修正：在 cpp 中定义析构函数
// Widget.cpp
Widget::~Widget() = default;  // 此时 Impl 是完整类型
```

### 陷阱 3：CRTP 中的 this 指针

```cpp
template<typename Derived>
class Base {
public:
    void foo() {
        // 危险：如果 Derived 不是 Base<Derived> 的子类
        static_cast<Derived*>(this)->bar();
    }
};

class Wrong : public Base<int> {  // 错误的模板参数
public:
    void bar() {}
};

Wrong w;
w.foo();  // UB：将 Wrong* 转换为 int*
```

### 陷阱 4：拷贝赋值的自赋值

```cpp
class Bad {
    int* data_;
public:
    Bad& operator=(const Bad& other) {
        delete data_;  // 先删除
        data_ = new int(*other.data_);  // 如果 this == &other，other.data_ 已失效！
        return *this;
    }
};

// 修正：Copy-and-Swap
class Good {
    int* data_;
public:
    Good& operator=(Good other) {  // 按值传递
        swap(data_, other.data_);
        return *this;
    }
};
```
