# 移动语义与右值引用详细解析

> **核心结论**：移动语义不是"移动数据"，而是**转移资源所有权**。`std::move` 本质是 static_cast 到右值引用，真正的移动发生在移动构造函数中。理解值类别（value categories）是掌握移动语义的基石。

---

## 1. Why — 为什么需要移动语义

### 1.1 传统拷贝的痛点

```cpp
// C++98/03：昂贵的拷贝操作
std::vector<std::string> createLargeVector() {
    std::vector<std::string> result;
    for (int i = 0; i < 100000; ++i) {
        result.push_back(std::string(1000, 'x'));  // 每个字符串1KB
    }
    return result;  // 返回时发生深拷贝！O(n) 复杂度
}

// 调用方
std::vector<std::string> data = createLargeVector();  // 拷贝构造，性能灾难
```

**问题分析**：
- 返回局部变量时触发拷贝构造函数
- 100MB 数据的深拷贝，时间复杂度 O(n)
- 实际上 `result` 即将销毁，我们只需要转移其资源所有权

### 1.2 移动语义的价值

```cpp
// C++11 起：移动而非拷贝
std::vector<std::string> data = createLargeVector();  // 移动构造，O(1) 复杂度
```

**移动的本质**：
- 源对象的资源指针直接赋给目标对象
- 源对象的指针置空（进入"有效但未指定状态"）
- 时间复杂度 O(1)，不涉及堆内存分配

## 2. What — 值类别（Value Categories）

### 2.1 值类别的完整分类

C++11 引入了精确的值类别系统：

```
                      表达式
                        │
            ┌───────────┴───────────┐
            │                       │
         glvalue                  rvalue
      (泛左值)                  (右值)
            │                       │
    ┌───────┴───────┐       ┌───────┴───────┐
    │               │       │               │
 lvalue          xvalue   prvalue       xvalue
(传统左值)      (将亡值)   (纯右值)      (将亡值)
```

**分类标准**：

| 类别 | 定义 | 特征 | 示例 |
|------|------|------|------|
| **lvalue** | 有身份（identity），不可移动 | 可取地址，可赋值 | 变量名、解引用 `*p`、函数返回左值引用 |
| **prvalue** | 无身份，可移动 | 临时对象，字面量 | 字面量 `42`、函数返回非引用类型、`a + b` |
| **xvalue** | 有身份，可移动 | 即将消亡的资源 | `std::move(x)`、返回右值引用的函数 |
| **glvalue** | 有身份（lvalue + xvalue） | 泛左值 | lvalue 和 xvalue 的并集 |
| **rvalue** | 可移动（prvalue + xvalue） | 右值 | prvalue 和 xvalue 的并集 |

### 2.2 值类别判定表

```cpp
// 完整示例：判断每个表达式的值类别

int global = 10;
int getValue() { return 42; }
int& getLValueRef() { return global; }
int&& getRValueRef() { return std::move(global); }

struct Widget {
    Widget operator+(const Widget&) const;
};

void testValueCategories() {
    int x = 10;
    int* p = &x;
    Widget w1, w2;
    
    // lvalue 示例
    x;              // lvalue：有身份的变量
    *p;             // lvalue：解引用结果
    getLValueRef(); // lvalue：返回左值引用
    "hello";        // lvalue：字符串字面量是数组，退化为指针
    
    // prvalue 示例
    42;             // prvalue：纯右值字面量
    x + 1;          // prvalue：内置运算符返回临时值
    getValue();     // prvalue：返回非引用类型
    w1 + w2;        // prvalue：运算符重载返回对象
    Widget();       // prvalue：临时对象
    
    // xvalue 示例
    std::move(x);   // xvalue：将亡值
    getRValueRef(); // xvalue：返回右值引用
    static_cast<int&&>(x);  // xvalue：强制转换为右值引用
}
```

### 2.3 值类别与引用绑定规则

```cpp
// 引用绑定规则表

void testBinding() {
    int x = 10;
    
    // 左值引用（只能绑定到 lvalue）
    int& lref1 = x;              // OK: lvalue 绑定到左值引用
    // int& lref2 = 42;          // ERROR: prvalue 不能绑定到左值引用
    // int& lref3 = std::move(x); // ERROR: xvalue 不能绑定到左值引用
    
    // const 左值引用（万能引用，可绑定到任何值类别）
    const int& clref1 = x;              // OK
    const int& clref2 = 42;             // OK: prvalue 可绑定到 const 左值引用
    const int& clref3 = std::move(x);   // OK: xvalue 可绑定到 const 左值引用
    
    // 右值引用（只能绑定到 rvalue：prvalue 或 xvalue）
    // int&& rref1 = x;          // ERROR: lvalue 不能绑定到右值引用
    int&& rref2 = 42;            // OK: prvalue 绑定到右值引用
    int&& rref3 = std::move(x);  // OK: xvalue 绑定到右值引用
    
    // 注意：右值引用变量本身是 lvalue！
    int&& rref4 = 42;
    // int&& rref5 = rref4;      // ERROR: rref4 是 lvalue
    int&& rref5 = std::move(rref4);  // OK: 需要再次 move
}
```

## 3. How — 右值引用与移动语义实现

### 3.1 右值引用语法

```cpp
// 右值引用：类型 &&
int&& rref = 42;  // 绑定到临时值

// 关键洞察：右值引用变量本身是左值！
void process(int&& val) {
    // 在函数内部，val 是 lvalue（有名字）
    // 如果要保持其右值属性，需要 std::move(val)
}
```

### 3.2 std::move 的本质

```cpp
// std::move 的标准实现（概念版）
template<typename T>
constexpr typename std::remove_reference<T>::type&& 
move(T&& t) noexcept {
    return static_cast<typename std::remove_reference<T>::type&&>(t);
}

// C++14 简化版
template<typename T>
constexpr std::remove_reference_t<T>&& move(T&& t) noexcept {
    return static_cast<std::remove_reference_t<T>&&>(t);
}
```

**核心要点**：
- `std::move` **不移动任何东西**，只是类型转换
- 将任何值转换为 xvalue（将亡值）
- 转换后对象可以绑定到右值引用
- 被 move 的对象进入"有效但未指定状态"

```cpp
// std::move 使用示例
#include <iostream>
#include <string>
#include <vector>

void demonstrateMove() {
    std::string str = "Hello, World!";
    std::cout << "Before move:" << std::endl;
    std::cout << "  str = \"" << str << "\"" << std::endl;
    std::cout << "  str.size() = " << str.size() << std::endl;
    std::cout << "  str.capacity() = " << str.capacity() << std::endl;
    
    // std::move 只是类型转换
    std::string&& rref = std::move(str);  // rref 绑定到 str 的 xvalue
    
    // 注意：str 本身还没有被修改！
    std::cout << "\nAfter std::move (but before actual move):" << std::endl;
    std::cout << "  str = \"" << str << "\"" << std::endl;  // 仍然有效
    
    // 真正的移动发生在移动构造函数
    std::string newStr = std::move(str);  // 调用移动构造函数
    
    std::cout << "\nAfter move construction:" << std::endl;
    std::cout << "  newStr = \"" << newStr << "\"" << std::endl;
    std::cout << "  str = \"" << str << "\"" << std::endl;  // 有效但未指定
    std::cout << "  str.size() = " << str.size() << std::endl;
}
```

### 3.3 移动构造函数与移动赋值运算符

```cpp
// 自定义类的移动语义实现
#include <iostream>
#include <cstring>
#include <utility>

class Buffer {
private:
    char* data_;
    size_t size_;
    
public:
    // 默认构造函数
    Buffer() : data_(nullptr), size_(0) {
        std::cout << "Default constructor" << std::endl;
    }
    
    // 带参数构造函数
    explicit Buffer(size_t size) : size_(size) {
        std::cout << "Constructor(size_t)" << std::endl;
        data_ = new char[size];
    }
    
    // 析构函数
    ~Buffer() {
        std::cout << "Destructor" << std::endl;
        delete[] data_;
    }
    
    // 拷贝构造函数（深拷贝）
    Buffer(const Buffer& other) : size_(other.size_) {
        std::cout << "Copy constructor" << std::endl;
        if (other.data_) {
            data_ = new char[size_];
            std::memcpy(data_, other.data_, size_);
        } else {
            data_ = nullptr;
        }
    }
    
    // 拷贝赋值运算符
    Buffer& operator=(const Buffer& other) {
        std::cout << "Copy assignment" << std::endl;
        if (this != &other) {
            delete[] data_;
            size_ = other.size_;
            if (other.data_) {
                data_ = new char[size_];
                std::memcpy(data_, other.data_, size_);
            } else {
                data_ = nullptr;
            }
        }
        return *this;
    }
    
    // ===== 移动构造函数（核心）=====
    Buffer(Buffer&& other) noexcept 
        : data_(other.data_), size_(other.size_) {
        std::cout << "Move constructor" << std::endl;
        // 窃取资源
        other.data_ = nullptr;  // 置空源对象
        other.size_ = 0;
    }
    
    // ===== 移动赋值运算符（核心）=====
    Buffer& operator=(Buffer&& other) noexcept {
        std::cout << "Move assignment" << std::endl;
        if (this != &other) {
            delete[] data_;  // 释放当前资源
            
            // 窃取资源
            data_ = other.data_;
            size_ = other.size_;
            
            // 置空源对象
            other.data_ = nullptr;
            other.size_ = 0;
        }
        return *this;
    }
    
    size_t size() const { return size_; }
    bool empty() const { return data_ == nullptr; }
};

// 使用示例
void testMoveSemantics() {
    std::cout << "=== Creating buffer ===" << std::endl;
    Buffer buf1(1024);
    
    std::cout << "\n=== Move construction ===" << std::endl;
    Buffer buf2 = std::move(buf1);  // 移动构造
    // buf1 现在处于"有效但未指定状态"
    
    std::cout << "\n=== Creating another buffer ===" << std::endl;
    Buffer buf3(512);
    
    std::cout << "\n=== Move assignment ===" << std::endl;
    buf3 = std::move(buf2);  // 移动赋值
}
```

### 3.4 noexcept 与移动语义

```cpp
// noexcept 对移动语义至关重要

class SafeBuffer {
public:
    // 强异常安全保证的移动构造函数
    SafeBuffer(SafeBuffer&& other) noexcept  // 必须标记 noexcept
        : data_(other.data_), size_(other.size_) {
        other.data_ = nullptr;
        other.size_ = 0;
    }
    
    // 如果移动构造函数不 noexcept，vector 会退化为拷贝
};

// 标准库容器的行为
void testNoexcept() {
    std::vector<SafeBuffer> vec;
    vec.reserve(10);
    
    for (int i = 0; i < 5; ++i) {
        SafeBuffer buf(100);
        vec.push_back(std::move(buf));  // 如果移动是 noexcept，使用移动
    }
    
    // 当 vector 重新分配时
    vec.reserve(100);  // 如果移动构造函数是 noexcept，所有元素移动
                       // 否则，所有元素拷贝！
}
```

**为什么需要 noexcept**：
- 标准库容器在重新分配时优先使用 `noexcept` 移动
- 如果移动可能抛异常，容器会选择拷贝以保证强异常安全
- 移动操作通常只是指针交换，天然不抛异常

## 4. 完美转发与万能引用

### 4.1 万能引用（Forwarding Reference）

```cpp
// 万能引用：T&& 在模板推导上下文中

template<typename T>
void func(T&& param);  // 这是万能引用，不是右值引用

// 万能引用的识别规则
void test() {
    int x = 10;
    
    func(42);       // T 推导为 int，param 类型为 int&&（右值引用）
    func(x);        // T 推导为 int&，param 类型为 int&（左值引用）
    
    // 引用折叠：int& + && = int&
}
```

**万能引用的两个条件**：
1. 必须是模板参数 `T`
2. 形式必须是 `T&&`（注意空格：`const T&&` 不是万能引用）

```cpp
// 不是万能引用的情况

template<typename T>
void func1(std::vector<T>&& param);  // 右值引用，不是万能引用

template<typename T>
void func2(const T&& param);          // 右值引用，不是万能引用

void func3(int&& param);              // 右值引用，不是万能引用（非模板）

template<typename T>
class Widget {
    void func(T&& param);  // 右值引用！不是万能引用（在类模板中）
};
```

### 4.2 引用折叠规则

```cpp
// 引用折叠规则（Reference Collapsing）

// 在模板实例化时，编译器应用以下规则：
// &  +  &  = &   （左值引用 + 左值引用 = 左值引用）
// &  +  && = &   （左值引用 + 右值引用 = 左值引用）
// && +  &  = &   （右值引用 + 左值引用 = 左值引用）
// && +  && = &&  （右值引用 + 右值引用 = 右值引用）

// 总结：只要有左值引用参与，结果就是左值引用

template<typename T>
void forwarder(T&& param) {
    // param 是万能引用
    // 如果传入左值，T = int&, param = int&
    // 如果传入右值，T = int,  param = int&&
}
```

### 4.3 std::forward 实现完美转发

```cpp
// std::forward 的标准实现
template<typename T>
T&& forward(typename std::remove_reference<T>::type& param) noexcept {
    return static_cast<T&&>(param);
}

template<typename T>
T&& forward(typename std::remove_reference<T>::type&& param) noexcept {
    return static_cast<T&&>(param);
}

// 完美转发的使用
#include <iostream>
#include <utility>

void process(int& x) {
    std::cout << "lvalue: " << x << std::endl;
}

void process(int&& x) {
    std::cout << "rvalue: " << x << std::endl;
}

void process(const int& x) {
    std::cout << "const lvalue: " << x << std::endl;
}

// 不完美转发：丢失值类别信息
template<typename T>
void imperfectForward(T param) {  // 按值传递，总是拷贝
    process(param);  // param 总是 lvalue
}

// 完美转发：保持值类别
template<typename T>
void perfectForward(T&& param) {
    process(std::forward<T>(param));  // 保持原始值类别
}

void testForwarding() {
    int x = 10;
    const int cx = 20;
    
    std::cout << "=== Perfect Forwarding ===" << std::endl;
    perfectForward(42);      // 调用 process(int&&) - rvalue
    perfectForward(x);       // 调用 process(int&) - lvalue
    perfectForward(cx);      // 调用 process(const int&) - const lvalue
    perfectForward(std::move(x));  // 调用 process(int&&) - xvalue
}
```

### 4.4 完美转发的实际应用

```cpp
// 工厂函数：完美转发的经典应用
#include <memory>
#include <utility>

class Widget {
public:
    Widget(int x, double y, const std::string& s) {
        // 构造函数实现
    }
};

// 不完美版本：需要多个重载
template<typename T>
std::shared_ptr<T> makeShared1() {
    return std::shared_ptr<T>(new T());  // 两次分配
}

// C++11 完美转发版本
template<typename T, typename... Args>
std::shared_ptr<T> makeShared(Args&&... args) {
    return std::shared_ptr<T>(new T(std::forward<Args>(args)...));
}

// C++14 更优版本
template<typename T, typename... Args>
std::shared_ptr<T> makeSharedCpp14(Args&&... args) {
    return std::make_shared<T>(std::forward<Args>(args)...);  // 一次分配
}

// 使用示例
void testFactory() {
    auto w1 = makeShared<Widget>(10, 3.14, "hello");
    
    std::string name = "world";
    auto w2 = makeShared<Widget>(20, 2.71, name);  // 左值，拷贝
    auto w3 = makeShared<Widget>(30, 1.41, std::move(name));  // 右值，移动
}
```

### 5.2 emplace 与移动语义

```cpp
#include <vector>
#include <string>

class Person {
    std::string name_;
    int age_;
public:
    Person(std::string name, int age) 
        : name_(std::move(name)), age_(age) {
        std::cout << "Constructor" << std::endl;
    }
    
    Person(const Person&) { std::cout << "Copy" << std::endl; }
    Person(Person&&) noexcept { std::cout << "Move" << std::endl; }
};

void testEmplace() {
    std::vector<Person> people;
    people.reserve(10);
    
    std::string name = "Alice";
    
    // push_back：先构造临时对象，再移动
    people.push_back(Person(name, 25));  // Constructor + Move
    
    // emplace_back：原地构造，无移动
    people.emplace_back(name, 30);  // Constructor only
    
    // emplace_back 配合 std::move
    people.emplace_back(std::move(name), 35);  // 移动 name，原地构造
}
```

## 6. RVO/NRVO 与 C++17 Guaranteed Copy Elision

### 6.1 返回值优化（RVO）

```cpp
#include <iostream>

class Expensive {
public:
    Expensive() { std::cout << "Default" << std::endl; }
    Expensive(const Expensive&) { std::cout << "Copy" << std::endl; }
    Expensive(Expensive&&) noexcept { std::cout << "Move" << std::endl; }
};

// 具名返回值优化（NRVO）
Expensive createNamed() {
    Expensive e;       // 默认构造
    return e;          // NRVO：直接在调用方内存构造，无拷贝/移动
}

// 无名返回值优化（RVO）
Expensive createUnnamed() {
    return Expensive();  // RVO：直接在调用方内存构造
}

void testRVO() {
    std::cout << "=== NRVO ===" << std::endl;
    Expensive e1 = createNamed();  // 可能只输出 "Default"
    
    std::cout << "\n=== RVO ===" << std::endl;
    Expensive e2 = createUnnamed();  // 只输出 "Default"
}
```

### 6.2 C++17 Guaranteed Copy Elision

```cpp
// C++17 前：RVO 是可选优化，可能仍有移动操作
// C++17 起：prvalue 的实质化规则改变，保证省略拷贝/移动

class NoCopy {
public:
    NoCopy() = default;
    NoCopy(const NoCopy&) = delete;  // 删除拷贝构造
    NoCopy(NoCopy&&) = delete;       // 删除移动构造
};

// C++17 前：编译错误（需要可访问的拷贝/移动构造）
// C++17 起：OK，保证不调用任何构造
NoCopy createNoCopy() {
    return NoCopy();  // prvalue 直接实质化到目标位置
}

void testGuaranteedElision() {
    NoCopy nc = createNoCopy();  // C++17 起 OK
}
```

## 7. 面试要点

### 7.1 高频面试题

#### Q1: std::move 会移动对象吗？移动后对象的状态是什么？

**答案**：
- `std::move` **不会移动任何东西**，它只是将参数转换为 xvalue（将亡值）
- 真正的移动发生在移动构造函数或移动赋值运算符中
- 移动后对象进入"有效但未指定状态"（valid but unspecified state）
- 可以安全地析构、重新赋值，但不能依赖其具体值

```cpp
std::string s1 = "hello";
std::string s2 = std::move(s1);  // s1 现在是"有效但未指定"

// s1 可能为空，也可能不为空，取决于实现
// 但可以安全地：
s1.clear();      // OK
s1 = "new";      // OK
// 不要这样做：
// if (s1.empty())  // 未指定，结果不确定
```

#### Q2: 什么是万能引用？如何识别？

**答案**：
- 万能引用（Universal Reference）是 Scott Meyers 提出的术语
- 形式为 `T&&`，其中 `T` 是模板推导参数
- 可以绑定到左值或右值，保持值类别信息
- 不是万能引用的情况：`const T&&`、`std::vector<T>&&`、类模板中的 `T&&`

```cpp
template<typename T>
void func(T&& param);  // 万能引用

template<typename T>
void func(const T&& param);  // 不是万能引用（右值引用）

template<typename T>
void func(std::vector<T>&& param);  // 不是万能引用
```

#### Q3: 完美转发失败有哪些场景？

**答案**：

1. **大括号初始化列表**：
```cpp
template<typename T>
void func(T&& param);

func({1, 2, 3});  // 错误：无法推导 T
```

2. **0 或 NULL 作为空指针**：
```cpp
void func(int*);
void func(int);

template<typename T>
void forwarder(T&& param) {
    func(std::forward<T>(param));
}

forwarder(0);   // 调用 func(int)，不是 func(int*)
forwarder(NULL); // 同样问题
```

3. **重载的函数名或模板函数名**：
```cpp
void func(int);
void func(double);

forwarder(func);  // 错误：无法确定重载版本
```

4. **位域**：
```cpp
struct S { int x : 4; };
S s;
forwarder(s.x);  // 错误：无法绑定非 const 引用到位域
```

#### Q4: 为什么移动构造函数要标记 noexcept？

**答案**：
- 标准库容器在重新分配时优先使用 `noexcept` 移动操作
- 如果移动可能抛异常，容器会选择拷贝以保证强异常安全
- 移动操作通常只是指针交换，天然不抛异常，应该标记 `noexcept`

```cpp
// 如果移动构造函数不是 noexcept
std::vector<Widget> vec;
vec.reserve(1000);
// 添加元素...
vec.reserve(2000);  // 重新分配
// 如果 Widget 的移动不是 noexcept，所有元素被拷贝而非移动！
```

#### Q5: 解释 C++17 Guaranteed Copy Elision

**答案**：
- C++17 前：RVO/NRVO 是编译器优化，可选，可能仍有移动操作
- C++17 起：prvalue 的实质化规则改变，保证省略拷贝/移动
- 即使类型删除了拷贝/移动构造函数，返回 prvalue 也能编译通过
- 实质化点延迟到目标位置，直接在该位置构造对象

## 8. 最佳实践

### 8.1 移动语义使用规范

```cpp
// 1. 为资源管理类实现移动操作
class Resource {
public:
    Resource(Resource&& other) noexcept 
        : ptr_(other.ptr_), size_(other.size_) {
        other.ptr_ = nullptr;
        other.size_ = 0;
    }
    
    Resource& operator=(Resource&& other) noexcept {
        if (this != &other) {
            delete ptr_;
            ptr_ = other.ptr_;
            size_ = other.size_;
            other.ptr_ = nullptr;
            other.size_ = 0;
        }
        return *this;
    }
};

// 2. 移动构造函数必须标记 noexcept
class Widget {
public:
    Widget(Widget&&) noexcept = default;  // 显式标记
};

// 3. 使用 std::move 表明意图
void process(std::string str);

std::string s = "hello";
process(std::move(s));  // 明确表示转移所有权

// 4. 不要返回局部变量的 std::move
Widget createWidget() {
    Widget w;
    // ...
    return w;           // OK：NRVO
    // return std::move(w);  // 错误：阻止 NRVO！
}
```

### 8.2 完美转发最佳实践

```cpp
// 1. 使用万能引用 + std::forward
template<typename T>
void wrapper(T&& param) {
    process(std::forward<T>(param));
}

// 2. 可变参数模板转发
template<typename... Args>
auto make_unique(Args&&... args) {
    return std::unique_ptr<T>(new T(std::forward<Args>(args)...));
}

// 3. 避免转发引用重载（导致问题）
// 不要这样做：
template<typename T>
void func(T&& param);  // 万能引用

void func(int param);  // 重载

func(42);  // 调用哪个？可能出乎意料
```

## 9. 常见陷阱与 UB

### 9.1 移动后使用源对象

```cpp
// 陷阱：移动后继续使用源对象
std::vector<int> v1 = {1, 2, 3};
std::vector<int> v2 = std::move(v1);

// 危险：v1 处于"有效但未指定状态"
v1.push_back(4);  // 技术上安全，但逻辑上可能混乱

// 安全做法：移动后立即重置或离开作用域
std::vector<int> v3 = std::move(v1);
v1 = std::vector<int>();  // 明确重置
```

### 9.2 自移动赋值

```cpp
// 陷阱：自移动赋值
Buffer buf(1024);
buf = std::move(buf);  // 自赋值！

// 正确的移动赋值实现需要检查
Buffer& operator=(Buffer&& other) noexcept {
    if (this != &other) {  // 必须检查自赋值
        delete[] data_;
        data_ = other.data_;
        size_ = other.size_;
        other.data_ = nullptr;
        other.size_ = 0;
    }
    return *this;
}
```

### 9.3 返回 std::move 局部变量

```cpp
// 错误：阻止 NRVO
std::string createString() {
    std::string s = "hello";
    return std::move(s);  // 不要这样做！
}

// 正确：让编译器优化
std::string createString() {
    std::string s = "hello";
    return s;  // NRVO 或移动
}
```

### 9.4 右值引用变量是左值

```cpp
// 陷阱
void process(std::string&& s);

std::string&& rref = getString();  // rref 是左值！
process(rref);  // 错误：rref 是 lvalue

// 正确
process(std::move(rref));  // 需要再次 move
```

### 10.1 移动 vs 拷贝性能对比

|------|-----------|---------|---------|

| 移动构造 | O(1) | 不需要 | 转移所有权 |
| 拷贝赋值 | O(n) | 可能需要 | 需要独立副本 |

### 10.2 实际性能测试

```cpp
#include <chrono>
#include <iostream>
#include <vector>
#include <string>

template<typename Func>
auto measure(Func&& f) {
    auto start = std::chrono::high_resolution_clock::now();
    f();
    auto end = std::chrono::high_resolution_clock::now();
    return std::chrono::duration_cast<std::chrono::microseconds>(end - start).count();
}

void performanceTest() {
    const int N = 100000;
    
    // 测试拷贝
    auto copyTime = measure([&]() {
        for (int i = 0; i < N; ++i) {
            std::string s1(1000, 'x');
            std::string s2 = s1;  // 拷贝
        }
    });
    
    // 测试移动
    auto moveTime = measure([&]() {
        for (int i = 0; i < N; ++i) {
            std::string s1(1000, 'x');
            std::string s2 = std::move(s1);  // 移动
        }
    });
    
    std::cout << "Copy: " << copyTime << " us" << std::endl;
    std::cout << "Move: " << moveTime << " us" << std::endl;
    std::cout << "Speedup: " << (double)copyTime / moveTime << "x" << std::endl;
    // 典型结果：移动比拷贝快 10-100 倍
}
```

## 11. 总结

### 核心要点回顾

1. **值类别**：理解 lvalue、prvalue、xvalue 的区别是掌握移动语义的基础
2. **std::move**：只是类型转换，真正的移动在移动构造函数中发生
3. **万能引用**：`T&&` 在模板推导中可以绑定到左值或右值
4. **完美转发**：`std::forward<T>` 保持原始值类别，配合万能引用使用
5. **noexcept**：移动操作应标记 noexcept，否则容器会退化为拷贝
6. **C++17**：Guaranteed Copy Elision 保证省略拷贝/移动

### 学习路径

```
值类别 → 右值引用 → std::move → 移动构造/赋值 → 万能引用 → std::forward → 完美转发
```

掌握移动语义是编写高效现代 C++ 代码的关键技能。
