## 核心结论

**C++ 类型系统的核心价值在于：编译期类型检查 + 零开销抽象。**

1. **静态类型**：每个变量的类型在编译期确定，编译器帮助捕获类型错误
2. **auto/decltype**：让编译器推导类型，减少冗余代码，同时保持类型安全
3. **类型转换**：四种具名转换比 C 风格转换更安全、更明确、更易于搜索
4. **CTAD**：C++17 进一步简化模板类型指定，提升代码简洁性

## 第一部分：基本类型

### 1.1 整型家族

C++ 整型按**符号**和**大小**两个维度分类：

```
┌──────────────────────────────────────────────────────────────────────┐
│                         C++ 整型分类                                 │
├──────────────────────────────────────────────────────────────────────┤
│                                                                      │
│   按符号分类：                                                       │
│   ├─ signed   ：有符号，可表示负数                                   │
│   └─ unsigned ：无符号，仅非负数，范围翻倍                           │
│                                                                      │
│   按大小分类（从小到大）：                                           │
│   ├─ char        ：至少 8 位，通常 1 字节                            │
│   ├─ short       ：至少 16 位，通常 2 字节                           │
│   ├─ int         ：至少 16 位，通常 4 字节                           │
│   ├─ long        ：至少 32 位，32位系统 4 字节，64位 Linux 8 字节    │
│   ├─ long long   ：至少 64 位，通常 8 字节                           │
│   └─ __int128    ：扩展类型，128 位（GCC/Clang 扩展）                │
│                                                                      │
│   标准保证的大小关系：                                               │
│   1 == sizeof(char) <= sizeof(short) <= sizeof(int)                 │
│       <= sizeof(long) <= sizeof(long long)                          │
│                                                                      │
└──────────────────────────────────────────────────────────────────────┘
```

**平台差异速查表**：

| 类型 | 32位系统 | 64位 Linux | 64位 Windows | 标准保证 |
|------|----------|------------|--------------|----------|
| `char` | 1 | 1 | 1 | 1 |
| `short` | 2 | 2 | 2 | ≥ 2 |
| `int` | 4 | 4 | 4 | ≥ 2 |
| `long` | 4 | 8 | 4 | ≥ 4 |
| `long long` | 8 | 8 | 8 | ≥ 8 |
| 指针 | 4 | 8 | 8 | - |

**关键要点**：
- `char` 的大小永远是 1 字节（`sizeof(char) == 1`）
- `long` 在 64 位 Windows/Linux 上的大小不同！跨平台代码需注意
- 使用 `<cstdint>` 获取固定大小类型：`int32_t`, `int64_t`, `uintptr_t`

```cpp
#include <cstdint>

// 固定大小整型（跨平台推荐）
int32_t  i32 = 42;      // 保证 32 位
int64_t  i64 = 42LL;    // 保证 64 位
uint64_t u64 = 42ULL;   // 保证 64 位无符号

// 指针大小整型
uintptr_t ptr_val = reinterpret_cast<uintptr_t>(&i32);  // 存放指针值
```

### 1.2 浮点型

| 类型 | 大小（典型） | 精度 | 范围 |
|------|-------------|------|------|
| `float` | 4 字节 | ~7 位有效数字 | ±3.4e38 |
| `double` | 8 字节 | ~15 位有效数字 | ±1.7e308 |
| `long double` | 8-16 字节 | 平台相关 | 平台相关 |

**关键要点**：
- 默认浮点字面量是 `double`：`3.14` 是 `double`，`3.14f` 才是 `float`
- 比较 float/double 时避免 `==`，使用 epsilon 比较法
- `long double` 在不同平台实现不同（x86 是 80-bit 扩展精度，ARM 可能与 double 相同）

```cpp
#include <cmath>
#include <limits>

// 浮点数比较的正确方式
bool nearly_equal(double a, double b, 
                  double epsilon = std::numeric_limits<double>::epsilon() * 100) {
    return std::abs(a - b) < epsilon;
}

// 或使用相对误差
bool nearly_equal_relative(double a, double b, double rel_tol = 1e-9) {
    return std::abs(a - b) <= rel_tol * std::max(std::abs(a), std::abs(b));
}
```

### 1.3 字符型

C++ 提供五种字符类型：

| 类型 | 大小 | 用途 |
|------|------|------|
| `char` | 1 字节 | ASCII/UTF-8 单字节，符号由实现定义 |
| `wchar_t` | 平台相关 | 宽字符（Windows 2 字节，Linux 4 字节） |
| `char16_t` | 2 字节 | UTF-16 编码单元（C++11） |
| `char32_t` | 4 字节 | UTF-32 编码单元（C++11） |
| `char8_t` | 1 字节 | UTF-8 编码单元（C++20） |

**关键要点**：
- `char` 的符号性由实现定义：可能是 `signed char` 或 `unsigned char`
- 如需确定符号，显式使用 `signed char` 或 `unsigned char`
- `wchar_t` 跨平台兼容性差，推荐使用 `char8_t/char16_t/char32_t`

```cpp
// char 符号性测试
char c = '\x80';           // 可能是 -128（signed）或 128（unsigned）
signed char sc = '\x80';   // 保证是 -128
unsigned char uc = '\x80'; // 保证是 128

// 平台检测
#if CHAR_MIN == 0
    // char 是无符号的
#else
    // char 是有符号的
#endif
```

### 1.4 void 与 std::nullptr_t

```cpp
// void：不完全类型，不能创建对象
void v;              // 编译错误
void* ptr;           // OK：可以声明 void 指针

// std::nullptr_t：空指针字面量 nullptr 的类型
std::nullptr_t null = nullptr;
int* p = nullptr;    // 隐式转换为任意指针类型

// nullptr vs NULL/0
void f(int);
void f(int*);

f(0);        // 调用 f(int)
f(NULL);     // 可能调用 f(int) 或 f(int*)，取决于 NULL 定义
f(nullptr);  // 一定调用 f(int*)
```

## 第二部分：auto 类型推导

### 2.1 auto 基本用法

`auto` 让编译器根据初始化表达式推导变量类型：

```cpp
auto x = 42;           // int
auto y = 3.14;         // double
auto z = 42LL;         // long long
auto s = "hello";      // const char*
auto str = std::string("hello");  // std::string

// 引用与 const
int a = 42;
auto& ref = a;         // int&
const auto& cref = a;  // const int&
auto* ptr = &a;        // int*
```

### 2.2 auto 推导规则

auto 的类型推导与**模板实参推导**几乎相同：

```
┌──────────────────────────────────────────────────────────────────────┐
│                      auto 推导规则速查                               │
├──────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  规则1：忽略初始化表达式的顶层 const/volatile                        │
│  ┌────────────────────────────────────────────────────────────┐     │
│  │ const int ci = 42;                                          │     │
│  │ auto x = ci;      // int（顶层 const 被忽略）               │     │
│  │ const auto cx = ci; // const int（显式添加 const）          │     │
│  └────────────────────────────────────────────────────────────┘     │
│                                                                      │
│  规则2：忽略引用（除非显式声明 auto&）                               │
│  ┌────────────────────────────────────────────────────────────┐     │
│  │ int& ref = x;                                               │     │
│  │ auto a = ref;     // int（引用被忽略）                       │     │
│  │ auto& ar = ref;   // int&（显式声明引用）                    │     │
│  └────────────────────────────────────────────────────────────┘     │
│                                                                      │
│  规则3：数组退化为指针（除非声明 auto&）                             │
│  ┌────────────────────────────────────────────────────────────┐     │
│  │ int arr[5] = {1,2,3,4,5};                                   │     │
│  │ auto a = arr;     // int*（数组退化为指针）                  │     │
│  │ auto& ar = arr;   // int (&)[5]（保留数组引用）              │     │
│  └────────────────────────────────────────────────────────────┘     │
│                                                                      │
│  规则4：函数退化为函数指针（除非声明 auto&）                         │
│  ┌────────────────────────────────────────────────────────────┐     │
│  │ void func(int);                                             │     │
│  │ auto f = func;     // void(*)(int)                          │     │
│  │ auto& fr = func;   // void(&)(int)                          │     │
│  └────────────────────────────────────────────────────────────┘     │
│                                                                      │
└──────────────────────────────────────────────────────────────────────┘
```

### 2.3 auto 与初始化列表

```cpp
// 统一初始化
auto x1 = {42};        // std::initializer_list<int>
auto x2 = {42, 3.14};  // 编译错误：类型不一致
auto x3{42};           // C++17：int（N=1 时特殊规则）
auto x4{42, 43};       // C++17：std::initializer_list<int>

// 推导指引
auto a = 42;           // int
auto b = {42};         // initializer_list<int>
auto c{42};            // C++11/14：initializer_list<int>
                       // C++17：int
```

### 2.4 auto 在范围 for 中

```cpp
std::map<std::string, int> scores = {{"Alice", 90}, {"Bob", 85}};

// 错误：拷贝 pair
for (auto p : scores) { /* ... */ }

// 正确：const 引用
for (const auto& p : scores) { /* ... */ }

// 结构化绑定（C++17）
for (const auto& [name, score] : scores) { /* ... */ }
```

### 2.5 auto 作为函数返回值（C++14）

```cpp
// C++14：自动推导返回类型
auto add(int a, int b) {
    return a + b;  // 返回 int
}

// 多返回语句必须类型一致
auto getValue(bool flag) {
    if (flag) return 42;    // int
    return 3.14;           // 编译错误：类型不一致！
}

// decltype(auto)：保留表达式的精确类型
decltype(auto) getRef(int& x) {
    return (x);  // 返回 int&，而非 int
}
```

## 第三部分：decltype 与 decltype(auto)

### 3.1 decltype 基本用法

`decltype` 返回表达式的**精确类型**，不进行任何转换：

```cpp
int x = 42;

decltype(x) a = x;    // int a = x;
decltype((x)) b = x;  // int& b = x;  注意双层括号！
decltype(x + 1) c = 0;// int c = 0;

const int& cref = x;
decltype(cref) d = x; // const int& d = x;

int arr[5] = {};
decltype(arr) e;      // int[5]（保留数组类型）
```

### 3.2 decltype 推导规则

```
┌──────────────────────────────────────────────────────────────────────┐
│                    decltype 推导规则                                 │
├──────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  规则1：无括号的标识符 → 该标识符的声明类型                           │
│  ┌────────────────────────────────────────────────────────────┐     │
│  │ int x = 42;                                                 │     │
│  │ decltype(x) → int                                           │     │
│  │                                                             │     │
│  │ const int& ref = x;                                         │     │
│  │ decltype(ref) → const int&                                  │     │
│  └────────────────────────────────────────────────────────────┘     │
│                                                                      │
│  规则2：有括号的标识符 → 引用类型（lvalue expression）               │
│  ┌────────────────────────────────────────────────────────────┐     │
│  │ int x = 42;                                                 │     │
│  │ decltype((x)) → int&                                        │     │
│  │                                                             │     │
│  │ 解释：(x) 是一个左值表达式，不是标识符                       │     │
│  └────────────────────────────────────────────────────────────┘     │
│                                                                      │
│  规则3：表达式 → 根据值类别确定类型                                  │
│  ┌────────────────────────────────────────────────────────────┐     │
│  │ int x = 42;                                                 │     │
│  │ decltype(x + 1) → int（x + 1 是 prvalue）                   │     │
│  │ decltype(x = 0) → int&（赋值表达式是 lvalue）               │     │
│  │ decltype(std::move(x)) → int&&（std::move 返回 xvalue）     │     │
│  └────────────────────────────────────────────────────────────┘     │
│                                                                      │
└──────────────────────────────────────────────────────────────────────┘
```

### 3.3 decltype(auto)（C++14）

`decltype(auto)` 结合了 `auto` 的简洁和 `decltype` 的精确性：

```cpp
// 返回值保留引用
int x = 42;

auto f1() {
    return (x);  // 返回 int（拷贝）
}

decltype(auto) f2() {
    return (x);  // 返回 int&（引用）
}

// 用于变量声明
decltype(auto) a = x;   // int a = x;
decltype(auto) b = (x); // int& b = x;

// 完美转发返回值
template<typename T>
decltype(auto) forwardValue(T&& val) {
    return std::forward<T>(val);
}
```

### 3.4 decltype 与 SFINAE

```cpp
// 检测成员函数是否存在
template<typename T, typename = void>
struct has_size : std::false_type {};

template<typename T>
struct has_size<T, std::void_t<decltype(std::declval<T>().size())>> 
    : std::true_type {};

// 使用
static_assert(has_size<std::string>::value);
static_assert(!has_size<int>::value);
```

## 第四部分：类型转换

### 4.1 四种具名类型转换

C++ 提供四种具名类型转换，比 C 风格转换 `(Type)value` 更安全：

```
┌──────────────────────────────────────────────────────────────────────┐
│                    C++ 类型转换速查                                   │
├──────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  static_cast<Type>(value)                                            │
│  ├─ 用途：相关类型间的显式转换                                       │
│  ├─ 示例：double→int, void*→T*, 枚举→整型                            │
│  └─ 风险：可能丢失精度，向下转型无运行时检查                          │
│                                                                      │
│  dynamic_cast<Type>(value)                                           │
│  ├─ 用途：多态类间的安全向下转型                                      │
│  ├─ 条件：必须有虚函数                                               │
│  ├─ 返回：成功则指针/引用，失败则 nullptr/bad_cast                   │
│  └─ 开销：运行时类型检查（RTTI）                                     │
│                                                                      │
│  const_cast<Type>(value)                                             │
│  ├─ 用途：添加或移除 const/volatile                                  │
│  ├─ 限制：只能改变 cv-限定符，不能改变类型本身                        │
│  └─ 风险：修改原本 const 的对象是 UB                                  │
│                                                                      │
│  reinterpret_cast<Type>(value)                                       │
│  ├─ 用途：不相关类型间的位重新解释                                    │
│  ├─ 示例：int*→char*, 指针→整数                                      │
│  └─ 风险：高度平台相关，可能导致 UB                                   │
│                                                                      │
└──────────────────────────────────────────────────────────────────────┘
```

### 4.2 static_cast 详解

```cpp
// 1. 数值类型转换
double d = 3.14;
int i = static_cast<int>(d);  // 精度丢失，但合法

// 2. 指针上下转换（同一继承体系）
class Base {};
class Derived : public Base {};

Derived derived;
Base* base = static_cast<Base*>(&derived);  // 向上转型，安全

Base* b = new Base;
Derived* d = static_cast<Derived*>(b);  // 向下转型，无运行时检查！危险

// 3. void* 与其他指针转换
void* ptr = malloc(100);
int* ip = static_cast<int*>(ptr);  // OK

// 4. 枚举与整型
enum class Color { Red, Green, Blue };
int color_val = static_cast<int>(Color::Red);  // OK

// 5. 明确构造函数调用
class String {
public:
    String(const char* s);
};
String s = static_cast<String>("hello");  // 显式调用构造函数
```

### 4.3 dynamic_cast 详解

```cpp
class Base { 
public:
    virtual void foo() {}  // 必须有虚函数
};
class Derived : public Base {};

// 指针转换：失败返回 nullptr
Base* b1 = new Derived;
Derived* d1 = dynamic_cast<Derived*>(b1);  // 成功

Base* b2 = new Base;
Derived* d2 = dynamic_cast<Derived*>(b2);  // 返回 nullptr

// 引用转换：失败抛出 std::bad_cast
try {
    Base& ref = *b2;
    Derived& dref = dynamic_cast<Derived&>(ref);  // 抛出异常
} catch (const std::bad_cast& e) {
    std::cout << "Cast failed: " << e.what() << std::endl;
}

// 交叉转换（多重继承）
class A { virtual void f() {} };
class B { virtual void g() {} };
class C : public A, public B {};

C* c = new C;
A* a = c;
B* b = dynamic_cast<B*>(a);  // 交叉转换，成功
```

### 4.4 const_cast 详解

```cpp
const int ci = 42;

// 移除 const
int* p = const_cast<int*>(&ci);
*p = 100;  // 编译通过，但是 UB！

// 合法用途：调用遗留 API
void legacyFunc(int* p);  // 不修改 *p，但参数不是 const

void safeWrapper(const int* p) {
    legacyFunc(const_cast<int*>(p));  // 安全：我们知道 legacyFunc 不修改
}

// 添加 const（较少需要）
int x = 42;
const int* cp = const_cast<const int*>(&x);  // 通常直接 &x 即可
```

**关键陷阱**：修改原本 `const` 的对象是 UB！

```cpp
const int ci = 42;
const_cast<int&>(ci) = 100;  // UB！

// 原因：编译器可能将 ci 放在只读内存，或进行常量传播优化
int x = ci * 2;  // 编译器可能优化为 x = 42 * 2 = 84
```

### 4.5 reinterpret_cast 详解

```cpp
// 1. 指针类型重解释
int* ip = new int(42);
char* cp = reinterpret_cast<char*>(ip);  // 将 int* 解释为 char*

// 2. 指针与整数转换（足够大的整数类型）
uintptr_t addr = reinterpret_cast<uintptr_t>(ip);
int* ip2 = reinterpret_cast<int*>(addr);

// 3. 函数指针转换（极少需要）
typedef void (*FuncPtr)();
void normalFunc(int);
FuncPtr fp = reinterpret_cast<FuncPtr>(normalFunc);  // 条件支持

// 4. 常见陷阱
int a = 0x12345678;
char* p = reinterpret_cast<char*>(&a);
std::cout << +p[0] << std::endl;  // 结果取决于字节序！
// 小端：0x78 (120)，大端：0x12 (18)
```

### 4.6 隐式转换与 narrowing conversion

```cpp
// 隐式转换序列
// 1. 标准转换（数值提升、数值转换）
// 2. 用户定义转换（构造函数、转换运算符）
// 3. 标准转换（第二阶段）

int i = 3.14;        // double → int：隐式 narrowing（可能警告）
double d = 42;       // int → double：隐式提升（安全）

// 列表初始化禁止 narrowing
int j{3.14};         // 编译错误！
int arr[]{1, 2, 3.0};// 编译错误！

// 算术转换规则
short s1 = 1, s2 = 2;
auto r = s1 + s2;    // int（整数提升）
// 规则：short/char → int，unsigned → 更大类型
```

## 第五部分：CTAD 类模板参数推导

### 5.1 CTAD 基本用法（C++17）

C++17 之前，类模板参数必须显式指定：

```cpp
// C++17 之前
std::pair<int, double> p1(42, 3.14);
std::tuple<int, double, char> t1(1, 2.0, 'a');
std::vector<int> v1{1, 2, 3};

// C++17：编译器自动推导
std::pair p2(42, 3.14);       // std::pair<int, double>
std::tuple t2(1, 2.0, 'a');   // std::tuple<int, double, char>
std::vector v2{1, 2, 3};      // std::vector<int>
std::array arr{1, 2, 3};      // std::array<int, 3>

// 智能指针
std::shared_ptr sp = std::make_shared<int>(42);  // std::shared_ptr<int>
std::optional opt = 42;       // std::optional<int>
```

### 5.2 自定义推导指引

```cpp
template<typename T>
struct Wrapper {
    T value;
    Wrapper(T v) : value(v) {}
};

// 隐式生成的推导指引等价于：
// Wrapper(T) -> Wrapper<T>;

Wrapper w1(42);      // Wrapper<int>
Wrapper w2(3.14);    // Wrapper<double>

// 显式自定义推导指引
template<typename T>
Wrapper(T) -> Wrapper<T>;  // 通常自动生成

// 复杂示例：从迭代器推导
template<typename Iter>
Wrapper(Iter, Iter) -> Wrapper<typename std::iterator_traits<Iter>::value_type>;
```

### 5.3 CTAD 陷阱

```cpp
// 陷阱1：无法推导聚合类型的模板参数
template<typename T>
struct Point {
    T x, y;
};

Point p{1.0, 2.0};  // C++17：编译错误！

// 需要显式推导指引
template<typename T>
Point(T, T) -> Point<T>;

Point p{1.0, 2.0};  // 现在 OK：Point<double>

// 陷阱2：initializer_list 的歧义
std::vector v1{5};      // std::vector<int>，1 个元素：5
std::vector v2(5);      // std::vector<int>，5 个元素：0
std::vector v3{5, 3};   // std::vector<int>，2 个元素：5, 3
std::vector v4(5, 3);   // std::vector<int>，5 个元素：3
```

## 面试要点

### Top 3 高频考点

#### 1. auto 类型推导规则

**Q：`auto x = {42}` 的类型是什么？**

```cpp
auto x = {42};     // std::initializer_list<int>
auto y = 42;       // int
auto z{42};        // C++17：int
auto w{42, 43};    // std::initializer_list<int>
```

**追问：`auto&` 和 `auto&&` 的区别？**

```cpp
int x = 42;

auto& r1 = x;      // int&：左值引用
auto&& r2 = x;     // int&：转发引用，绑定左值
auto&& r3 = 42;    // int&&：转发引用，绑定右值

// auto&& 是转发引用（forwarding reference）
// 根据初始化表达式的值类别决定最终类型
```

#### 2. 四种类型转换的使用场景

| 转换 | 场景 | 安全性 |
|------|------|--------|
| `static_cast` | 相关类型转换 | 编译期检查，无运行时检查 |
| `dynamic_cast` | 多态向下转型 | 运行时检查，安全但慢 |
| `const_cast` | 移除 const | 危险，可能导致 UB |
| `reinterpret_cast` | 位重解释 | 极危险，平台相关 |

**追问：什么时候必须用 `const_cast`？**

- 调用遗留 API（参数非 const 但实际不修改）
- 实现 `operator[]` 的 const 版本复用非 const 版本

#### 3. decltype((x)) vs decltype(x)

```cpp
int x = 42;
decltype(x) a;      // int
decltype((x)) b=x;  // int&

// 关键：双层括号将标识符变成表达式
// (x) 是左值表达式，decltype 返回引用类型
```

**追问：为什么 `decltype(auto)` 有用？**

```cpp
// 返回值完美转发
template<typename T>
decltype(auto) wrapper(T&& x) {
    return std::forward<T>(x);
}

// 返回表达式保留引用
decltype(auto) getRef(int& x) {
    return (x);  // int&
}
```

## 最佳实践

### 1. 类型选择

```cpp
// ✓ 推荐：使用固定大小类型
#include <cstdint>
int32_t counter;
int64_t large_value;

// ✓ 推荐：使用 auto 简化迭代器声明
for (auto it = vec.begin(); it != vec.end(); ++it)

// ✓ 推荐：使用 const auto& 避免拷贝
for (const auto& item : container)

// ✗ 避免：混用 signed/unsigned
std::vector<int> vec;
for (int i = 0; i < vec.size(); i++)  // 警告：有符号/无符号比较

// ✓ 修正：使用 size_t 或 auto
for (size_t i = 0; i < vec.size(); i++)
for (auto i = 0u; i < vec.size(); i++)
```

### 2. 类型转换

```cpp
// ✓ 推荐：优先使用具名转换
double d = 3.14;
int i = static_cast<int>(d);  // 明确意图

// ✓ 推荐：多态转型用 dynamic_cast
Base* b = getBase();
if (Derived* d = dynamic_cast<Derived*>(b)) {
    // 安全使用 d
}

// ✗ 避免：C 风格转换
int* p = (int*)malloc(100);  // 难以搜索，可能隐藏错误

// ✓ 修正：使用具名转换
int* p = static_cast<int*>(malloc(100));
```

### 3. auto 使用

```cpp
// ✓ 推荐：消除冗余类型名
auto ptr = std::make_unique<Widget>();
auto lambda = [](int x) { return x * 2; };

// ✓ 推荐：避免意外拷贝
for (const auto& [key, value] : map) { /* ... */ }

// ✗ 避免：隐藏复杂的初始化
auto x = computeSomething();  // 类型不明显时，考虑显式写类型

// ✗ 避免：auto 持有 proxy 对象
auto proxy = container[0];  // 可能是 proxy 类型（如 vector<bool>）
bool value = container[0];  // 显式类型更安全
```

## 常见陷阱

### 陷阱 1：无符号整数下溢

```cpp
// UB 风险：无符号整数下溢
unsigned int x = 10;
if (x - 20 > 0) {  // 永远为真！x - 20 是一个巨大的正数
    std::cout << "Unexpected!" << std::endl;
}

// 修正：先检查或使用有符号类型
if (x > 20 && x - 20 > 0) { /* ... */ }
```

### 陷阱 2：const_cast 修改常量对象

```cpp
// UB：修改原本 const 的对象
const int ci = 42;
const_cast<int&>(ci) = 100;  // UB！

// 编译器可能优化为：
int x = ci * 2;  // 直接替换为 x = 84
```

### 陷阱 3：整型提升导致的符号扩展

```cpp
char c = '\x80';  // 如果 char 是 signed，值为 -128
int i = c;        // 符号扩展：i = -128 (0xFFFFFF80)

unsigned char uc = '\x80';  // uc = 128
int j = uc;       // 零扩展：j = 128 (0x00000080)
```

### 陷阱 4：narrowing conversion

```cpp
// 隐式 narrowing 可能导致数据丢失
long long big = 1LL << 40;  // 约 1TB
int small = big;            // 截断！结果不确定

// 列表初始化禁止 narrowing
int x{big};  // 编译错误：保护你免于截断
```

### 陷阱 5：auto 与 initializer_list

```cpp
auto x = {42};     // initializer_list<int>，不是 int！
auto y = {1, 2};   // initializer_list<int>

// 突然想要一个 vector？
std::vector v{1, 2, 3};  // CTAD：vector<int>
auto v2 = std::vector{1, 2, 3};  // 同上
```

### 陷阱 6：reinterpret_cast 的对齐问题

```cpp
// UB：未对齐访问
char buffer[10];
int* ip = reinterpret_cast<int*>(buffer);  // 可能未对齐！
*ip = 42;  // UB：某些架构上崩溃

// 修正：使用 memcpy 或 alignas
int value;
std::memcpy(&value, buffer, sizeof(int));

// 或确保对齐
alignas(int) char buffer[10];
```
