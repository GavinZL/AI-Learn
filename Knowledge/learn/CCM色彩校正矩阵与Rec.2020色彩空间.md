## 核心概念

### CCM（色彩校正矩阵）
**白话解释**：CCM是一个3×3的数学矩阵，作用是把相机传感器"看到"的颜色（传感器原生RGB）转换成标准颜色空间（如sRGB、Rec.709、Rec.2020）定义的"正确"颜色。就像把方言翻译成普通话，让不同相机的颜色表现一致。

**关键理解**：
- CCM目标是**标准RGB线性空间**（如线性sRGB、线性Rec.709），而非直接到XYZ
- 同一传感器针对不同目标颜色空间需要**不同的CCM矩阵**
- CCM标定使用ColorChecker色卡，通过最小化ΔE来优化矩阵系数

### Rec.2020 Linear vs 非线性
**白话解释**：Rec.2020定义了广色域（三原色坐标），但信号可以有两种表示方式：
- **Linear**：与光强度成正比，适合计算但浪费位深
- **非线性**：通过传递函数（Gamma/PQ/HLG）压缩，适合传输和显示

**权威依据**（ITU-R BT.2087-0）：
> "Linear to non-linear conversion from linearly represented, normalized RGB colour signals E_R E_G E_B (Rec. 2020) to normalized R'G'B' colour signals E'_R E'_G E'_B"

### 传递函数（OETF/EOTF）
**白话解释**：
- **OETF**（光转电）：相机端把场景光线转换成电信号的函数（如Gamma、PQ）
- **EOTF**（电转光）：显示器端把电信号还原成光线的函数
- **为什么需要**：人眼对暗部更敏感，非线性编码能更高效利用位深

### 颜色空间的层次结构
```
色域（Gamut）    → 三原色坐标定义（线性）
     ↓
传递函数（TF）   → OETF/EOTF（非线性编码）
     ↓
完整色彩空间     → 色域 + 传递函数
```

**例子**：
- Rec.2020 + Gamma = SDR广色域
- Rec.2020 + PQ = HDR10
- Rec.2020 + HLG = 广播HDR

## 比喻 & 例子

### 比喻：CCM就像"色彩翻译官"
想象不同品牌的相机是说不同方言的人：
- 传感器A说"四川话"（特定的RGB响应）
- 传感器B说"广东话"（另一种RGB响应）
- CCM就是翻译官，把它们都翻译成"普通话"（标准sRGB/Rec.709）
- **关键点**：翻译成"普通话"需要一套规则，翻译成"英语"（Rec.2020）需要另一套规则

### 工作例子：相机ISP管线中的CCM配置
```
场景：开发一款支持HDR/SDR双模式的相机

配置1：SDR模式
- CCM目标：线性sRGB
- 传递函数：Gamma 2.2
- 位深：10bit
- 输出：sRGB非线性

配置2：HDR模式  
- CCM目标：线性Rec.2020
- 传递函数：PQ (ST 2084)
- 位深：12bit
- 输出：Rec.2020 PQ

关键发现：两套完全不同的CCM矩阵！
```

### 生活例子：照片的"原片"与"成片"
- **Linear Rec.2020** = 摄影师的RAW原片：信息完整，文件大，需要专业软件打开
- **Rec.2020 + PQ** = 处理后的HDR照片：压缩过，适合在HDR电视上看，文件小
- **Rec.709** = 普通JPG：最通用，但色彩和动态范围有限

## 边界 & 反例

### 什么情况下CCM不适用？
1. **传感器色域与目标色域完全不重叠**：CCM无法凭空创造颜色
2. **极端光照条件**：标定时的光源与实际场景差异过大
3. **多光源混合场景**：CCM基于单一光源假设

### Rec.2020 Linear的局限
- **不适合直接显示**：显示器需要非线性信号
- **文件体积大**：需要高位深（12-16bit）才能避免色阶断层
- **兼容性差**：大多数播放器不支持线性Rec.2020

## 常见误区

| 误区 | 正确理解 |
|------|----------|
| CCM直接把传感器RGB转换到XYZ | CCM目标是**标准RGB线性空间**（如线性sRGB），XYZ是中间参考空间 |
| sRGB/Rec.709/Rec.2020都是非线性的 | 它们都有**线性版本**和**非线性版本**，传递函数是独立的概念 |
| 同一个CCM矩阵可以用于所有输出模式 | **错误**！CCM与目标颜色空间强相关，HDR和SDR需要不同的CCM |
| Rec.2020就是HDR | Rec.2020是**色域标准**，HDR需要配合PQ/HLG传递函数 |
| Gamma校正只是简单的"幂函数" | sRGB Gamma是**分段函数**，并非纯幂函数 |
| 线性RGB可以直接用于显示 | 线性信号需要**EOTF**转换才能在显示器上正确呈现 |

## 自测题

1. **CCM的目标色彩空间是什么？**
   > 答：标准RGB线性空间（如线性sRGB、线性Rec.709、线性Rec.2020），而非直接到XYZ。XYZ是设备无关的中间参考空间。

2. **为什么同一个传感器需要多套CCM矩阵？**
   > 答：因为CCM与目标颜色空间强相关。不同目标空间（sRGB vs Rec.2020）有不同的三原色定义，需要不同的矩阵来实现准确映射。

3. **Rec.2020 Linear和Rec.2020 PQ有什么区别？**
   > 答：Rec.2020 Linear是线性光强度表示，适合计算但位深要求高；Rec.2020 PQ是通过ST 2084传递函数编码的非线性表示，支持HDR（0.0001-10000 nit），适合传输和显示。

4. **HDR输出配置如何影响CCM选择？**
   > 答：HDR通常配合Rec.2020色域，因此CCM目标空间需要选择线性Rec.2020，后续处理链路也完全不同（PQ/HLG传递函数、12bit位深、色调映射等）。
